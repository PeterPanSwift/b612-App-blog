解析 Google 表單提交的網路請求，開發上傳表單資料的 App

最近經過 AI 的指導，彼得潘學會當我們提交 Google 表單時，其實是發送x-www-form-urlencoded 格式的資料。

ps: x-www-form-urlencoded 格式長得像這樣。

name=Peter&age=25&city=London

提交 Google 表單時每個欄位的名稱是神秘的 entry.一串數字，因此只要能解析 Google 表單提交的網路請求，找到每個欄位的名稱，即可開發上傳表單資料的 App。

以下我們以訂飲料表單為例，一步步示範實現的步驟。

  • 在 Google Forms 建立表單。
  • 連到表單，顯示 Web Inspector。
  • 填寫表單後提交，查看表單提交的網路請求。
  • 表單提交的 curl 指令和 Swift 程式。
  • 請 AI 開發上傳 Google 表單的 SwiftUI & Flutter App。

在 Google Forms 建立和分享表單

Google Forms: Sign-in
Access Google Forms with a personal Google account or Google Workspace account (for business use).docs.google.com

發布分享表單時將 Responder view 的權限改成 Anyone with the link。

連到表單，顯示 Web Inspector

表單的分享連結如下。

https://docs.google.com/forms/d/e/1FAIpQLSflc1IeGcrAe2QpX2twr1-2QpY-UzmgO8oGSMf4PU80gx72xA/viewform

從分享連結連到表單，從 Safari 的 menu 點選 Develop > Show Web Inspector。

ps: 如果 menu 上找不到 Develop，請從 Safari 的 Settings 打開 Show features for web developers。

網頁下方出現 Web Inspector，從現在開始記錄網路的相關活動。

填寫表單後提交，查看表單提交的網路請求

填寫表單後按提交。

切換到 Web inspector 的 Network 分頁，找到對應表單提交請求的 formResponse，然後點選 Headers 分頁。

從 Headers 分頁可得知網址(URL)，HTTP method 和上傳的資料。

網址為 https://docs.google.com/forms/d/e/1FAIpQLSflc1IeGcrAe2QpX2twr1-2QpY-UzmgO8oGSMf4PU80gx72xA/formResponse，HTTP method 為 POST。網址其實是當初表單的分享網址，只是結尾的 viewform 改成 formResponse。

從 Request Data 可看到上傳的資料內容，欄位名的對應如下。

  • entry.1331430243: 飲料。
  • entry.59557013: 訂購人。
  • entry.887937259: 甜度。

表單提交的 curl 指令和 Swift 程式

從剛剛 formResponse 的相關資訊，我們可透過以下的 curl 指令和 Swift 程式訂飲料。

  • curl 指令。
curl -X POST \
  "https://docs.google.com/forms/d/e/1FAIpQLSflc1IeGcrAe2QpX2twr1-2QpY-UzmgO8oGSMf4PU80gx72xA/formResponse" \
  -H "Content-Type: application/x-www-form-urlencoded" \
 --data-urlencode "entry.1331430243=玫瑰鮮奶茶" \
 --data-urlencode "entry.59557013=小王子" \
 --data-urlencode "entry.887937259=微糖"
  • Swift 程式。

由於上傳的資料可能會有中文，因此要用 addingPercentEncoding 進行轉換。

func buildDrinkOrderRequest(drink: String, name: String, sweet: String) -> URLRequest {
    let url = URL(string: "https://docs.google.com/forms/d/e/1FAIpQLSflc1IeGcrAe2QpX2twr1-2QpY-UzmgO8oGSMf4PU80gx72xA/formResponse")!
    var request = URLRequest(url: url)
    request.httpMethod = "POST"
    let postString = "entry.1331430243=\(drink)&entry.59557013=\(name)&entry.887937259=\(sweet)".addingPercentEncoding(withAllowedCharacters: .urlQueryAllowed)!
    request.httpBody = postString.data(using: .utf8)
    return request
}

以下彼得潘將 entry.1331430243 設為珍珠奶茶，entry.59557013 設為狐狸，entry.887937259 設為全糖，不管是 curl 指令或 Swift 程式都能成功訂飲料。

請 AI 開發上傳 Google 表單的 SwiftUI & Flutter App

使用 Windsurf Cascade，prompt 如下。

依據以下 curl 指令設計一個漂亮的訂飲料 App 頁面

curl -X POST \
  "https://docs.google.com/forms/d/e/1FAIpQLSflc1IeGcrAe2QpX2twr1-2QpY-UzmgO8oGSMf4PU80gx72xA/formResponse" \
  -H "Content-Type: application/x-www-form-urlencoded" \
 --data-urlencode "entry.1331430243=玫瑰鮮奶茶" \
 --data-urlencode "entry.59557013=小王子" \
 --data-urlencode "entry.887937259=微糖"

甜度有全糖、半糖、微糖、無糖。
