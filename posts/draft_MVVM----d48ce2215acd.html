  • ViewModel: Contains fields that are to be displayed in the view.
  • MVVM is considered suitable when you need to transform models into another representation for a view, and slims down view controllers that require several model-to-view transformations
  • Binding

binding fields within the view model to a view, three methods

  1. Use simple tools to create the bindings (in this case closures); and arguably this makes it wholly an MVP implementation

  2. Use a third party library like RXSwift to bind the view to the viewmodel

  3. The the Key-Value Observing pattern (KVO) to perform the binding
