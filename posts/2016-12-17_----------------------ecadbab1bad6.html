寫程式跟寫作文很像，都是打字。不過它比寫作簡單多了，因為現在的程式編輯器都有強大的自動完成 ( Autocomplete、code completion suggestion)功能。當我們輸入程式碼時，它會貼心地出現提示選單任君挑選，減少我們打錯字的機率。比方以下例子，當我們在 Xcode 裡輸入 function prepare 的前幾個字 prepa 後，在提示選單裡利用上下鍵移動選擇目標，很快就可找到我們想輸入的 function， prepare(for segue: UIStoryboardSegue, sender: Any?)。

找到目標，快狠準地按下 Enter 後，Xcode 將馬上貼心為我們輸入 function prepare 的相關程式碼，並將滑鼠游標體貼地停在 { } 裡的提示輸入框 ( 藍色長方形，白色文字code )，方便我們直接輸入程式碼。

然而程式初學者有時會習慣照著範例的程式碼一字一字輸入，想要每個字都自己打，於是就忘了自動完成。雖然大家都有多年的打字經驗，小時候都得過 xxx 打字冠軍。不過程式語言是個被惡魔詛咒，極容易打錯字的語言，當你愈努力地用真心一字一字輸入時，有 99 % 機率打錯，比方將剛剛例子的 prepare 打成 prepere。

自己打真的很容易打錯，讓我們再看看另一個例子。當我們想生成 UIImage 物件，輸入 UIImage(n 後，即可從提示選單找到目標，按下 Enter 鍵完成輸入。

然而當你自己輸入時，你極有可能將參數打成 name，忘了加 d，出錯以下錯誤訊息。

Argument labels '(name:)' do not match any available overloads

相反的，當你懶惰地搭配自動完成，反而不太會出錯。所以請千萬記得以下這句程式界的名言:

不要每個字都自己打，請交給專業的來自動完成 !

基本上，程式裡你需要自己輸入每一個字的時機，主要是以下三種情況:

1 宣告變數或常數

例子。

var age = 18

2 定義 function

func sing() {
}

3 定義發明型別

class Baby {

}

其它時候，你大部分都在使用自己或別人發明的變數，function 或型別，貼心的 Xcode 將記得它們的名字，所以你只要偷入前一兩個字，再從跳出的提示選單選擇對象後，按下 enter 即可自動完成。
