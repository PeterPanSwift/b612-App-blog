SwiftUI 問題

  • 請 AI 幫我 debug。
  • GPT 4 的回答。

The error message “Cannot assign to property: ‘self’ is immutable” typically indicates that you’re trying to mutate a property of a structure or enumeration without the proper permission to do so. In Swift, structs are value types and are immutable by default.

To fix the error, you can mark the number variable with the @State property wrapper. This tells SwiftUI that number is a state variable which can be changed from within the view. When a @State variable changes, SwiftUI will automatically update the views that depend on it.

Here’s how you should update your code:

struct ContentView: View {
    @State var number = 1 // Use @State to declare a mutable state variable

    var body: some View {
        VStack {
            Image(systemName: "die.face.\(number).fill")
                .resizable()
                .scaledToFit()
                .frame(width: 300)
            Button("roll") {
                number = Int.random(in: 1...6) // This will now work correctly
            }
        }
    }
}

根據錯誤訊息 “Cannot assign to property: ‘self’ is immutable”，問題在於你試圖對 Swift 中的 self 屬性進行賦值。self 是指向當前實例的隱式屬性,是不可變更的。

在你的程式碼中,Button 的動作中包含了 number = Int.random(in: 1...6) 這一行。這裡的 number 應該是想要修改 ContentView 結構中的 number 屬性。但是由於你是在 Button 的動作範圍內,這裡的 number 被視為一個局部變數,而不是外部結構的屬性。

要解決這個問題,你需要正確存取外部結構的屬性。一種常見的方式是在 ContentView 內定義一個方法,並在該方法中修改 number 屬性,然後讓 Button 的動作呼叫該方法。例如:

struct ContentView: View {
    @State private var number = 1
    
    var body: some View {
        // ...
        Button("Roll") {
            rollDice()
        }
    }
    
    private func rollDice() {
        number = Int.random(in: 1...6)
    }
}

這樣就可以正確地修改 number 屬性,而不會觸發 "Cannot assign to property: 'self' is immutable" 錯誤。

框框中應該輸入要連接的正在運行的 Dart 或 Flutter 應用程式的 URL。這個 URL 通常是用於 Dart DevTools 的 WebSocket 連接，用於調試應用程式。從提供的截圖中，我們可以看到終端機已經輸出了一個 WebSocket 連接 URL：

ws://127.0.0.1:54690/wpBZD3ZT4fg=/ws

你應該將這個 URL 複製並粘貼到「Connect to a Running App」框框中，然後點擊「Connect」以便連接到你的應用程式進行調試。
