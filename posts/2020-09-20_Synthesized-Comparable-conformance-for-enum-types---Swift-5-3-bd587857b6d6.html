Synthesized Comparable conformance for enum types — Swift 5.3

Swift 的 enum 型別很適合用來描述清單，用一個個 case 陳列清單的成員，比方以下例子以 enum Suit 描述撲克牌的四種花色。

enum Suit {
    case clubs, diamonds, hearts, spades
}

有時我們希望 enum 裡的 case 可以比大小，例如剛剛的撲克牌花色由小到大依序為 clubs, diamonds, hearts, spades。可惜天不從人願，如下圖所示，當我們傻傻地比較 suit1 & suit2 時將出現紅色錯誤，它們愛好和平，不肯比大小 !

從前想讓 enum 的 case 比大小我們要自己寫程式，要遵從 protocol Comparable & 定義比大小的 function <。在 Swift 5.3，SE-0266 的 Synthesized Comparable conformance for enum types 將讓 enum case 更容易比大小。

根據 SE-0266，當 enum 型別遵從 protocol Comparable，而且滿足以下其中一個條件時，它的 case 即可比大小，我們不用再定義 function <。

  • 沒有 associated value 的 enum。
  • enum 有 associated value，但 associated value 的型別遵從 protocol Comparable。

因此剛剛代表撲克牌花色的 enum Suit 遵從 protocol Comparable 後即可比大小。

enum Suit: Comparable {
    case clubs, diamonds, hearts, spades
}

此時 suit1 終於可跟 suit2 PK 比大小。

let suit1 = Suit.diamonds
let suit2 = Suit.spades
if suit1 < suit2 {
    print("suit1 < suit2")
} else {
    print("suit1 >= suit2")
}

除了使用常見的 > & < 比大小，比較相等也不是問題。

let suit1 = Suit.diamonds
let suit2 = Suit.diamonds
if suit1 == suit2 {
    print("suit1 == suit2")
} else {
    print("suit1 >= suit2")
}

因為可以比大小，排序當然也 ok。

var array: [Suit] = [.diamonds, .spades, .clubs, .hearts]
array.sort()
