使用 SwiftUI 的 toolbar(content:) 在 navigation bar，tool bar ＆ keyboard 上方的 bar 加入元件

開發 iOS App 時，我們時常會在上方的 navigation bar & 下方的 tool bar 加入元件，比方下圖 Calendar App 上方的放大鏡和下方的 Today。

接下來我們將介紹使用 SwiftUI 的 toolbar(content:)，利用它在 navigation bar，tool bar & keyboard 上方的 bar 加入元件。

toolbar(content:) | Apple Developer Documentation
Populates the toolbar or navigation bar with the specified items.developer.apple.com

在上方的 navigation bar 加入 button

透過 modifier toolBar 可在 navigation bar 上加入元件。

struct ContentView: View {
    var body: some View {
        NavigationStack {
            VStack {
                Text("小王子的藝想世界")
                    .font(.largeTitle)
                Image(.ticket)
                    .resizable()
                    .scaledToFit()
            }
            .toolbar {
                Button {
                        
                } label: {
                    Image(systemName: "plus")
                }
            }
        }
    }
}

如下圖所示，toolbar { } 裡的元件預設將顯示在 navigation bar 的右上角。值得注意的，toolbar 一定要搭配 NavigationStack，因為 NavigationStack 才會長出上方的 navigation bar 跟下方的 tool bar。

剛剛 toolbar { } 裡的元件也可以包在 ToolbarItem { } 裡，透過它我們將可調整元件的位置。

struct ContentView: View {
    var body: some View {
        NavigationStack {
            VStack {
                Text("小王子的藝想世界")
                    .font(.largeTitle)
                Image(.ticket)
                    .resizable()
                    .scaledToFit()
            }
            .toolbar {
                ToolbarItem {
                    Button {
                        
                    } label: {
                        Image(systemName: "plus")
                    }
                }
            }
        }
    }
}

設定 navigation bar 上元件的位置

透過 ToolbarItem 的參數 placement 可設定元件的位置。

  • 在 navigation bar 的左右各加一個 button

參數 placement 傳入 navigationBarLeading & navigationBarTrailing。

.toolbar {
     ToolbarItem(placement: .navigationBarLeading) {
          Button("edit") {
                        
          }
     }
     ToolbarItem(placement: .navigationBarTrailing) {
          Button {
                        
          } label: {
              Image(systemName: "plus")
          }
     }
}
  • 在 navigation bar 的右邊加入兩個 button

由左而右依序為 edit & +。

.toolbar {
    ToolbarItem(placement: .navigationBarTrailing) {
       Button("edit") {
                        
       } 
    }
    ToolbarItem(placement: .navigationBarTrailing) {
       Button {
                        
       } label: {
           Image(systemName: "plus")
       }
    }
}
  • 利用 ToolbarItemGroup 加入多個元件

我們也可以用 ToolbarItemGroup 加入多個元件，以下例子在 ToolbarItemGroup 裡加入兩個 button，設定它們的位置在 trailing。

.toolbar {
     ToolbarItemGroup(placement: .navigationBarTrailing) {
                    
          Button("edit") {
                        
          } 
     
          Button {
                        
          } label: {
               Image(systemName: "plus")
          }
     }               
}
  • 在 navigation bar 的中間加入元件

參數 placement 傳入 principal 可讓元件出現在 navigation bar 的中間。

.toolbar {
    ToolbarItem(placement: .principal) {
       HStack {
          Image(.peter)
             .resizable()
             .scaledToFit()
             .frame(width: 100, height: 100)
          Text("Peter")
       }
    }               
}

能不能讓 bar 中間顯示的帥氣 peter 圖片大一點而且不被上方的瀏海遮到呢 ? 當然可以，以下例子利用 offset 將圖片下移。

.toolbar {
    ToolbarItem(placement: .principal) {
       HStack {
          Image(.peter)
             .resizable()
             .scaledToFit()
             .frame(width: 200, height: 200)
             .offset(y: 50)
          Text("Peter")
       }
    }               
}
  • primaryAction，cancellationAction，confirmationAction & destructiveAction & navigation

參數 placement 的型別是 ToolbarItemPlacement，它還有很多可以設定的選項，有興趣的可進一步參考 ToolbarItemPlacement 的說明。

ToolbarItemPlacement | Apple Developer Documentation
A structure that defines the placement of a toolbar item.developer.apple.com

比方在 cancellationAction 代表取消，在 iOS 上取消通常出現在 leading。

.toolbar {
    ToolbarItem(placement: .cancellationAction) {
       Button("Cancel") {
       }
    }
}

在下方的 tool bar 加入元件

參數 placement 傳入 bottomBar，此時 button 將出現在下方中間的位置。

.toolbar {
    ToolbarItemGroup(placement: .bottomBar) {
                    
       Button {
                        
       } label: {
          Image(systemName: "plus")
       }
    }
               
}

我們可以加入 Spacer，讓 button 靠左。

.toolbar {
    ToolbarItemGroup(placement: .bottomBar) {
        Button {
                        
        } label: {
            Image(systemName: "plus")
        }
        Spacer()
    }               
}

在鍵盤上方的 bar 加入元件

從 iOS 15 開始，在 placement 參數傳入 keyboard，我們還可在鍵盤上方的 bar 加入元件。

struct ContentView: View {
    @State private var name = ""
    
    var body: some View {
        NavigationStack {
            VStack {
                TextField("name", text: $name)
                    .textFieldStyle(.roundedBorder)
                Image(.ticket)
                    .resizable()
                    .scaledToFit()
            }
            .toolbar {
                ToolbarItemGroup(placement: .keyboard) {
                    Button("Cancel") {
                        
                    }
                    Button("Ok") {
                        
                    }
                }
            }
        }
    }
}

動態設定 bar 上元件的顯示

有時我們希望 bar 的元件在某些情況顯示，某些情況隱藏，此時我們可在 ToolbarItem 的 closure 參數裡利用 if 控制是否加入元件。

struct ContentView: View {
    @State private var isNew = false
    
    var body: some View {
        NavigationStack {
            VStack {
                Text("小王子的藝想世界")
                    .font(.largeTitle)
                Image(.ticket)
                    .resizable()
                    .scaledToFit()
            }
            .toolbar {
                ToolbarItem(placement: .cancellationAction) {
                    if isNew {
                        Button("Cancel") {
                           
                        }
                    }
                }
            }
        }
    }
}
