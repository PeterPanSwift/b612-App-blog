程式初學者容易遺忘的 }

對程式初學者來說，} 是個他們常常不小心遺忘或殺掉的可憐蟲。請記得 { } 一定要成對，不要拆散他們。

少一個 } 真的滿可怕的 ! 雖然只是少一個符號，卻會如下圖所示，產生一堆紅色錯誤 !

在剛剛的例子裡，其實只不過是 for 的 { 少了搭配的 }，為何會產生這麼多的錯誤呢 ?

這是因為 { 很壞，當它找不到搭配的 } 後，將主動搶走別人的女朋友 } 。所以 for 的 { 把 function updateUI 搭配的 } 搶走了。證明的方法很簡單，有以下兩個方法:

方法1: 雙擊 for 的 { ，發現它果然跟 function 的 } 成了一對。

方法2: 滑鼠游標移到 for 上，按住 cmd 鍵，發現 for 的 { 跟 function 的 } 成了一對。

因為 for 的 { 搶了 function updateUI 的 } ，所以就天下大亂了 ! 這時 updateUI 只能接著搶別人的女朋友，所以它把 class 的 } 搶走了。

於是其它的 function，比方 viewDidLoad，就變成定義在 function updateUI 裡。但是 viewDidLoad 有加 override，只有 class 裡的 function 會加 override 呀，所以就產生錯誤了。

剛剛少了 } 的 for 只是一個例子，其它像是 if, function 等，少了 } 時，也一樣會產生類似的問題。

所以請記得，千萬不要少了 }。一般我們在輸入 class , for , func , if 的 { 後按下 enter，Xcode 會自動幫我們補上 }，所以只要你不要誤刪，應該是不會漏掉 } 啦。

真的少了的話也不用太慌張，只要用剛剛的方法， 雙擊 { 檢查，找出誰的 { } 配對出問題後，補上 } 即可。
