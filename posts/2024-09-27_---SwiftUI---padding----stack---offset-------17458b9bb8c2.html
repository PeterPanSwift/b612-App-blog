利用 SwiftUI 的 padding 移除 stack 裡 offset 造成的空白

使用 SwiftUI 的 VStack & HStack 排列元件時，通常元件是不會重疊的，例如以下 VStack 的例子。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.macbook)
                .resizable()
                .scaledToFill()
                .frame(width: 200, height: 200)
                .clipShape(.circle)
            Image(systemName: "swift")
                .resizable()
                .scaledToFit()
                .frame(width: 100)
                .foregroundStyle(.white)
                .padding()
                .background(.orange.gradient, in: .rect(cornerRadius: 10))
            Text("swift")
                .font(.largeTitle)
                .bold()
            Text("The powerful programming language that's also easy to learn")
        }
        .padding()
    }
}

要重疊也不是問題，比方透過 offset 將 Image swift 上移，讓它跟 Image macbook 重疊。

 Image(systemName: "swift")
     .resizable()
     .scaledToFit()
     .frame(width: 100)
     .foregroundStyle(.white)
     .padding()
     .background(.orange.gradient, in: .rect(cornerRadius: 10))
     .offset(y: -30)

Image swift 上移時並不會影響其它元件，造成 Image 和 Text 間多出一塊空白。有方法讓 Image swift 下的元件全都上移，不要有空白嗎 ?

將 Image swift 下的元件全都添加 offset(y: -30) 是一種解法，不過挺麻煩的。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.macbook)
                .resizable()
                .scaledToFill()
                .frame(width: 200, height: 200)
                .clipShape(.circle)
            Image(systemName: "swift")
                .resizable()
                .scaledToFit()
                .frame(width: 100)
                .foregroundStyle(.white)
                .padding()
                .background(.orange.gradient, in: .rect(cornerRadius: 10))
                .offset(y: -30)
            Text("swift")
                .font(.largeTitle)
                .bold()
                .offset(y: -30)
            Text("The powerful programming language that's also easy to learn")
                .offset(y: -30)
        }
        .padding()
    }
}

另一種較簡單的寫法是在 offset(y: -30) 後加入 padding(.bottom, -30)。為什麼這樣可以呢 ?

讓我們先看未加 padding 前的情況。當 Image 呼叫 offset(y: -30) 後，它將上移 30，但是 Image 本身佔據的框框並沒有移動，因此 Image 和 Text 間會多出空白。

當我們在 offset(y: -30) 後加入 padding(.bottom, -30) 時，Text 本身框框的高度將減少 30，因此剛好移除掉多出的空白。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.macbook)
                .resizable()
                .scaledToFill()
                .frame(width: 200, height: 200)
                .clipShape(.circle)
            Image(systemName: "swift")
                .resizable()
                .scaledToFit()
                .frame(width: 100)
                .foregroundStyle(.white)
                .padding()
                .background(.orange.gradient, in: .rect(cornerRadius: 10))
                .offset(y: -30)
                .padding(.bottom, -30)
            Text("swift")
                .font(.largeTitle)
                .bold()
            Text("The powerful programming language that's also easy to learn")
        }
        .padding()
    }
}

當我們用 stack 搭配 ForEach 顯示多筆資料時，則可利用負數的 padding 讓它們重疊。

struct ContentView: View {
    var body: some View {
        HStack {
           ForEach(0..<3) { index in
              Image("peter\(index)")
                 .resizable()
                 .scaledToFill()
                 .frame(width: 100, height: 100)
                 .clipShape(.circle)
                 .padding(.trailing, -30)
           }
        }
    }
}

以上程式的 Image 因為 -30 的 padding 讓元件本身的框框變小，所以圖片之間可以有部分重疊。
