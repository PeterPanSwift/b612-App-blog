swift ‘s async await network

Book: Practical Swift Concurrency

An async function that’s not bound to any actor will always run on the default executor. An executor in Swift concurrency is a low-level mechanism that takes jobs (pieces of asynchronous code) and runs them somewhere. The default executor will run jobs on the shared thread pool. In practical terms this means that, unless otherwise specified, an async function will always run on some thread that is not the main thread.

Note that it doesn’t matter where we call our asynchronous function from. What matters is whether or not the function itself is somehow tied to an actor.
