利用 Code Snippet & Predictive Code Completion 快速建立預覽 UIKit controller 的 preview

使用 #Preview 可以方便地預覽和操作 UIKit view controller 設計的畫面，不過手動輸入 preview 裡的程式還是有點辛苦，以下我們結合 Code Snippet & Predictive Code Completion，之後只要打幾個字即可生成預覽 UIKit view controller 的 preview。

使用 #Preview macro 定義預覽畫面
透過 Xcode 的 preview 功能，我們可以方便地從 preview 預覽和操作畫面，不用啟動模擬器，而且修改程式後 preview 也會立即更新，馬上就能測試畫面和功能是否正確。medium.com

使用 storyboard 和 id 建立 view controller

輸入以下程式，利用注解告訴 Predictive Code Completion 我們想生成的程式。

@available(iOS 17.0, *)
#Preview {
    // create main storyboard
    // create controller from id, type casting
    // return controller
    <#code#>
}

iOS 17 之後才支援在 #Preview { } 顯示 UIKit 的元件，因此我們要在 #Preview 前加上 @available(iOS 17.0, *)。

#Preview macro 搭配 iOS 的版本問題
透過 #Preview macro 我們可以方便地預覽 App 畫面，不過當專案的 Minimum Deployments 小於 iOS 17 時，在某些情況會遇到版本問題。medium.com

選取程式後從右鍵選單點選 Create Code Snippet。

輸入 code snippet 的名字，在 Completion 欄位輸入快速鍵 prevcid。

在 TableTennisViewController.swift 測試，輸入 prevc，點選 prevcid。

在 code 按下 enter，然後按下 option 鍵觸發 Predictive Code Completion 的多行提示。

按下 tab 完成輸入。

@available(iOS 17.0, *)
#Preview {
    // create main storyboard
    // create controller from id, type casting
    // return controller
    let storyboard = UIStoryboard(name: "Main", bundle: nil)
    let controller = storyboard.instantiateViewController(withIdentifier: "TableTennisViewController") as! TableTennisViewController
    return controller
}

以上程式假設採用的 storyboard 是 Main.storyboard，如果是其它的 storyboard，可再修改建立 UIStoryboard 時傳入的 name 參數，或是修改注解的內容。
