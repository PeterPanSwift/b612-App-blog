Xcode 11 體貼的拼字檢查(spell checking)

寫程式時我們時常會為東西取名，比方變數，function 和類別的名字。而對程式來說，好的名字往往是由淺顯易懂，清楚表達變數，function 和類別目的的英文單字組成。因此輸入這些英文單字時，記得要特別注意拼字問題，如果不小心打錯字，將變數名 name 打成 nane 可就糗大了。

Xcode 11 加入了拼字檢查的功能，大幅減少我們打錯字的機率。接下來就讓我們看看它如何發揮作用吧。

比方我們不小心將常數名 name 打成 nane。

let nane = "peter"
print(nane)

預設 Xcode 並不會產生任何錯誤，但我們可點選 Xcode menu 的 Edit > Format > Spelling & Grammar > Check Document Now 強制檢查目前編輯的檔案(快速鍵 cmd + ; )。

檢查後馬上可看到 nane 底下出現紅線，提示我們此單字可能拼錯。

若想更快地發現單字拼錯，還可以勾選 Edit > Format > Spelling & Grammar > Check Spelling While Typing。

勾選後輸入以下程式馬上看到拼錯的單字底下出現紅線。

有時就算知道單字拼錯，我們還是不知道正確的拼法。沒關係，我們可以點選 Edit > Format > Spelling and Grammar > Show Spelling and Grammar，請 Xcode 告訴我們可能的正確拼法(快速鍵 cmd + : )。

如下圖所示，從 Spelling & Grammar 視窗裡，聰明的 Xcode 提示我們 name 可能的拼法有 name，none，nine 等單字。
