Allow interchangeable use of CGFloat and Double types — Swift 5.5 新功能

從 Swift 5.5(搭配 Xcode 13)開始，CGFloat & Double 可以自動轉換，因此以下程式的 numberDouble & numberCGFloat 不會再因型別不同造成問題。

func test() {
    var numberDouble: Double = 2.7
    var numberCGFloat: CGFloat = 2.7
    numberCGFloat = numberDouble
    numberDouble = numberCGFloat
}

在 Swift 5.5 之前，以上程式將出現以下錯誤訊息。

接著讓我們透過一個實際的例子，認識 CGFloat & Double 自動轉換的好處。

struct ContentView: View {
    var body: some View {
        PeterImage(degrees: 30, width: 200)
    }
}

struct PeterImage: View {
    let width: CGFloat
    let degrees: Double
    
    var body: some View {
        Image("peter")
            .resizable()
            .scaledToFit()
            .frame(width: width)
            .rotationEffect(.degrees(degrees))
    }
}

在 Swift 5.5 之前，以上程式 PeterImage 的 width & degrees 型別分別宣告為 CGFloat & Double，因為 frame & degrees 的參數型別分別為 CGFloat & Double。

在 Swift 5.5 CGFloat & Double 可以自動轉換，因此我們可以將 width & degrees 都宣告成熟悉的 Double，不用再煩惱該用 Double 還是 CGFloat。

struct PeterImage: View {
    let width: Double
    let degrees: Double
    
    var body: some View {
        Image("peter")
            .resizable()
            .scaledToFit()
            .frame(width: width)
            .rotationEffect(.degrees(degrees))
    }
}

ps: 值得注意的，只有 CGFloat & Double 可以自動轉換，Float & CGFloat 之間並沒有自動轉換。
