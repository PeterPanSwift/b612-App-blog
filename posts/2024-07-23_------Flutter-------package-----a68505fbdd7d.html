修正下載的 Flutter 專案找不到 package 的問題

當我們從網路上下載 Flutter App 專案後，第一次從 IDE 打開它時通常會出現滿滿的錯誤。

修正下載的 Flutter 專案找不到 package 的問題

當我們從網路上下載 Flutter App 專案後，第一次從 IDE 打開它時通常會出現滿滿的錯誤。

比方彼得潘抓取 https://github.com/PeterPanSwift/peter_profile 上的專案，使用 VS Code 打開後出現滿滿的錯誤。

此錯誤通常跟專案需要的 package 找不到有關，比方上圖的 Target of URI doesn’t exist: ‘package:flutter/material.dart’. 表示找不到 Flutter package，難怪整個專案出現滿滿的錯誤。

修正的方法也很簡單，只要執行 Get Packages 抓取專案需要的 package 即可。以下我們介紹 4 種抓取 package 的方法。

  • 方法 1: 點選右下的小鈴檔通知，點選 Run ‘pub get’。

通知的訊息 Some packages are missing or out of date, would you like to get them now 明白告訴我們錯誤的原因和修正的方法。

  • 方法 2: 從 command palette 點選 Flutter: Get Packages。
  • 方法 3: 從 pubspec.yaml 點選 Get Packages。
  • 方法 4: 從 Terminal 輸入 flutter pub get。

打開專案時自動 get packages

如果懶得手動觸發 get packages，我們也可以設定每次打開專案時自動 get packages，方法如下。

  • 新增資料夾 .vscode，在 .vscode 下新增 tasks.json。
  • 在 tasks.json 輸入想要 VS Code 執行的任務。

以下設定將讓 VS Code 打開專案時執行 flutter pub get。

{
    "version": "2.0.0",
    "tasks": [
      {
        "label": "Get Flutter Packages",
        "type": "shell",
        "command": "flutter pub get",
        "group": "build",
        "problemMatcher": [],
        "runOptions": {
          "runOn": "folderOpen"
        }
      }
    ]
  }
