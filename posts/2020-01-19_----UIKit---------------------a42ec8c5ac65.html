為什麼 UIKit 元件可以用 == 比較是否為同一個元件

開發 iOS App 時，我們有時會想判斷變數是否為某一個 UI 元件，比方以下 Lesley 同學的調色 App，前三個 slider 負責調整顏色的 RGB，由於它們的功能類似，因此我們將它們連到同一個 IBAction function changeColor。

在 function changeColor 裡我們必須判斷 sender 是哪個顏色的 slider，然後更新它對應的 label 數字，因此我們利用 == 比較 sender & slider 的 outlet，然後判斷 sender 是哪個 slider。

class ViewController: UIViewController {
    @IBOutlet weak var redSlider: UISlider!
    @IBOutlet weak var blueSlider: UISlider!
    @IBOutlet weak var greenSlider: UISlider!
    @IBOutlet weak var redLabel: UILabel!
    @IBOutlet weak var greenLabel: UILabel!
    @IBOutlet weak var blueLabel: UILabel!

    @IBAction func changeColor(_ sender: UISlider) {
        if sender == redSlider {
            redLabel.text = String(format: "%.2f", sender.value)
        } else if sender == greenSlider {
            greenLabel.text = String(format: "%.2f", sender.value)
        } else if sender == blueSlider {
            blueLabel.text = String(format: "%.2f", sender.value)
        }
    }
    
}

不過為什麼可以用兩個 == 比較呢 ? 在 Swift 裡 == 應該是比較物件內容是否相等，=== 才是比較是否為同一個物件呀。

其實 UIKit 元件都可以用 == 比較是否為同一個元件，讓我們利用以下例子說明。

let slider1 = UISlider()
let slider2 = slider1
if slider1 == slider2 {
}

以上程式的 == 是利用 NSObject 裡定義的 function == 做比較。

extension NSObject : Equatable, Hashable {
/// Returns a Boolean value indicating whether two values are
    /// equal. `NSObject` implements this by calling `lhs.isEqual(rhs)`.
    ///
    /// Subclasses of `NSObject` can customize Equatable conformance by overriding
    /// `isEqual(_:)`. If two objects are equal, they must have the same hash
    /// value, so if you override `isEqual(_:)`, make sure you also override the
    /// `hash` property.
    ///
    /// - Parameters:
    ///   - lhs: A value to compare.
    ///   - rhs: Another value to compare.
    public static func == (lhs: NSObject, rhs: NSObject) -> Bool

由於所有的 UIKit 元件都繼承自 NSObject，NSObject 是 UISlider 的祖先，因此 UISlider 可以利用 NSObject 的 == 做比較。從以上程式的說明，我們了解 NSObject 的 == 其實是呼叫 function isEqual，而當兩個變數儲存同一個 NSObject 物件時，isEqual 的判斷將回傳 true。因此我們可以直接用 == 比較 sender & slider 的 outlet，然後判斷 sender 是哪個 slider。

Returns a Boolean value that indicates whether the receiver and a given object are equal.developer.apple.com
@IBAction func changeColor(_ sender: UISlider) {
    if sender == redSlider {
        redLabel.text = String(format: "%.2f", sender.value)
    } else if sender == greenSlider {
        greenLabel.text = String(format: "%.2f", sender.value)
    } else if sender == blueSlider {
        blueLabel.text = String(format: "%.2f", sender.value)
    }
}

One more thing, 搭配 switch

既然 UIKit 元件可以用 == 比較相等，我們也可以將剛剛的例子改成 switch 的寫法。

@IBAction func changeColor(_ sender: UISlider) {
    switch sender {
    case redSlider:
        redLabel.text = String(format: "%.2f", sender.value)
    case greenSlider:
        greenLabel.text = String(format: "%.2f", sender.value)
    case blueSlider:
        blueLabel.text = String(format: "%.2f", sender.value)
    default:
        break
    }
}
