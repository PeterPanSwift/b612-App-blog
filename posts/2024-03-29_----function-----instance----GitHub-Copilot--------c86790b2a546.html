先呼叫 function 和建立 instance，再由 GitHub Copilot 自動生成定義

感覺未來結合 AI 寫程式的方法會跟以前差很多，甚至有可能跟原本的習慣相反。

開發程式時，我們通常會先定義 function 和型別，然後再呼叫 function 和建立 instance。 最近彼得潘使用 GitHub Copilot 時，發現有時順序顛倒也不錯，我們可以先呼叫 function 和建立 instance，再由 GitHub Copilot 自動生成定義。

先呼叫 function，再由 GitHub Copilot 自動生成 function

以加入漸層為例，先寫出呼叫 function addGradient 的程式。在寫的過程中， GitHub Copilot 也會聰明地建議可能的參數。

addGradient(x: 30, y: 30, width: 200, height: 200, colors: [UIColor.red, UIColor.blue])

GitHub Copilot 聰明地自動在下方生成 function addGradient 的定義。

func addGradient(x: CGFloat, y: CGFloat, width: CGFloat, height: CGFloat, colors: [UIColor]) {
     let gradient = CAGradientLayer()
     gradient.frame = CGRect(x: x, y: y, width: width, height: height)
     gradient.colors = colors.map { $0.cgColor }
     view.layer.addSublayer(gradient)
}

先建立 instance，再請 GitHub Copilot 生成型別

以電影為例，先寫出電影 array，在 [ ] 裡生成 Movie 物件。在寫的過程中， GitHub Copilot 也會聰明地建議可能的參數。

無間道是彼得潘偶像劉德華和梁朝偉的代表作，聰明的 Copilot 也心有靈犀地知道演員名單。

let movies = [
    Movie(name: "無間道", director: "劉偉強", actors: ["劉德華", "梁朝偉"]),
]

輸入 struct Movie 後，GitHub Copilot 聰明地加入相關的 property。

struct Movie {
  let name: String
  let director: String
  let actors: [String]
}
