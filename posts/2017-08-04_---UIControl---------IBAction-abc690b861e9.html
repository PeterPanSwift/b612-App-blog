繼承 UIControl 的類別才可以拉 IBAction

UIControl | Apple Developer Documentation
The base class for controls, which are visual elements that convey a specific action or intention in response to user…developer.apple.com

當類別繼承 UIControl 時，表示它們是有溫度的人，可以依據人類的觸碰產生事件，做適當的回應，例如以下這些是 iOS SDK 裡繼承 UIControl 的類別，其實它們就是 App 畫面上我們時常點選的按鈕，開關等元件。

  • UIButton
  • UISegmentedControl
  • UISwitch
  • UISlider
  • UIStepper
  • UIDatePicker
  • UIPageControl

以上元件皆可透過 action 設定觸發的 function，Send Events 底下的事件即是它們可以產生的事件。

相反的，一般單純繼承 UIView 的類別，就無法拉 action 連線了，只能拉 outlet。
