iOS App 表格點選收鍵盤的方法

開發 iOS App 時，輸入內容的表單頁面很適合用 table view 實現。而當內容輸入後，我們通常希望點選表格就能收鍵盤，這樣的功能可以透過 UITableViewDelegate 或 UITapGestureRecognizer 的幫忙實現。

iOS App table & collection view 點選收鍵盤的方法

開發 iOS App 時，輸入內容的表單頁面很適合用 table view 實現。而當內容輸入後，我們希望點選空白的地方就能收鍵盤。通常這可透過在 controller 裡定義 function touchesEnded(_:with:) 寫收鍵盤的程式實現，不過由於 table & collection view 會接收點選的事件，所以 touchesEnded(_:with:) 不會被觸發。

因此接下來我們將介紹另外兩個方法，透過 UITableViewDelegate 或 UITapGestureRecognizer 幫忙實現。(ps: 接下來我們就以下圖的訂飲料表單為例，分別說明這兩種方法。

透過 UITableViewDelegate 的 tableView(_:didSelectRowAt:)

cell 點選時將觸發 tableView(_:didSelectRowAt:)， 我們在裡面呼叫 endEditing 收鍵盤。

override func tableView(_ tableView: UITableView, didSelectRowAt indexPath: IndexPath) {
     view.endEditing(true)
}

雖然我們想在 cell 點選時收鍵盤，但是我們並不想 cell 點選時變色，因此請記得 table view 的 selection 設為 Single Selection，讓表格可點選。table view cell 的 Selection 設為 None，讓 cell 點選時不會變色。

透過 UITapGestureRecognizer

UITapGestureRecognizer 可偵測點選的手勢，我們將它加到 table view 上，然後設定 UITapGestureRecognizer 觸發的 IBAction function。

點選畫面將觸發 UITapGestureRecognizer 連結的 function tableTap(_:)，我們在裡面呼叫 endEditing 收鍵盤。

@IBAction func tableTap(_ sender: Any) {
     view.endEditing(true)
}

想要 UITapGestureRecognizer 偵測點選，也想要點選 cell 跳到下一頁

UITapGestureRecognizer 可以幫我們偵測畫面點選收鍵盤，不過有時我們希望表格的某個 cell 點選時做別的事，比方跳到下一頁，但是 UITapGestureRecognizer 預設卻會攔截點選的動作，不讓 cell 點選發揮作用，因此下圖從 cell 白玉珍珠連到下一頁的 segue 將失去作用。

還好，此問題只要將 UITapGestureRecognizer 的 Cancels touches in view 取消即可讓 cell 點選繼續發揮作用。(或是從程式將 UITapGestureRecognizer 的 property cancelsTouchesInView 設為 false。
