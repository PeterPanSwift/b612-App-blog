將專案程式模組化的 local Swift package

開發 iOS App 時，我們可以將常用的功能整理成 Swift package，之後即可方便地在不同的 App 專案使用，而且也能將 package 上傳到 GitHub，分享給需要的人下載使用。

關於 Swift package 的製作，使用和分享，彼得潘在之前的文章有分享過。

Swift package 的製作，使用和分享
開發 iOS App 時，若能將常用的功能整理成重覆使用的套件，我們就能方便地在不同 App 使用，而且也能提供給需要的人使用。medium.com

不過有時我們只想將 App 專案裡的程式模組化，不需要讓不同的 App 專案共享，而且也不想分享給其它人使用呢 ?

我們還是可以依功能將程式包成 Swift package，只是它們是 App 專案底下的 local package。以 Apple 的範例程式 Food Truck App 為例，從下圖可看到 Food Truck App 專案有個 FoodTruckKit package，專案的程式只要 import FoodTruckKit 即可使用 FoodTruckKit 提供的功能。

Apple Developer Documentation
Edit descriptiondeveloper.apple.com

FoodTruckKit package 的資料夾在 App 的專案資料夾下，因此 package 程式的版本管理是由 App 專案的版本管理控制的。

要如何製作將專案程式模組化的 local Swift package 呢 ? 方法其實跟以下連結介紹的差不多。

Swift package 的製作，使用和分享

唯一要注意的地方是 Swift package 要存在 App 專案下。如下圖所示，打開 App 專案，點選 File > New > Package 後，將 package 存在 App 資料夾下，Add to 欄位選擇 App 專案。
