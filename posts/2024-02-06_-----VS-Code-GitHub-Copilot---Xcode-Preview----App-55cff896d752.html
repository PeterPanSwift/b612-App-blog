同時使用 VS Code GitHub Copilot 和 Xcode Preview 開發 App

GitHub Copilot 提供方便的 AI 程式助手，不過很可惜 Xcode 並沒有內建 copilot，因此目前開發 iOS App 想使用 copilot 有兩種方法。

  • 在 Xcode 安裝第三方套件 CopilotForXcode。
在 Xcode 使用 Copilot 幫忙寫程式
最近聽說 GitHub 推出了厲害的 AI 程式碼工具 Copilot(副駕駛)，它可以提供不錯的程式碼建議，大幅加快程式的開發速度。medium.com
  • 使用 VS Code 開發。

使用 VS Code 有優點也有缺點，它能享用最完整的 GitHub Copilot，但是它也少了 Xcode 方便的 preview。

彼得潘平常工作時會外接大螢幕，因此想到也許可以試試左邊用 VS Code 寫程式，右邊用 Xcode 顯示 preview。

以下我們試試火力全開，同時使用 VS Code GitHub Copilot 和 Xcode preview 開發簡單的童話故事 App 吧。

  • 用 VS Code 和 Xcode 打開專案，一個在左，一個在右。

Xcode 只要顯示 preview，所以它可以犧牲自己，視窗小一點，大部分的空間留給 VS Code。

  • 定義 Book 型別。

copilot 自動生成 Book 相關的 property。

彼得潘微調後的程式如下。

import Foundation

struct Book: Identifiable {
    let id = UUID()
    let title: String
    let author: String
    let summary: String
}

copilot 自動生成顯示清單的 List，而且還搭配 NavigationLink 跳到 BookView 顯示詳情。

彼得潘微調後的程式如下。

struct ContentView: View {
    let fairyTaleBooks = [
        Book(title: "彼得潘", author: "J·M·巴里", summary: "彼得潘是一個永遠不會長大的男孩，他住在一個叫做「樂園」的地方。"),
        Book(title: "小紅帽", author: "格林兄弟", summary: "很久很久以前，有一個小女孩，她的外套是一件紅色的斗篷，所以大家都叫她小紅帽。"),
        Book(title: "灰姑娘", author: "查爾斯·佩羅", summary: "灰姑娘的母親生病了，父親又娶了一個惡毒的後母。"),
        
    ]

    var body: some View {
        NavigationStack {
            List(fairyTaleBooks) { book in
                NavigationLink(destination: BookView(book: book)) {
                    Text(book.title)
                }
            }
            .navigationTitle("童話故事")
        }
    }
}

大功告成 ! 我們成功透過 GitHub Copilot 產生大部分的程式，而且還能一邊修改程式，一邊從右邊的 Xcode preview 看到自動更新的預覽畫面。

ps: 預設按下 cmd + s 存檔才能在 Xcode preview 看到更新的畫面，建議設定 VS Code 修改程式後自動存檔。

設定 VS Code 修改程式後自動存檔(Auto Save)
VS Code 預設不會自動存檔，我們必須手動按下 cmd + s 才會存檔。不過 VS Code 其實也能設定修改程式後自動存檔，方法如下。medium.com

分割視窗

macOS 支援兩個 App 的視窗左右並排，因此我們也可以用分割模式設定 VS Code 在左，Xcode 在右，唯一的小缺點是此時 Xcode 的程式會露出一點，不能完全關閉。
