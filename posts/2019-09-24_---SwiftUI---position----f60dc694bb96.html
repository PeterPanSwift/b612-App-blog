利用 SwiftUI 的 position 定位

利用 position 我們可以指定元件的 x, y 座標(相對於 parent)，但有個要特別小心的地方，它設定的是元件中心點的座標，而不是左上角的座標。

position(x:y:) | Apple Developer Documentation
Positions the center of this view at the specified coordinates in its parent's coordinate space.developer.apple.com
ZStack {
   Image(.peter0)
      .resizable()
      .scaledToFill()
      .frame(width: 100, height: 100)
      .clipShape(.circle)
      .position(x: 0, y: 0)
   Image(.peter1)
      .resizable()
      .scaledToFill()
      .frame(width: 100, height: 100)
      .clipShape(.circle)
      .position(x: 200, y: 200)
}
.frame(width: 200, height: 200)
.background(.yellow)

因此以上的程式將讓 peter 圖片的中心點落在 (0, 0) & (200, 200) 。

若我們希望 peter 圖片剛好落在 ZStack 的左上角和右下角，我們要設定的座標應該是 (50, 50) & (150, 150)。

ZStack {
   Image(.peter0)
      .resizable()
      .scaledToFill()
      .frame(width: 100, height: 100)
      .clipShape(.circle)
      .position(x: 50, y: 50)
   Image(.peter1)
      .resizable()
      .scaledToFill()
      .frame(width: 100, height: 100)
      .clipShape(.circle)
      .position(x: 150, y: 150)
   }
   .frame(width: 200, height: 200)
   .background(.yellow)
}

當我們事先知道元件的大小時，設定中心點座標並不是太難的一件事。比方 peter 圖片的大小為 122 * 170，想讓圖片的左上角落在 ZStack 的左上角時，中心點座標即為圖片寬高的一半。

ZStack {
   Image(.peter0)
      .position(x: 61, y: 85)
}
.frame(width: 300, height: 300)
.background(.yellow)

position 的副作用: 改變元件大小

雖然 position 可以幫我們固定元件的位置，但它卻有一個不可小看的副作用，它還會改變元件的大小。

沒有設定 position 時， Text 的大小是文字內容的大小。

struct ContentView: View {
    var body: some View {
        Text("position 不可小看的副作用")
            .background(.yellow)
    }
}

設定 position 後，Text 變成佔據整個可用空間的大小。

struct ContentView: View {
    var body: some View {
        Text("position 不可小看的副作用")
            .background(.yellow)
            .position(x: 200, y: 100)
            .background(.green)
    }
}
