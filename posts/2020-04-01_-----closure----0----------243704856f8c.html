手動輸入 closure & $0 比自動完成還方便

在 Xcode 輸入 Swift 程式時，我們習慣偷懶，時常從自動完成選單選取輸入，不過最近彼得潘發現在某些 closure 搭配 $0 的參數省略 case 時，手動輸入的速度反而更快呢。

closure 參數的省略，神奇的 $0，$1 & $2 (Shorthand Argument Names)
Swift 的 closure 有很多精簡省略的寫法，幫助我們加班打 code 不再手酸。接下來我們將介紹其中最特別，最像火星文的參數省略寫法。medium.com

比方以下 array singers 存了三個深情款款的男歌手，我們想利用 map 取出他們的名字，組合成一個新的 array。

struct Singer {
    let name: String
    let gender: String
}

let singers = [
    Singer(name: "周興哲", gender: "男"),
    Singer(name: "周杰倫", gender: "男"),
    Singer(name: "周華健", gender: "男"),
]

使用自動完成

使用自動完成時步驟如下:

  1. 輸入 ma，從選單選擇 map。

2. 在框框按下 enter。

3. 輸入參數名 singer。

4. 在 closure 裡輸入 singer.name。

let names = singers.map { singer in
    singer.name
}

結果:

print(names)

使用手動輸入 closure & $0

  1. 手動輸入 map { 後按空白鍵。

此時 Xcode 還貼心地幫我們輸入 }。

2. 在 closure 裡輸入 $0.name。

let names = singers.map { $0.name }

想不到手動輸入的速度更快，而且打的字更少。因此如果遇到一些適合採用 closure 參數省略的 case，不妨考慮拋棄自動完成，改用手動輸入，比方 map，filter 等 higher order function 就還滿適合的。
