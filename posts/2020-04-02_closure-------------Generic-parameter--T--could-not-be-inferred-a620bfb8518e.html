舊版的問題 & 解法

撰寫 Swift 程式時，當我們將 closure 當參數傳入 function，通常會省略回傳型別，例如以下例子:

func loveEvent(count: Int, event: (Int, String) -> String) {
    for i in 1...count {
        let message = event(i, "沈佳宜")
        print(message)
    }
}
loveEvent(count: 3) { index, name  in
    let activity = ["看星星", "看月亮", "看太陽"].randomElement()!
    return "第\(index)次和\(name)\(activity)"
}

結果

不過當參數的型別是回傳 generic type 的 function 型別時，有時 closure 的回傳型別是不可省略的。讓我們看看以下 map function 的例子。

map function 的宣告如下，參數 transform 是 function 型別，回傳的型別 T 是 generic type

func map<T>(_ transform: (Element) throws -> T) rethrows -> [T]

當 closure 裡只有一行程式時，省略回傳型別是 ok 的。

struct Song {
    let name: String
    let singer: String
}
let songs = [
    Song(name: "龍捲風", singer: "周杰倫"),
    Song(name: "愛笑的眼睛", singer: "徐若瑄")
]
let messages = songs.map { song  in
    "\(song.singer)唱著\(song.name)"
}
print(messages)

結果

然而當 closure 的程式超過一行時，回傳型別省略將造成問題，出現錯誤 Generic parameter ‘T’ could not be inferred。因為它沒那麼聰明，無法猜出 generic type T 的型別。

此問題的解法有兩種:

  • 解法1: 描述 closure 的回傳型別

明白地寫出 -> String，讓 Swift 知道 T 是 String。

let messages = songs.map { song -> String in
    let message = "\(song.singer)唱著\(song.name)"
    print(message)
    return message
}
  • 解法2: 宣告變數的型別

宣告 messages 的型別是 [String]，因此 Swift 將知道 map 的 generic type T 是 String。

let messages: [String] = songs.map { song in
    let message = "\(song.singer)唱著\(song.name)"
    print(message)
    return message
}

值得注意的，如果想採用 $0 的參數省略語法，只能使用解法 2，因為此時我們無法在 closure 的 { } 裡描述回傳型別。

let messages: [String] = songs.map {
    let message = "\($0.singer)唱著\($0.name)"
    print(message)
    return message
}
