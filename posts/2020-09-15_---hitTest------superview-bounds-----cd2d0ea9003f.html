當我們點選 iOS App 畫面上的元件時，正常情況下超出 superview 範圍(bounds)的元件是不能點選的，例如下圖 X button 只有在黃色 view 裡的範圍可以點擊。

定義 function hitTest(_:with:) 決定收到 touch 事件的元件

class DemoView: UIView {
    
    @IBOutlet weak var closeButton: UIButton!

    override func hitTest(_ point: CGPoint, with event: UIEvent?) -> UIView? {
        let view = super.hitTest(point, with: event)
        print(view)
        return view
    }
}

function hitTest 會回傳 view，它回傳的 view 代表處理 touch 事件的元件。super.hitTest(point, with: event) 將從 DemoView 元件自己 & 它的 subview 裡找出應該接收 touch 事件的元件回傳，比方以下是我們實際測試後列印的結果。

當我們點選黃色 view 時，hitTest 回傳 DemoView。當我們點選 button 在黃色 view 裡的範圍時，hitTest 回傳 UIButton。然而當我們點選 button 超出黃色 view 的範圍時，hitTest 將回傳 nil，因此沒有元件可以收到 touch 事件，難怪此時 button 點選毫無反應。

接著讓我們稍稍修改 hitTest，讓超出 superview bounds 的 button 可以點選吧。

override func hitTest(_ point: CGPoint, with event: UIEvent?) -> UIView? {
        
    let pointForButton = self.convert(point, to: closeButton)
        
    if closeButton.bounds.contains(pointForButton) {
        return closeButton.hitTest(pointForButton, with: event)
    }
        
    return super.hitTest(point, with: event)
}

說明。

let pointForButton = self.convert(point, to: closeButton)

將點選的座標轉換為以 button 左上角為原點的座標。關於 function convert(_:to:) 的相關說明，可參考以下連結。

if closeButton.bounds.contains(pointForButton) {
    return closeButton.hitTest(pointForButton, with: event)

當點選的座標位在 button 範圍內時，表示點選的是 button，回傳 closeButton.hitTest(pointForButton, with: event)。因此就算我們點選 button 超出黃色 view 的範圍，button 一樣可以收到 touch 事件，因為此時 DemoView 的 hitTest(_:with:) 將回傳 button 元件，而不是 nil。
