利用 static let shared & private init 建立共用物件(globally shared objects)

貼心的 iOS SDK 提供許多方便的共用物件，讓我們開發功能時能直接使用，不用麻煩地另外建立物件，比方連線網路的 URLSession.shared、管理檔案的 FileManager.default。

我們自己定義的類別其實也可以跟 SDK 學習，模仿它將整個 App 只需要一個的物件宣告成共用物件，比方串接 API 的物件，提供假資料的物件等。

宣告 static let shared

以串接電影 API 的物件為例，以下例子在 MovieService 宣告 static let shared，之後串接 API 時只要透過 MovieService.shared 即可呼叫 fetchNowPlayingMovies & fetchUpcomingMovies。

class MovieService {
    static let shared = MovieService()
    
    func fetchNowPlayingMovies() async {
    }

    func fetchUpcomingMovies() async {        
    }
}

定義 private init

有時我們還會將 init 宣告為 private，它將帶來以下幾個好處。

  • 確保類別只有一個實例。

以下例子在串接 API 時只能透過 MovieService.shared。

class MovieService {
    static let shared = MovieService()
    
    private init() {
    }
    
    func fetchNowPlayingMovies() async {
    }
    
    func fetchUpcomingMovies() async {
    }
}

此時另外以 MovieService( ) 生成物件將產生錯誤，顯示 initializer is inaccessible due to ‘private’ protection level。

  • 確保生成的物件不會有問題。

以下例子我們希望 MovieService 的 isFetching 預設為 false，宣告為 private 的 init 將防止使用者用 MovieService(isFetching: true) 建立物件。

class MovieService {
    private(set) var isFetching: Bool
    static let shared = MovieService(isFetching: false)
    
    private init(isFetching: Bool) {
        self.isFetching = isFetching
    }
    
    func fetchNowPlayingMovies() async {
    }
    
    func fetchUpcomingMovies() async {
    }
}

Apple 官方範例

Apple 的 SwiftData 範例利用 static let shared & private init 建立 SampleData 的共用物件。

import Foundation
import SwiftData


@MainActor
class SampleData {
    static let shared = SampleData()
    
    let modelContainer: ModelContainer
    
    var context: ModelContext {
        modelContainer.mainContext
    }
    
    private init() {
        let schema = Schema([
            Movie.self,
        ])
        let modelConfiguration = ModelConfiguration(schema: schema, isStoredInMemoryOnly: true)
        
        do {
            modelContainer = try ModelContainer(for: schema, configurations: [modelConfiguration])
        } catch {
            fatalError("Could not create ModelContainer: \(error)")
        }
    }
}
