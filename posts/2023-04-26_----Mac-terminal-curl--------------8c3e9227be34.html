為什麼 Mac terminal curl 指令的網址通常要加雙引號

在 Mac 開發程式時我們時常利用 curl 指令測試 API 資料的抓取，不過在輸入網址時要特別注意，通常要像以下例子一樣加上雙引號。

curl "https://itunes.apple.com/search?term=apple"

剛剛的例子若是沒有將網址包在雙引號將遇到問題，比方我們在 terminal 輸入以下指令。

curl https://itunes.apple.com/search?term=apple

它將顯示以下的錯誤訊息。

zsh: no matches found: https://itunes.apple.com/search?term=apple

這是因為 Mac 的 terminal 預設採用的 shell 是 zsh ，當網址裡有 ?，& 等特殊符號時，它會無法解析。因此我們要將網址包在雙引號裡，讓它可以正確解析網址。

ps: 若是 shell 採用 bash，它將可以處理不加雙引號的網址。在 terminal 輸入以下指令再重開 terminal 可切換成 bash 模式。

chsh -s /bin/bash

輸入以下指令再重開 terminal 可切換回 zsh。

chsh -s /bin/zsh
