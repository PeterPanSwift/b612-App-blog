SwiftUI 將多個元件組合的 Group

SwiftUI 提供了擅長將多個元件組合的 Group，接下來讓我們認識它主要應用的 2 個地方:

SwiftUI 將多個元件組合的 Group

Group | Apple Developer Documentation
A type that collects multiple instances of a content type - like views, scenes, or commands - into a single unit.developer.apple.com

SwiftUI 提供了擅長將多個元件組合的 Group，它可以將多個 view 組合在一起而且不影響排版，接下來讓我們認識它主要應用的 3 個地方。

  • 將 modifier 作用在多個元件。
  • 作用在條件判斷區塊。
  • 解決 Extra argument in call 問題(舊版)。

將 modifier 作用在多個元件

當我們從 Group 呼叫 modifier 時，modifier 將套用到 Group { } 裡的每個元件，我們不用再辛苦地從每個元件呼叫 modifier。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.littlePrince)
                .resizable()
                .scaledToFit()
            Group {
                Text("But the eyes are blind. One must look with the heart.")
                Text("Only the children know what they are looking for.")
            }
            .font(.title)
            .foregroundStyle(.blue)
            .background(Color.yellow)
        }
    }
}

作用在條件判斷區塊

設計 SwiftUI 畫面時，我們有時會用 if else，optional binding、switch 等條件判斷式控制呈現的內容。

比方以下例子，有資料時顯示 List，無資料時顯示 ContentUnavailableView。然而這兩種情況我們想套用一樣的 foregroundStyle 和 navigationTitle，造成同樣的程式要寫兩次。

struct ContentView: View {
    var names = ["小王子", "玫瑰", "狐狸"]
    
    var body: some View {
        NavigationStack {
            if names.isEmpty {
                ContentUnavailableView("無資料", systemImage: "book.circle.fill")
                    .foregroundStyle(.red)
                    .navigationTitle("小王子")
            } else {
                List(names, id: \.self) { name in
                    Text(name)
                        .foregroundStyle(.red)
                }
                .navigationTitle("小王子")
            }
        }
    }
}

用 Group 包含條件判斷的區塊可以簡化程式，此時共同的 modifier 加在 Group 即可。

struct ContentView: View {
    var names = ["小王子", "玫瑰", "狐狸"]
    
    var body: some View {
        NavigationStack {
            Group {
                if names.isEmpty {
                    ContentUnavailableView("無資料", systemImage: "book.circle.fill")
                } else {
                    List(names, id: \.self) { name in
                        Text(name)
                    }
                }
            }
            .foregroundStyle(.red)
            .navigationTitle("小王子")
        }
    }
}

解決 Extra argument in call 問題(舊版)

從 Xcode 15 開始，搭配 Swift 5.9 的 ViewBuilder 不再有元件數量的限制。

SwiftUI ViewBuilder 不再有 10 個 view 的限制了 ~
從 Xcode 15 開始，隨著 Swift 5.9 推出的 Value and Type Parameter Packs， SwiftUI ViewBuilder 不再有 10 個 view 的限制了。medium.com

不過以下程式在舊版的 Xcode 會出現錯誤 Extra argument in call，因為舊版的 ViewBuilder 有 10 個 view 的限制。

struct ContentView: View {
    var body: some View {
        VStack {
            Text("不懂")
            Text("江南")
            Text("害怕")
            Text("一千年以後")
            Text("簡簡單單")
            Text("原來")
            Text("西界")
            Text("小酒窩")
            Text("她說")
            Text("愛笑的眼睛")
            Text("不潮不用花錢")
        }
    }
}

利用 Group 我們可將多個元件組合成一個元件，讓 ViewBuilder 的元件數量不會超過上限十個。因此我們可用 Group 組合十個 Text ，然後將它跟另一個 Text 放在 VStack { } 裡，如此 VStack 的 { } 裡只有兩個元件，沒有超過十個的限制。

struct ContentView: View {
    var body: some View {
        VStack {
            Group {
                Text("不懂")
                Text("江南")
                Text("害怕")
                Text("一千年以後")
                Text("簡簡單單")
                Text("原來")
                Text("西界")
                Text("小酒窩")
                Text("她說")
                Text("愛笑的眼睛")
            }
            Text("不潮不用花錢")
        }
    }
}

有了 Group，我們再也不用擔心數量問題，我們甚至可以在 Group 裡包 Group，比方以下例子。

struct ContentView: View {
    var body: some View {
        VStack {
            Group {
                Group {
                    Text("不懂")
                    Text("江南")
                    Group {
                        Text("怎樣")
                        Text("愛瘋了")
                        Text("街角的祝福")
                    }
                }
            }
        }
    }
}
