如何自行研究，學會使用 UIPickerView

從課堂上 table view 的例子，我們已經對 protocol，data source 和 delegate 的概念有了一定程度的認識，了解了程式世界跟現實世界一樣，請人幫忙都很重要。而 iOS SDK 其實大量使用這樣的技巧，許多 UI 元件都是透過 data source 的幫忙呈現內容，透過 delegate 的幫忙實現功能。因此現在你們已經有能力自行研究，學會這些 UI 元件的使用，接下來彼得潘就以 UIPickerView 為例子，介紹學習的幾個方法:

方法1: google 
雖然 google 是 Apple 的敵人，但它還是有點利用價值。直接 google 的確很快，不過最好不是單純把別人的程式碼複製貼上，如果可以找一些教學文章閱讀理解背後的原理會更好，比方搜尋 UIPikcerView swift tutorial。

方法2. 從 iOS SDK 找答案。
因為 picker 和 table 類似，可以猜想 picker 也是透過 data source 的幫忙呈現內容，因此關鍵就藏在 dataSource 的型別。

也有你會猜到型別名稱是 UIPickerViewDataSource，Apple 在這部分的取名還滿好猜的，常常都是 UI 元件的類別名加上 DataSource 或 Delegate。就算猜不到，你可以按 cmd + shift + o，打開 open quickly 視窗後，輸入 UIPickerView，跳到 UIPickerView 裡找到 dataSource 的型別。

open class UIPickerView : UIView, NSCoding {
weak open var dataSource: UIPickerViewDataSource? // default is nil. weak reference

知道型別是 UIPickerViewDataSource 後，問題就簡單了，決定 picker 內容的，應該就是 UIPickerViewDataSource 裡宣告的 function。於是你可以查詢 UIPickerViewDataSource 裡有哪些 function，這些應該就是你在遵從 protocol UIPickerViewDataSource 的 controller 裡要定義的 function。

public protocol UIPickerViewDataSource : NSObjectProtocol {
// returns the number of ‘columns’ to display.
@available(iOS 2.0, *)
public func numberOfComponents(in pickerView: UIPickerView) -> Int
// returns the # of rows in each component..
@available(iOS 2.0, *)
public func pickerView(_ pickerView: UIPickerView, numberOfRowsInComponent component: Int) -> Int
}

當你不明白 function 的意思和作用時，你可以將滑鼠游標移到 function 上，按住 option 鍵後按左鍵，即可看到 function 的說明。

方法3: 問彼得潘
FB 或 LINE 問彼得潘當然也可以，不過這應該是最後一招，如果能試著自己研究出來，得到的收獲和成長將會更多。
