自動將 asset 的圖片和顏色變成 ImageResource 和 ColorResource 的型別屬性 — Xcode 15 & 16 的新功能

從 Xcode 15 開始，我們可以更方便地從程式生成圖片和顏色了。Xcode 會自動將 asset 的圖片和顏色變成 ImageResource 和 ColorResource 的型別屬性，我們不用再擔心打錯圖片和顏色的名字，而 Xcode 16 則更進一步在自動完成的提示顯示圖片和顏色的縮圖(Asset thumbnail completions)。

以下我們使用 asset 裡的 baby pink 和 peter 說明。

我們想要呈現搭配粉紅色背景的 peter。

在舊版的 Xcode，我們必須手動輸入圖片和顏色名字的字串，以下分別為 SwiftUI & UIKit 的寫法。

  • SwiftUI
struct ContentView: View {
    var body: some View {
        VStack {
            Image("peter")
                .background(Color("baby pink"))
        }
        .padding()
    }
}
  • UIKit
class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        // Do any additional setup after loading the view.
        
        let image = UIImage(named: "peter")
        let imageView = UIImageView(image: image)
        imageView.backgroundColor = UIColor(named: "baby pink")
        view.addSubview(imageView)
    }
}

將 asset 的圖片和顏色變成型別屬性

從 Xcode 15 開始，Build Settings 的 Generate Asset Symbols 預設為 Yes，因此 Xcode 會自動將 asset 的圖片和顏色變成 ImageResource 和 ColorResource 的型別屬性。

以下分別為 SwiftUI & UIKit 的例子。

  • SwiftUI

產生 Image 時可傳入型別 ImageResource 的資料。

輸入 .p 時，自動完成出現了 peter 的選項。

產生 Color 時可傳入型別 ColorResource 的資料，在此我們傳入 .babyPink。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.peter)
                .background(Color(.babyPink))
        }
        .padding()
    }
}

ps: 以下為另一種產生 ImageResource & ColorResource 的方法，參數 name 傳入圖片或顏色的名字，bundle 傳入 .main。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(ImageResource(name: "peter", bundle: .main))
                .background(Color(ColorResource(name: "baby pink", bundle: .main)))
        }
        .padding()
    }
}
  • UIKit

產生 UIImage 時可傳入型別 ImageResource 的資料。

輸入 .pet 時，自動完成出現了 peter 的選項。

產生 UIColor 時可傳入型別 ColorResource 的資料，在此我們傳入 .babyPink。

class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        // Do any additional setup after loading the view.
        
        let image = UIImage(resource: .peter)
        let imageView = UIImageView(image: image)
        imageView.backgroundColor = UIColor(resource: .babyPink)
        view.addSubview(imageView)
    }
}

方便生成圖片和顏色的 extension

由於 Generate Swift Asset Symbol Extensions 預設為 Yes，Xcode 還會自動定義方便生成圖片和顏色的 extension。

以下分別為 SwiftUI & UIKit 的例子。

  • SwiftUI

Xcode 自動將 asset 的顏色定義成 Color 的 extension，不過圖片的部分倒是沒有自動定義。

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.peter)
                .background(.babyPink)
        }
        .padding()
    }
}
  • UIKit

Xcode 自動將 asset 的圖片和顏色定義成 UIImage 和 UIColor 的 extension。

class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        // Do any additional setup after loading the view.
        
        let imageView = UIImageView(image: .peter)
        imageView.backgroundColor = .babyPink
        view.addSubview(imageView)
    }
}

asset 的圖片和顏色名字

asset 的圖片和顏色名字支援各種語言，輸入中文或日文也不是問題。不過有以下幾點要注意。

  • 產生的型別屬性會變成 lowerCamelCase，所以開頭會小寫，空白會被移除。

範例:

baby pink 變成 babyPink，Peter 變成 peter。

  • 圖片的名字若出現 image / Image，產生的型別屬性會移除 image / Image。

範例:

peter image 和 peterImage 都會變成 peter。

  • 顏色的名字若出現 color / Color，產生的型別屬性會移除 color / Color。
  • 名字不要用數字開頭。

雖然不會有錯誤，但是產生的型別屬性開頭會加上有點礙眼的底線。

範例:

123 變成 _123。

點選圖片 / 顏色名字跳到 assets

在圖片 / 顏色名字上按住 cmd 鍵，點選後將跳到 assets 的圖片 / 顏色，方便我們查看圖片 / 顏色的模樣。

比方下面三行的 .rabbit 按住 cmd 鍵，點選後將跳到 assets 的圖片。

Image(.rabbit)
UIImage(resource: .rabbit)
UIImageView(image: .rabbit)

相容舊版的 iOS

此功能必須搭配 Xcode 15 以上的版本開發，不過令人開心的，它不限於 iOS 17，可相容舊版的 iOS。以下是相關的程式定義。

#if canImport(UIKit)
@available(iOS 11.0, tvOS 11.0, *)
@available(watchOS, unavailable)
extension UIKit.UIColor {

    /// Initialize a `UIColor` with a color resource.
    convenience init(resource: ColorResource) {
#if !os(watchOS)
        self.init(named: resource.name, in: resource.bundle, compatibleWith: nil)!
#else
        self.init()
#endif
    }

}

@available(iOS 11.0, tvOS 11.0, *)
@available(watchOS, unavailable)
extension UIKit.UIImage {

    /// Initialize a `UIImage` with an image resource.
    convenience init(resource: ImageResource) {
#if !os(watchOS)
        self.init(named: resource.name, in: resource.bundle, compatibleWith: nil)!
#else
        self.init()
#endif
    }

}
#endif

#if canImport(SwiftUI)
@available(iOS 13.0, macOS 10.15, tvOS 13.0, watchOS 6.0, *)
extension SwiftUI.Color {

    /// Initialize a `Color` with a color resource.
    init(_ resource: ColorResource) {
        self.init(resource.name, bundle: resource.bundle)
    }

}

@available(iOS 13.0, macOS 10.15, tvOS 13.0, watchOS 6.0, *)
extension SwiftUI.Image {

    /// Initialize an `Image` with an image resource.
    init(_ resource: ImageResource) {
        self.init(resource.name, bundle: resource.bundle)
    }

}
#endif
