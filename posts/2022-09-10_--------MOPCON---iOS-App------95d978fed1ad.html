點選 Mopcon.xcworkspace 打開專案

移除 GoogleService-Info.plist

下載的專案少了 GoogleService-Info.plist，造成可怕的紅色錯誤。由於我們測試時不會用到 Firebase 的功能，因此請直接刪除下圖的 GoogleService-Info.plist。

刪除 AppDelegate.swift 的 FirebaseApp.configure()

由於少了 GoogleService-Info.plist，因此要記得刪除 FirebaseApp.configure()，否則會造成程式閃退。

FirebaseApp.configure()

修正模擬器無法執行 App 的錯誤

現在 App 已經可以在實機測試，不過如果 Mac 是 M1 或 M2，套件 FirebaseAnalytics 將害我們無法在模擬器執行 App。

此問題可參考以下連結修正。請先將專案關閉，將 Xcode 設為 Rosetta 模式，然後再打開 Mopcon.xcworkspace。

模擬器無法執行 App 的 architecture arm64 問題
開發 iOS App 時，我們時常會加入套件快速實現某些功能。不過有時套件也會造成一些問題，比方使用 CocoaPods 安裝 GoogleMaps 或 GooglePlaces 套件後，App 將無法在模擬器執行，出現以下錯誤訊息。medium.com

打開專案後，點選 Product > Build 重新生成 App。

執行 MOPCON App

現在 App 已可順利啟動在最新最潮的 iPhone 14 Pro 模擬器，不過它顯示的是過時的 2020 資料。

抓取 MOPCON 2022 的資料

原本 App 抓取的資料是 MOPCON 2020 的資料。如下圖所示，API 的網址都是 api/2020。

我們想抓取 MOPCON 2022 的資料，因此利用 Find and Replace in Workspace，將 api/2020 更新成 api/2022。

點選 Replace All。

現在 API 網址全都更新成 api/2022，可以抓取 MOPCON 2022 的資料。

值得注意的，由於 api/2020/initial 和 api/2022/initial 的 JSON 格式不同，api/2022/initial 將造成 JSON 解析失敗，因此請將 InitialAPI.swift 裡的 API 網址改回 api/2020。

由於 API /api/2022/session 回傳的 JSON 資料有缺少，會造成 JSON 解析失敗，請將 SessionObject.swift 的 struct Room 做以下更新。為了讓修改不影響其它地方讀取 Room 的 property，我們將 JSON 對應的 property 加上底線，然後宣告 computed property 方便其它地方讀取，比方 _startedAt & startedAt 的例子。

struct Room: Codable {
    
    let topic: String
    
    let topicEn: String
    
    let summary: String
    
    let summaryEn: String
    
    let isKeynote: Bool
    
    let isOnline: Bool
    
    let _startedAt: Int?
    var startedAt: Int {
        _startedAt ?? 0
    }
    let _endedAt: Int?
    var endedAt: Int {
        _endedAt ?? 0
    }
    
    let room: String
    
    let _floor: String?
    
    var floor: String {
        _floor ?? ""
    }
    
    let _sponsorId: Int?
    var sponsorId: Int {
        _sponsorId ?? 0
    }
    
    let recordable: Bool
    
    let level: String
    
    let sessionId: Int
    
    let _communityPartner: String?
    var communityPartner: String {
        _communityPartner ?? ""
    }
    
    let tags: [Tag]
    
    let sponsorInfo: SponsorInfo?
    
    let _speakers: [SessionSpeaker]?
    var speakers: [SessionSpeaker] {
        _speakers ?? []
    }
    
    var isLiked: Bool = false
    
    enum CodingKeys: String, CodingKey {
        
        case topic, summary, room, recordable, level, tags
        case _floor = "floor"
        case _speakers = "speakers"
        
        case topicEn = "topic_e"
        
        case summaryEn = "summary_e"
        
        case isKeynote = "is_keynote"
        
        case isOnline = "is_online"
        
        case _startedAt = "started_at"
        
        case _endedAt = "ended_at"
        
        case _sponsorId = "sponsor_id"
        
        case sessionId = "session_id"
        
        case sponsorInfo = "sponsor_info"
        
        case _communityPartner = "community_partner"
    }
}

另外從 API 回傳的講者資訊，我們發現講者圖片的格式如下:

api/2022/speaker/images/speaker_214

為了抓取講者圖片，我們還要調整 SponsorObject.swift 的 SpeakerImage，讓圖片加上 MPConstant.baseURL。(ps: baseURL 是 https://mopcon.org)

struct SpeakerImage: Codable {
    
    let web: String
    
    let _mobile: String
    
    var mobile: String {
        "\(MPConstant.baseURL)/\(_mobile)"
    }
    
    enum CodingKeys: String, CodingKey {
        case web
        case _mobile = "mobile"
    }
}
