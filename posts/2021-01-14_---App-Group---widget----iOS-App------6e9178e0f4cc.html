利用 App Group 讓 widget 跟 iOS App 共享資料

App 跟它搭配的 widget 雖然有親密的關係，不過彼此之間就像隔著一道牆的戀人，無法輕易存取對方的資料。

想打破牆讓雙方共享資料有可能嗎 ？其實還是做得到的，比方搭配專門讓不同 App 共享資料的 App Group。接下來我們將一步步示範如何透過 App Group 讓 widget 存取 App 存入 AppStorage 的資料。

在 App & widget 設定 App Group

參考以下連結，在 App & widget 加入 App Group，綁定同一個 group id，方便之後 App & widget 共享資料。

讓不同 App 共享資料的 App Group
一般的情況下，iOS App 只能存取自己資料夾下的東西，無法存取其它 App 的內容。Apple 這麼做主要是為了安全，比方彼得潘如果不小心裝了虎克船長開發的邪惡 App，它也無法讀取我在 LINE 裡傳給溫蒂的私密照片。medium.com

比方以下的 App WidgetDemo & widget ShareDataWidgetExtension 都設定了 group id group.com.peter.WidgetDemo。

從 App 將資料存入 AppStorage 宣告的變數

import SwiftUI
import WidgetKit
struct ContentView: View {
    @State private var name = ""
    @AppStorage("zombie", store: UserDefaults(suiteName: "group.com.peter.WidgetDemo")) var zombie = ""
    var body: some View {
        VStack {
            HStack {
                TextField("殭屍的名字", text: $name)
                    .textFieldStyle(.roundedBorder)
                Button("Save") {
                    zombie = name
                    WidgetCenter.shared.reloadAllTimelines()
                }
            }
            Image("沒人會特地去殺殭屍")
                .resizable()
                .scaledToFit()
        }
        .padding()
    }
}

說明:

@AppStorage("zombie", store: UserDefaults(suiteName: "group.com.peter.WidgetDemo")) var zombie = ""

宣告 AppStorage 變數時，在參數 store 裡傳入 UserDefaults，suiteName 傳入 group id，到時候 widget 宣告 AppStorage 變數時只要搭配相同的 key & group id，即可存取同一份資料。在我們的例子裡，key 是字串 zombie，group id 是 group.com.peter.WidgetDemo。

zombie = name
WidgetCenter.shared.reloadAllTimelines()

將資料存入 AppStorage 變數時，記得呼叫 WidgetCenter.shared.reloadAllTimelines，如此 widget 才能更新讀取 AppStorage 裡儲存的新資料。

從 widget 讀取 AppStorage 裡共享的資料

struct ShareDataWidgetEntryView : View {
    var entry: Provider.Entry
    @AppStorage("zombie", store: UserDefaults(suiteName: "group.com.peter.WidgetDemo")) var zombie = ""
    
    var body: some View {
        
        Text(zombie != "" ? "\(zombie) 就在你身邊" : "殭屍尚未出現")
    }
}

假設 widget 顯示的 view 是 ShareDataWidgetEntryView，我們只關心殭屍的名字，所以宣告 AppStorage 存取共享的殭屍名字，然後用 Text 顯示。

定義 widget 的 getTimeline function

widget 利用 getTimeline 加入要顯示的資料，我們定義以下程式簡單地加入資料。之後 App 在 AppStorage 存入殭屍名字後，將呼叫 reloadAllTimelines 觸發 widget 更新，觸發 widget 執行 getTimeline。

func getTimeline(in context: Context, completion: @escaping (Timeline<Entry>) -> ()) {
        var entries: [SimpleEntry] = []
        let currentDate = Date()
        let entry = SimpleEntry(date: currentDate)
        entries.append(entry)
        let timeline = Timeline(entries: entries, policy: .atEnd)
        completion(timeline)
}

測試

尚未儲存殭屍名字時，widget 顯示殭屍尚未出現。

從 App 輸入史上最有名的殭屍 Dracula，然後按下 Save。
