SwiftUI 的 modifier 順序問題

問題 1: 結果不一樣

由於 modifier 會依原本的元件調整生成新的 view，modifier 串聯的順序將大大影響最後生成的東西。比方我們看看以下例子:

以下程式先利用 background 設定背景顏色，然後再用 rotationEffect 旋轉，最後順利生成藍色背景並旋轉 30 度的文字。

Text("知否？知否？應是綠肥紅瘦")  
   .font(.title) 
   .foregroundColor(.orange) 
   .background(Color.blue)  
   .rotationEffect(Angle(degrees: 30))

但如果我們先用 rotationEffect 得到旋轉的文字，然後再用 background 產生藍色背景的文字呢 ?

Text("知否？知否？應是綠肥紅瘦")  
   .font(.title) 
   .foregroundColor(.orange)    
   .rotationEffect(Angle(degrees: 30))   
   .background(Color.blue)

結果變得十分可怕 ! 因為旋轉後文字的框框依然是水平的，所以接著設定背景色時只有水平的區塊填滿藍色。

var body: some View {
        
        let text = Text("知否？知否？應是綠肥紅瘦")
        print(text)
        return text
}
