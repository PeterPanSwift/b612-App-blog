Swift operator 一邊有空白，一邊沒有的錯誤

撰寫 Swift 程式時，通常對空白沒有太多的限制，我們想寫幾個空白都可以。不過在使用 operator(運算子)時，則要特別注意，否則會造成紅色錯誤。

寫程式常用到的=、+、-、*、/、%、==、!=、>、<、>=、<= 都是 operator，operator 的前後可以都有空白，也可以都沒有空白，因此以下例子是 ok 的。

3+3

3 + 3

if 6 > 3 {
    
}

if 6>3 {
    
}

但是你不能一邊有空白，一邊沒有空白。

以下例子都會造成紅色錯誤。

彼得潘自己是習慣 operator 的前後都加空白，這樣子程式看起來的可讀性也會比較清楚。

3 + 3

if 6 > 3 {
    
}
