像 lazy 一樣偷懶延後設定內容的 global variable & type property

當我們宣告 property 時，利用 lazy 可以讓 property 的內容等到第一次存取時才設定，讓程式的效能更好，例如以下 Baby 的 dog。

class Dog {
   init() {
      print("create dog")
   }
   func run() {
   }
}
class Baby {
   lazy var dog = Dog()
}
var cuteBaby = Baby()

當我們建立 Baby 後，小狗還未生成。要等到第一次存取 dog，比方 cuteBaby.dog.run() 時小狗才會產生。

cuteBaby.dog.run()

其實 Swift 裡有兩種特別的東西不用加上 lazy，與生俱來就有 lazy 的特性，也就是 global variable(constant) & type property。(ps: 就像彼得潘也是與生俱來就有偷懶賴床的天性。)

global variable(constant) 的例子
let name: String = {
   print("取名為周冬雨")
   return "周冬雨"
}()
class ViewController: UIViewController {
   @IBAction func buttonTapped(_ sender: Any) {
      print(name)
   }
}

name 是 global constant，因此要等到點擊 button，觸發 function buttonTapped，第一次存取 name 時，才會執行 closure { } 的程式。

type property 的例子
class ViewController: UIViewController {
   static let name: String = {
      print("取名為周冬雨")
      return "周冬雨"
   }()
   @IBAction func buttonTapped(_ sender: Any) {
      print(ViewController.name)
   }
}

name 是 type property，因此要等到點擊 button，觸發 function buttonTapped，第一次存取 name 時，才會執行 closure { } 的程式。
