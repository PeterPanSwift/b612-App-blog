利用 CGRect 的 minX，minY，width & height 快速讀取位置大小

開發 iOS App 時，我們時常利用 CGRect 描述畫面上元件的位置大小，它的 origin 表示位置，size 表示大小。

CGRect | Apple Developer Documentation
A structure that contains the location and dimensions of a rectangle.developer.apple.com
var frame = CGRect(x: 10, y: 20, width: 300, height: 400)
frame.origin.x
frame.origin.y
frame.size.width
frame.size.height

如以上例子，我們可用 origin.x & origin.x 讀取 x,y 的座標，size.width & size.height 讀取寬高。

然而其實有更快的寫法，我們可用 CGRect 的 minX & minY 讀取座標，width & height 讀取寬高，省略 origin & size。

frame.minX
frame.minY
frame.width
frame.height

為什麼可以這樣呢 ? 其實 minX 是 CGRect 的 computed property，如下圖所示，{ get } 表示它是 computed property。所以當我們存取 minX 時，它會再另外執行程式，從 origin 讀出 x 呀。

CGRectGetMinX(_:) | Apple Developer Documentation
Returns the smallest value for the x-coordinate of the rectangle.developer.apple.com

關於 computed property，不熟的朋友可參考以下連結。

精於算計的 computed property
在 Swift 的型別定義裡，我們可宣告屬性儲存寶貴的資訊，例如以下例子，真男人 RealMan 類別的 name 和 age 儲存著永遠 18 歲的彼得潘。medium.com

不過有一點要特別注意，minX，minY，width，height 都是只有 get，沒有 set 的 computed property，所以只能讀取，不能修改，修改將出現錯誤 Cannot assign to property: xxx is a get-only property。

想修改的話，還是得勤勞一點，利用 frame 的 origin & size。

frame.origin.x = 0
frame.origin.y = 0
frame.size.width = 0
frame.size.height = 0
