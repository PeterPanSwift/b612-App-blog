SwiftUI Image 的縮放

Image - SwiftUI | Apple Developer Documentation
Creates a labeled image based on a Core Graphics image, that you can use as content for controls.developer.apple.com

當我們利用 SwiftUI 加入 Assets 裡的圖片時，預設它會將圖片以原本的大小顯示。

因此當我們將小彼得潘 smallPeter 從 Library 視窗拖曳到 Preview 畫面時，圖片將完美地置中顯示在中間。

當圖片很大時

然而當圖片很大時，它卻會傻傻地維持原本的尺寸置中顯示，造成我們只能看到圖片中間的區塊，比方以下例子。

struct ContentView : View {
   var body: some View {
      Image("bigPeter")
   }
}

如下圖所示，圖片維持原尺寸置中顯示，由於圖片比 iPhone 螢幕大很多，因此它無法完整顯示，小彼得可愛的臉只露出一半。

利用 resizable 縮放圖片

Image 原本是無法縮放的，我們須透過 modifier resizable() 生出可以縮放改變大小的圖片。

resizable(capInsets:resizingMode:) - Image | Apple Developer Documentation
This documentation contains preliminary information about an API or technology in development. This information is…developer.apple.com

呼叫 resizable 後圖片將可縮放改變大小，它將縮放到佔滿所在的空間，因此在以下的例子它將幾乎佔滿整個螢幕，只留下上方 status bar 的空間。

var body: some View {
   Image("bigPeter")
      .resizable()
}

利用 edgesIgnoringSafeArea 忽略 Safe Area

為何會保留上方 status bar 的空間呢 ? 因為 SwiftUI 預設會認為我們希望元件呈現在 Safe Area 裡，如此才不會被 status bar 的時間電量檔到。不過如果我們還是想利用 Safe Area 之外的空間，不在乎被檔到的話，則可呼叫 modifier edgesIgnoringSafeArea。

var body: some View {
   Image("bigPeter")
      .resizable()
   
      .edgesIgnoringSafeArea(.all)
}

ps: 目前從 Preview 畫面無法看到 edgesIgnoringSafeArea 的效果，必須從模擬器執行 App 後才能看到圖片佔據 status bar。

利用 frame 控制圖片尺寸

一般加了 resizable() 讓圖片可以縮放後，我們還會再呼叫 modifer frame 指定圖片的大小，比方以下將圖片變成 300 * 600。

var body: some View {
   Image("bigPeter")
      .resizable()
      .frame(width: 300, height: 600)
}

要記得沒有呼叫 resizable() ，圖片是無法縮放的，就好像不吃飯體重不會增加一樣。因此以下程式雖然設定 frame 寬高為 300 * 600，圖片卻依然佔滿螢幕不會變小。

var body: some View {
   Image("bigPeter")
      .frame(width: 300, height: 600)
}

讓圖片維持比例的 scaledToFill，scaledToFit & aspectRatio

剛剛我們看到許多圖片縮放的例子，但是它卻有個很大的致命傷。圖片縮放後因為跟原本圖片的比例不同而變形了 !

想要圖片維持原本的比例不變形，在 SwiftUI 有三種 modifier 可實現，scaledToFill，scaledToFit & aspectRatio。

維持比例填滿空間

呼叫 modifier scaledToFill() 得到維持比例並填滿空間的圖片。

var body: some View {
   Image("bigPeter")
      .resizable()
      .scaledToFill()
}

我們也可以使用 modifier aspectRatio，只要在它的參數傳入 .fill 即可。

var body: some View {
   Image("bigPeter")
      .resizable()
      .aspectRatio(contentMode: .fill)
}

此時圖片完美地維持比例佔滿空間，但它為了維持比例填滿空間，將有部分區塊超出畫面，只是因為它在螢幕外所以我們看不到。

讓我們看個更明顯的例子，比方以下例子除了維持比例佔滿還指定大小。

var body: some View {
   Image("bigPeter")
      .resizable()
      .scaledToFill()
      .frame(width: 300, height: 300)
}

我們指定大小為正方形 300 * 300，但 scaledToFill() 將讓圖片除了填滿 300 * 300 的空間，還會有部分超出的區塊露出來。

覺得超出 300 * 300 的部分很礙眼嗎 ? 沒問題，我們可加入 modifier clipped 切除超出的區塊。

var body: some View {
   Image("bigPeter")
      .resizable()
      .scaledToFill()
      .frame(width: 300, height: 300)
      .clipped()
}

維持比例，左右或上下留白

modifier scaledToFit() 也可以維持圖片比例，不過它將製造左右或上下留白的效果。

var body: some View {
   Image("bigPeter")
      .resizable()
      .scaledToFit()
}

同樣的，我們也可以呼叫 modifier aspectRatio，只要改成在它的參數傳入 .fit。

var body: some View {
   Image("bigPeter")
   .resizable()
   .aspectRatio(contentMode: .fit)
}

我們也可以維持比例留白並且指定大小。

var body: some View {
   Image("bigPeter")
      .resizable()
      .scaledToFit()
      .frame(width: 300, height: 300)
}
