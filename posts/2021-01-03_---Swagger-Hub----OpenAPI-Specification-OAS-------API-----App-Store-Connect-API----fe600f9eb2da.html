利用 Swagger Hub 瀏覽 OpenAPI Specification(OAS) & 測試 API — 以 App Store Connect API 為例

開發 iOS App 時，我們時常串接第三方的 API，不過在真正寫程式串接前，我們得先看懂 API 的文件說明。

目前大部分的 RESTful API 的文件都以 OpenAPI Specification(OAS) 格式撰寫，接下來我們將以 App Store Connect API 為例說明如何利用 Swagger Hub 瀏覽 OAS & 測試 API。

OAS 文件的格式是 JSON 或 YAML

OAS 文件的格式有 JSON & YAML 兩種，App Store Connect API 的 OAS 是 JSON 格式。

熟悉 JSON 的朋友可以直接打開檔案閱讀內容，不過這樣實在太累了，因此待會我們將透過 Swagger Hub 瀏覽文件。

連到 Swagger Hub 網站

SwaggerHub | API Design and Documentation with OpenAPI
Accelerating API delivery and quality through standards and collaboration, built on OpenAPI. Create Free Account The…swagger.io

註冊帳號

上傳 OAS 文件

點選側邊欄的 Create New > Import and Document API

點選 UPLOAD FILE

點選 IMPORT OPENAPI。

Cool，Swagger 變出了適合人類閱讀的 API 文件 !

從 Swagger Hub 瀏覽 API

從 Swagger Hub 我們可以舒適地瀏覽 API。

看到有興趣的 API，可進一步點選查看細節，比方以下建立 AppInfoLocalizations 的 request body & response。

從 Swagger Hub 測試 API

Swagger Hub 十分厲害，它的能力可不只是顯示 API 文件，我們還可以直接在上面連線測試 API，比方以下取得 App 價錢的例子。

  • 點選 Try it out
  • 輸入欄位內容，點選 Execute 執行。

不過眼尖的同學也許注意到 API 回傳的結果是可怕的 error ! 別緊張，這不是 Swagger 的錯，問題出在 App Store Connect API 必須加上 token 驗證身份後才能串接。
