Xcode 13 更方便的自動完成(code completion) & 自動 import

Xcode 13 的自動完成變得更方便了，主要包含以下幾點:

  • 搜尋二層的 property 或 method
  • if let & guard let
  • for in
  • switch 搭配 enum
  • 自動 import
  • 打 su 出現父類別 function 的自動完成。
  • function 型別搭配參數名字，呼叫時 Xcode 將自動輸入 closure 接收的參數名

搜尋二層的 property 或 method

struct Cat {
    var name: String
}
struct Woman {
    var cat: Cat
    func sing() {
    }
}
struct Man {
    var girlFriend: Woman
}
let cat = Cat(name: "kitty")
let cuteWoman = Woman(cat: cat)
let superMan = Man(girlFriend: cuteWoman)

在 superMan 後輸入 c，找到 girlFriend.cat。

在 superMan 後輸入 si，找到 girlFriend.sing()。

目前測試最多找到第二層，貓的名字在第三層，因此無法列出 girlFriend.cat.name。

if let & guard let

if let & guard let 的自動完成 — Xcode 13 新功能
Xcode 13 變得更方便了。當我們輸入 guard let 跟 if let 時，只要在 let 後輸入 optional 的名字，比方 name，就可以從選單選擇 name = name 的選項了。medium.com

for in

let books = ["湯姆歷險記", "彼得歷險記", "蘋果歷險記"]

array 的名字是 books，當我們輸入 for b 時，自動完成出現 book in books 的選項。

按 enter 後自動輸入 for book in books { }。

令人佩服的，Xcode 除了程式厲害，也是個英文高手，比方 array 的名字是複數的 cities，它知道單數要變成 city。

let cities = ["台北", "台中", "高雄"]

switch 搭配 enum

enum Animal {
    case cat
    case dog
    case rabbit
}

在 switch 後輸入 a，自動列出 animal { 的選項。

按 enter 後自動輸入 enum 的每個 case。

輸入 s 也會自動列出 self { 的選項，按 enter 後自動輸入 enum Animal 的每個 case。

自動 import

當 Xcode 檢查到不認得的型別時，它將找到需要 import 的 module，幫我們自動 import。

如下圖所示，我們想要遵從 protocol UIViewRepresentable，由於它定義在尚未 import 的 SwiftUI 裡，所以 Xcode 將顯示灰色的驚嘆號，按下 enter 後自動 import SwiftUI。

說明訊息提到 This protocol is defined in SwiftUI. Choosing this completion will import the module.

第三方套件也不是問題，比方輸入 kfi 時自動 import Kingfisher。

在 override 的 function 裡打 su 會出現父類別 function 的自動完成

以下圖為例，想在 function viewWillAppear 裡呼叫 super.viewWillAppear(animated)，只要打 su 加 enter，Xcode 就可以幫我們輸入 super.viewWillAppear(animated)。
