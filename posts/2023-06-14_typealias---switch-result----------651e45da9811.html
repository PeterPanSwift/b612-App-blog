typealias 讓 switch result 的自動完成不見了

開發 iOS App 時，我們時常會搭配 completion handler 和 Result type 回傳 API 抓取的資料，然後再利用 switch 判斷 result 是成功或失敗。

這樣子的程式寫起來滿簡單的，因為 Xcode 會幫我們自動寫出 result 成功跟失敗的 case。不過最近為了簡化程式，彼得潘進一步用 typealias 幫 completion handler 定義新的型別名後，卻發現它會影響 switch result 的自動完成，造成小小的不方便。

以下我們做一個實驗說明。

  • 參數 completion 的型別以 (Result<Int, Error>)->Void 宣告。
class ApiManager {
    static let shared = ApiManager()
    
    func test(completion: (Result<Int, Error>)->Void) {
        let number = Int.random(in: 1...100)
        completion(.success(number))
    }
}

自動完成順利地出現了 result { …，按下 enter 後 Xcode 自動幫我們輸入 success 和 failure 的 case。

  • 參數 completion 的型別以 CompletionHandler 宣告。

CompletionHandler 是我們另外用 typealias 定義的型別名稱，對應 (Result<Int, Error>)->Void。

class ApiManager {
    typealias CompletionHandler = (Result<Int, Error>)->Void
    static let shared = ApiManager()
    
    func test(completion: CompletionHandler) {
        let number = Int.random(in: 1...100)
        completion(.success(number))
    }
}

自動完成不再出現 result { …。

我們還是可以請 Xcode 幫忙補上 success 和 failure 的 case，不過要多一步，按下 fix 修正錯誤時才會補上。
