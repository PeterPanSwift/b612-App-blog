Xcode 快速將程式搬到新檔的方法 — Xcode 16 新功能

開發 SwiftUI App 時，我們常常用 Extract Subview 將某段畫面的程式切出來變成另一個 struct，此時我們可選擇讓它跟原來的程式待在同一個檔案，或是將它搬到新的檔案。

將程式搬到新檔的傳統方法是先建立新檔，然後再將程式搬過去，不過最近彼得潘剛看完 WWDC24 的 Xcode essentials，學到 Xcode 16 有 4 招更快的方法，而且前 3 個方法還會聰明地依據程式內容設定檔名，不用自己輸入呢。

方法 1 可參考剛剛連結的說明，以下我們用顯示 Apple logo 的 Image 當例子說明方法 2 ~ 方法 4，先將 Image 切出來變成 struct AppleLogo。

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "apple.logo")
                .resizable()
                .scaledToFit()
                .frame(width: 100, height: 100)
                .padding()
                .foregroundStyle(.orange.gradient)
                .background(.yellow)
                .clipShape(Circle())
            Text("Apple")
                .font(.largeTitle)
        }
        .padding()
    }
}

選取 AppleLogo 程式後按 cmd + x 剪下。

之後使用以下三個方法即可快速將程式搬到新檔。

方法 2: 點選 Project navigator，按下 cmd + v 貼上程式

因為程式是 struct AppleLogo 的定義，因此檔名自動設為 AppleLogo。

輸入 import SwiftUI 修正問題。

import SwiftUI

struct AppleLogo: View {
    var body: some View {
        Image(systemName: "apple.logo")
            .resizable()
            .scaledToFit()
            .frame(width: 100, height: 100)
            .padding()
            .foregroundStyle(.orange.gradient)
            .background(.yellow)
            .clipShape(Circle())
    }
}

方法 3: 在 Project navigator 按出右鍵選單，點選 New File from Clipboard

因為程式是 struct AppleLogo 的定義，因此檔名自動設為 AppleLogo。

輸入 import SwiftUI 修正問題。

方法 4: 用 New Empty File 建立新檔再貼上程式

在 Project navigator 按出右鍵選單，點選 New Empty File。

輸入檔名。

貼上程式和 import SwiftUI。

import SwiftUI

struct AppleLogo: View {
    var body: some View {
        Image(systemName: "apple.logo")
            .resizable()
            .scaledToFit()
            .frame(width: 100, height: 100)
            .padding()
            .foregroundStyle(.orange.gradient)
            .background(.yellow)
            .clipShape(Circle())
    }
}
