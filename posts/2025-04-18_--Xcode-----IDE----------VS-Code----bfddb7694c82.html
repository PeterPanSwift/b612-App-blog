從 Xcode 叫其它 IDE 打開專案 — 以 VS Code 為例

彼得潘開發 iOS App 時，除了使用 Xcode，也常搭配其它內建方便 AI 功能的 IDE。

由於從 Xcode 開啟專案後，再用其它 IDE 打開專案是常做的小動作，因此彼得潘很想加快這個流程。跟 AI 討論後，終於找到快速的方法，透過設定 Behavior 即可從 Xcode 一鍵叫其它 IDE 打開專案。

以下我們以 VS Code 為例說明。

  • 撰寫叫 VS Code 打開專案的 openVSCode.sh。

寫法 1

#!/bin/sh

open -a "Visual Studio Code" "${XcodeProjectPath%/*}"

寫法 2

#!/bin/sh

open -a "Visual Studio Code" "$(dirname "$XcodeProjectPath")"

XcodeProjectPat 取得的是 xcodeproj 的路徑，然而我們想要的是專案的資料夾路徑。因此我們必須用 /* 或 dirname 取得父目錄。

ps: /* 會將最後一個 / 後的字串(也就是 .xcodeproj)砍掉，因此留下的即是專案的資料夾路徑。

open -a 後指定打開資料夾的 App，因此想用 Windsurf、Cursor、Zed 打開也不是問題。

  • 設定檔案的執行權限。
chmod +x openVSCode.sh
  • 新增 Behavior

從 Quick Actions 點選 Edit Behaviors。

點選 Custom。

點選左下的 +。

取名 Open VS Code。

點選左下的 +，點選 Run Script。

點選 Choose Script。

從 Finder 選擇 openVSCode.sh。

從 Key Binding 輸入快速鍵，輸入 cmd + shift + v。

叫 VS Code 打開專案

現在我們可以一鍵呼喚 VS Code 開專案了 ! 當 Xcode 正在編輯某個專案時，按下 cmd + shift + v 即可叫 VS Code 打開專案。
