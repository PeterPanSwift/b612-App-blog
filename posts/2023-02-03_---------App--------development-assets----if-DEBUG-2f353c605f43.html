建立不會加入上架 App 的假資料 — development assets & #if DEBUG

開發 iOS App 時，我們時常會產生一些假資料，方便在 SwiftUI preview 或測試時使用。

然而 App 實際執行時這些資料是用不到的，若能將這些資料從上架的 App 移除，將可進一步縮減 App 的大小。

以下我們介紹兩種移除假資料的方法。

  • 在 development assets 設定假資料。
  • 利用 #if DEBUG 設定假資料。

在 development assets 設定假資料

方便存放測試檔案的 Xcode development assets
開發 iOS App 時，我們時常會在專案裡準備一些方便測試的圖片或 JSON 檔，這樣當後台還不能串接或是彼得潘回 Neverland 過年沒有網路時，依然可以測試 App。medium.com

利用 #if DEBUG 設定假資料

struct Song {
    let name: String
    let singer: String
    let lyrics: String
}

#if DEBUG
extension Song {
    static let sampleSongs = [
        Song(name: "對的時間點", singer: "林俊傑", lyrics: "如果 愛情是場遠程的渦旋 儘管繞著圈子 也要走向前 不離心太遠 我要面朝最藍的 晴天 不脫離軌道 有你在身邊"),
        Song(name: "想見你想見你想見你", singer: "八三夭", lyrics: "想見你 只想見你 未來過去 我只想見你 穿越了 千個萬個 時間線裡 人海裡相依")
    ]

    static let sampleSong = sampleSongs[0]
}
#endif
