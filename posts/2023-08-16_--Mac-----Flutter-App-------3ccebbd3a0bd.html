在 Mac 上準備 Flutter App 的開發環境

開發 Flutter App 前，我們必須先準備好開發環境。以下彼得潘以自己的 Mac 為例，介紹如何在 Mac 上從零開始準備 Flutter App 的開發環境。

  • 安裝 Flutter SDK。
  • 在 PATH 加入 Flutter 的執行路徑。
  • 測試是否已準備好開發 Flutter App 的環境。
  • 修正 Xcode 的 CocoaPods 問題。
  • 修正 Android toolchain 問題。

安裝 Flutter plugin

為了更方便開發 Flutter App，我們最好在 IDE 上安裝 Flutter plugin，它將提供更好的自動完成和除錯功能，而且也能方便啟動 iOS 或 Android 的模擬器執行 Flutter App。

以下分別介紹 VS Code & Android Studio 安裝 Flutter plugin 的方法。

  • VS Code

打開 VS Code，切換到 Extensions 分頁。

輸入 flutter，點選 Install 安裝。

  • Android Studio

打開 Android Studio，切換到 Plugins 分頁。在搜尋框輸入 flutter，點選 Install 安裝。

點選 Accept。

點選 Install。

點選 Restart IDE。

點選 Restart。

安裝 Flutter SDK

  • 方法 1: 從 VS Code 安裝。

啟動 VS Code 後打開 Command Palette(cmd + shift + p)，輸入 flutter，點選 Flutter: New Project。

此時會出現找不到 Flutter SDK 的錯誤，別擔心，點選 Download SDK 即可下載 SDK。

選擇儲存 SDK 的路徑，在此彼得潘點選個人帳號資料夾下的 Developer，然後點選 Clone Flutter。

此時會開始下載 SDK，請稍等片刻，等待是值得的。

點選 Copy SDK path to Clipboard，等下會用到。

  • 方法 2: 下載安裝 Flutter SDK 的 zip 檔。。

參考網頁的說明下載安裝 Flutter SDK 的 zip 檔。

Start building Flutter iOS apps on macOS
Configure your system to develop Flutter mobile apps on macOS and iOS.docs.flutter.dev

在 PATH 加入 Flutter 的執行路徑

為了方便在 terminal 執行 flutter 指令，我們可以在 PATH 加入 Flutter 的執行路徑，方法如下。

打開 Terminal App，輸入以下指令。PATH: 後是 Flutter SDK 的安裝路徑。請貼上剛剛 Copy SDK path 得到的路徑，結尾加上 /bin，比方彼得潘的路徑如下。(ps: 路徑的規則是儲存 SDK 的資料夾加上 /flutter/bin)

echo 'export PATH="$PATH:/Users/shih-yingpan/Developer/flutter/bin"' >> ~/.zshenv

重開 VS Code 和 terminal，之後即可在 VS Code 和 Mac 的 terminal 執行 flutter 指令。

ps: Mac 預設的 shell 是 zsh，因此剛剛的指令會將 Flutter 路徑存在 .zshenv。如果 shell 不是 zsh，可用以下指令將 shell 改成 zsh。

chsh -s /bin/zsh

修正 Xcode 的 CocoaPods 問題

開發 Flutter App 時，我們常常安裝一些第三方的 package。當 package 是純 dart 程式時，不需要用到 CocoaPods。然而當 package 包含 iOS 原生的程式時，通常需要另外透過 CocoaPods 安裝。

ps: CocoaPods 是 iOS 舊的套件管理方法，建議參考以下連結將 Flutter 改成採用新的 SPM 做法 。

讓 Flutter 的 iOS App 使用 SPM 安裝套件
Flutter 最近開始支援 SPM(Swift Package Manager)，我們可以設定 iOS App 使用 SPM 安裝套件，而不是 CocoaPods。medium.com

若執行 Flutter App 時出現以下的 CocoaPods 錯誤，請再安裝管理 iOS App 套件安裝的 CocoaPods。

Launching lib/main.dart on iPhone 16 Pro in debug mode...
Warning: CocoaPods not installed. Skipping pod install.
  CocoaPods is used to retrieve the iOS and macOS platform side's plugin code that responds to your plugin usage on the Dart side.
  Without CocoaPods, plugins will not work on iOS or macOS.
  For more info, see https://flutter.dev/platform-plugins
To install see https://guides.cocoapods.org/using/getting-started.html#installation for instructions.

CocoaPods not installed or not in valid state.
Error launching application on iPhone 15 Pro.
使用 CocoaPods 管理第三方套件
隨著 Xcode 11 的推出，未來開發 iOS App 時最方便的套件管理工具變成 Xcode 內建的 SPM。不過目前還有很多套件尚未支援 SPM，只能透過 Cocoapods…medium.com

安裝 Cocoapods 後再執行 flutter doctor，Xcode 的部分出現了完美的 [✓]。

[✓] Xcode - develop for iOS and macOS (Xcode 26.0)

修正 Android toolchain 問題

輸入 flutter doctor 時，若 Android toolchain 顯示以下問題，可參考以下方法安裝 cmdline-tools 和同意 Android license。

[!] Android toolchain - develop for Android devices (Android SDK version 33.0.1)
    ✗ cmdline-tools component is missing
      Run `path/to/sdkmanager --install "cmdline-tools;latest"`
      See https://developer.android.com/studio/command-line for more details.
    ✗ Android license status unknown.
      Run `flutter doctor --android-licenses` to accept the SDK licenses.
      See https://flutter.dev/docs/get-started/install/macos#android-setup for
      more details.
  • 安裝 cmdline-tools。

啟動 Android Studio，切換到 Projects 分頁，點選右上的三個點，然後點選 SDK Manager。

ps: 如果沒看到右上的三個點，請點選下圖的 More Actions，然後點選 SDK Manager。

勾選 SDK Tools 分頁下的 Android SDK Command-line Tools (latest)，然後點選 OK。

ps: 如果想在 terminal 使用 Android 的常用指令，可在 PATH 加入 Android SDK 的路徑，比方彼得潘輸入的路徑如下。(ps: 路徑的規則是儲存 Android SDK 的資料夾加上 /platform-tools，因此是 /Users/shih-yingpan/Library/Android/sdk/platform-tools)

echo 'export PATH="$PATH:/Users/shih-yingpan/Library/Android/sdk/platform-tools"' >> ~/.zshenv
  • 同意 Android license。

在 Terminal 輸入以下指令。

flutter doctor --android-licenses

輸入 y 同意 license。

安裝 cmdline-tools 和同意 Android license 後，再次執行 flutter doctor，開心地看到滿滿的綠色勾勾和 No issues found。
