利用date(byAdding:value:to:wrappingComponents:) 計算對的時間點

利用 Calendar 的 function date(byAdding:value:to:wrappingComponents:) 我們可以方便地計算日期，再也不怕算錯。

比方彼得潘想追求真命天女，鐵板神算告訴我，20 天後是對的時間點，只要在那天告白就能成功。由於每個月的天數不同，為了怕算錯，彼得潘決定將影響這輩子幸福的重要計算交給永不算錯的 Swift。

import Foundation
let today = Date()
let dateFormatter = DateFormatter()
dateFormatter.dateStyle = .long
let rightTime = Calendar.current.date(byAdding: .day, value: 20, to: today, wrappingComponents: false)
print(dateFormatter.string(from: rightTime!))

今天是 2019/8/26，透過 Calendar 的 date(byAdding:value:to:wrappingComponents:)，我們可以成功算出 20 天後是 2019/9/15。

  • component

傳入時間調整的單位，比方傳入 .day 表示以天為單位，傳入 .month 表示以月為單位。

Calendar.Component

  • value

時間調整的數字，因此 Calendar.current.date(byAdding: .day, value: 20, to: today, wrappingComponents: false) 表示 20 天之後。

  • date

開始的時間點。

  • wrappingComponents

預設為 false，因此我們也可以省略它。

let rightTime = Calendar.current.date(byAdding: .day, value: 20, to: today)

讓我們試試傳入 true 會如何 ?

let rightTime = Calendar.current.date(byAdding: .day, value: 20, to: today, wrappingComponents: true)

此時得到的時間是 2019/8/15。明明現在是 8/26，20 天後怎麼會變成 8/15 呢 ? 因為當 wrappingComponents 為 true 時，它將不會改變更高一層的時間，所以月份將永遠保持 8 月，只會改變幾號的日期。
