UserDefaults 的 integer(forKey:) 回傳 0 的問題

利用 UserDefaults 我們可以方便地存取一些簡單的資料，然而當我們存取的資料型別是 Int，Bool，Float，Double 時，卻會遇到一個特別的問題。因為它們回傳的型別不是 optional，所以當我們傳入的 key 找不到對應的 value 時，它還是會回傳某個預設值。

open func integer(forKey defaultName: String) -> Int

比方找不到 key number 對應的數字時，UserDefaults.standard.integer(forKey: "number") 將回傳 0。因此我們無法判斷它是內容為 0，還是尚未儲存資料。(ps: 如果儲存的內容是字串倒是沒這個問題，因為 string(forKey:) 回傳的型別是 String?，找不到時可以回傳 nil。)

此問題有很多解法，其中一個是利用 register(defaults:) 設定找不到 key 對應的 value 時回傳的預設值。

UserDefaults 的預設值
UserDefaults 很適合用來儲存簡單的資料，比方記錄使用者是否第一次使用 App，上次打開 App 的時間等。medium.com

另一種解法則是透過回傳 Any? 的 object(forKey:) 搭配 as? 轉型判斷。

object(forKey:) - UserDefaults | Apple Developer Documentation
This method searches the domains included in the search list in the order in which they are listed and returns the…developer.apple.com

比方以下例子:

if let number = UserDefaults.standard.object(forKey: "number") as? Int {
   print(number)
} else {
   print("no value")
}

由於 object(forKey:) 回傳的型別是 Any?，所以找不到 key number 對應的 value 時，它將回傳 nil，因此我們可以用 optional binding 判斷是否找到東西。另外由於回傳的型別是 Any?，所以我們要額外用 as? 轉型成資料真正的型別，比方 Int。
