The new build system doesn’t yet support On Demand Resources

有朋友在開發 iOS App 時，出現以下的紅色錯誤，造成 App 無法執行。

The new build system doesn’t yet support On Demand Resources; if you require ODR, the legacy build system can be reenabled in File > Workspace/Project Settings….

問題發生的原因出在 Xcode 10 新的 build system 尚未支援 On Demand Resources。

On-Demand Resources Guide: On-Demand Resources Essentials
Describes how apps use remote on-demand resources.developer.apple.com

ps: On Demand Resources 技術可以讓我們將 App 用到的一些資源(比方圖片)放在 App Store，減少 App 本身的大小，等 App 需要存取時再從 App Store 下載。

App 預設並不會有任何資源設成 On Demand Resources，除非我們另外設定。然而有時我們不小心將某個檔案設成 On Demand Resources，產生剛剛看到的紅色錯誤。比方如下圖所示，Main.storyboard File inspector 頁面的 On Demand Resource Tags 不小心輸入了文字，造成 Main.storyboard 被當成 On Demand Resources。

此時我們將發現 Main.storyboard 出現在 App 的 Resource Tags 頁面。

想解決此問題，有兩種解法:

解法1: 清空 On Demand Resources。

如果確定不想有任何的 On Demand Resources，只要從 App 的 Resource Tags 頁面點選右上角的 X 移除所有的資源，即可修正 Xcode 10 不支援 On Demand Resources 的問題。

解法2: 用舊的 build system。

如果希望某些檔案是另外下載的 On Demand Resources，則可選擇採用舊版的 build system。

(1) 點選 File > Project Settings。

(2) 將 Build System 從 New Build System 改成 Legacy Build System。
