利用 Firebase MCP server 我們可以更方便地透過 AI 管理 Firebase，實現建立 project、管理 user、管理 Cloud Firestore 等功能。

Firebase MCP server | Firebase Documentation
You can use the Firebase MCP server to give AI-powered development tools the ability to work with your Firebase…firebase.google.com

有了 Firebase MCP server，開發串接 Firebase 的 App 變得簡單許多。以開發 Firebase 帳密登入的 iOS App 為例，Firebase MCP server 可建立 Firebase project 和下載 iOS App 需要的 GoogleService-Info.plist，AI 可撰寫程式，我們唯一要做的事只有打開 Firebase 專案的帳密登入功能和用 SPM 安裝 Firebase 套件。

以下彼得潘一步步說明使用 GitHub Copilot & Firebase MCP server 開發 Firebase 登入功能的相關流程。

  • 使用 Firebase CLI 登入 Firebase。
  • 在 VS Code 加入 Firebase MCP server 的相關設定。
  • 建立 Firebase project 和下載 GoogleService-Info.plist。
  • 使用 SPM 加入Firebase iOS SDK。
  • 打開 Firebase 專案的帳密登入功能。
  • 請 AI 撰寫 Firebase 註冊登入程式。

使用 Firebase CLI 登入 Firebase

使用 Firebase MCP server 前必須先用 Firebase CLI 登入 Firebase，相關步驟可參考以下連結。

方便管理 Firebase 的 Firebase CLI(Command Line Interface)
透過 Firebase CLI 我們可以方便地從 Terminal 下指令管理 Firebase 專案和部署網站。以下彼得潘簡單介紹如何在 macOS & Windows 安裝 Firebase CLI。medium.com

在 VS Code 加入 Firebase MCP server 的相關設定

為了讓 VS Code 能使用 Firebase MCP server，我們必須加入它的相關設定。請在 User Settings 加入以下設定。

"mcp": {
  "servers": {
    "firebase": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "firebase-tools@latest", "experimental:mcp"]
    }
  }
}

ps: 其它 IDE 的設定可參考以下連結。

Firebase MCP server | Firebase Documentation
You can use the Firebase MCP server to give AI-powered development tools the ability to work with your Firebase…firebase.google.com

成功設定後，將 GitHub Copilot 切到 Agent 模式，點選下圖的 Select Tools 可看到 MCP Server: firebase 支援的各種指令。

ps: 如果找不到 MCP Server: firebase，請先點選下圖的 New tools available ，然後再點選 Select Tools。

目前 AI 幫 Xcode 專案加入 SPM 管理的套件還是常出問題，因此建議自己從 Xcode 手動加入套件。

我們的目標是帳密登入，因此請設定 FirebaseAuth 的 Add to Target。

從 Package Dependencies 可看到 FirebaseAuth 用到超多套件。
