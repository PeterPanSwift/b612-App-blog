UI 元件預設的 action event

當我們在 storyboard 上連結 IBAction，生成元件的特定事件觸發的 function 時，它將預設綁定元件主要的事件。接下來，就讓我們看看各種不同元件預設綁定的事件吧。

button

預設 Touch Up Inside，表示點選按鈕，按鈕按下後手離開的動作。

事件選單裡的 Primary Action Triggered，代表元件主要的事件，所以對 button 來說，選 Primary Action Triggered 跟選 Touch Up Inside 是一樣的意思。

slider

預設 Value Changed，function 將在 slider 滑動時觸發。

segmented control

預設 Value Changed，function 將在使用者點擊不同的 segment 時觸發。

switch

預設 Value Changed，function 將在 switch 被點擊，由開變關，關變開時觸發。

text field

預設 Editing Did End，function 將在使用者結束 text filed 的文字輸入，游標離開 text field 時觸發。

stepper

預設 Value Changed，function 將在使用者點擊 + 或 -時觸發。

Date Picker

預設 Value Changed，function 將在使用者滑動 picker，picker 滑動停止時觸發。

bar button item

function 將在使用者點選 bar button item 時觸發。

各種 gesture recognizer

偵測到特定手勢時觸發。

其它，比方 image view，label

別傻了，不是每個元件都會有反應呀。像 label 和 image view 都是鐵石心腸，你怎麼按它，它都不會理你的。所以它們只能連 outlet，不能連 action。
