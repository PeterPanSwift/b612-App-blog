@available 裡的 deprecated: 100000.0 是什麼意思

iOS 每年推出新的 SDK，包含許多的 API，為我們帶來厲害的新技術。不過舊的不去，新的不來，因此 Apple 會將一些舊的 API 加上 deprecated，警告我們別再使用。（deprecated 的意思是不建議使用）

比方以下例子，UIWebView 前的 @available(iOS, introduced: 2.0, deprecated: 12.0 說明從 iOS 12 開始，最好別再使用 UIWebView。

@available(iOS, introduced: 2.0, deprecated: 12.0, message: "No longer supported; please adopt WKWebView.")
open class UIWebView : UIView, NSCoding, UIScrollViewDelegate {

若是一意孤行使用 UIWebView，Xcode 將溫柔地產生黃色警告，提醒我們不要使用。

不過彼得潘最近發現一種很特別的 deprecated，它後面的數字是驚人的 100000.0 ！

比方以下 function navigationBarTitle(_:) 的例子。

@available(iOS, introduced: 13.0, deprecated: 100000.0, renamed: "navigationTitle(_:)")  
public func navigationBarTitle(_ title: Text) -> some View

難道是要等 iOS 100000 才不建議使用嗎 ? 應該不太可能，在這之前應該已經世界末日了。它的意思一樣是提醒我們不要使用，只是當我們在程式裡使用時並不會產生黃色警告。

如果我們查詢 Apple 的官方文章，可看到 Apple 明白地告訴我們它 deprecated 了。

至於為什麼是 100000.0 呢？ 秘密藏在 availability.h 裡，裡頭的 API_TO_BE_DEPRECATED 正是 100000。

#ifndef API_TO_BE_DEPRECATED
#define API_TO_BE_DEPRECATED 100000
#endif
