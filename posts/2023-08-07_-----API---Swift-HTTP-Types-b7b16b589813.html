方便串接 API 的 Swift HTTP Types

Apple 最近推出了方便串接 API 的 package Swift HTTP Types，它幫我們定義好常用的 HTTP method 和 header name，讓我們不用再手動輸入字串，而且它的 HTTPRequest 還能搭配我們熟悉的 URLSession 串接 API。

以下彼得潘一步步示範如何使用 Swift HTTP Types 串接 API。

  • 加入 Swift HTTP Types package。
  • HTTPRequest.Method & HTTPField.Name。
  • 產生 HTTPRequest & 串接 API。

加入 package Swift HTTP Types

使用 SPM 加入 Swift HTTP Types。

https://github.com/apple/swift-http-types

HTTPRequest.Method & HTTPField.Name

import HTTPTypes 後，我們可以使用 HTTPRequest.Method 和 HTTPField.Name 設定 HTTP method 和常見的 HTTP header 名稱。

例如以下例子。

import HTTPTypes

let url = URL(string: "https://reqres.in/api/login")!
var request = URLRequest(url: url)
request.httpMethod = HTTPRequest.Method.post.rawValue
request.setValue("application/json", forHTTPHeaderField: HTTPField.Name.contentType.rawName)

產生 HTTPRequest & 串接 API

加入 Swift HTTP Types 後，使用 HTTPRequest 串接 API 會比原本的 URLRequest 方便。以下我們以幾個不同的 API 舉例說明。

  • iTunes API。
https://itunes.apple.com/search?term=swift

產生 HTTPRequest 有以下兩種寫法。

方法 1: 設定 method，scheme，authority 和 path。

let request = HTTPRequest(method: .get, scheme: "https", authority: "itunes.apple.com", path: "/search?term=swift")

方法 2: 設定 URL。

如果剛剛的 path 有需要 URL encode 的字元，建議採用方法 2，因為 iOS 17 開始，生成 URL 時可以自動做 URL encode。

let request = HTTPRequest(url: URL(string: "https://itunes.apple.com/search?term=swift")!)

串接 API有以下兩種寫法。

方法1: async/await。


import HTTPTypes
import HTTPTypesFoundation

func test() async throws {
    let request = HTTPRequest(method: .get, scheme: "https", authority: "itunes.apple.com", path: "/search?term=swift")
    let (responseBody, _) = try await URLSession.shared.data(for: request)
    if let content = String(data: responseBody, encoding: .utf8) {
        print(content)
    }
}

記得要 import HTTPTypesFoundation，它將幫我們進行 HTTPRequest 和 URLRequest 間的轉換，讓我們用熟悉的 URLSession.shared.data 串接 API。

方法2: completion handler。

import HTTPTypes
import HTTPTypesFoundation

func test() {
    let request = HTTPRequest(method: .get, scheme: "https", authority: "itunes.apple.com", path: "/search?term=swift")
    if let request = URLRequest(httpRequest: request) {
        URLSession.shared.dataTask(with: request) { data, response, error in
            if let data,
               let content = String(data: data, encoding: .utf8) {
                print(content)
            }
        }.resume()
    }
}

我們必須自己將 HTTPRequest 變成 URLRequest，然後才能用 URLSession.shared.dataTask 串接 API。

  • ReqRes API。

curl 指令如下。

curl -X 'POST' \
  'https://reqres.in/api/login' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
  "email": "eve.holt@reqres.in",
  "password": "cityslicka"
}'

設定 headerFields 跟利用 URLSession.shared.upload 上傳資料，contentType 和 accept 都是 HTTPField.Name 事先定義好的名字。

struct LoginBody: Encodable {
    let email: String
    let password: String
}

func test() async throws {
    var request = HTTPRequest(method: .post, scheme: "https", authority: "reqres.in", path: "/api/login")
    request.headerFields[.contentType] = "application/json"
    request.headerFields[.accept] = "application/json"
    let loginBody = LoginBody(email: "eve.holt@reqres.in", password: "cityslicka")
    if let data = try? JSONEncoder().encode(loginBody) {
        let (responseBody, _) = try await URLSession.shared.upload(for: request, from: data)
        if let content = String(data: responseBody, encoding: .utf8) {
            print(content)
        }
    }

}
  • The Cat API。

curl 指令如下。

curl "https://api.thecatapi.com/v1/images/search?limit=20" -H "x-api-key:DEMO-API-KEY"

由於 HTTPField.Name 沒有內建 x-api-key，因此我們必須自己建立 HTTPField.Name("x-api-key")。


func test() async throws {
    var request = HTTPRequest(method: .get, scheme: "https", authority: "api.thecatapi.com", path: "/v1/images/search?limit=20")
    request.headerFields[HTTPField.Name("x-api-key")!] = "DEMO-API-KEY"
    let (responseBody, _) = try await URLSession.shared.data(for: request)
    if let content = String(data: responseBody, encoding: .utf8) {
        print(content)
    }
}
