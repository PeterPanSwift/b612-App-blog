使用 Swift Vapor 開發回傳隨機名字的 API

隨著各式各樣 server-side Swift web framework 的推出，現在的 Swift 不只能寫 iOS App，還可以寫 server。利用 Swift 寫 server 程式時，目前最受歡迎的 framework 應該非 Vapor 莫屬，因此接下來我們將介紹如何從零開始用 Vapor 開發回傳隨機名字的 API。

安裝 Homebrew

為了安裝 Vapor 的開發環境，我們必須先安裝 Homebrew。Homebrew 是 macOS 上知名的套件管理工具，透過它我們將可方便地安裝許多有趣厲害的套件。

Homebrew
The Missing Package Manager for macOS (or Linux).brew.sh

打開 Terminal，輸入以下指令安裝 Homebrew。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

成功安裝後，依序輸入以下指令將 Homebrew 加到 PATH，這樣 Terminal 才能使用 brew 指令安裝套件。

(echo; echo 'eval "$(/opt/homebrew/bin/brew shellenv)"') >> /Users/peterpan/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

安裝 Vapor

打開 Terminal，輸入以下指令安裝 Vapor。

brew install vapor

建立 Vapor 專案

打開 Terminal，輸入以下指令建立專案，RandomNameAPI 是專案名稱。

vapor new RandomNameAPI

由於我們只要建立一個回傳隨機名字的 API，不會使用進階的功能，因此 Would you like to use Fluent? 和 Would you like to use Leaf? 都回答 n 就好。

專案建立後將產生資料夾 RandomNameAPI，資料夾內有以下檔案。

打開專案和在 Mac 本機啟動 server

用 Xcode 打開 RandomNameAPI 資料夾下的 Package.swift。

選擇 My Mac，然後點選三角形按鈕啟動 Server。

啟動後 Xcode 的 Console 會顯示以下訊息，表示我們可以連到 http://127.0.0.1:8080 讀取 server 回傳的資料。

[ WARNING ] No custom working directory set for this scheme, using /Users/peterpan/Library/Developer/Xcode/DerivedData/RandomNameAPI-fqcrfofffqgkpndarzxvhbambjqr/Build/Products/Debug
[ NOTICE ] Server starting on http://127.0.0.1:8080

連線到 server

現在我們可以在 Safari 輸入 http://127.0.0.1:8080 或 http://localhost:8080連到剛剛啟動的 server，網頁上將顯示訊息 It works。

若是輸入 http://localhost:8080/hello 或 http://127.0.0.1:8080/hello，網頁將顯示 Hello, world!。

設定 server 如何回應的 routes.swift

當我們連線到 server 時，Sources > App 下的 routes.swift 控制著 server 如何回應。

以下面這段程式為例，它表示連線到路徑 hello 時會回傳 "Hello, world!"，因此我們剛剛連線到 http://localhost:8080/hello 時網頁會顯示 Hello, world!。

app.get("hello") { req async -> String in
    "Hello, world!"
}

新增回傳隨機名字的 API randname

我們目標產生一個隨機名字，因此彼得潘先請 AI 幫忙產生迪士尼可愛角色的名字 array disneyCharacters，然後在 routes.swift 加入 disneyCharacters 和 app.get("randname")。

func routes(_ app: Application) throws {
    let disneyCharacters = ["米奇老鼠", "唐老鴨", "高飛", "米妮老鼠", "普路托", "黛西鴨", "厄休拉", "愛麗兒", "貝兒", "野獸", "阿拉丁", "茉莉", "辛巴", "史卡", "莫安娜", "艾莎", "安娜", "歐樂福"]

    app.get { req async in
        "It works!"
    }

    app.get("hello") { req async -> String in
        "Hello, world!"
    }
    
    app.get("randname") { req -> String in
        disneyCharacters.randomElement()!
    }
}

現在我們每次連到路徑 randname 時都會看到隨機顯示的迪士尼角色。

連線 server 的 Swift 程式

  • 使用 await。

import Foundation

let url = URL(string: "http://localhost:8080/randname")!
Task {
    let (data, _) = try await URLSession.shared.data(from: url)
    if let name = String(data: data, encoding: .utf8) {
        print(name)
    }
}
  • 使用 completion handler。

import Foundation

let url = URL(string: "http://localhost:8080/randname")!
URLSession.shared.dataTask(with: url) { data, response, error in
    if let data,
       let name = String(data: data, encoding: .utf8) {
        print(name)
    }
}.resume()

部署 Vapor 應用程式

值得注意的，localhost & 127.0.0.1 都是本機的網址，因此只能從本機的程式連線，其它的機器無法連進來。若想讓外面的機器可以連到 server，可參考以下連結的說明。
