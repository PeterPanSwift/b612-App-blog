修改 id 也能實現 SwiftUI 元件的 transition 動畫

當 view 出現或消失時，SwiftUI 的 transition 可以為我們帶來淡入淡出、移動、縮放等生動的轉場效果。

transition 只會在元件生成顯示或從畫面移除時發揮作用，因此 transition 通常會搭配使用 if 生成 / 移除的 view。以下程式的 Image 一直在畫面上，點選 button 換圖時沒有造成 Image 的生成或移除，因此加上 .transition(.slide) 不會產生滑動的動畫，換圖時只會有預設的淡入淡出動畫。

struct ContentView: View {
    var imageResources: [ImageResource] = [
        .yellowPeter,
        .redPeter,
        .bluePeter,
        .greenPeter
    ]
    @State private var index = 0
    
    var body: some View {
        VStack {
            Image(imageResources[index])
                .transition(.slide)

            Button("next") {
                withAnimation {
                    index = (index + 1) % imageResources.count
                }
            }
            .buttonStyle(.borderedProminent)
        }
    }
}

然而 Apple 在 WWDC 的 Demystify SwiftUI 提到當 view 的 id 改變時，view 的生命會結束，因此即使沒有 if，剛剛的 Image 也能實現 transition 的效果。只要我們改變 Image 的 id，即可生成新的 Image，觸發 transition 的動畫效果。

以下程式在 Image 加上 .id(index)，因此點選 button 改變 index 時也會改變 Image 的 id，造成新的 Image 生成，觸發 transition 的滑動動畫。

struct ContentView: View {
    var imageResources: [ImageResource] = [
        .yellowPeter,
        .redPeter,
        .bluePeter,
        .greenPeter
    ]
    @State private var index = 0
    
    var body: some View {
        VStack {
            Image(imageResources[index])
                .id(index)
                .transition(.slide)
            
            Button("next") {
                withAnimation {
                    index = (index + 1) % imageResources.count
                }
            }
            .buttonStyle(.borderedProminent)
        }
    }
}
