利用 #Preview 從 Canvas 快速操作 UIKit App

當我們使用 UIKit & storyboard 開發 App 時，常常會在修改後將 App 裝到模擬器測試，不過說實話，等待 App 從模擬器啟動的時間真的有點久。

現在有了 #Preview 的 App 預覽操作功能，我們有更快的方法。我們可在專案的某個檔案新增顯示 App 初始畫面的 #Preview，如此即可方便地隨時從 Canvas 操作 App， 速度比裝到模擬器還快。

以下我們以 Adam 同學練習的的神隱少女 App 為例說明。

#3 Storyboard 電子書 App _ 神隱少女
這次的作業我打算做一個介紹電影《神隱少女》的電子書APP。medium.com

如下圖所示，Main.storyboard 設計了 App 的每個畫面。

在 AppDelegate.swift 新增 #Preview，在 #Preview { } 裡回傳 App 的初始 controller，如此 Canvas 顯示的畫面即代表整個 App，可在 Canvas 操作 App 和切換到任何頁面。

  • 寫法 1: 當 App 的初始畫面來自 storyboard。

以初始畫面來自 Main.storyboard 的 initail view controller 為例。

#Preview {
    let storyboard = UIStoryboard(name: "Main", bundle: nil)
    return storyboard.instantiateInitialViewController()!
}
  • 寫法 2: 當 App 的初始畫面來自程式。

假設初始畫面為 navigation controller 管理的 ViewController。

#Preview {
    let viewController = ViewController()
    let navController = UINavigationController(rootViewController: viewController)
    return navController
}

ps: AppDelegate 定義了 App 啟動完成的 function，因此很適合定義代表整個 App 的 #Preview，若有其它考量也可將 #Preview 寫在其它檔案。

之後修改 storyboard 或程式後，只要切回 AppDelegate.swift 即可操作更新後的 App。

如果想在編輯任何程式檔時都能操作 App，還可點選 Canvas 左上的圖釘將畫面釘起來。

若是遇到 Canvas 顯示的 App 沒有更新畫面或功能，可透過以下兩種方法強制更新。

  • 從 Quick Actions 點選 Refresh Canvas。
  • 按 cmd + b 重新編譯專案，再按 Canvas paused 旁的重新整理按鈕。
