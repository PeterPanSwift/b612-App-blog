A build only device cannot be used to run this target 問題

有些同學在 GitHub 上看到別人開發的 iOS App 程式，下載後準備好好研究研究，不過卻發現根本跑不起來，模擬器選單裡沒有模擬器可選，而且還出現錯誤訊息。

The run destination xxx is not valid for Running the scheme 或 A build only device cannot be used to run this target 問題

有些朋友在從 Xcode 啟動 App 時，遇到以下兩種問題。

問題 1 : App 無法裝到實機

裝置名稱的右邊顯示 OS version lower than development target。

不理它暴力地點選三角形將出現以下錯誤。

問題 2: 模擬器選單裡沒有模擬器可選

A build only device cannot be used to run this target
No supported iOS devices are available. Connect a device to run your application or choose a simulated device as the destination.

以上問題發生的原因是因為 App 支援的最低 iOS 版本較新，比方 App 支援 iOS 14.5 以上，但你的實機跟模擬器的 iOS 版本是 14.4。

解決的方法有兩種:

解法1: 調降 App 支援的最低 iOS 版本，修改 App 的支援版本

Xcode 14 以上

  • 點選 TARGETS 下的 App，切換到 App 的 General 頁面。
  • 找到 Minimum Deployments 區塊。
  • 選擇版本，比方選擇 17，表示 App 支援的最低 iOS 版本是 17。

此時它將自動填為 17 的最新版，因此是 17.6。

若想改小數點後的數字，則可手動編輯修改。

舊版 Xcode

  • 點選 TARGETS 下的 App，切換到 App 的 General 頁面。
  • 找到 Development Info 下的 Target 區塊。
  • 選擇 Target 的版本，比方選擇 13.3。

解法2: 升級 Xcode

若是模擬器選單裡沒有模擬器可選，有可能是 Xcode 版本較舊，無法支援新版的 iOS。比方 Xcode 16 無法支援 iOS 26。若要支援 iOS 26，須將 Xcode 升級到 26。
