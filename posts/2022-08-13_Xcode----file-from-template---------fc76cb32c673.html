Xcode 新增 file from template 時要選哪一個呢

開發 iOS App 時，我們時常需要新增檔案，為 App 加入更多的功能或畫面。

從 Quick Actions，Xcode menu (快速鍵 cmd + N) 或 Project navigator 的右鍵選單可以點選 file from template 打開檔案的選擇視窗。

檔案選擇視窗的畫面如下，雖然檔案很多，但是常用的就那幾種，就像彼得潘 buffet 晃了一圈常常只夾牛排一樣。以下我們介紹幾種常用的檔案。

SwiftUI View

以 struct 定義 SwiftUI 的 view，可能是整個畫面，或是畫面裡的某個元件。

Cocoa Touch Class

定義 class，並且繼承來自 Cocoa Touch framework 的類別。當我們想自訂 view controller，table view controller，cell 或 UIKit 的 view 時，會選擇 Cocoa Touch Class。

從 Subclass of 可選擇繼承的類別，常用的父類別包含 UIViewController，UITableViewCell，UICollectionViewCell，UIView，UITableViewController，UICollectionViewController，NSObject 等。

Swift File

當我們想新增 swift 檔，而且目的跟 SwiftUI View & Cocoa Touch Class 無關時，通常會選擇 Swift File。它將產生一個空白的 swift 檔，方便我們在其中寫下任何想寫的 swift 程式。

常見的應用如下:

  • 以 Swift 語法定義 struct，class，enum，protocol 或 extension
  • 定義 model 型別，通常是資料的型別，比方為電影 App 定義儲存電影資料的 Movie 型別。

Storyboard

產生設計 App 畫面的 storyboard。

View

產生設計 App 畫面的 xib 檔，裡面將預設有個空白的 view。

Empty

產生設計 App 畫面的 xib 檔，裡面什麼都沒有，我們必須自己加入 UI 元件。
