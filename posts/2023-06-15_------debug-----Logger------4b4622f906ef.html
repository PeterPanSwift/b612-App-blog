使用更方便 debug 程式的 Logger 列印訊息

開發 iOS App 時，列印訊息主要有 print 和 Logger 兩種方法。從 Xcode 15 開始，Xcode 的 console 在顯示 Logger 列印的訊息有了非常大的進步。相較於 print，採用 Logger 列印有以下好處:

  • 搭配彩色背景顯示重要訊息。
  • 一鍵跳到印出訊息的程式。
  • 顯示細節，像是列印的時間，列印程式所在的檔案跟行數。
  • 依重要度分類訊息。
  • 方便過濾尋找特定訊息。

以下我們從產生 Logger 開始，一步步介紹使用 Logger 列印訊息的各項功能。

  • 產生 Logger。
  • 呼叫 function log 列印訊息。
  • 依重要度列印不同等級的訊息。
  • 查看 log 訊息的詳細內容。
  • 一鍵跳到印出 log 訊息的程式。
  • 過濾尋找特定的 log 訊息。
  • Logger 無法列印 optional 的資料。
  • 版本限制。

產生 Logger

為了使用 Logger 列印訊息，我們必須先 import OSLog。

import OSLog

依據 Apple 的範例，儲存 Logger 的常數通常宣告在以下兩個地方。

  • 將 logger 宣告為只在檔案裡使用的常數。
private let logger = Logger()

struct ContentView: View {
  • 將 logger 宣告為 property。
struct ContentView: View {
    private let logger = Logger()

呼叫 function log 列印訊息

import SwiftUI
import OSLog

private let logger = Logger()

struct ContentView: View {

    var body: some View {
        Button("tap", action: tap)
    }
    
    func tap() {
        logger.log("tap")
    }
}

當我們點選三次 button 時，console 將印出三個 tap，而且 console 還會以分隔線區分訊息，不會像 print 那樣，列印的訊息全都混在一起。

依重要度列印不同等級的訊息

列印訊息時，我們可依重要的程度列印不同等級的訊息，方便之後在茫茫訊息海中查看重要的訊息。

log 訊息有以下五種等級，重要的程度由低而高依序為 Debug，Info，Notice(Default)，Error 和 Fault。

Generating Log Messages from Your Code | Apple Developer Documentation
Record useful debugging and analysis information, and include dynamic content in your messages.developer.apple.com

呼叫 log 時，從參數 level 可傳入 debug，info，default，error & fault 五種等級，省略 level 則會採用 default 等級。

logger.log(level: .info, "tap")

除了呼叫 log，Logger 也提供方便我們列印不同等級訊息的 function，像是 debug，info，notice，error & fault。

logger.debug("debug tap")
logger.info("info tap")
logger.notice("notice tap")
logger.error("error tap")
logger.fault("fault tap")

不同等級的訊息顯示時會搭配不同的背景色，特別重要的 error 和 fault 會搭配顯眼的黃色和紅色背景。

使用 string interpolation 列印變數的內容

Logger 列印訊息時接受的參數型別其實是 OSLogMessage，只是我們用雙引號傳入字串時，它會自動被轉成 OSLogMessage。

若是直接傳入變數，它並不會自動轉成 OSLogMessage，錯誤訊息顯示 Cannot convert value of type ‘String’ to expected argument type ‘OSLogMessage’。

因此若要印出變數的內容，我們必須使用 string interpolation 的寫法。

let name = "peter"
logger.log("\(name)")

如果變數的型別沒有遵從 CustomStringConvertible，我們還需另外遵從 CustomStringConvertible，定義 property description，如此 logger 才知道如何將物件變成字串印出。

class Baby: CustomStringConvertible {
    var name = "peter"
    var age = 1
    var description: String {
        "Baby named \(name), age \(age)"
    }
}
let cuteBaby = Baby()
logger.log("\(cuteBaby)")

查看 log 訊息的詳細內容

比起 print，Logger 列印的訊息多了許多細節。在印出的訊息上按空白鍵可查看詳細內容。

訊息詳情包含了 Type，Timestamp，Library，PID:TID，Subsystem，Category 和 Call Site。從上圖的 Call Site 可知道列印 error tap 的程式來自 ContentView 的 function tap。

我們也可以讓訊息顯示時直接包含細節。點選下圖的 Metadata Options icon，可勾選 Type，Timestamp，Process Name，Library，PID:TID，Subsystem 和 Category。

比方我們勾選 Type，Timestamp，Library，Subsystem & Category。

訊息的下方多了勾選的細節。

Library Demo & Subsystem Demo 對應 App 的 target 名字，Category 名稱則預設為 default。

當 log 訊息很多時，我們也可以分類訊息，產生自訂 subsystem 和 category 的 Logger，方便之後查詢不同分類的訊息。

以 Apple 的 Backyard Birds 專案為例，它定義了多種 Logger，比方 subsystem 有 BackyardBirds，BackyardBirdsData & Widgets，category 有 Account Information，Bird Brain，BackyardBirdsShopViewModifier。

跳到印出 log 訊息的程式

當我們列印許多訊息時，我們常會忘了列印訊息的程式在哪。從 Xcode 15 開始，我們不用再煩腦了，現在可以一鍵跳到列印訊息的程式。

  • 方法 1: 點選訊息右下的箭頭。

當訊息的下方有顯示細節時，訊息右下會顯示所在的程式檔案和行數，下圖顯示 error tap 來自 ContentView.swift 的 23 行。

點選箭頭將直接跳到印出 log 訊息的程式碼。

  • 方法 2: 從訊息的右鍵選單點選 Jump To Source。

過濾尋找特定的 log 訊息

從 filter 框框可以過濾尋找特定的 log 訊息。以下我們以 Apple 的 Backyard Birds 專案說明。Backyard Birds App 啟動後，console 印出豐富的訊息。

  • 顯示特定等級的訊息。

點選下圖的紅色框框可指定顯示特定等級的訊息。

比方選擇 TYPE Debug 後，只會顯示等級為 Debug 的訊息。

  • 輸入關鍵字搜尋。
  • 顯示特定 Subsystem，Category，Library 的訊息。

輸入關鍵字時也可以過濾特定 Subsystem，Category，Library 的訊息。

  • 顯示 / 隱藏相關的訊息。

從訊息的右鍵選單點選 Show Similar Items 或 Hide Similar Items 可以顯示 / 隱藏相關的訊息，依據 Type，PID，Library，Subsystem，Category，TID 過濾。

版本限制

  • 方法 1: 使用 os_log。

 os_log(.error, "tap")
