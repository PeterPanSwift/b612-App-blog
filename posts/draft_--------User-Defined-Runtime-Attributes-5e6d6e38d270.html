設定元件內容的 User Defined Runtime Attributes

在 Interface Builder 設計畫面時，我們習慣從 Attributes inspector 設定元件的內容，比方從下圖的 Image 欄位設定顯示的圖片。

不過 Attributes inspector 並不會列出元件所有可設定的欄位，因此很多進階的設定我們必須透過程式設定。

然而除了從程式設定，Interface Builder 另外提供一個特別的窗口，我們可以從 Identity inspector 的 User Defined Runtime Attributes 設定元件的內容。

如下圖所示，我們在 User Defined Runtime Attributes 輸入 layer.cornerRadius，設定圓片的圓角弧度為 150。

執行 App 後，圖片順利地變成了圓形。

User Defined Runtime Attributes 可以輸入哪些內容

有朋友問到，我們要怎麼知道 User Defined Runtime Attributes 可以輸入哪些內容呢 ?

它可以輸入的內容必須符合以下規則

  • key path 通常是元件能存取的 property，可以不只一層。

剛剛的例子裡，layer.cornerRadius 是 UIView 能存取的 property。

  • property 的型別是 Type 清單裡的其中一種。

剛剛的例子裡，cornerRadius 的型別是 CGFloat，是浮點數，因此可以選擇 Type Number。
