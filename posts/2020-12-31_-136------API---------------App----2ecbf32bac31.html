#136 結合網路 API 串接，登入跟資料儲存的轉職 App 作品

有許多朋友問彼得潘，想轉職成為有趣的 iOS App 魔法師，最好準備什麼樣的 iOS App 作品呢 ?

想成為一個新手的 iOS App 工程師，需要具備的技術有很多，比方 swift 語法，auto layout 等，不過我覺得其中最重要的是網路 API 的串接，帳號登入跟資料儲存，因為以下兩個原因:

  • 不管去哪一家公司上班，幾乎都會用到這三個技術。
  • 結合這三個技術已經可以做出功能滿完整的 App。

因此有興趣準備轉職 App 作品的朋友，可以考慮在 App 裡完成以下功能需求。

  • 登入功能。
  • 搭配第三方的 HTTP get API 抓取資料。
  • 搭配 REST API 實現 HTTP 的 post、delete & put。
  • 搭配 Imgur API 上傳圖片。
  • 將資料存在 App 裡。

上傳圖片也是工作時常開發的功能，這部分可以搭配 Imgur API。

ps: 以上網路 API 串接也可以使用 Firebase，不過比較不建議。因為工作時大部分公司會雇用工程師寫後台，而不是用 Firebase，因此建議練習時多使用 URLSession 串接 API 的寫法，而非 Firebase SDK。

將資料存在 App 裡

使用 SwiftData、Core Data 或 document 資料夾儲存資料。(ps: 有些資料也可以使用 UserDefaults 或 keychain 儲存)

範例: 自己收藏的公車清單除了存在網路上，也存一份在 App 裡，沒有網路時打開 App 還可以找到回家要坐哪台公車。

One more thing, 轉職作品要創意跟精美畫面嗎 ?

應該是不用，因為重點是技術，只要表現出你有足夠的技術，就能順利找到工作。因此雖然公車，天氣，電影 App 很常見，但是只要你做得跟 App Store 上排行榜的公車，天氣，電影 App 一樣好，相信會有很多公司對你有興趣的。

ps: 通常工作時工程師不見得有機會發揮 App idea 的創意跟設計精美畫面，因為工程師主要的任務是將老闆想要的功能做出來。而且說實話，老闆也不一定有創意跟美感呀，但是只要他給你很多很多錢，就算他的創意跟美感很爛你也會開心地將它完成的。
