將 iPad App 變成 Mac App

iOS 的 App Store 上有許多方便好用的 App，有沒有可能在 Mac 使用這些 App 呢 ?

有的 ! 在 macOS 10.15 這個夢想成真了。開發者不需要特別花時間另外開發 Mac App，只要原本的 App 能在 iPad 上流暢地操作運行，花一點點的時間心力調整，即可讓它變成 Mac App。

一鍵將 iPad App 變成 Mac App

沒錯，一鍵就能轉換，就跟美肌 App 一樣。

點選專案檔後，從 TARGETS App 的 General 分頁找到 Development Info 區塊，勾選 Mac。

接著 Xcode 將自動幫我們進行變身 Mac App 的相關設定。

ps: 值得注意的，當 Development Info 沒有勾選 iPad 時，Mac 是無法勾選的。因為 iPad 的操作比較接近 Mac，如果我們連 iPad 都無法支援，想支援 Mac 有點像還不會走路就想跑一樣，實在是太不自量力了。

執行Mac App

從左上角的執行裝置選單選擇 My Mac 啟動 Mac App。

不過請先等等，此時還有一個小小的紅色錯誤要先處理。如下圖所示，錯誤訊息提到 Signing for DemoMac requires a development team。

此問題有兩種修正的方法:

方法 1: 將 Signing Certificate 設為 Sign to Run Locally。

方法 2: 設定 Team 帳號，輸入自己的 Apple ID。

下圖採用方法 1 Sign to Run Locally 的解法。

這次真的可以啟動 Mac App 了 ! Single View App 啟動後的樣子如下圖所示。

處理 compile 錯誤

有時運氣不好，我們還是會遇到 compile 錯誤，比方某些 iOS 相關的程式在 mac 不支援，或是我們希望程式只在 mac 或 iOS 執行，此時可利用 targetEnvironment(UIKitForMac) 做判斷。

只在 mac 執行的程式

#if targetEnvironment(UIKitForMac)
#endif

只在 iOS 執行的程式

#if !targetEnvironment(UIKitForMac)
#endif

為 Mac App 加入 menu 選項

接著讓我們以加入 menu 選項為例，試試加入一些 Mac 獨有的功能吧。

1 在 Main.storyboard 裡加入 Main Menu。

Main menu 代表 App 在 mac 啟動後，在上方顯示的 menu bar。

如下圖所示，Main Menu 一開始有 Application，File，Edit，Format，View，Window & Help 七種選項。

2 在 Help menu 下加入選項 Learn Swift。

Menu Command 代表 menu 裡的選項。我們從 Library 視窗找到 Menu Command，將它拖曳加到 Help 底下。

不過此時 Learn Swift 還沒有任何功能，所以接著我們利用 IBAction 設定它執行的程式。

以下我們在 AppDelegate.swift 裡定義 function learnSwift。

@IBAction func learnSwift(_ sender: Any) {
   print("learnSwift")
}

接著從 storyboard 的 Learn Swift 連線到上方三個按鈕中間的 First Responser，選擇觸發的 function learnSwift:。

執行 App

現在當我們點選 menu Help > Learn Swift 時，將在 Xcode 的 Console 看到 learnSwift 訊息順利印出。

以上只是將 iPad App 變成 Mac App 的簡單範例。如果想讓變身後的 Mac App 有更好的使用者體驗，搭配更多方便的 Mac 操作習慣的話，可再進一步參考以下 Apple 的文件說明。
