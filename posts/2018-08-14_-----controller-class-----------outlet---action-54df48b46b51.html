產生新的 controller class 後，請稍等片刻再拉 outlet & action

身為一個 iOS App 工程師，有件我們幾乎每天都會做的事:

1 新增 controller 的類別。

2 將 storyboard 畫面上 controller 的 class 改成新增的類別。

3 從 storyboard 的 controller 畫面拉 outlet 和 action 到新增的類別程式。

然而有個小地方要注意一下。當我們產生新的 controller class 後，請稍等片刻再拉 outlet & action。

因為 Xcode 的動作沒那麼快，他不像我們是亞洲撲克牌界的第一快手。如果你剛產生新的 controller class，此時是無法馬上拉 outlet & action 的，而且右上角選單的 Automatic 也會無法選取。

或是顯示 Automatic，然而畫面裡卻出現令人絕望的 No Assistant Results。

不用太心急，不用等太久，大概只要稍等幾秒鐘，automatic 就會可以選取，Assistant editor 也會貼心地自動切換為 Automatic，顯示跟左邊 storyboard 畫面有關的 controller 程式。
