SwiftUI preview 的檔案儲存路徑

SwiftUI 的 live preview 功能讓我們不用啟動模擬器就能測試 App，而且大部分的功能都能測試，就連檔案的儲存跟讀取也不是問題。

當我們從 live preview 啟動 App 時，檔案會在 Previews 資料夾下，跟從模擬器啟動的 App 路徑不同，因此我們在 live preview 跟模擬器存取的檔案是不同的。接下來就讓我們親眼看看 UserDefaults，document directory & bundle file 在 live preview 的實際路徑。

UserDefaults

struct ContentView: View {
    
    @State private var number = UserDefaults.standard.integer(forKey: "number")
    
    
    var body: some View {
        
        Text("\(number)")
            .onTapGesture {
                
                self.number += 1
                
                UserDefaults.standard.set(self.number, forKey: "number")
                
        }
        .onAppear {
            print(NSHomeDirectory())
        }
    }
}

UserDefaults 的檔案在 NSHomeDirectory 下，在 live preview 時 NSHomeDirectory 在 /Users/shih-yingpan/Library/Developer/Xcode/UserData/Previews/Simulator Devices/122BF202–0731–4E31–9B58–29793EE20973 下。

/Users/shih-yingpan/Library/Developer/Xcode/UserData/Previews/Simulator Devices/122BF202-0731-4E31-9B58-29793EE20973/data/Containers/Data/Application/4F9F4097-49EB-45A7-853C-F6BAC7C5B8A0

在模擬器執行時，它在 /Users/shih-yingpan/Library/Developer/CoreSimulator/Devices/55B59729–7B10–4EC9-BDA1-FE427E1064B4 下。

/Users/shih-yingpan/Library/Developer/CoreSimulator/Devices/55B59729-7B10-4EC9-BDA1-FE427E1064B4/data/Containers/Data/Application/7BBB00FF-E5AE-4BF1-9808-BE48A20ACAE5

因此很明顯的，live preview 時 UserDefaults 的檔案在 Previews 資料夾下，我們在 live preview 存取的 UserDefaults 跟在模擬器存取的 UserDefaults 是不同的檔案。比方在 live preview 讀到 number 10，在模擬器讀到 number 3。

document directory

var body: some View {
        
        Text("document directory")
        .onAppear {
            print(FileManager.default.urls(for: .documentDirectory, in: .userDomainMask)[0])
        }
}

在模擬器執行時，路徑如下

file:///Users/shih-yingpan/Library/Developer/CoreSimulator/Devices/55B59729-7B10-4EC9-BDA1-FE427E1064B4/data/Containers/Data/Application/68A48AD2-8ED0-4ED5-84CF-39A230DB2774/Documents/
