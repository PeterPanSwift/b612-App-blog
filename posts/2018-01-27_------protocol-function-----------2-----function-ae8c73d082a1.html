為什麼實作 protocol function 時，自動完成會出現 2 個同名 function

Xcode 提供貼心的自動完成功能，讓我們輸入程式時容易許多。不過當我們實作 protocol 宣告的 function 時，卻可能出現一個奇怪的現象。

如下圖所示，來自 protocol UITableViewDelegate 的 function tableView(_:didSelectRowAt:) 竟然出現了兩次 ! 程式裡出現同名的 function 跟現實生活交兩個同名的女朋友一樣，都是很麻煩的事，因為你不知道要選哪一個 !

為什麼同樣的 function 會出現兩次呢 ? 因為我們自訂的型別繼承 UITableViewController，而 UITableViewController 又遵從 protocol UITableViewDelegate，因此它會顯示兩次 function tableView(_:didSelectRowAt:)，一個代表我們要遵從 protocol，實作 function。如果你選擇它，輸入的程式將如下圖:

但是很不幸地，這個選擇是錯的。因為 UITableViewController 已經定義了 function，所以你必須額外補上 override。

另一個則代表我們要覆寫原本 UITableViewController 定義的 function，因此如果選擇它，一開始就會貼心幫我們加上 override。

不過很可惜的，哪一個會加上 override ，目前並沒有一定的順序。比方剛剛的例子，第一個沒加 override, 第二個有加。然而當我們輸入另一個 function tableView(_:willDisplay:forRowAt:) 時，卻變成第一個有加，第二個沒加。

所以我們只好認命點，如果運氣不好選到沒有 override 的，再另外自己補上吧。其實也沒有很難補啦，懶得自己打的話，可以選擇 Fix，Xcode 會幫你補的。
