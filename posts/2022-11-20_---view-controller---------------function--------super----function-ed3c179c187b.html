覆寫 view controller 畫面出現/消失/排版的相關 function 時，記得呼叫 super 執行 function

開發 iOS UIKit App 時，我們時常覆寫 UIViewController / UITableViewController 畫面出現/消失/排版的相關 function，比方 viewDidLoad，viewWillAppear，viewDidAppear，viewWillDisappear，viewDidDisappear，viewWillLayoutSubviews & viewDidLayoutSubviews 等。

UIViewController / UITableViewController 會在這些 function 裡做一些事情，因此在覆寫這些 function 時，記得呼叫 super 執行 function，如此在父類別定義的程式才會被執行。

Apple 也在文件裡提到，

If you override this method, you must call super at some point in your implementation.

比方以下例子覆寫 viewWillAppear 時，記得要加上 super.viewWillAppear(animated)。

class TableViewController: UITableViewController {

    override func viewWillAppear(_ animated: Bool) {
        super.viewWillAppear(animated)
    }

若是忘了呼叫 super 執行 function，可能會產生一些特別的問題。假設 storyboard 的畫面設計如下，點選第一個 cell 會顯示美麗的福州山公園夜景。

我們故意覆寫 viewWillAppear，但是沒有呼叫 super 執行 function。

class TableViewController: UITableViewController {

    override func viewWillAppear(_ animated: Bool) {

    }

當 App 執行後，點選 cell 福州山公園會顯示美麗的夜景，但是別高興地太早。當我們返回表格頁面時，cell 將繼續維持點選狀態的灰色，沒有正確地恢復成未選取的狀態。為什麼會這樣呢 ? 因為我們忘了加上 super.viewWillAppear(animated)呀。
