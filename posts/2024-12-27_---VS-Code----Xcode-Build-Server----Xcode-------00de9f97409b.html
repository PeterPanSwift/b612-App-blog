使用 VS Code 搭配 Xcode-Build-Server 開發 Xcode 專案的程式

VS Code 只要安裝 Swift extension 即可開發 Swift 程式。不過如果使用 VS Code 打開 Xcode 建立的專案資料夾，卻會發現以下問題。

  • 偵測錯誤出問題。

常常無法判斷程式的錯誤，比方以下程式 globe 少了雙引號的簡單錯誤竟然沒偵測到。

  • 自動完成有問題。

常常無法提示可輸入的選項，比方以下程式輸入 .ba 時並沒有出現任何相關的選項。

以上問題可透過 Xcode-Build-Server 解決。它將幫我們在專案資料夾下生成 buildServer.json，讓 VS Code 能正確解析和識別 Xcode 專案的程式。

GitHub — SolaWing/xcode-build-server: a build server protocol implementation for integrate xcode…
a build server protocol implementation for integrate xcode with sourcekit-lsp — SolaWing/xcode-build-servergithub.com

在 Terminal 輸入以下指令安裝 Xcode-Build-Server。

brew install xcode-build-server

ps: 如果無法執行 brew 指令，請先安裝 Homebrew。

Homebrew
The Missing Package Manager for macOS (or Linux).brew.sh

用 VS Code 打開專案資料夾，在它的 Terminal 依照以下格式輸入指令可生成 buildServer.json。

# *.xcworkspace or *.xcodeproj should be unique. can be omit and will auto choose the unique workspace or project.
xcode-build-server config -workspace *.xcworkspace -scheme <XXX> 
xcode-build-server config -project *.xcodeproj -scheme <XXX>

比方我們的專案檔是 Demo.xcodeproj，輸入以下指令將生成 buildServer.json。

xcode-build-server config -project Demo.xcodeproj -scheme Demo

結果。

find root: /Users/shih-yingpan/Library/Developer/Xcode/DerivedData/Demo-egmvhhfzunqgifbpmitcjzxeaxfh
updated buildServer.json

有了 buildServer.json，重新用 VS Code 打開專案後，偵錯和自動完成變正常了 !
