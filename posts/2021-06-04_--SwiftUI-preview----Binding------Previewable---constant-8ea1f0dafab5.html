在 SwiftUI preview 產生 Binding 資料 — Previewable & constant

開發 SwiftUI App 時，為了能修改同一份資料，我們時常使用 Binding，比方以下例子 SizeSlider 裡宣告的 property scale 是 Binding，因此它可以修改 ContentView 裡宣告的 scale。

import SwiftUI

struct ContentView: View {
    @State private var scale: Double = 2
    
    var body: some View {
        VStack {
            Image(.fox)
                .resizable()
                .scaledToFit()
                .frame(width: 200, height: 200)
                .scaleEffect(scale)
            
            ScaleSlider(scale: $scale)
                .padding(.top, 100)
        }
        
    }
}

struct ScaleSlider: View {
    @Binding var scale: Double
    
    var body: some View {
        HStack {
            Slider(value: $scale, in: 1...2)
            Text("\(scale, specifier: "%.2f")")
        }
        .padding()
    }
}

剛剛例子裡 ScaleSlider 的 Binding 資料來自 ContentView 的 State property scale，那如果我們想用 preview 單獨呈現 ScaleSlider，該如何傳入 Binding 呢 ?

  • 方法 1: 使用 iOS 17 的 Previewable。
  • 方法 2: 定義包裝元件的 view。
  • 方法 3: 利用 constant 產生固定的 Binding。

方法 1: 使用 iOS 17 的 Previewable

在 #Preview 的 { } 裡宣告 State property 原本是無效的，它將產生固定的 scale，slider 無法移動。

#Preview {
    @State var scale: Double = 2
    return ScaleSlider(scale: $scale)
}

不過從 iOS 17 開始，在 @State 前加上 @Previewable 即可產生可改變的狀態，讓 slider 可以滑動。

#Preview {
    @Previewable @State var scale: Double = 2
    return ScaleSlider(scale: $scale)
}

Tag allowing a dynamic property to appear inline in a preview.developer.apple.com

Previewable 的原理是它會在背後幫我們定義包裝元件的 view，用這個 view 來宣告 State property，實作細節可參考以下方法 2 的說明。

方法 2: 定義包裝元件的 view

透過定義包裝 ScaleSlider 的 ScaleSliderView，在其中宣告 State property，即可在產生 ScaleSlider 時傳入能改變的 Binding。

#Preview {
    struct ScaleSliderView: View {
        @State private var scale: Double = 2
        
        var body: some View {
            ScaleSlider(scale: $scale)
        }
    }
    return ScaleSliderView()
}

方法 3: 利用 constant 產生固定的 Binding

利用 Binding 的 function constant 產生固定的 Binding。

#Preview {
    ScaleSlider(scale: .constant(2))
}

此方法產生的 Binding 資料無法改變，因此 preview 顯示的 slider 將無法滑動。
