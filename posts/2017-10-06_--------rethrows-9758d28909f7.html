再次丟出錯誤的 rethrows

Swift 的 Error Handling 有個跟 throws 很像的兄弟，叫做 rethrows。

我們都知道，英文單字的前面加上 re 表示再一次的意思，所以 rethrows 應該是再次將錯誤丟出的意思吧。賓果 ! 就是這樣，讓我們看看以下的例子:

enum GoAfterGirlError: Error {
    case poorProblem
    case tooYoungProblem
    case notAquariusProblem
    case falseHeartProblem
}
func goAfterAngelababy(money: Int, age: Int) throws {
    guard money > 1000 else {
        throw GoAfterGirlError.poorProblem
    }
    guard age > 18 else {
        throw GoAfterGirlError.tooYoungProblem
    }
    print("我追到 Angelababy 了!")
}
func myDream(goAfter: (Int, Int) throws -> ()) rethrows {
    try goAfter(100, 25)
    print("成功追到女生!")
}
do {
    try myDream(goAfter: goAfterAngelababy)
} catch {
    print("差一點點點就追到 Angelababy 了")
}

當 function 後面加了 rethrows，即表示此 function 會丟出錯誤。但它丟出的錯誤很特別，它只能將呼叫參數產生的錯誤再次丟出。剛剛例子裡，try goAfter 因為錢太少丟出錯誤，此錯誤將被 function myDream 再次丟出，所以 try myDream 會失敗，印出 catch 裡的”差一點點點就追到 Angelababy 了”。

值得注意的，加了 rethrows 的 myDream 只能將參數產生的錯誤再次丟出，不能自己 throw 錯誤，如以下例子，錯誤訊息告訴我們， A function declared ‘rethrows’ may only throw if its parameter does。

因此，當你呼叫加了 rethrows 的 function 產生錯誤時，請記得真正產生錯誤的罪魁禍首，其實是傳入 function 的參數，那個加了 throws 的 closure 或 function 參數。
