利用 Optional 的 flatMap 將多行 optional binding 變成一行

開發 iOS App 時，我們時常遇到串接多個 optional binding 的情況，比方以下例子先將網址字串變成合法格式的網址，然後再將它轉成 URL。

if let urlStr = "https://itunes.apple.com/search?term=戴佩妮&media=music".addingPercentEncoding(withAllowedCharacters: .urlQueryAllowed),
   let url = URL(string: urlStr) {
            
       URLSession.shared.dataTask(with: url) { data, response, error in
                
       }.resume()
}

若覺得以上程式有點麻煩，也可以考慮搭配 Optional 的 flatMap function。

func flatMap<U>(_ transform: (Wrapped) throws -> U?) rethrows -> U?
flatMap(_:) | Apple Developer Documentation
Evaluates the given closure when this instance is not , passing the unwrapped value as a parameter.developer.apple.com

它的作用如下:

  • 當 Optional 有值時，transform 代表的 closure 程式將執行。此時 closure 的參數將是 Optional 的 value。(unwrapped value) closure 回傳的東西將成為 flatMap 回傳的東西。若 closure 回傳 nil，flatMap 也會回傳 nil。
  • 當 Optional 等於 nil 時，flatMap 將直接回傳 nil。

讓我們看看以下的例子:

var numberString: String? = "18"
if let number = numberString.flatMap({Int($0)}) {
    print(number)
}

結果: 印出 18

numberString 有值，因此 flatMap ( ) 裡的 closure {Int($0)} 會執行，此時 $0 是 "18"，因此 Int("18")將得到數字 18，flatMap 將回傳 18(型別 Int?)。

相反的，以下例子的 $0 是 "hook" ，因此 Int("hook")將得到 nil，flatMap 將回傳 nil。

了解 flapMap 的妙用後，我們可以利用它改寫剛剛將網址字串變成 URL 的範例。

if let url = "https://itunes.apple.com/search?term=戴佩妮&media=music".addingPercentEncoding(withAllowedCharacters: .urlQueryAllowed).flatMap({ URL(string: $0) }) {
     URLSession.shared.dataTask(with: url) { data, response, error in
                
     }.resume()
} else {
}

當 addingPercentEncoding 成功轉成合法網址的字串時，flatMap( ) 傳入的 closure 將執行。$0 代表合法網址的字串，它是 unwrapped value，型別是 String，所以我們可用 URL(string: $0) 生成型別 URL? 的資料。此時 flatMap 回傳的資料型別也將是 URL?，因此我們再搭配 if let 讀取內容。

相反的，當 addingPercentEncoding 得到的結果為 nil，或是 URL(string: $0) 的結果為 nil 時，將讓 flatMap 回傳 nil，因此執行 else 的程式。
