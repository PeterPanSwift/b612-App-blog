  • 將專案的 git 版本管理功能打開。

建立專案時，勾選 Create Git repository on my Mac。

如果一開始忘了勾，之後可參考以下連結為專案加上版本管理的功能。

Xcode 的 no Git Repositories 問題
如果一開始在 Xcode 建立專案時忘了勾選 Create Git repository on my Mac，專案將沒有 git 版本管理的功能。如上圖所示，左邊的 Source Control Navigator 分頁將顯示 no…medium.com
  • 修改程式後做版本記錄。

當程式寫到某個段落時，我們可以將目前的版本做記錄，方便未來查看之前寫的版本。記錄的方法如下。

  1. 執行 commit。

從 Quick Actions 選單(cmd + shift + a)點選 Commit。

2. 點選 Stage All，把所有修改的檔案加入「準備記錄」的狀態。

3. 在 Commit message 的框框輸入文字，說明做了哪些修改，然後點選 commit 完成記錄。

輸入文字後，我們才能點選右邊的 commit 完成新版本的記錄。

  • 將專案上傳到 GitHub。

切換到 Source Control navigator 頁面的 Repositories 分頁。

滑鼠游標移到專案名字上，從右鍵選單點選 New xxx Remote。

點選右下角的 Create 按鈕將專案上傳到 GitHub。

成功後 Remote 的下方會出現 origin，專案已經成功上傳到 GitHub。

  • 更新程式。

程式上傳後，我們常常會再做修改，此時一樣先點選 Stage All 再點選 commit 記錄版本。

此時修改的版本只存在我們的電腦，所以 GitHub 的程式還是之前上傳的版本。若想將新版本上傳到 GitHub，可透過以下方法執行 push 上傳。

方法 1: 點選 Push。

方法 2: 從 Quick Actions 點選 Push。

方法 3: commit 時同時 push。

從 Commit 的下拉箭頭點選 Commit and Push。
