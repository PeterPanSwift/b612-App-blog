Interface Builder 的圖片名字長度限制

彼得潘最近教課時開心地用 AI 畫圖，結果不小心發現了 Interface Builder 的圖片名竟然有長度限制。

以下我們一步步重現問題。彼得潘用 DALL·E 生成美麗的圖片，圖片下載後有著超長的檔名， DALL·E 2024–01–03 01.25.34 — In a charming animated evening scene, a boy in a green jacket and backpack kneels and presents a MacBook to a girl in a white dress. They are in an ur.png

彼得潘開心地將圖片加入 asset。

然後在 Interface Builder 用超長的名字設定 Image View 的 Image。

在 storyboard 圖片看起來一切正常，但是 App 執行時圖片卻消失無蹤。

研究一下後，彼得潘發現問題出在圖片名不能太長，而且只有 Interface Builder 的圖片名有限制。若是從程式生成 UIImge，再長的名字也不是問題。

class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        // Do any additional setup after loading the view.
        let imageView = UIImageView(frame: CGRect(x: 0, y: 0, width: 300, height: 300))
        imageView.image = UIImage(named: "DALL·E 2024-01-03 01.25.34 - In a charming animated evening scene, a boy in a green jacket and backpack kneels and presents a MacBook to a girl in a white dress. They are in an ur")
        view.addSubview(imageView)
    }
}

那到底多長會有問題呢 ? 彼得潘耐心研究後，發現名字長度 127 個字時正常，128 個字就不行了。看來以後一定要記得，圖片名最好不要超過 127 個字 !
