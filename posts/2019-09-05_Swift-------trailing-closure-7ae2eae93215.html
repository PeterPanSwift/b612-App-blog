Swift 最後一名的 trailing closure

小時候唸書時，我們都不喜歡當最後一名，因為只有壞事，沒有好事。然而在 Swift 的世界，如果傳入 function 的 closure 參數剛好是最後一個參數，反而會有好事發生呢! 我們可以採用 trailing closure 的寫法讓程式的可讀性更好。

當 closure 是最後一個參數

func repeatTask(count: Int, task: () -> Void) {
   for _ in 1...count {
      task()
   }
}

呼叫 function repeatTask 時，標準的寫法如下:

repeatTask(count: 3, task: { () -> Void in
   print("吃飯皇帝大")
})

我們可以省略 ()->Void in，變成

repeatTask(count: 3, task: { 
   print("吃飯皇帝大")
})

然而因為參數 task 是最後一個參數，我們可以採用以下 trailing closure 的寫法。

repeatTask(count: 3) {
   print("吃飯皇帝大")
}

trailing closure 把代表 closure 的整段 { } 區塊獨立出來，將它搬到 ( ) 後，然後再把 closure 對應的參數名稱和冒號移除。(因此 task: 被移除了。)此寫法將讓程式的可讀性更好，尤其當 closure { } 裡程式碼很多時，原本將 closure 寫在 () 裡的方式，將讓整個呼叫 function 的程式碼顯得格外冗長複雜，不易區分 closure 和其它參數。

自動採用 trailing closure 的 Xcode 自動完成

當我們呼叫 function，將 closure 當參數傳入時，Xcode 的自動完成將聰明地採用最合適的寫法，因此當我們在 repeatTask 的參數 task 的藍色框框按下 Enter 鍵時，它將自動採用 trailing closure 的寫法。

predictive code completion 的提示也會採用 trailing closure 的寫法。

當 closure 是唯一的參數

當我們傳入 function 的 closure 是 function 唯一的參數時，此時還有更精簡的寫法。

func repeatTask(task: () -> Void) {
   for _ in 1...10 {
      task()
   }
}

呼叫 function 時，我們可利用 trailing closure 將 closure 移出 () 外，將參數名字省略，讓 () 變成被掏空的失意人。

repeatTask() {
   print("吃飯皇帝大")
}

不過我們還能更精簡，狠心甩掉沒有利用價值的 ()，變成以下程式。

repeatTask {
   print("吃飯皇帝大")
}

Xcode 的自動完成也會聰明地採用最精簡的寫法，如下圖所示，自動完成列出的選項出現了沒有 () 的 repeatTask { code } 。

iOS SDK 搭配 trailing closure 的 function

iOS SDK 很多 function 都可以搭配 trailing closure，比方以下例子:

scheduledTimer(withTimeInterval:repeats:block:)
Timer.scheduledTimer(withTimeInterval: 1, repeats: true) { timer in
}
present(_:animated:completion:)
present(controller, animated: true) {
}
async(group:qos:flags:execute:)
DispatchQueue.main.async {
}
