在 init 裡初始 Binding property

開發 SwiftUI App，我們有時會用到 Binding property，例如以下例子，代表亮度數值的 brightness 利用 Binding 傳入 BrightnessSlider，因此 BrightnessSlider 能夠修改來自 ContentView 的 brightness。

struct ContentView: View {
    @State private var brightness: Double = 0
    
    var body: some View {
        VStack {
            Image("the morning show")
                .resizable()
                .scaledToFit()
                .brightness(brightness)
            
            BrightnessSlider(brightness: $brightness)
                .padding()
        }
    }
}
struct BrightnessSlider: View {
    @Binding var brightness: Double
    
    var body: some View {
        VStack {
            Text("亮度")
            Slider(value: $brightness) {
                Text("亮度")
            } minimumValueLabel: {
                Image(systemName: "sun.min.fill")
            } maximumValueLabel: {
                Image(systemName: "sun.max.fill")
            }
            
        }
    }
}

不過如果我們要在 BrightnessSlider 裡定義 init，在 BrightnessSlider 生成時額外做一些事呢 ? 我們要如何初始 @Binding property 呢 ?

以下寫法將出現錯誤，因為 self.brightness 的型別是 Bool，不是 Binding<Bool>。

@Binding 是 property wrapper，Swift 會在背後產生加了底線的變數，它的型別才會是 Binding。剛剛的例子裡，_brightness 的型別為 Binding<Double>，它才是我們要初始的 property。

struct BrightnessSlider: View {
    @Binding var brightness: Double
    
    init(brightness: Binding<Double>) {
        _brightness = brightness
        // do something
    }
