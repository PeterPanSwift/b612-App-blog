利用 netfox 查看 iOS App 的網路封包資訊

開發 iOS App 時，我們有時會想查詢送出的 request & 收到的 response 是否正確，比方當送出的 request 得不到正確的結果時，我們可以比對 App 發送的 request 跟 Postman 發送的有哪裡不同。

利用 function dump，我們可以直接印出 URLRequest 的內容。

let url = URL(string: "https://sheetdb.io/api/v1/5af5d484a11f3")
var urlRequest = URLRequest(url: url!)
urlRequest.httpMethod = "POST"
urlRequest.setValue("application/json", forHTTPHeaderField: "Content-Type")
struct Student: Encodable {
    var name = "peter"
    var grade = 13
}
struct StudentData: Encodable {
    var data: Student
}
var cuteStudent = Student()
var studentData = StudentData(data: cuteStudent)
let jsonEncoder = JSONEncoder()
if let data = try? jsonEncoder.encode(studentData) {
   urlRequest.httpBody = data
   dump(urlRequest)
}
URLSession.shared.dataTask(with: urlRequest) { (data, response, error) in
}.resume()

從 Console 我們可以看到完整的 URLRequest 內容，包含它的網址， http header & body，不過它不是很好看，尤其 body 的部分它印出每個 byte 的數字，而不是原始的 JSON 內容。

接下來我們將試試大大開發的套件 netfox，透過它我們可以方便地檢視 App 送出的 request & 收到的 response。

kasketis/netfox
Netfox provides a quick look on all executed network requests performed by your iOS or OSX app. It grabs all requests …github.com
  • 從 CocoaPods 安裝 netfox
pod 'netfox'
  • 在 AppDelegate.swift import netfox & 在 application(_:didFinishLaunchingWithOptions:) 裡利用 NFX.sharedInstance().start() 開始記錄 App 送出的 request & 收到的 response。
import UIKit
import netfox
@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {
   func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        // Override point for customization after application launch.
        NFX.sharedInstance().start() 
        return true
}
  • 執行 App 測試。

現在我們搖晃 iPhone 即可查看 request & response 的相關資訊。

如下圖所示，列表上會顯示我們發送的 API，點進去後可看到它的詳細資訊。

從 Request & Response 分頁可看到詳細的資訊，比方下圖的 Body 顯示我們上傳的 JSON 內容。

netfox 會聰明地判斷如何顯示內容，比方當 Response 回傳大量的 JSON 資料時，它的 Body 會變成 Show Response body，點選後可到下一頁查看它的完整內容。

當 response 是圖片時，點進去將看到美麗的圖片。
