Swift 的 was never used 黃色警告

許多初學者在剛開始學習 Swift 程式時，會看到 was never used 的黃色警告，因此覺得有點緊張，擔心自己是否做錯了什麼。

比方以下四種是最常出現 was never used 的程式例子。

例子 1

func sing() {
   let song = "原來你是我最想留住的幸運"
}
Initialization of immutable value 'song' was never used

例子 2

func sing() {
   let song: String
   song = "原來你是我最想留住的幸運"
}
Immutable value 'song' was never used

例子 3

func sing() {
   var song = "原來你是我最想留住的幸運"
}
Initialization of variable 'song' was never used

以上三個例子都提到 was never used，意思是從未使用。變數和常數都可能遇到這個問題，它代表我們宣告了變數(常數)，也許還存了一些東西，但之後就不理它了。

Swift 覺得你這樣不太好，他覺得你既然宣告了變數(常數)，之後要使用它才有意義。就好像你中了樂透，得到十億，將它存到銀行，之後就不管它，不把錢領出來 Shopping，這樣是不對的，老天爺會看不過去，早知道就不讓你中樂透，應該讓會好好利用，拿錢去環遊世界的人中樂透。

只要使用它即可消除 never used 的警告，比方加上 print，把 song 印出來。

func sing() {
    var song = "原來你是我最想留住的幸運"
    print(song)
}

不過剛剛的例子現在還是有警告，顯示 Variable ‘song’ was never mutated; consider changing to ‘let’ constant。因為我們沒有改變 song 的內容，所以 Xcode 建議我們宣告為常數。

最後讓我們再看一種特別的警告例子。

func sing() {
   var song = "原來你是我最想留住的幸運"
   song = "與你相遇好幸運"
}
variable 'song' was written to, but never read

警告變成了 but never read，不是剛剛介紹的 was never used。but never used 表示你產生了一個變數，存了一些東西，因此你有 write，但你從未讀取，沒有 read。

Xcode 覺得這樣沒有意義，你將東西存到變數，但卻從不讀取。我們將東西存到變數，就是為了之後能讀取內容呀 。只要再加上一行 print，讀取 song 的內容印出，即可消除警告。

func sing() {
    var song = "原來你是我最想留住的幸運"
    song = "與你相遇好幸運"
    print(song)
}
