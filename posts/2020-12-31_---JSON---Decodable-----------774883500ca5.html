解析 JSON 時，Decodable 型別加 ? 的情況

當我們利用 JSONDecoder 將 JSON 資料變成自訂的 Decodable 型別時，property 在以下兩種情況必須加上問號，宣告為 optional，因為有可能找不到對應的 value。

  • key 不一定會出現時。
  • key 的 value 有可能是 null 時。

接下來讓我們以實際的例子說明，假設解析 JSON 的程式如下:

struct Book: Decodable {
    let name: String
    let author: String
    let price: Int?
}
do {
    let books = try JSONDecoder().decode([Book].self, from: data)
    dump(books)
} catch  {
    print(error)
}
