SwiftUI View 的 body 和 PreviewProvider 的 previews 宣告時搭配了 ViewBuilder

開發 SwiftUI App 時，我們最常寫的程式莫過於 View 的 body 和 PreviewProvider 的 previews，比方以下剛建立新專案後的 ContentView 程式。

import SwiftUI
struct ContentView: View {
    var body: some View {
        Text("Hello, world!")
            .padding()
    }
}
struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        ContentView()
    }
}

body & preview 初看是一般的 computed property，其實它們很特別，他們是加了 @ViewBuilder 的 computed property。

  • protocol View 裡的 body 宣告。
@ViewBuilder var body: Self.Body { get }
  • PreviewProvider 裡的 previews 宣告。
@ViewBuilder static var previews: Self.Previews { get }

由於它們添加了 @ViewBuilder ，所以 { } 裡的程式可以實現一些 ViewBuilder 允許的寫法。

不只一行程式，卻可以省略 return

body { } 裡的程式不只一行，卻可以省略 return，這是因為 @ViewBuilder 會另外處理 { } 裡的程式，然後回傳 Text。

struct ContentView: View {
    var body: some View {
        let name = "peter"
        Text("Hello, \(name)!")
    }
}

因此在撰寫 body 或 previews 的程式時，我們可以在裡面宣告一些變數常數，然後卻依然可以省略 return。

可以將多個 UI 元件組合回傳

var body: some View {
        Text("SwiftUI")
        Text("真好玩")
}

雖然這個寫法不會有錯誤，但是它有以下 2 個缺點。

  • preview 顯示不正常。

變成 2 個 preview 分別顯示 Text。

  • 從程式看不出畫面的排版。

不知道 Text 會上下排列，左右排列，還是其它的排列方式。

因此在 body 裡還是建議另外用容器元件包裝要顯示的內容，比方用 VStack 將 2 個 Text 包起來，

SwiftUI body 裡有多個元件時，記得包在容器裡
開發 SwiftUI App 時，我們透過變數 body 設計畫面的內容。當 body 裡有多個 UI 元件時，記得它們要包在某個容器元件的 { } 裡，這樣 App 才知道如何排列元件。medium.com

可以回傳不同型別的元件

以下程式可能回傳 Text，也可能回傳 Image。原本這樣的寫法是不行的，因為 some View 必須明確知道回傳的型別。

var body: some View {
        let dice = Int.random(in: 1...6)
        if dice > 3 {
            Text("大")
        } else {
            Image(systemName: "\(dice).circle")
        }
}

不過由於 body 添加 ViewBuilder，所以它可以幫我們把 body { } 裡的程式組合成一個 view 回傳。

值得注意的，剛剛的 Text & Image 前若加上 return，則會產生錯誤。因為此時 ViewBuilder 無法幫我們處理， Swift 將認為我們可能回傳兩種不同型別的資料。

struct ContentView: View {
    var body: some View {
        let dice = Int.random(in: 1...6)
        if dice > 3 {
            return Text("大")
        } else {
            return Image(systemName: "\(dice).circle")
        }
    }    
}

可以不回傳元件

此時 body 將顯示空白畫面。

struct ContentView: View {
    var body: some View {
        if Bool.random() {
            Text("Hello, world!")
        }
    }
}
