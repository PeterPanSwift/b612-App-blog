抓取 IG 動態的 JSON 資料

IG 動態牆有許多有趣的動態資料，不過很可惜，官方並沒有提供公開的 API 讓我們抓取。

雖然如此，如果想練習解析 IG 動態的 JSON，我們可透過以下方法抓取 JSON 測試。

注意: 之前抓取 IG 資料的方法已經失效，若想直接練習，可從以下網址抓取最近退休的費神 Roger Federer 的 IG 動態 JSON

https://raw.githubusercontent.com/AppPeterPan/CutePic/main/rogerfederer.json

從瀏覽器連到 IG 網站，登入 IG 帳號

https://www.instagram.com/

在瀏覽器輸入 API 網址抓取 JSON

公開帳號的 API 網址

https://www.instagram.com/公開帳號的id/?__a=1&__d=dis

範例:

抓取小王子官方帳號的 po 文。

https://www.instagram.com/lepetitprinceofficiel/?__a=1&__d=dis

包含使用者的名字，大頭照，簡介，followers，following & po 文的數量， po 文(edge_owner_to_timeline_media)。

po 文內容包含圖片網址，like & 留言的數量等。

tag 的 API 網址

https://www.instagram.com/explore/tags/tag名稱/?__a=1&__d=dis

範例

抓取 tag 小王子語錄的 po 文。

https://www.instagram.com/explore/tags/小王子語錄/?__a=1&__d=dis

從 edge_hashtag_to_media & edge_hashtag_to_top_posts 可抓取 tag 的相關動態。

抓取下一頁的 API 網址

從 JSON 裡的 page_info 可查詢是否有下一頁的資料。比方我們利用以下網址抓取 SwiftUI tag 的相關 po 文。

https://www.instagram.com/explore/tags/swiftui/?__a=1&__d=dis

下圖的 has_next_page 為 true，表示有下一頁，end_cursor 可用來抓取下一頁的資料。

因此抓取下一頁資料的網址如下，包含 __a，__d & end_cursor 3 個參數。

https://www.instagram.com/explore/tags/swiftui/?__a=1&__d=dis&end_cursor=QVFBWV9BaWxkVmZXRWxycllldzVWMDRnV2M4aHlXc01vd04yek12YUthdkRfcUdpSERxdzV0OWhXb1pzWERRMFo0czJnZ3EzelA2aUNlTVBQUHE5c1Y1Vg==

下載瀏覽器顯示的 JSON

在瀏覽器的網址列按住 option，然後按 enter 下載 JSON。以下圖為例，我們將下載最近退休的費神 Roger Federer 的 IG 動態 JSON。

讀取專案或 playground 的檔案測試 JSON 解碼

  • 將 JSON 檔加到專案
讀取專案裡的檔案測試 JSON 解碼
為了測試 JSON 資料對應的 Decodable(或 Codable) 型別是否有問題，除了實際串接 API 抓資料，我們也可以先將 JSON 檔加入專案裡，方便沒有網路時進行測試。medium.com
  • 將 JSON 檔加到 playground

加到 Resources 資料夾下。

在 playground 讀取 JSON 的範例程式如下:

let url = Bundle.main.url(forResource: "rogerfederer", withExtension: "json")!
do {
    let data = try Data(contentsOf: url)
} catch {
    print(error)   
}
