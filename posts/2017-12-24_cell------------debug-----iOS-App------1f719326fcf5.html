cell 的高度問題 — 訓練 debug 能力的 iOS App 開發問題

App 執行後，表格的 cell 高度不夠高，圖片無法完整顯示 。

但是在 storyboard 裡，cell 的高度明明可以完整顯示圖片。

問題範例連結

AppPeterPan/TableCellHeightProblem

解答:

歡迎有興趣的朋友嘗試看看，試試能不能自己找出問題並修正。

問題原因:

表格的 cell 高度預設依據 Auto Layout 條件自動計算高度。如下圖所示，點選 table view 後，它的 Size Inspector 顯示 Row Height 的 Automatic 被勾選。然而在剛剛的例子裡，我們在 cell 上加入元件時，完全沒有設定 auto layout 的條件，因此 App 執行時，cell 的高度將變回預設的 44，造成 cell 裡的內容被切到。

解法 1:

將 Row Height 的 Automatic 取消勾選，並把 Row Height 設為我們想要的 cell 高度 172，如此 App 執行時，cell 的高度即可固定 172。

解法 2:

替 cell 上的元件加入 auto layout 條件，讓 cell 能依據條件計算高度。值得注意的，image view 的高度條件，priority 要設為 999，否則會產生衝突。
