Xcode 9 的 table view cell 預設就是自動計算高度了 !

在 Xcode 9，一行程式碼都不用寫，table view cell 的高度自動就會依據 auto layout 條件自動計算了 !

如下圖所示， Table View 元件的 Row Height & Estimate 一開始就被勾選 Automatic，果然是超自動的貼心表格 !

ps: 經過實測，在 iOS 11 & iOS 10 都可以實現。所以看來在 Xcode 9，table 預設就將高度設為 automatic，省掉我們另外從程式設定的麻煩。

倘若懷念以前沒有自動高度的表格，也可以參考以下連結，將 cell 設為固定高度。

如何讓 table view cell 固定高度
從 Xcode 9 開始，table view cell 預設搭配 Auto Layout 條件計算高度。若是沒有在 cell 設定任何 auto layout 條件，cell 將無從計算它的高度，只能採用預設的 cell 高度 44。medium.com

對剛剛的 App 範例有興趣的，可參考底下 cell 裡元件的 Auto Layout 條件和程式碼。

如圖所示，image view 和 label 都和 cell bottom margin 有著 ≥ 10 的間距。

程式碼

class IdolTableViewCell: UITableViewCell {
    @IBOutlet weak var photoImageView: UIImageView!
    @IBOutlet weak var lyricsLabel: UILabel!
}
struct Idol {
    var lyrics: String
    var imageName: String
}

class DemoTableViewController: UITableViewController {
    struct PropertyKeys {
        static let loverCell = "idolCell"
    }

    var idols = [Idol]()
    override func viewDidLoad() {
        super.viewDidLoad()
        idols.append(Idol(lyrics: """
有一段走過的路我不會忘
有一個愛過的人放在心上
""", imageName: "Penny"))
        idols.append(Idol(lyrics: """
快樂的一刻
勝過永恆的難過
黑夜過後就有日出和日落
兩個人走不會寂寞
每一刻都會珍惜
都會把握
慶幸有你愛我
""", imageName: "Joi"))
        idols.append(Idol(lyrics: """
Hey Peter Pan
帶我飛往世界的角落
緊握住手中的夢
別讓那陣風吹走
Where's Neverland
傳說中守護夢的樂園
我乘著白色翅膀飛多遠
""", imageName: "Paggy"))
}
    override func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
        return idols.count
    }
    override func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
    guard let cell = tableView.dequeueReusableCell(withIdentifier: PropertyKeys.loverCell, for: indexPath) as? IdolTableViewCell else {
        fatalError("create cell error")
    }
    let idol = idols[indexPath.row]
    cell.lyricsLabel.text = idol.lyrics
    cell.photoImageView.image = UIImage(named: idol.imageName)
    return cell
}
