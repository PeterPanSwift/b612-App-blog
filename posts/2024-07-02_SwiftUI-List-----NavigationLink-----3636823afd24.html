SwiftUI List 包多層 NavigationLink 的問題

開發 SwiftUI App 時我們常用 NavigationLink 切換頁面，不過當 List 裡有多層 NavigationLink，NavigationLink 裡包著另一個 NavigationLink 時，卻會發生特別的問題。

以下我們以 List 顯示經典的世界文學名著說明。

定義 Book。

import Foundation

struct Book: Identifiable {
    let id = UUID()
    let name: String
    let author: String
}

定義 BookRow，透過 NavigationLink 點選作者名字顯示作者介紹。

import SwiftUI

struct BookRow: View {
    let book: Book
    
    var body: some View {
        VStack(alignment: .leading) {
            HStack {
                Image(book.name)
                    .resizable()
                    .scaledToFit()
                    .frame(height: 100)
                VStack(alignment: .leading) {
                    Text(book.name)
                    NavigationLink {
                        Text("\(book.author)介紹")
                    } label: {
                        Text(book.author)
                    }
                }
                Spacer()
            }
            HStack {
                Button {
                    print("share")
                } label: {
                    Image(systemName: "square.and.arrow.up")
                }
                Button {
                    print("comment")
                } label: {
                    Image(systemName: "bubble.right")
                }
            }
        }
    }
}

定義 BookList，透過 NavigationLink 點選 BookRow 顯示書籍內容介紹。

import SwiftUI

struct BookList: View {
    let books = [
        Book(name: "小王子", author: "聖修伯里"),
        Book(name: "彼得潘", author: "詹姆斯")
    ]
    
    var body: some View {
        NavigationStack {
            List {
                ForEach(books) { book in
                    NavigationLink {
                        Text("\(book.name)介紹")
                    } label: {
                        BookRow(book: book)
                    }
                }
                
            }
        }
    }
}

結果如下，發生了一些可怕的問題。

  • 兩個向右的箭頭。
  • 點擊 BookRow 的任何地方都會跳到書籍介紹頁面，點選作者名也是跳到書籍介紹頁面。
  • 分享和留言按鈕失效，點選也是跳到書籍介紹頁面。

這些問題和 List 包多層 NavigationLink 有關，如果不是搭配 List，而是用 VStack、ScrollView、Table 包多層 NavigationLink 並不會有這些問題。

比方將剛剛的 List 改成 ScrollView，畫面上不再有向右的箭頭，點選作者名會跳到作者介紹頁面，分享和留言按鈕也能正常運作。

struct BookList: View {
    let books = [
        Book(name: "小王子", author: "聖修伯里"),
        Book(name: "彼得潘", author: "詹姆斯")
    ]
    
    var body: some View {
        NavigationStack {
            ScrollView {
                ForEach(books) { book in
                    NavigationLink {
                        Text("\(book.name)介紹")
                    } label: {
                        BookRow(book: book)
                    }
                }
                
            }
        }
    }
}

讓我們將焦點放回 List 包多層 NavigationLink 的問題。分享和留言按鈕失效的問題較好解決，以下兩種解法都可順利偵測分享和留言的點擊。

  • 解法 1: 從 NavigationLink 呼叫 buttonStyle(.plain)。
struct BookList: View {
    let books = [
        Book(name: "小王子", author: "聖修伯里"),
        Book(name: "彼得潘", author: "詹姆斯")
    ]
    
    var body: some View {
        NavigationStack {
            List {
                ForEach(books) { book in
                    NavigationLink {
                        Text("\(book.name)介紹")
                    } label: {
                        BookRow(book: book)
                    }
                    .buttonStyle(.plain)
                }
                
            }
        }
    }
}
  • 解法 2: 將 BookRow 裡的 Button 改用 onTapGesture 實現。
struct BookRow: View {
    let book: Book
    
    var body: some View {
        VStack(alignment: .leading) {
            HStack {
                Image(book.name)
                    .resizable()
                    .scaledToFit()
                    .frame(height: 100)
                VStack(alignment: .leading) {
                    Text(book.name)
                    NavigationLink {
                        Text("\(book.author)介紹")
                    } label: {
                        Text(book.author)
                    }
                }
                Spacer()
            }
            
            HStack {
                Image(systemName: "square.and.arrow.up")
                    .onTapGesture {
                        print("share")
                    }
                Image(systemName: "bubble.right")
                    .onTapGesture {
                        print("comment")
                    }
            }
        }
    }
}
