新手剛開始學習串接網路 API 解析 JSON 時，常常會遇到以下兩個問題。

搭配 AI 建立自己感興趣的 JSON API

新手剛開始學習串接網路 API 解析 JSON 時，常常會遇到以下兩個問題。

  • 範例的 API 已經沒有在維護，無法串接。
  • API 回傳的 JSON 內容不是自己感興趣的主題。

最近隨著 AI 的興起，彼得潘想到我們可以透過以下步驟快速建立自己喜歡的 JSON API。

  1. 請 AI 創作 JSON 或讀取網頁內容生成 JSON。

2. 在 GitHub 新增檔案，貼上 AI 生成的 JSON。

3. 取得 JSON API 的網址。

ps: 現在這三步都可以請 AI 一次完成了，方法可參考以下連結。

請 AI 生成 JSON API — 生成 JSON、建立 gist，取得 API 網址
透過 AI 我們可以快速生成 JSON API ，請它生成 JSON、建立 gist，取得 API 網址。medium.com

以下我們請 AI 產生 5 種常見的 JSON 例子。

  • 內容是字串的 array。
  • 第一層是 object( { } )。
  • 第一層是 array( [ ] )，array 的成員是物件( { } )。
  • 多層的 JSON，比方至少 4 層的 JSON。
  • 讀取網頁內容產生 JSON。

使用 GPT Web browsing 搜尋網路資料
最近 OpenAI 推出了讓 GPT 連網的 Web browsing 功能，只要是訂閱的用戶都可以享用。彼得潘實測後，發現連網功能的確滿方便的，有著一定的準確度，雖然它會花費較長的時間，而且有時會找不到想要的資訊。medium.com

請 AI 寫出 JSON 對應的 Swift Codable 型別

練習寫出 JSON 對應的 Codable 型別後，我們也可以請 AI 寫出 JSON 對應的 Codable 型別，瞧瞧 AI 寫得有沒有比自己好。

比方以下是 AI 從劉德華 JSON 寫出的 Codable 型別。(ps: 原始的 JSON key 有底線，因此記得要將 JSONDecoder 的 keyDecodingStrategy 設為 .convertFromSnakeCase。)

import Foundation

struct AndyLau: Codable {
    let name: String
    let romanizedName: String
    let englishName: String
    let nicknames: [String]
    let nationality: String
    let origin: String
    let birthDate: String
    let birthPlace: String
    let profession: [String]
    let languages: [String]
    let education: [String]
    let honoraryDegrees: [String]
    let religion: String
    let spouse: String
    let marriageYear: String
    let children: Int
    let musicGenres: [String]
    let debutYear: String
    let debutWorks: DebutWorks
    let representativeWorks: RepresentativeWorks

    struct DebutWorks: Codable {
        let tvShow: String
        let movie: String
        let album: String
    }

    struct RepresentativeWorks: Codable {
        let musicAlbums: [String]
        let songs: [String]
        let movies: [String]
    }
}
