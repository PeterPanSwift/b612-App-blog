從 Android Studio 上傳程式到 GitHub 和用 commit 記錄修改

Android Studio 內建版本管理功能，因此我們可以方便地對專案進行 git 操作。以下文章我們將針對兩個部份做簡單地說明。

  • 上傳程式到 GitHub。
  • 用 commit 記錄修改的程式並上傳到 GitHub。

上傳程式到 GitHub

  • 從 Find Action 點選 Share Project on GitHub。
  • 輸入 Repository 的名稱(預設等於專案名稱)，勾選是否要 Private。

交作業的同學記得取消 Private 的勾選。

  • 設定 Android Studio 連結的 GitHub 帳號。

若是還沒設定連結的 GitHub 帳號，請先點選右下的 Add account，點選 Log In via GitHub。

此時會打開網頁，連到 JetBrains 頁面。請點選 Authorize in GitHub。

點選 Authorize JetBrains。

連結成功。

回到 Android Studio，下方的 Share by 出現了連結的 GitHub 帳號。

  • 點選 Share 上傳。

點選 Add。

成功上傳 ! 右下方出現通知 Successfully shared project on GitHub。

點選通知裡的連結將開啟專案的 GitHub 網頁，從上方的網址列可看到網址。

ps: 之後想從 Android Studio 打開專案的 GitHub 網頁，則可從專案的右鍵選單點選 Open In > GitHub。

剛剛連結 GitHub 帳號的動作只須設定一次，未來新的專案上傳到 GitHub 時將自動綁定之前連結的帳號。

之後想新增或刪除連結的 GitHub 帳號，可從 Settings 的 GitHub 分頁設定。

用 commit 記錄修改的程式並上傳到 GitHub

將專案的程式上傳到 GitHub 後，我們可能還會做修改。

當程式修改到某個段落時我們可執行 git 的 commit 做記錄。如下圖所示，切換到 Commit 分頁，Changes 的下方將列出目前被修改的相關檔案。

勾選要記錄的檔案，在 Commit Message 框框輸入訊息說明做了哪些修改，然後再點選 Commit and Push。如果想請 AI 幫忙，也可以點選 Gemini 圖示，請 Gemini 生成訊息。

點選 Push 上傳到 GitHub。
