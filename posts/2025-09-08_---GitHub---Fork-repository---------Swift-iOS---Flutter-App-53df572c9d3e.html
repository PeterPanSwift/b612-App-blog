之前有個困擾彼得潘許久的難題，如何找出某個 App 謎題的同學作業 repository 清單呢 ?

彼得潘請教 AI 後，終於找到可行的方法，解答就在 GitHub 的 Fork repository 功能。

以下彼得潘以最適合同學練習的第一個題目，#242 使用 AI 創作人生第一個 App(Vibe coding) 為例說明。

#242 使用 AI 創作人生第一個 App(Vibe coding)
彼得潘在教 SwiftUI、UIKit、Flutter 程式時，會出很多很多的作業。不過第一堂課的作業往往是最難想的，因為剛上完第一堂課的同學會的東西還不多，能做的東西有限。medium.com

老師建立作業的 GitHub Repository

首先彼得潘從 Xcode 建立 SwiftUI App 專案，取名為 CreatingMyFirstAppWithAISwiftUI242，然後透過 Xcode 的 New Remote 將專案上傳到 GitHub，Repository Name 設為 Creating-My-First-App-With-AI-SwiftUI-242，owner 選擇 Swift B612 Neverland。

GitHub 的 Swift-B612-Neverland 下新增了 repository Creating-My-First-App-with-AI-SwiftUI-242。

https://github.com/Swift-B612-Neverland/Creating-My-First-App-with-AI-SwiftUI-242

幫 repository 加上 README，顯示作業的說明連結和精美插圖。

同學 fork(複製)作業的 GitHub Repository 進行開發

有 GitHub 帳號的同學都可連到剛剛的 repository 頁面點選右上的 Fork 複製專案到自己帳號底下。

接下來輪到認真的 AppPeterPan 同學登場了，他認真地點選 Fork。

然後再認真地點選 Create fork。

認真的 AppPeterPan 同學成功在自己帳號下生成 repository Creating-My-First-App-with-AI-SwiftUI-242。

https://github.com/AppPeterPan/Creating-My-First-App-with-AI-SwiftUI-242

認真的 AppPeterPan 同學準備開始寫作業了，他從 Xcode 的 Quick Actions 點選 Clone 抓取自己 GitHub 上的 Creating-My-First-App-with-AI-SwiftUI-242。

輸入 repository 的網址，然後點選 Clone。

選擇儲存的位置，然後點選 Clone。

AppPeterPan 同學認真地利用 Xcode AI 完成打地鼠 App。

透過 Xcode 的 commit & push 將修改上傳到 GitHub。

查詢同學們開發的 App

現在我們可從 Swift-B612-Neverland 的 repository Creating-My-First-App-with-AI-SwiftUI-242 查詢同學們開發的 App。

點選右上的 Insights。

從 Forks 分頁可看到同學的 repository 清單。

ps: 必須 Fork 後有做過 commit & push 才會出現在清單上。
