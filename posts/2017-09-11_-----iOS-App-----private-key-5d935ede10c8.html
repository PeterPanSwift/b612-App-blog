輸出上架 iOS App 必備的 private key

想要上架 iOS App，實現躺著賺錢的夢想，有一個必備的材料，iOS Distribution Certificate。如果沒有它，你的 App 是不可能上架的 !

但是當你的 App 是多人合作，在多台不同 Mac 開發時，問題就來了。只有安裝 iOS Distribution Certificate 的 Mac 才能生成可以上架的 App，但一個開發帳號最多只能產生 3 個 iOS Distribution Certificate，因此你不能每台 Mac 想上架 App 時就浪費地生一個新的 Distribution Certificate。你要養成節檢的好習慣，多台 Mac 共用一個 Certificate。

從 Xcode 的 Manage Certificate 頁面，點選 + 可新增 Apple Distribution。

然而當 Distribution Certificate 達到三個時，將無法再新增，出現錯誤 There is a problem with the request entity。You already have a current Distribution certificate or a pending certificate request.

那如果從 Xcode 或 Apple 開發網站下載 iOS Distribution Certificate 呢 ? 令人難過的，不行 ! 因為你少了 跟 Certificate 搭配的 private key。Apple 為了安全的考量，將 private key 存在當初生成 Certificate 的 Mac 上。如此除非 Mac 的主人主動從它的 Mac 輸出 private key 給你，你才能搭配 Distribution Certificate 製作上架的 App。

如下圖所示，從 Xcode 下載的 Distribution Certificate 顯示少了 private key 的紅色錯誤。

因此，接下來我們就來瞧瞧從 Xcode 輸出 private key 的兩個方法:

方法1: 輸出 developer profile。

此方法輸出的檔案將包含開發帳號相關的 certificate, private key 和 profile，十分方便。換句話說，如果你買了一台新 mac，只要安裝這個檔案，就具備開發上架 iOS App 需要的所有材料。

打開 Xcode Preferences 的 Accounts 頁面，選擇 Apple ID 後，點選左下角的齒輪 button，選擇 Export Apple ID and Code Signing Assets。

設定輸出檔案的名稱和安裝時輸入的密碼。

生成的 developer profile.developerprofile 包含搭配 private key 的 certificate 和 profile。

如果你手動將 profile.developerprofile 的附檔名加上 .zip，即可將它解壓縮，看到它裡面包含的檔案。

Identities 資料夾下的 p12 檔即是包含 private key 的 certificate。

方法2: 輸出 certificate。

此方法比較單純，它只會輸出包含 private key 的 certificate。

在 Accounts 頁面，選擇 Apple ID 後，點選右下角的 Manage Certificates。

選擇想要輸出的 certificate 後，從右鍵選單選擇 Export Certificate。

設定輸出檔案的名稱和安裝時輸入的密碼。

生成包含 private key 的 certificate。

之後想在另一台 Mac 製作上架的 App，只要安裝以上兩個方法生成的檔案，輸入密碼即可。
