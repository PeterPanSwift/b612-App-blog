為什麼 ForEach & List 需要區分 array 裡的成員 ?

當我們使用 SwiftUI 的 ForEach 或 List 搭配 array 設計畫面時，必須讓它能區分 array 裡的成員，比方以下例子建立 List 時傳入參數 id \.name，以專輯的名字當區分的 id。

struct Album {
   var name: String
   var singer: String
}
struct ContentView: View {
    @State var albums = [Album(name: "謝謝你的愛", singer: "劉德華"), Album(name: "忘情水", singer: "劉德華"), Album(name: "天意", singer: "劉德華")]
    @State var number = 0
    
    var body: some View {
        NavigationView {
            List(albums, id: \.name) { (album) in
                AlbumText(album: album)
            }
        }
    }
}
struct AlbumText: View {
    var album: Album
    var body: some View {
        print("create \(album.name)")
        return Text("\(album.name) \(album.singer)")
            .padding()
            .background(Color.yellow)
            .clipShape(Capsule())
    }
}

但是為什麼它需要區分 array 裡的成員呢 ? 這主要是為了有效率地管理 List 裡的元件。接下來讓我們以 App 常見的功能新增東西說明，我們利用 navigationBarItems 在畫面的右上角加個 button Add，點選後增加 array 裡的專輯，更新表格顯示的專輯清單。

List(albums, id: \.name) { (album) in
    AlbumText(album: album)
}
.navigationBarItems(trailing: Button("Add", action: {
    self.number += 1
    self.albums.insert(Album(name: "第 \(self.number) 次相思成災", singer: "劉德華"), at: 0)
}))

當我們點選 Add 時，state property albums 將增加專輯，因此會觸發 body 裡的 List 重新生成。

但請放心，它不會傻傻地重新生成表格裡每個專輯的 AlbumText，它會保留原本已經生成的 AlbumText，只為新加入的專輯產生 AlbumText。 而我們生成 List 時傳入的參數 id 將幫助 List 區分 array 裡的成員，因此它知道新加入的專輯需要生成 AlbumText。

如下圖的 Console 截圖所示，當我們點選 Add 增加新專輯時，只會產生新專輯的 AlbumText，不會重新生成之前的 AlbumText，因此 create 謝謝你的愛 只有一開始印出一次。

如果 array 裡成員的 id 有重覆，將會讓 List 錯亂。比方我們加入張學友的忘情水，但因為我們以專輯名當 id，所以它會分不出劉德華 & 張學友的忘情水，最後全部顯示成同一個專輯。

.navigationBarItems(trailing: Button("Add", action: {
    self.albums.insert(Album(name: "忘情水", singer: "張學友"), at: 0)
}))

如下圖所示，第一次點選 Add 時，全部顯示劉德華的忘情水，張學友輸入。但風水輪流轉，第二次點選 Add 時，變成劉德華輸了，全部顯示張學友的忘情水。
