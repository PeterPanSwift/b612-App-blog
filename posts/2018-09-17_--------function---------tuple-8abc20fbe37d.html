沒有 -> 的 function 其實回傳了空的 tuple

定義 function 時，當我們加上 -> ，即表示 function 會回傳東西，比方
-> String 會回傳字串。

func sing() -> String {
   return "我願變成童話裡，你愛的那個天使"
}
let lyrics = sing()

然而沒有 ->，就表示它沒有回傳東西嗎 ?

就好像 FB 沒寫感情狀態，卻可能有男女朋友一樣，沒有 -> 的 function 其實有回傳東西呀。它會回傳空的 tuple，()。

如下圖所示，事實擺在眼前，lyrics 變成了 ()。

甚至我們還可以用 if 比對，檢查它是否真的是 ()。

if lyrics == () {
  print("我是()")
}

由於沒有 -> 的 function 會回傳 ()，所以搭配 optional chaining 時，它會回傳的東西型別是 ()?，因此我們可以由它是否為 nil 判斷 function 是否有執行。

class Baby {
   func sing() {
      print("我願變成童話裡，你愛的那個天使")
   }
}
var cuteBaby: Baby?
if cuteBaby?.sing() != nil {
   print("寶寶唱情歌")
}
cuteBaby = Baby()
if cuteBaby?.sing() != nil {
   print("寶寶唱情歌")
}

另外我們時常在 swift 看到的 Void 其實就是空的 tuple，因為 swift 利用 typealias 發明 Void，說它等同於 ( )。

public typealias Void = ()

因此以下四種寫法其實是一樣的意思。

func sing() {
   print("我願變成童話裡，你愛的那個天使")
}
func sing() -> () {
   print("我願變成童話裡，你愛的那個天使")
}
func sing() -> Void {
   print("我願變成童話裡，你愛的那個天使")
}

func sing() -> Swift.Void {
   print("我願變成童話裡，你愛的那個天使")
}
