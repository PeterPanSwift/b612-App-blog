Separator None 造成的 cell 高度問題

當 cell 的高度由 auto layout 決定時，我們可在 cell 裡放入一個 view，然後設定 view 的高度條件控制 cell 的高度。

然而當 table view 的 Separator 設為 None，view 的高度條件設為 0 時，卻會發生奇怪的現象 !

App 執行後，Cell 依然有預設的高度 44，而不是我們期待的高度 0。

此時 Xcode 的錯誤訊息如下

TableCellHeight[25493:1183731] [LayoutConstraints] Unable to simultaneously satisfy constraints.
Probably at least one of the constraints in the following list is one you don't want.
Try this:
(1) look at each constraint and try to figure out which you don't expect;
(2) find the code that added the unwanted constraint or constraints and fix it.
(
"<NSLayoutConstraint:0x608000093600 UIView:0x7fc86d111170.height == 0   (active)>",
"<NSLayoutConstraint:0x608000093830 V:|-(0)-[UIView:0x7fc86d111170]   (active, names: '|':UITableViewCellContentView:0x7fc86d110b50 )>",
"<NSLayoutConstraint:0x608000093920 V:[UIView:0x7fc86d111170]-(0)-|   (active, names: '|':UITableViewCellContentView:0x7fc86d110b50 )>",
"<NSLayoutConstraint:0x600000093790 'UIView-Encapsulated-Layout-Height' UITableViewCellContentView:0x7fc86d110b50.height == 44   (active)>"
)
Will attempt to recover by breaking constraint
<NSLayoutConstraint:0x608000093600 UIView:0x7fc86d111170.height == 0   (active)>

但是，只要表格的 Separator 不是 None，比方設為 Default，即可正常顯示高度 0 的 cell。

因此，如果想要 cell 高度 0，而且不顯示 Separator 的話，可採用另一個方法，將 Separator 的顏色設為 Clear Color。
