和 AI 聯手開發 iOS App 的 table view 頁面 — Netflix 影集列表

初學者常常會覺得製作 iOS UIKit App 的表格頁面有點困難，因為必須理解 protocol，data source 和 table view 的相關技術才能從無到有做出表格頁面。

若是剛學會表格的相關技術，但還不是很熟練，開發時常常卡關，不妨先試試跟 AI 聯手開發表格頁面。比較難的程式交給 AI，有藝術細胞的我們負責用 storyboard 設計畫面。等聯手開發能夠順利完成，再來練習自己獨力完成表格頁面。

以下為彼得潘跟 GPT 4 聯手製作 Netflix 影集列表的過程。通常 AI 不會一次實現我們的需求，所以我們也要在過程中和 AI 溝通，請它進行相關的調整。(ps: 練習的同學請找自己有興趣的主題，不要學彼得潘做影集列表呀。)

用 UITableViewController 製作 Netflix 影集列表，顯示五部影集，請使用真實的影集資料，比方后翼棄兵。我已經在 storyboard 設計好畫面，cell 裡包含影集名稱，封面圖片，演員清單資訊，簡介。以下請幫我寫出相關的程式。

將 Series 的 image 改成儲存圖片名字，cast 儲存演員名稱的 array

struct Series {
    let name: String
    let imageName: String
    let cast: [String]
    let description: String
}

        Series(name: "六人行", imageName: "friends", cast: ["珍妮佛·安妮斯頓", "寇特尼·考克斯", "麗莎·庫卓"], description: "六位好友在紐約市的生活中尋找成功和幸福，他們一起經歷了愛情、工作和親情的挑戰。這部經典情景喜劇展示了他們之間的友誼和生活中的幽默時刻。")
    ]
    
    
    override func viewDidLoad() {
        super.viewDidLoad()
    }
    
    // MARK: - Table view data source
    
    override func numberOfSections(in tableView: UITableView) -> Int {
        return 1
    }
    
    override func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
        return seriesList.count
    }
    
    override func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
        let cell = tableView.dequeueReusableCell(withIdentifier: "SeriesCell", for: indexPath) as! SeriesTableViewCell
        
        let series = seriesList[indexPath.row]
        cell.seriesNameLabel.text = series.name
        cell.seriesImageView.image = UIImage(named: series.imageName)
        cell.seriesCastLabel.text = series.cast.joined(separator: ", ")
        cell.seriesDescriptionLabel.text = series.description
        
        return cell
    }
    
}
