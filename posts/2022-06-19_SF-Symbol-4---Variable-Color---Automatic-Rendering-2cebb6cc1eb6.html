SF Symbol 4 的 Variable Color & Automatic Rendering

從 iOS 16 的 SF Symbol 4 (搭配 Xcode 14)開始，SF Symbol 新增了兩個很棒的功能，Automatic Rendering & Variable Color。

Automatic Rendering: 自動採用的 rendering mode

SF Symbol 4 的 symbol 變聰明了。當我們沒有指定 rendering mode 時，它會自動採用最適合自己的 rendering mode。

利用 SF Symbols App 可查詢 symbol 自動採用的 rendering mode。以下圖的 square.and.arrow.up.trianglebadge.exclamationmark 為例，Rendering 選擇 Automatic 後，第一個 icon 下出現小圓點，因此它自動採用的 rendering mode 是 Monochrome。

ps: 上圖 4 個 icon 的順序依序為 Monochrome，Hierarchical，Palette & Multicolor。對 rendering mode 不熟的朋友可參考以下連結。

依據百分比呈現強度或進度變化的 Variable Color

透過 Variable Color 我們可以調整百分比，為 symbol 加上強度或進度的變化。這是什麼意思呢 ? 以下我們打開 SF Symbols App，實際看一些例子說明。

在側邊欄選擇 Variable，查看支援 Variable Color 的 symbol。只有需要呈現強度或進度變化的 symbol 才有支援 Variable Color。

以下我們以熟悉的 wifi symbol 為例子。wifi 訊號有不同的強度，因此很適合搭配 Variable Color。

點選下圖的紅色框框打開 variable color，拖曳 slider 可查看不同百分比的變化。

0% 是完全沒訊號，因此 wifi 是可憐的 0 格。

從 wifi symbol 的圖案可看出它的強度分三層，因此 1% ~ 33% 是 1 格。34% ~ 67% 是 2 格，68% ~ 100% 是 3 格。

等等，幾％對應幾格是怎麼決定的呢 ? wifi 強度有三層，由於 100/3 不會整除，第一格應該到 33.3 % ，第二格應該到 66.6 % 呀。不過在這裡我們可指定的百分比只能是整數，因此要用四捨五入的算法，第一格是 1% ~ 33%，第二格是 34% ~ 67%，第三格是 68% ~ 100%。

支援 Variable Color 的 symbol 會依據強度或進度分層，剛剛的 wifi symbol 分成 3 層，但也有分更多層的，比方下圖的 timelapse 分成 24 層(ps: 彼得潘數到眼睛要花掉了)。

若想要看得更清楚，也可以點選下圖的紅色框框進入 gallery 模式，放大展示 symbol。

接下來讓我們看看如何在 SwiftUI & UIKit 設定 SF Symbol 的百分比。

SwiftUI 的 Variable Color

從參數 variableValue 傳入百分比，傳入的數字範圍是 0 ~ 1，因此 0.1 代表 10%。

struct ContentView: View {
    var body: some View {
        HStack(spacing: 20) {
            Image(systemName: "wifi", variableValue: 0.1)
            Image(systemName: "wifi", variableValue: 0.4)
            Image(systemName: "wifi", variableValue: 0.7)
        }
        .font(.largeTitle)
        .foregroundColor(.purple)
    }
}

UIKit 的 Variable Color

  • 從 interface builder

從 value 設定百分比，傳入的數字範圍是 0 ~ 1，因此 0.1 代表 10%。

  • 從程式

從 variableValue 傳入百分比，傳入的數字範圍是 0 ~ 1，因此 0.1 代表 10%。

let wifiImage = UIImage(systemName: "wifi", variableValue: 0.1)
let wifiImageView = UIImageView(image: wifiImage)
view.addSubview(wifiImageView)

依強度或進度順序變化的動畫

由於 Variable Color 可依強度或進度變化，而且有順序性，因此很適合做成動畫。以下我們分別示範利用 TimeLineView & Timer 實現動畫效果。

  • SwiftUI 搭配 TimeLineView
struct ContentView: View {
    
    var body: some View {
        TimelineView(.periodic(from: .now, by: 0.3)) { context in
            BubbleView(date: context.date)
        }
    }
}

struct BubbleView: View {
    @State private var percent: Double = 0
    let date: Date
    
    var body: some View {
        Image(systemName: "ellipsis.bubble.fill", variableValue: percent)
            .font(.system(size: 100))
            .onChange(of: date) {
                percent += 1.0 / 3.0
                if percent > 1 {
                    percent = 0
                }
            }
    }
}

symbol ellipsis.bubble.fill 有三個小圓點，可分三層，因此我們以 percent += 1.0 / 3.0 變化 percent。其它的 symbol 可以此類推，比方 speaker.wave.2.fill 可分兩層，因此我們以 percent += 1.0 / 2.0 變化 percent。

  • UIKit 搭配 Timer
class ViewController: UIViewController {
    var percent: Double = 0
    var timer: Timer?
    
    override func viewDidDisappear(_ animated: Bool) {
        super.viewDidDisappear(animated)
        timer?.invalidate()
        
    }

    override func viewDidLoad() {
        super.viewDidLoad()
        
        let bubbleImage = UIImage(systemName: "ellipsis.bubble.fill", variableValue: percent)
        let bubbleImageView = UIImageView(image: bubbleImage)
        bubbleImageImageView.frame = CGRect(x: 100, y: 100, width: 100, height: 100)
        view.addSubview(bubbleImageView)
        
        timer = Timer.scheduledTimer(withTimeInterval: 0.3, repeats: true) { [self] _ in
            percent += 1.0 / 3.0
            if percent > 1 {
                percent = 0
            }
            bubbleImageView.image = UIImage(systemName: "ellipsis.bubble.fill", variableValue: percent)
        }
    }
}

symbol ellipsis.bubble.fill 有三個小圓點，可分三層，因此我們以 percent += 1.0 / 3.0 變化 percent。其它的 symbol 可以此類推，比方 speaker.wave.2.fill 可分兩層，因此我們以 percent += 1.0 / 2.0 變化 percent。
