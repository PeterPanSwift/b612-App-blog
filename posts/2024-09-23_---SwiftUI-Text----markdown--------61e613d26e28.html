使用 SwiftUI Text 搭配 markdown 實現連結按鈕

開發 SwiftUI App 時，使用 Link 元件可以做出點擊打開連結的按鈕。

SwiftUI 點選打開連結的 Link 按鈕
點選按鈕打開連結幾乎是每個 App 都會看到的功能，比方打開網頁，地圖，App，電話等。在 iOS 14，SwiftUI 提供了方便的 Link 按鈕幫我們快速實現這樣的功能。medium.com

不過最近彼得潘發現有個更簡單的寫法。SwiftUI 的 Text 從 iOS 15 開始支援部分的 Markdown 語法，因此我們可用 Text 結合 Markdown 的連結語法實現連結按鈕。

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(.iPhoneProMax16)
                .resizable()
                .scaledToFit()
            Text("[iPhone](https://www.apple.com/iphone/)")
            
        }
    }
}

Markdown 連結語法的格式為 [文字](網址)，因此 [iPhone](https://www.apple.com/iphone/) 將顯示標題為 iPhone 的按鈕，點擊後開啟 Safari 顯示網址 https://www.apple.com/iphone/ 的網頁。

值得注意的，preview 無法開啟 Safari，因此要從模擬器或實機操作。

當 Text 成為連結時，它的顏色預設將和 App 的 AccentColor 一致，可透過 modifier tint 修改顏色。

Text("[iPhone](https://www.apple.com/iphone/)")
    .tint(.orange)

連結不見得是開啟網頁，某些特殊的網址將開啟特別的 App，比方 maps.apple.com 將開啟 Apple Map。

Text("[iPhone](https://maps.apple.com/?address=臺北市信義區信義路5段7號)")

值得注意的，若在 ( ) 裡使用字串置換傳入網址，點擊時將無法打開連結。

struct ContentView: View {
    let name = "iPhone"
    let url = "https://www.apple.com/iphone/"
    
    var body: some View {
        Text("[\(name)](\(url))")
    }
}

( ) 裡傳入字串置換的寫法會讓 Markdown 解析網址出問題，此時必須改成以下寫法，使用 AttributedString 生成 Markdown 的內容。

struct ContentView: View {
    let name = "iPhone"
    let url = "https://www.apple.com/iphone/"
    
    var body: some View {
        Text(try! AttributedString(markdown: "[\(name)](\(url))"))
    }
}
