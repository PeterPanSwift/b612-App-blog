從 Xcode 設定 App 執行的語言環境

開發 App 時，我們常常需要測試 App 在不同語言環境的執行結果。除了手動調整 iOS 的系統語言，其實有個不影響系統語言的快速方法。

我們可以從 Xcode 編輯 Scheme 設定 App 執行的語言環境，步驟如下。

ps: 此設定只會影響從 Xcode 啟動 App 時的語言環境，不影響系統的語言環境。若是從桌面點選 App icon 啟動 App，仍然會採用系統的語言。

  • 從 Quick Actions 點選 Edit Scheme。
  • 切換到 Run 的 Options 分頁，找到 App Language。
  • 從 App Language 選擇 App 執行的語言環境。

比方選擇 Chinese, Traditional，即使 iPhone 的語言設為英文，從 Xcode 啟動 App 時將變成中文環境。

  • 測試 App。
struct ContentView: View {
    var body: some View {
        HStack {
            Image(.rose)
                .resizable()
                .scaledToFit()
            Text("因為你為你的玫瑰花了那麼多時間，它才變得那麼重要")
        }
        .padding()
    }
}

用以上的 SwiftUI 程式為例，當語言環境是中文、英文等由左到右的語言時，HStack 裡的元件將由左而右排列，因此圖在左字在右。

當語言環境是阿拉伯等由右到左的語言時，HStack 裡的元件將由右而左排列，因此圖在右字在左。

ps: 為了方便測試右到左的環境，App Language 可直接選擇 Right-to-Left Pseudolanguage。
