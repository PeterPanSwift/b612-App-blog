模擬器無法執行 App 的 architecture arm64 問題

開發 iOS App 時，我們時常會加入套件快速實現某些功能。不過有時套件也會造成一些問題，比方使用 CocoaPods 安裝 GoogleMaps 或 GooglePlaces 套件後，App 將無法在模擬器執行，出現以下錯誤訊息。

Building for iOS Simulator, but linking in object file built for iOS, file '/Users/peterpan/Desktop/DemoMap/Pods/GoogleMaps/Maps/Frameworks/GoogleMaps.framework/GoogleMaps' for architecture arm64

ps: 新版的 GoogleMaps 或 GooglePlaces 已可透過 SPM 安裝，不需要透過 CocoaPods。

此問題可透過將 Xcode 以 Open using Rosetta 方式打開解決，方法如下。

找到 Xcode App，從右鍵選單點選 Get Info

勾選 Open using Rosetta，然後重新啟動 Xcode。

啟動後若發現還是有問題，可先點選 Xcode menu 的 Product > Clean Build Folder，讓它之後可以重新 build App。
