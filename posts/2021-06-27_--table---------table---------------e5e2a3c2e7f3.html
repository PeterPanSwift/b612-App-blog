當 table 的內容沒有超出 table 畫面時，不顯示多餘的分隔線

以下分別介紹 SwiftUI & UIKit 的做法。

SwiftUI

不用做任何事，預設不會顯示多餘的分隔線。

struct ContentView: View {

    var body: some View {
        List {
            HStack(spacing: 20.0) {
                Image("Apple Regent Street")
                    .resizable()
                    .scaledToFill()
                    .frame(width: 120, height: 120)
                    .clipped()
                VStack(alignment: .leading) {
                    Text("Regent Street")
                    Text("London")
                        .foregroundStyle(.secondary)
                }
            }
            
            HStack(spacing: 20.0) {
                Image("Apple Tower Theatre")
                    .resizable()
                    .scaledToFill()
                    .frame(width: 120, height: 120)
                    .clipped()
                VStack(alignment: .leading) {
                    Text("Tower Theatre")
                    Text("Los Angeles")
                        .foregroundStyle(.secondary)
                }
            }
        }
    }
}

UIKit

iOS 15 以上

不用做任何事，預設不會顯示多餘的分隔線。

舊版

什麼事都不做的話，將看到多餘的分隔線唷。

解法

加入 table view 的 footer，並將 footer 的高度設為 0。footer 的目的只是為了消除多餘的分隔線，不需要顯示，因此將它的高度設為 0。

  • 從 Interface Builder 加入

從 Objects Library 拖曳 view 到畫布 table view 的最後一個 cell 後面。

此時 view 將成為 Table View 底下的最後一個成員，成為 Table View 的 footer。

接著將 view 的高度設為 0，讓它成為看不到的 footer。

  • 從程式設定
tableView.tableFooterView = UIView()
