Xcode 11.4 的 SwiftUI Create Preview

Xcode 11.4 多了一個方便的 SwiftUI Create Preview 功能，幫我們自動生成顯示 preview 畫面的型別。接下來就讓我們看看如何操作吧。

定義包裝 WKWebView 的型別 WebView

import SwiftUI
import WebKit
struct WebView: UIViewRepresentable {
    typealias UIViewType = WKWebView
    
    let urlString: String
    
    func makeUIView(context: UIViewRepresentableContext<WebView>) -> WKWebView {
        let webView = WKWebView()
        if let url = URL(string: urlString) {
            let request = URLRequest(url: url)
            webView.load(request)
        }
        
        return webView
    }
    
    func updateUIView(_ uiView: WKWebView, context: UIViewRepresentableContext<WebView>) {
    }
}

點選 button Create Preview

由於尚未定義顯示 preview 畫面的型別，所以我們無法在右邊看到 preview 畫面。此時 Xcode 貼心地提供 button Create Preview，點選它即可自動生成顯示 preview 畫面的型別。

在變數 previews 的 { } 裡回傳 WebView

Create Preview 會幫我們定義型別 WebView_Previews，不過它沒那麼聰明，它不明白我們想顯示什麼，所以我們得自己產生 WebView。

struct WebView_Previews: PreviewProvider {
    static var previews: some View {
        WebView(urlString: "https://medium.com/@apppeterpan")
    }
}
