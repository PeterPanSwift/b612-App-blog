如何從程式控制 Interface Builder 的畫面元件 ? 將元件的類別設成自訂類別

開發 iOS App 時，透過畫面製作工具 Interface Builder 的幫忙，我們可以從 storyboard 和 xib 快速製作 App 畫面。但是若希望 App 有特別的功能，還是需要寫程式呀。

透過 IBAction 和 IBOutlet，我們可以設定元件事件觸發的 function 和存取元件。但在這之前，還有一個重要的準備動作，我們得先將元件的類別設成自訂的類別，然後才能在類別裡寫程式。不然原本元件的類別都是 iOS SDK 內建的類別，我們是無法修改的。

連結 IBAction 和 IBOutlet 的步驟說明
以 Yaya 同學的小費 App 為例medium.com

依不同的情境，可分為以下幾種常見的類別設定:

view controller

我們看到的每一個 App 畫面，都是由 controller 控制。因此只要將 controller 設為自訂的類別，即可在 controller 裡寫程式控制畫面。

(1) 繼承 UIViewController

class TrueLoveViewController: UIViewController

(2) 繼承 UITableViewController

class MealTableViewController: UITableViewController
cell

table view 的 cell 若想拉 outlet，必須設定 cell 的類別。(collection view 也一樣)

繼承 UITableViewCell

class MealTableViewCell: UITableViewCell
Objects Library 的其它元件

我們在 Objects Library 看到的所有元件，都可以設成自訂的類別，在類別裡撰寫相關的程式。

比方 button 元件可以設為繼承自 UIButton 的 LoginButton 類別。

xib 的 File’s Owner

想控制 xib 裡的元件，除了自訂 UI 元件的類別，也可以將 File’s Owner 設成自定類別，如此即可從 File’s Owner 寫程式控制。
