iOS 10 存取使用者私密資料都要加上 usage description

Apple 十分重視使用者的個人隱私，在 iOS 10 更是做到滴水不漏的地步。我們開發的App想要存取使用者私密資料時，一定要在 App 的 Info 頁面新增相關的 usage description，如此 App 執行，想要取用使用者的資料時，才能跳出 usage description 設定的詢問訊息，徵求使用者的同意。

如果想偷偷地存取使用者的資料，沒有設定 usage description，那會怎麼樣呢? 人在做，天在看，沒有乖乖設定 usage description 的後果很可怕，結局只有死路一條，App 閃退 !

以下我們就不怕死地以存取使用者的照片為例實驗吧:

let imagePicker = UIImagePickerController()
imagePicker.sourceType = .photoLibrary
self.present(imagePicker, animated: true, completion: nil)

以上三行程式碼利用我們熟悉的 UIImagePickerController 物件存取使用者的照片。但是很不幸地，當 App 執行這三行程式碼後，馬上倒地閃退 !

閃退並不可恥，知錯能改，善莫大焉。讓我們趕緊看看 Console 的錯誤訊息。

This app has crashed because it attempted to access privacy-sensitive data without a usage description. The app’s Info.plist must contain an NSPhotoLibraryUsageDescription key with a string value explaining to the user how the app uses this data.

Xcode 貼心地告訴我們 App 閃退是因為存取私密資料，沒有設定 usage description，而且她連解決的方法都說了，要我們在 Info.plist 補上 NSPhotoLibraryUsageDescription 。

哇，NSPhotoLibraryUsageDescription 那麼長，應該很容易打錯字吧。別怕，Xcode 早為我們設想好，切換到 App 的 Info 頁面，點選 + 新增一行，在下拉選單即可看到 usage description 的相關欄位，凡以 Privacy 開頭的都是。Xcode 把所有需徵求使用者同意，需設定 usage description 的都陳列在選單裡。

我們的目的是存取照片，所以目標很明顯，我們該選的是 Privacy — Photo Library Usage Description。 Value 則輸入為什麼要存取照片的理由，為了提高使用者同意的機率，我們只好說善意的謊言，”你的照片將被用於男神女神選拔大賽”。只要有心，人人都能成為男神女神。

加了 usage description 後，App 終於不再閃退，存取照片前跳出徵求使用者同意的訊息。
