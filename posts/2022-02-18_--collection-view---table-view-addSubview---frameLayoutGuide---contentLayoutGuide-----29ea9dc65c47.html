從 collection view 或 table view addSubview 和 frameLayoutGuide / contentLayoutGuide 的關係

當我們用 collection view 或 table view 製作 iOS App 畫面時，大部分都是從 cell 跟 header 設計內容，因此很少會直接從 collection view 或 table view 呼叫 addSubview 貼上另一個 view。

不過既然是很少，表示還是有機會，以下我們以 table view 為例子，說明從 table view 呼叫 addSubview 貼上另一個 view 時，使用 tableView.frameLayoutGuide & tableView.contentLayoutGuide 設定 auto layout 的差異。

假設 table 的畫面如下，navigation bar 是透明的。

在 TableViewController 的 viewDidLoad 裡加入黃色的 contentLayoutGuideLabel & 綠色的 frameLayoutGuideLabel。主要差異在於 contentLayoutGuideLabel 的上邊界跟 tableView.contentLayoutGuide 的上邊界對齊，frameLayoutGuideLabel 的上邊界跟 tableView.frameLayoutGuide 的上邊界對齊。

class TableViewController: UITableViewController {
   override func viewDidLoad() {
        super.viewDidLoad()
        let contentLayoutGuideLabel = UILabel()      contentLayoutGuideLabel.translatesAutoresizingMaskIntoConstraints = false
        contentLayoutGuideLabel.backgroundColor = .yellow
        contentLayoutGuideLabel.text = "contentLayoutGuide"
        tableView.addSubview(contentLayoutGuideLabel)
       contentLayoutGuideLabel.topAnchor.constraint(equalTo: tableView.contentLayoutGuide.topAnchor).isActive = true      contentLayoutGuideLabel.leadingAnchor.constraint(equalTo: tableView.frameLayoutGuide.leadingAnchor).isActive = true       contentLayoutGuideLabel.trailingAnchor.constraint(equalTo: tableView.frameLayoutGuide.trailingAnchor).isActive = true       contentLayoutGuideLabel.heightAnchor.constraint(equalToConstant: 40).isActive = true
        
        let frameLayoutGuideLabel = UILabel()       frameLayoutGuideLabel.translatesAutoresizingMaskIntoConstraints = false
        frameLayoutGuideLabel.backgroundColor = .green
        frameLayoutGuideLabel.text = "frameLayoutGuideLabel"
        tableView.addSubview(frameLayoutGuideLabel)
        frameLayoutGuideLabel.topAnchor.constraint(equalTo: tableView.frameLayoutGuide.topAnchor).isActive = true         
     frameLayoutGuideLabel.leadingAnchor.constraint(equalTo: tableView.frameLayoutGuide.leadingAnchor).isActive = true       frameLayoutGuideLabel.trailingAnchor.constraint(equalTo: tableView.frameLayoutGuide.trailingAnchor).isActive = true       frameLayoutGuideLabel.heightAnchor.constraint(equalToConstant: 40).isActive = true
        
        
    }
}

結果

說明

  • frameLayoutGuideLabel 將對齊螢幕頂端，因為 tableView.frameLayoutGuide 指的是 tableView 本身 frame 的框框。在我們的例子裡，table view 佔滿整個畫面，因此 tableView.frameLayoutGuide.topAnchor 代表 table 本身的上邊界，也就是螢幕頂端。
  • contentLayoutGuideLabel 將對齊 navigation bar 的底端，因為 tableView.contentLayoutGuide 指的是 tableView 捲動內容的框框。而 table view 捲動內容的框框預設將在 Safe Area 裡，防止 table view 裡的東西被蓋到，因此 tableView.contentLayoutGuide.topAnchor 代表捲動內容框框的上邊界，它一開始將對齊 Safe Area 的上邊界(也就是 navigation bar 的底端)。

值得注意的，當我們捲動 table 時，只有 contentLayoutGuideLabel 會跟著捲動，frameLayoutGuideLabel 是固定不動的。因為 tableView.frameLayoutGuide.topAnchor 是固定的，它代表 table 本身的上邊界，因此 frameLayoutGuideLabel 不會跟著捲動。

ps: 若是條件改成對齊 tableView.topAnchor，效果將跟剛剛使用 tableView.contentLayoutGuide.topAnchor 一樣。

contentLayoutGuideLabel.topAnchor.constraint(equalTo: tableView.topAnchor).isActive = true
