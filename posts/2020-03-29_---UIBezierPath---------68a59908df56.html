利用 UIBezierPath 繪製路經的原理

開發 iOS App 時，我們可以利用 UIBezierPath 繪製路徑，然後搭配 CAShapeLayer 做出任意形狀的 view。接下來我們就以繪製三角形為例，一步步說明利用 UIBezierPath 繪製路經的原理。

三角形程式範例

let path = UIBezierPath()
var point = CGPoint(x: 0, y: 0)
path.move(to: point)
point = CGPoint(x: 100, y: 0)
path.addLine(to: point)
point = CGPoint(x: 100, y: 100)
path.addLine(to: point)

(0, 0) 為左上角的座標，愈向右 x 愈大，愈向下 y 愈大。以上的程式將從 (0, 0) 畫線到 (100, 0)，再從 (100, 0) 畫線到 (100, 100)，產生以下路徑。

說明:

let path = UIBezierPath()

產生 UIBezierPath。

var point = CGPoint(x: 0, y: 0)
path.move(to: point)

利用 move 移動到座標 (0, 0) 的位置，以 (0, 0) 為起點。

point = CGPoint(x: 100, y: 0)
path.addLine(to: point)

利用 addLine 畫線，從 (0,0) 畫線到 (100, 0)。

等等，我們的三角形還沒畫完，只有兩邊呀，應該要再用 addLine 畫回 (0,0)，才會變成真真正正的三角形。

其實用不著這麼麻煩，呼叫 UIBezierPath 的 close() 即可從目前所在的 (100, 100) 連線回起點 (0,0)，組合出完美的三角形路徑。

path.close()

剛剛的程式由於三角形的座標都只使用一次，之後不會再使用，所以也可以採用另一種寫法，呼叫 move & addLine 的 ( ) 裡直接生成 CGPoint。

let path = UIBezierPath()
path.move(to: CGPoint(x: 0, y: 0))
path.addLine(to: CGPoint(x: 100, y: 0))
path.addLine(to: CGPoint(x: 100, y: 100))
path.close()

此寫法的好處在於打的字更少，而且可以按下 enter 鍵自動輸入 CGPoint。(相關說明可參考以下連結)

在參數的藍色框框按下 enter 鍵，加快程式開發速度
當我們撰寫 Swift 程式時，時常需要在呼叫 function 或生成東西的 ( ) 裡傳入參數。當我們輸入參數時，若能在某些時候在藍色框框按下 enter 鍵，將大大加快開發的速度。medium.com

在 Playground 顯示 UIBezierPath

剛剛的 path，如果直接從 Playground 觀看它的結果，將發現產生的三角形和剛剛不太一樣。這是因為 UIBezierPath 以左下為原點 (0,0)，x & y 愈向右和愈向上愈大，但 iOS App 的 View 元件則以左上為圓點，x & y 愈向右和愈向下愈大。

不過別擔心，想將繪製的 UIBezierPath 顯示到畫面上，我們必須透過 CAShapeLayer 或 view，到時候呈現的結果將變成以左上為原點繪製。
