利用 enum 的 associated value 和 struct 定義包含更多資訊的 Swift error

在 Swift 裡當我們想自訂錯誤型別時，通常用 enum 搭配簡單的 case 定義遵從 Error protocol 的型別。不過如果想在錯誤裡包含更多資訊，不妨試試運用 enum 的 associated value 和 struct。

比方以下彼得潘用多年經驗，以 GoAfterGirlError 定義追女生的各種錯誤:

enum GoAfterGirlError: Error {
   case poorProblem
   case tooYoungProblem
   case notAquariusProblem
   case falseHeartProblem
}

然後我們定義可能會丟出錯誤的 function goAfterAngelababy，在 do catch 裡呼叫 goAfterAngelababy。

func goAfterAngelababy(money: Int, age: Int) throws {
   guard money > 10000 else {
      throw GoAfterGirlError.poorProblem
   }
   guard age > 18 else {
      throw GoAfterGirlError.tooYoungProblem
   }
   print("我追到 Angelababy 了!")
}
do {
   try goAfterAngelababy(money: 100, age: 25)
   print("為了給她幸福，我要白天寫 iOS，晚上寫 Android")
} catch GoAfterGirlError.poorProblem {
   print("為了多賺一點，我要白天寫 iOS，晚上寫 Android，半夜寫 Windows")
} catch {
   print("我知道她不愛我，她的眼神，說出她的心。")
}

在剛剛的例子，我們只有一百塊，所以追不到 Angelababy。我們利用 catch GoAfterGirlError.poorProblem 補捉到錯誤，知道失敗的原因是錢太少，但我們無法在 catch 的 { } 裡知道當初我們有多少錢。如果能知道，我們就能知道還需要接多少 App case 賺錢。

搭配 enum 的 associated value 包含錯誤的相關資訊，可幫我們解決這個煩惱。

我們將 poorProblem 加上 Int 型別的 associated value 儲存金額，age 加上 Int 型別的 associated value 儲存年紀。

enum GoAfterGirlError: Error {
   case poorProblem(Int)
   case tooYoungProblem(Int)
   case notAquariusProblem
   case falseHeartProblem
}

此時在 function goAfterAngelababy 裡丟出錯誤 poorProblem & tooYoungProblem 時，即可包含金額和年紀。

func goAfterAngelababy(money: Int, age: Int) throws {
   guard money > 10000 else {
      throw GoAfterGirlError.poorProblem(money)
   }
   guard age > 18 else {
      throw GoAfterGirlError.tooYoungProblem(age)
   }
   print("我追到 Angelababy 了!")
}

有了 associated value，之後我們利用 catch let GoAfterGirlError.poorProblem(money) 補捉到錯誤時，即可透過 money 取得金額。

do {
   try goAfterAngelababy(money: 100, age: 25)
   print("為了給她幸福，我要白天寫 iOS，晚上寫 Android")
} catch let GoAfterGirlError.poorProblem(money) {
   if money < 1000 {
      print("還要賺很多，我要白天寫 iOS，晚上寫 Android，半夜寫 Windows")
   } else {
      print("只要再賺一點，只要白天寫 iOS 就可以了")
   }
} catch {
   print("我知道她不愛我，她的眼神，說出她的心。")
}

利用 associated value，我們可以幫 GoAfterGirlError 的各種錯誤加上相關資訊，比方 poorProblem 加上 money，tooYoungProblem 加上 age。不過有時我們還希望有其它資訊是這些錯誤都有的，比方追求對象的名字。此時可考慮用 struct 定義錯誤型別，例如以下例子，GoAfterGirlError 的 girlName 描述女生的名字，kind 則是 enum 型別，記錄它是哪一種錯誤。

struct GoAfterGirlError: Error {
   enum ErrorKind {
      case poorProblem(Int)
      case tooYoungProblem(Int)
      case notAquariusProblem
      case falseHeartProblem
    }
    let kind: ErrorKind
    let girlName: String
}

修改 function goAfterAngelababy，丟出 struct 型別 GoAfterGirlError 定義的錯誤。

func goAfterAngelababy(money: Int, age: Int) throws {
   guard money > 10000 else {
      throw GoAfterGirlError(kind: .poorProblem(money), girlName: "Angelababy")
   }
   guard age > 18 else {
      throw GoAfterGirlError(kind: .tooYoungProblem(age), girlName: "Angelababy")
   }
   print("我追到 Angelababy 了!")
}

利用 catch let 搭配 as 可判斷不同的錯誤，比方 catch let error as GoAfterGirlError 將補捉錯誤 GoAfterGirlError。當錯誤的型別是 GoAfterGirlError 時，錯誤將存在常數 error 裡，方便我們在 { } 裡讀取它的 property，比方以下例子判斷 girlName 是否為 Angelababy。

do {
   try goAfterAngelababy(money: 100, age: 25)
   print("為了給她幸福，我要白天寫 iOS，晚上寫 Android")
} catch let error as GoAfterGirlError {
   if error.girlName == "Angelababy" {
      print("為了 Angelababy，我要白天寫 iOS，晚上寫 Android，半夜寫 Windows")
   } else {
      print("那就這樣吧")
   }
} catch {
   print("我知道她不愛我，她的眼神，說出她的心。")
}

catch let 的後面也可以搭配 where 做進一步的判斷，比方剛剛用 if 判斷名字是否為 Angelababy 的程式，其實可以寫在 where ，變成 catch let error as GoAfterGirlError where error.girlName == “Angelababy” 。

do {
   try goAfterAngelababy(money: 100, age: 25)
   print("為了給她幸福，我要白天寫 iOS，晚上寫 Android")
} catch let error as GoAfterGirlError where error.girlName == "Angelababy" {
   print("為了 Angelababy，我要白天寫 iOS，晚上寫 Android，半夜寫 Windows")
} catch {
   print("我知道她不愛我，她的眼神，說出她的心。")
}
