array，dictionary，傻傻分不清

初學者剛學程式時，時常會遇到許多長得很像，容易搞混分不清的東西。就像蔡依林唱的，老虎 老鼠 傻傻分不清楚。

Swift 的 array & dictionary，都是以 [ ] 裝著多筆資料，資料間以逗號分隔，因此初學者常會把 dictionary 誤認為 array。

讓我們看看以下例子。zodiacs 是 array，signDic 是 dictionary。

let zodiacs: [String] = ["鼠", "牛", "虎", "兔", "龍", "蛇", "馬", "羊", "猴", "雞", "狗", "豬"]
let signDic: [String: String] = ["名字": "水瓶座", "象限": "風向", "日期": "1/20 ~ 2/18", "emoji": "♒️"]

請記得，array 長這樣，dictionary 長那樣 ! signDic 的資料是一對一對的，它以 key 跟 value 組合成一對，中間以冒號分隔。因此 key 名字將對應 value 水瓶座，key 象限將對應 value 風向。

看出差異後，最簡單的判斷方法大概就是冒號了。之後只要在 [ ] 裡看到冒號，有很大的機率它會是 dictionary。

我們可從以下兩個地方判斷它是否是 dictionary

  • 當型別是 [ ] ，而且 [ ] 裡有冒號時，表示它是 dictionary。就像剛剛例子的 [String: String]。
  • 當資料有 [ ]，而且 [ ] 裡有冒號時，有很大的機率它會是 dictionary。就像剛剛例子的"名字": "水瓶座"。

ps: 還是有一絲絲的機率它會是 array，比方冒號出現在雙引號裡，例如以下例子:

let messages = [
    "狐狸說: 只有用心看，才能看清楚。重要的東西是眼睛看不見的。",
    "小王子說: 只有小孩子知道他們在找什麼。他們把時間花在布條縫成的洋娃娃上面，因此洋娃娃變得很重要。"
]
