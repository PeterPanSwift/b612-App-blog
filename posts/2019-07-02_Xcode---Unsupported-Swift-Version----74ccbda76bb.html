Xcode 的 Unsupported Swift Version 問題

對開發 iOS App 的我們來說，GitHub 就像氧氣一樣重要，因為大部分我們想開發的功能都能在上面找到程式範例。

不過因為 Swift 每年不斷地推陳出新，有時我們開心地點選 GitHub 專案網頁上的 Download 按鈕下載，然後從 Xcode 打開後卻跑不起來，出現錯誤訊息 Unsupported Swift Version。

此問題是因為專案採用的 Swift 版本太舊了，Xcode 不支援。上圖裡的文字提到可用舊版的 Xcode 10.1 將它升級到 Swift 4，但這需要另外安裝舊版的 Xcode。

有沒有方法直接用新版的 Xcode 將專案升級到新版的 Swift 呢 ? 其實是可以的，只是要手動設定，步驟如下:

1 從 Project navigator 點選專案檔，接著點選 TARGETS 下的 App，然後切換到 Build Settings 頁面。
2. 輸入關鍵字 Swift 過濾資訊，找到 Swift Language Version 的欄位。

果然凶手在這裡，Swift 3.0 (unsupported) 告訴我們 Xcode 並不支援 Swift 3。

3. 將 Swift Language Version 改成最新的 Swift 5。

這樣應該大功告成了吧，已經調成 Xcode 支援的 Swift 版本了。別急，最麻煩的事還在後頭呢。

4. 逐一修正舊版 Swift 語法造成的紅色錯誤。

此時應該會有大量的紅色錯誤產生，因為 Swift 5 認為專案裡某些 Swift 3 的程式語法有問題。

請耐心地一個個修正，如果專案沒有太複雜的話，應該很快就能修正完畢。若是專案比較龐大的話，嗯 …，有志者事竟成，God bless you，相信總會有改完的一天的。

另外遇到有些錯誤不知怎麼修正的話，我們也可點選紅色錯誤的 icon，請 Xcode 提供修正的建議。

比方點選上圖的 Fix 後，UIApplicationLaunchOptionsKey 被改成 UIApplication.LaunchOptionsKey，順利地成為 Swift 5 下正確的版本。
