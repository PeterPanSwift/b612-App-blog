將 data source / delegate 指定為 controller 以外的物件，簡化 controller 的程式

開發 UIKit iOS App 時，我們時常透過 data source 或 delegate 實現 App 的功能。通常初學的範例都是把 view controller 設為 data source 或 delegate，例如以下顯示世界知名景點的例子。

使用 view controller 當 data source

view controller 是 table view 的 data source。

table view 的 style 為 Inset Grouped，實現圓角 cell 的效果。

cell 的類別和 id 為 AttractionTableViewCell。

定義 cell 的類別 AttractionTableViewCell。

class AttractionTableViewCell: UITableViewCell {
    static let reuseIdentifier = "\(AttractionTableViewCell.self)"
    @IBoutlet weak var nameLabel: UILabel!
    @IBoutlet weak var photoImageView: UIImageView!
}

定義景點的型別 Attraction。

struct Attraction {
    let name: String
    let image: String
}

定義 ViewController。

class ViewController: UIViewController {
    
    let attractions = [
        Attraction(name: "倫敦塔橋", image: "pic1"),
        Attraction(name: "福克斯冰河", image: "pic2"),
        Attraction(name: "泰姬瑪哈陵", image: "pic3")
    ]

    override func viewDidLoad() {
        super.viewDidLoad()
    }
}

extension ViewController: UITableViewDataSource {

    func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
        return attractions.count
    }
    
    func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
        let cell = tableView.dequeueReusableCell(withIdentifier: AttractionTableViewCell.reuseIdentifier, for: indexPath) as! AttractionTableViewCell
        let attraction = attractions[indexPath.row]
        cell.photoImageView.image = UIImage(named: attraction.image)
        cell.nameLabel.text = attraction.name
        return cell
    }
    
    
}

將 data source / delegate 指定為 controller 以外的物件

然而當 App 愈來愈複雜，一個畫面很多功能時，我們的 view controller 時常會變得過於肥胖，程式碼愈來愈多。

為了讓 view controller 瘦身，讓它的程式碼少一點，有許多方法可以簡化它的程式。其中一招是將 data source / delegate 指定為 controller 以外的物件，將 data source / delegate 的相關程式搬出 controller。

以下我們以剛剛的例子為例，自訂 AttractionDataSource 類別，使用它當提供景點資料的 data source。

  • 定義遵從 protocol UITableViewDataSource 的 AttractionDataSource

AttractionDataSource 負責提供表格顯示的資料，因此我們將資料的 array 搬到 AttractionDataSource 裡。它遵從 UITableViewDataSource，因此顯示表格內容的相關程式都搬到 AttractionDataSource，不用再寫在 view controller。

import UIKit

class AttractionDataSource: NSObject, UITableViewDataSource {
    
    let attractions = [
        Attraction(name: "倫敦塔橋", image: "pic1"),
        Attraction(name: "福克斯冰河", image: "pic2"),
        Attraction(name: "泰姬瑪哈陵", image: "pic3")
    ]
    
    func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
        return attractions.count
    }
    
    func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
        let cell = tableView.dequeueReusableCell(withIdentifier: AttractionTableViewCell.reuseIdentifier, for: indexPath) as! AttractionTableViewCell
        let attraction = attractions[indexPath.row]
        cell.photoImageView.image = UIImage(named: attraction.image)
        cell.nameLabel.text = attraction.name
        return cell
    }
       
}

ps: 也許有人好奇，為什麼 AttractionDataSource 要繼承 NSObject ? 若是沒有繼承 NSObject，將產生以下錯誤，Cannot declare conformance to ‘NSObjectProtocol’ in Swift，相關說明可參考以下連結。

  • 將 AttractionDataSource 設為 table view 的 data source。

現在 view controller 的程式將精簡許多，因為顯示表格內容的相關程式都被搬出去了。我們只差最後一件事要做，將 AttractionDataSource 設為 table view 的 data source。

設定 data source 有以下兩個方法。

  1. 從程式設定。
  2. 從 Interface Builder 設定。

方法 1: 從程式設定。

import UIKit

class ViewController: UIViewController {
    
    @IBOutlet weak var tableView: UITableView!
    let attractionDataSource = AttractionDataSource()
    
    override func viewDidLoad() {
        super.viewDidLoad()
        tableView.dataSource = attractionDataSource
    }
}

連結 table view 的 outlet，然後在 viewDidLoad 設定 tableView 的 data source。值得注意的，因為 tableView 的 dataSource 是 weak，因此我們要將 AttractionDataSource 宣告為 controller 的 property。若是使用以下寫法，沒有用 property 另外儲存 AttractionDataSource，將造成 AttractionDataSource 產生之後馬上可憐地死掉 (Instance will be immediately deallocated because property ‘dataSource’ is ‘weak’)。

方法 2: 從 Interface Builder 設定。

將 Object 加到 ViewController 的 Exit 下。

將 Object 的類別設為 AttractionDataSource。

從 Table View 連線到 Attraction Data Source，將 Attraction Data Source 設為 Table View 的 data source。

現在 ViewController 的程式變得更簡單了，什麼都不用寫，就可以顯示表格。

class ViewController: UIViewController {
