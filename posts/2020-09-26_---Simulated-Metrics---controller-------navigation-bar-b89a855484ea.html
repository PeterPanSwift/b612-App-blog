開發 iOS App 時，我們時常透過 storyboard 的 segue 切換頁面，不過有時我們會採用其它方式，比方從程式呼叫 navigation controller 的 pushViewController 切換頁面，例如以下例子:

點選 button 想見你想見你想見你，切換到想見你的圖片頁面。

想見你圖片頁面的 storyboard ID 為 SeeYouViewController。

點選 button 時，利用 navigation controller 的 pushViewController 切換到 SeeYouViewController。

@IBAction func buttonPressed(_ sender: Any) {
        
        if let controller = storyboard?.instantiateViewController(identifier: "\(SeeYouViewController.self)") {
            navigationController?.pushViewController(controller, animated: true)
        }
     
}

以上程式完全沒問題，App 執行點選 button 後將出現想見你的圖片頁面，上方也會有 navigation bar & back 鍵。不過此時有個小小的問題，storyboard 的想見你頁面並不會有 navigation bar，因此有些功能不方便從 storyboard 實現，比方設定 bar 的標題或加入 Bar Button Item。

其實我們還是可以在 storyboard 設定 bar 的標題或加入 Bar Button Item，只要透過 Simulated Metrics 的幫忙，步驟如下:

點選 SeeYouViewController，切換到它的 Attributes inspector 頁面

將 Simulated Metrics 下的 Top Bar 改成 Translucent Navigation Bar

設定 Top Bar 為 Translucent Navigation Bar，讓畫面的上方出現 navigation bar，方便我們待會在 bar 上加入 Bar Button Item。

值得注意的，Simulated Metrics 的設定只影響 storyboard 顯示的畫面，不影響 App 執行後的畫面。由於此頁面將透過 navigation controller 的 pushViewController 呈現，因此 App 執行時，上方的確會有 navigation bar。倘若它不是以 navigation controller 加入，App 執行時，頁面的上方將不會出現 navigation bar。
