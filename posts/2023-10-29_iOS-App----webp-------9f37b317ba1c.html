iOS App 顯示 webp 格式的圖片

從 iOS 14 開始，iOS App 也能顯示 webp 格式的圖片。以下我們逐一說明如何在三種不同情況用 SwiftUI 和 UIKit 顯示 webp 圖片，

  • asset 裡的 webp 圖片。
  • Project navigator 下的 webp 圖片。
  • 網路上的 webp 圖片。

我們的目標是以下 AI 繪製的可愛 Pair Programming 圖片。

asset 裡的 webp 圖片

當圖片的附檔名是 webp 時，加到 asset 時將無法被識別為圖片。

因此請將圖片的附檔名改成 png 或 jpg，然後再加到 asset，即可成功識別為圖片。

  • SwiftUI
struct ContentView: View {
    var body: some View {
        Image(.pic)
    }
}
  • UIKit
class ViewController: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        let imageView = UIImageView(image: .pic)
        view.addSubview(imageView)
    }
}

Project navigator 下的 webp 圖片

  • SwiftUI

當圖片加到 Project Navigator 下，沒有放在 asset 時，使用Image("pic.webp") 是無法讀到圖片的。因此我們必須先產生 UIImage，再用 Image(uiImage:) 生成圖片。

struct ContentView: View {
    var body: some View {
        Image(uiImage: UIImage(named: "pic.webp")!)
    }
}
  • UIKit
class ViewController: UIViewController {
    override func viewDidLoad() {
        super.viewDidLoad()
        let imageView = UIImageView(image: UIImage(named: "pic.webp"))
        view.addSubview(imageView)
    }
}

網路上的 webp 圖片

  • SwiftUI

使用 AsyncImage。

struct ContentView: View {
    var body: some View {
        AsyncImage(url: URL(string: "https://raw.githubusercontent.com/PeterPanSwift/JSON_API/main/pic.webp")!)
    }
}
  • UIKit
class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()
        if let url = URL(string: "https://raw.githubusercontent.com/PeterPanSwift/JSON_API/main/pic.webp") {
            Task {
                do {
                    let (data, _) = try await URLSession.shared.data(from: url)
                    if let image = UIImage(data: data) {
                        let imageView = UIImageView(image: image)
                        view.addSubview(imageView)
                    }
                } catch {
                    print(error)
                }
            }
        }
        
    }

}
