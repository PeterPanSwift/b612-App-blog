# Swift Optional 心智圖

## Optional 基本概念
- 定義：一種可以包含值或 nil 的類型
- 表示方法：在類型後加上 `?` 例如：`String?`

## Optional 類型
- `Optional<T>`: Swift 中的 Optional 實際上是一個枚舉，有兩種可能的狀態：
  - `.some(T)`：包含一個值
  - `.none`：表示 nil

## 宣告 Optional
- 明確宣告：`var name: String?`
- 隱式解包：`var name: String!`

## 操作 Optional
- 解包 (Unwrapping)
  - 強制解包：`name!`
  - 可選綁定 (Optional Binding)：`if let name = name { ... }`
  - 守護語句 (Guard Statement)：`guard let name = name else { ... }`
  
## 可選鏈 (Optional Chaining)
- 用法：`person?.address?.city`
- 意義：如果任何一個節點是 nil，整個表達式會返回 nil

## Nil 合併運算符
- 用法：`a ?? b`
- 意義：如果 `a` 不為 nil，返回 `a`，否則返回 `b`

## 隱式解包 Optional
- 定義：`var name: String!`
- 用途：通常用於初始化之後不會變為 nil 的變數

## 可選值的模式匹配
- 使用 `switch` 語句匹配 `.some` 和 `.none`

## Optional 的應用場景
- 處理可能為 nil 的返回值
- 延遲初始化
- 資料轉換

## Optional 的進階用法
- map 和 flatMap 操作：轉換和鏈式處理 Optional
  - `optionalValue.map { $0.someTransformation() }`
  - `optionalValue.flatMap { someFunctionReturningOptional($0) }`
