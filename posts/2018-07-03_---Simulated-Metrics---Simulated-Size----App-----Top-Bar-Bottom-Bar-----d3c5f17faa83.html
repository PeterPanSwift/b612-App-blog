利用 Simulated Metrics 和 Simulated Size 模擬 App 畫面的 Top Bar，Bottom Bar 和大小

在 Interface Builder 製作 App 畫面時，我們可利用 Simulated Metrics 模擬畫面有 Top Bar 和 Bottom Bar ，或是利用 Simulated Size 模擬特定尺寸的樣子，幫助我們更方便地設計 App。接下來，就讓我們實際操作看看吧。

模擬 Top Bar & Bottom Bar

1 點選 Storyboard 裡的 controller 後，可在它的 Attributes inspector 頁面看到 Simulated Metrics 區塊。

Simulated Metrics 有三個欄位，Size，Top Bar 和 Bottom Bar。預設它們都是 Inferred，表示 controller 的畫面將經由推理決定大小和是否有 Top Bar & Bottom Bar。比方如下圖所示，當有 navigation controller 時，聰明的 Xcode 將認為上方應長出 navigation bar。

2 將 Top Bar 設為 Translucent Navigation Bar。

倘若 navigation controller 是從程式生成，再跟 storyboard 的 controller 串接，要如何在 Interface Builder 的 controller 畫面看到 navigation bar 呢 ?

此時 Top Bar 欄位就派上用場了，只要將它設為 Translucent Navigation Bar，即可模擬上方有 bar 的模樣。

ps: 請記得 Simulated Metrics 只是模擬， App 執行時想看到 navigation bar 還是需要搭配 navigation controller。

2 將 Bottom Bar 設為 Translucent Tab Bar 可模擬下方有 tab bar 的模樣。

調整畫面的大小

Interface Builder 顯示的畫面預設將為 iPhone 的大小，可由下方的 device configuration pane 設定。

然而當我們設計比 iPhone 螢幕大的畫面時，卻會有一些缺點。比方如下圖所示，我們想在內容尺寸超過螢幕的 scroll view 放很多東西，但因為 Interface Builder 只會笨笨地顯示 iPhone 的大小，而且無法捲動 scroll view，所以我們在設計畫面時有點難操作，也無法一眼看出 scroll view 的內容。

因此，我們希望能調整 controller 畫面顯示的大小，看到 scroll view 的完整內容。

點選 controller 後，可從 Size inspector 的 Simulated Size 調整畫面的大小。預設 Fixed 表示不能更改。

只要將它改為 Freeform，即可指定任意的寬高。

指定寬高後，終於可以看到剛剛 scroll view 裡完整的內容。

ps: 請記得 Simulated Size 只是模擬，所以並不影響 App 執行時真正的大小。

xib 的操作

如果不是在 storyboard ，而是在 controller 的 xib 設計畫面，也一樣可以模擬 Top Bar & Bottom Bar。比方 App 執行時，下方將有 Tab bar，因此希望在設計時能看到 Tab bar。

點選 xib 裡的 view 後，可在它的 Attributes inspector 頁面看到 Simulated Metrics 區塊。

若想調整畫面的大小，則需先將 Size 改為 Freeform，然後再於 Size inspector 頁面修改 Width & Height。
