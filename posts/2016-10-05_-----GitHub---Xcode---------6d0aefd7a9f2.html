下載開啟 GitHub 上 Xcode 專案的各種方法

分成不保留 git 版本管理資訊 & 保留 git 版本管理資訊兩種做法。

  • 不保留專案的 git 版本管理資訊

通常用在下載別人的專案，只是為了研究別人的程式。

  • 保留專案的 git 版本管理資訊

通常用在下載自己的專案或跟別人合作的專案，想修改後做記錄並上傳到 GitHub。

方法 1: 不保留專案的 git 版本管理資訊

  • 連到 App 專案的網址，比方以下連結。

  • 點選綠色按鈕 Code，然後點選 Download ZIP 下載。
  • 開啟專案

進入專案的資料夾，滑鼠雙擊附檔名 xcodeproj 的專案檔，啟動 Xcode 開啟專案。

點選 Trust and Open。

此方法下載的專案不包含版本管理資訊，因此 Source Control navigator 下的 Repositories 顯示 No Working Copies。

方法 2: 保留專案的 git 版本管理資訊

  • 連到 App 專案的網址，比方以下連結。
GitHub - meikumiko/eBook-for-NZ
You can't perform that action at this time. You signed in with another tab or window. You signed out in another tab or…github.com
  • 點選綠色按鈕 Code，然後點選 Open with Xcode 下載。

如果沒看到 Open with Xcode，可參考以下連結解決:

Open with Xcode 都去哪兒了 ?
Xcode 和 GitHub 深度整合，當我們連到 GitHub 的 Xcode 專案頁面，點選綠色的 Clone or download 按鈕時，將出現貼心的 Open with Xcode，方便我們下載後直接開啟它的 Xcode…medium.com

點選允許。

選擇儲存專案的位置，然後點選 Clone。

點選 Trust and Open。

下載成功後，專案將自動從 Xcode 啟動打開 !

此方法下載的專案包含版本管理資訊，因此 Source Control navigator 下的 Repositories 顯示專案的 commit 修改記錄清單。

由於有版本管理資訊，我們之後修改程式後，可進行 commit 記錄。不過若要做 push 上傳到 GitHub，則會分成以下兩種情況:

  • case 1: GitHub 上的專案是我們寫的，當初是以我們自己的 GitHub 帳號上傳。

修改更新自己的專案，當然沒有問題呀。

  • case 2: GitHub 上的專案是別人寫的，當初是別人上傳的。

修改更新別人的專案，當然是不行呀。我們每個人都有一些仇人跟情敵，如果可以的話，辛苦寫的程式很容易被他們毀於一旦。

比較可行的方式是點選 New Remote，另外在 GitHub 建立一個儲存專案的 repository，這樣可以儲存我們的修改，而且也不會改到別人存在 GitHub 的專案。
