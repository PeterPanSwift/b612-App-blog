Swift function 的參數名字

在定義 function 時，function 的名字最好能表達 function 裡程式做的事情，參數的名字則表達呼叫 function 時，傳入的資料代表的意義。比方我們看看以下例子，想一下不同參數名字的差別。

func 給我一杯(忘情水: String) {
   print("給我一杯" + 忘情水)
   print("換我一夜不流淚")
   print("所有真心真意")
   print("任它雨打風吹")
   print("付出的愛收不回")
}
給我一杯(忘情水: "微笑忘情水")

參數取名為忘情水，看起來就像我們要傳入的東西是各式各樣的忘情水，比方微笑忘情水，暖心忘情水。

func 給我一杯(water: String) {
   print("給我一杯" + water)
   print("換我一夜不流淚")
   print("所有真心真意")
   print("任它雨打風吹")
   print("付出的愛收不回")
}
給我一杯(water: "礦泉水")

參數名為 water 就彈性多了，代表我們可以傳入各種水，比方礦泉水，阿里山的水等。

另一個判斷參數名是否取得好的方法是看呼叫 function 時，整行程式看起來像不像一個完整的句子，愈像一般人看得懂的句子愈好，因為這樣代表別人很容易看懂你寫的程式。

func sing(song: String) {
   print("到 KTV 唱" + song)
}
sing(song: "你好不好")

sing(song: “你好不好”) 一眼就能看出唱了一首歌，歌名是你好不好。

相反的，如以下例子，參數名改成 song 的縮寫 s，看到 sing(s: “你好不好”) 時，令人看了一萬年，也不明白 s 是什麼。

func sing(s: String) {
   print("到 KTV 唱" + s)
}
sing(s: "你好不好")
