安裝 Firebase CLI 和登入 Firebase

我們將在 Terminal 透過 Firebase CLI 執行 Firebase 的相關指令，請參考以下連結安裝 Firebase CLI 和登入 Firebase。

方便管理 Firebase 的 Firebase CLI(Command Line Interface)
透過 Firebase CLI 我們可以方便地從 Terminal 下指令管理 Firebase 專案和部署網站。以下彼得潘簡單介紹如何在 macOS & Windows 安裝 Firebase CLI。medium.com

安裝 flutterfire_cli

在 Terminal 輸入指令安裝方便 Flutter App 串接 Firebase 的指令工具 flutterfire_cli。

dart pub global activate flutterfire_cli

ps:

如果 Mac 出現警告訊息 Pub installs executables into $HOME/.pub-cache/bin, which is not on your path.，請在 Terminal 輸入以下指令。

echo 'export PATH="$PATH:$HOME/.pub-cache/bin"' >> ~/.zshenv

如果 Windows 出現警告訊息 Pub installs executables into C:\Users\shih-yingpan\AppData\Local\Pub\Cache\bin, which is not on your path.，請在環境變數 Path 加入警告訊息提到的路徑。

flutterfire configure 執行成功後將在 Flutter 專案的 lib 下生成 firebase_options.dart，它包含 Flutter App 初始 Firebase 需要的相關資料。

此時連到 Firebase 的專案頁面將看到專案包含了 App 支援的平台。

不過 firebase_options.dart 的程式有一些要修正的錯誤，它 import 了 package firebase_core，因此我們必須先加入套件 firebase_core。

  • 方法 1: 從 Quick Fix 點選 Add firebase_core。
  • 方法 2: 從 Terminal 輸入指令。
flutter pub add firebase_core

在 Flutter App 加入初始 Firebase 的程式

import 'package:flutter/material.dart';
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);
  runApp(const MyApp());
}
