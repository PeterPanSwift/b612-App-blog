開發 SwiftUI App 時，我們時常利用方便的 State 或 Published 變數觸發畫面更新，不過也因為修改它們的內容會造成畫面更新，因此我們修改內容時要特別小心，並不是任何地方都能修改。

Modifying state during view update, this will cause undefined behavior

開發 SwiftUI App 時，我們時常利用方便的 State 或 Published 變數觸發畫面更新，不過也因為修改它們的內容會造成畫面更新，因此我們修改內容時要特別小心，並不是任何地方都能修改。

比方我們看看以下有問題的例子。

struct ContentView: View {
    
    @State private var imageFrame = CGRect.zero
    @State private var offset = CGSize.zero
       
    var body: some View {
        
        VStack(spacing: 10) {
            Image("密碼的故事")
                .resizable()
                .scaledToFit()
                .overlay(
                    GeometryReader(content: { geometry in
                        let _ = updateFrame(geometry: geometry)
                        Color.clear
                    })
                )
                .offset(offset)
            Button(action: {
                offset = CGSize(width: CGFloat.random(in: -100...100), height: CGFloat.random(in: -100...100))
            }, label: {
                Text("change")
                    .background(Color.yellow)
            })
            Button(action: {
                print(imageFrame)
            }, label: {
                Text("print frame")
                    .background(Color.yellow)
　　　　　　　})
        }
        .font(.largeTitle)
　　}
    
    func updateFrame(geometry: GeometryProxy) {
        imageFrame = geometry.frame(in: .global)
    }
}

我們想知道圖片的位置大小，因此我們利用 overlay & GeometryReader 取得它的位置大小，然後存在 State 變數 imageFrame 裡。

以上程式將在產生 GeometryReader 時呼叫 updateFrame，將圖片目前的位置大小存在 imageFrame。不過由於 imageFrame 是 state 變數，所以將產生問題，因為修改它會觸發畫面更新，然後重新產生 GeometryReader，然後又會再修改 imageFrame，變成了無窮迴圈。

產生 GeometryReader > 修改 imageFrame > 畫面更新 > 產生 GeometryReader > 修改 imageFrame > 畫面更新 > 產生 GeometryReader ...

雖然 App 跑得起來，但是 Xcode 將顯示特別的紫色錯誤，提醒我們

Modifying state during view update, this will cause undefined behavior

而且點選 button 印出 imageFrame 時，將發現結果都是 (0.0, 0.0, 0.0, 0.0)。

改用 onAppear 呢 ?

在 onAppear 修改 State 變數是 ok 的，不會造成 Modifying state during view update, this will cause undefined behavior，但是它只會在圖片出現時執行一次，所以我們只能取得圖片一開始的位置，當圖片位置移動時，imageView 並不會更新。

VStack(spacing: 10) {
   Image("密碼的故事")
      .resizable()
      .scaledToFit()
      .overlay(
          GeometryReader(content: { geometry in
             Color.clear
                .onAppear(perform: {
                    imageFrame = geometry.frame(in: .global)
                 })
           })
      )
      .offset(offset)

正確的解法

我們必須利用變數儲存圖片的位置大小，並且確保不會觸發畫面更新，因此我們不能用 value type 存資料，因為在 SwiftUI view 裡修改 value type 必須加上會造成畫面更新的 State。

我們改以 class 定義資料的型別，然後在其中宣告儲存位置大小的 imageFrame。到時候我們將可在 SwiftUI view 裡直接修改 DemoData 的 imageFrame，而且它也不會觸發畫面更新，不再產生可怕的無窮迴圈。

class DemoData {
    var imageFrame = CGRect.zero
}
struct ContentView: View {
    
    private var demoData = DemoData()
    @State private var offset = CGSize(width: 200, height: 0)
    
    
    var body: some View {
        
        VStack(spacing: 10) {
            Image("密碼的故事")
                .resizable()
                .scaledToFit()
                .overlay(
                    GeometryReader(content: { geometry in
                        let _ = updateFrame(geometry: geometry)
                        Color.clear
                    })
                )
                .offset(offset)
                
            Button(action: {
                offset = CGSize(width: CGFloat.random(in: -100...100), height: CGFloat.random(in: -100...100))
            }, label: {
                Text("change")
                    .background(Color.yellow)
            })
            Button(action: {
                print(demoData.imageFrame)
            }, label: {
                Text("print frame")
                    .background(Color.yellow)
　　　　　　　})
        }
        .font(.largeTitle)
　　}
    
    func updateFrame(geometry: GeometryProxy) {
        demoData.imageFrame = geometry.frame(in: .global)
    }
}
