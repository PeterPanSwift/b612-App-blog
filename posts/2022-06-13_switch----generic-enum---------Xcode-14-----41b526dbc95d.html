switch 搭配 generic enum 的自動完成 — Xcode 14 新功能

從 Xcode 14 開始，當我們將 switch 作用在 generic enum 時，Xcode 的自動完成會貼心地幫我們輸入每個 case 的內容。

以下我們以時常使用的 Result 型別說明。 Result 是 generic enum，它的 宣告如下:

enum Result<Success, Failure> where Failure : Error {
    case success(Success)
    case failure(Failure)

假設我們想呼叫以下 function

func fetchCategories(completion: @escaping (Result<[String], Error>) -> Void) {
        
}

呼叫 fetchCategories，在輸入框按 enter。

輸入 switch re，點選自動輸入每個 case 的 result { 。

Xcode 自動幫我們輸入 case success & failure，而且連 ( ) 裡的參數名字都寫好了。

如果不喜歡預設的參數名字 success，我們也可以手動修改，比方改成 categories。

若是使用舊版的 Xcode，就沒那麼幸運了。如下圖所示，輸入 switch re 時，Xcode 只會笨笨地幫我們輸入 result，不會包含 { } 裡的 case。
