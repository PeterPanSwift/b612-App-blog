Swift 的參數名字，順序和數量

撰寫 Swift 程式時，不管是生成東西還是呼叫 function，( ) 裡傳入的參數名時都要完全和當初 init 或 function 裡宣告的一致。所謂的一致十分嚴格，參數的名稱，順序和數量都要一模一樣。接下來我們就以代表歌曲的 struct Song 為例，看看幾個初學者容易不小心遇到的問題。

struct Song {
   var trackName: String
   var artistName: String
   var trackPrice: Int
}
參數的數量不對

Missing argument for parameter

我們忘了輸入參數 trackPrice。

參數的順序不對

Argument ‘artistName’ must precede argument ‘trackPrice’

參數 artistName 應該要在 trackPrice 之前。

參數的名字不對

Incorrect argument label

參數 name 應該改成 trackName。

另外還有兩種錯誤訊息，也可能在一些特別的情況下出現:

1 如果型別是 array 時，當參數的名字，順序或數量不一致時，出現的錯誤訊息會變成 tuple 無法轉換。

2 當型別遵從 protocol Codable，而我們給的參數不對時，將顯示錯誤 Cannot invoke initializer for type xxx with an argument list of type。

struct Song: Codable {
var trackName: String
var artistName: String
var trackPrice: Int
}
struct Song: Codable {
var trackName: String
var artistName: String
var trackPrice: Int
}

Swift 看似要求很多，但只要我們在輸入時盡量使用 Xcode 的自動完成，不要自己打，即可大大降低錯誤的機率，比方輸入 Song( 後，選擇參數 trackName，artistName & trackPrice。
