IndexPath 在 switch 裡可以當成 array 來比對

在開發 table 相關功能時，我們時常使用 IndexPath 元件，判斷它的 section 和 row 屬性來決定如何呈現 cell 的內容，所以我們一般會找最熟悉的好朋友 if else 來幫忙判斷，類似以下例子:

override func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
let cell = tableView.dequeueReusableCell(withIdentifier: "reuseIdentifier", for: indexPath)
if indexPath.section == 0 && indexPath.row == 0 {
print("section 0 row 0")
}
else if indexPath.section == 1 && indexPath.row == 0 {
print("section 1 row 0")
}
return cell
}

其實，用 switch 來比可以更簡單 !

當 switch 比對 IndexPath 元件時，可以把它當成 array 來比對。case 後接的 array 有兩個成員，第一個是 section 的數字，第二個是 row 的數字。所以 [0,0] 可比對 section 0 row 0，[1, 0] 比對 section 1 row 0。

override func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
let cell = tableView.dequeueReusableCell(withIdentifier: "reuseIdentifier", for: indexPath)
switch indexPath {
case [0,0]:
print("section 0 row 0")
case [1,0]:
print("section 1 row 0")
default:
break
}
return cell
}
