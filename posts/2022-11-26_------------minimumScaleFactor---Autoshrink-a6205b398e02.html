文字塞不下時自動縮小的 minimumScaleFactor & Autoshrink

開發 iOS App 時，有時我們會希望字體大一點。比方下圖畫面上顯示超好看的影集阿達一族外傳:星期三，為了強調它真的好看，彼得潘特意將字體放大到 42。

在 iPhone 14 Pro 它可以完整地顯示，但在較小的 iPhone SE 卻會變成點點點，看不出可愛的女主角是星期幾。

其實文字本來就有塞不下時自動縮小的功能，以下我們分別介紹 SwiftUI & UIKit 的做法，讓星期三在小小的 iPhone SE 也能完美顯示。

SwiftUI

呼叫 modifier minimumScaleFactor，比方傳入 0.2，文字塞不下時會自動縮小，最多縮小到 0.2 倍。值得注意的，文字的行數要設為 1，不然文字一行塞不下時將會換行，不會有文字縮小的效果。

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image("wednesday")
                .resizable()
                .scaledToFit()
            Text("阿達一族外傳:星期三")
                .font(.system(size: 42))
                .lineLimit(1)
                .minimumScaleFactor(0.2)
        }
    }
}

UIKit

  • 從 Interface Builder。

設定 Autoshrink，選擇 Minimum Font Scale 或 Minimum Font Size。

設定 Minimum Font Scale，文字塞不下時會自動縮小，最多縮小到 0.5 倍。

設定 Minimum Font Size，文字塞不下時會自動縮小，最多縮小到 9 的大小。

  • 從程式。

adjustsFontSizeToFitWidth 設為 true，讓文字塞不下時自動縮小，minimumScaleFactor 設為 0.5 表示最多縮小到 0.5 倍。

label.adjustsFontSizeToFitWidth = true 
label.minimumScaleFactor = 0.5
