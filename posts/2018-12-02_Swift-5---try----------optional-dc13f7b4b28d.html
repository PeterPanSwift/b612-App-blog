Swift 5 的 try? 不再產生多層的 optional

使用 try? 呼叫可能產生錯誤的 function 將讓程式更安全，不像 try! 一失敗會讓 App 馬上閃退。但在某些時候它卻會造成一些不便，比方產生兩個問號的多層 optional。

if let url = URL(string: "https://itunes.apple.com/search?term=swift&media=music") {
   let task = URLSession.shared.dataTask(with: url) { (data, response , error) in
      if let data = data {
         let dic = try? JSONSerialization.jsonObject(with: data) as? [String: Any]
      }
   }
   task.resume()
}

因為 as? [String: Any] 會產生 optional ，try ? 也會產生 optional，所以威力加倍，變成了百年難得一見的雙層 optional，??。

變成兩個問號其實也不是很大的問題，只是讀取它的內容時會很麻煩，因為要解開兩次的包裝，比方我們時常利用 JSONSerialization 解析 JSON，但為了得到 dictionary 卻須麻煩地使用多次的 optional binding ，先讀出型別 [String: Any]? 的 optionalDic，再讀出型別 [String: Any] 的 dic。

if let data = data, let optionalDic = try? JSONSerialization.jsonObject(with: data) as? [String: Any], let dic = optionalDic {
}

將多層的 optional 變成一層 optional 的概念其實在 optional chaining 已經出現過，我們不用擔心多層的 optional chaining 會得到多層的 optional。

class Rabbit {
var name = "peter"
}
class Baby {
var cuteRabbit: Rabbit? = Rabbit()
}
var cuteBaby: Baby? = Baby()
let name = cuteBaby?.cuteRabbit?.name

在 Swift 5，SE-0230 的 Flatten nested optionals resulting from ‘try?’ 模仿剛剛 optional chaining 的概念， 現在 try? 也不會再產生多層的 optional 了 !

比方以下例子，try? JSONSerialization.jsonObject(with: data) as? [String: Any] 得到的東西型別將為 [String: Any]?，因此我們只需要一次 optional binding，即可得到型別 [String: Any] 的 dic。

let task = URLSession.shared.dataTask(with: url) { (data, response , error) in
   if let data = data, let dic = try?    JSONSerialization.jsonObject(with: data) as? [String: Any] {
   }
}
