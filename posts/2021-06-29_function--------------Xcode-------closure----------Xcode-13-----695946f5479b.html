function 型別搭配參數名字，呼叫時 Xcode 將自動輸入 closure 接收的參數名 — Xcode 13 新功能

最近彼得潘使用 Xcode 13，發現一個方便的地方。

平常定義 function 時，我們可以宣告 function 型別的參數，例如以下例子的參數 completion。

func fetchImage(url: URL, completion: @escaping (UIImage?) -> Void) {
        URLSession.shared.dataTask(with: url) { data, response, error in
            if let data,
               let image = UIImage(data: data) {
                completion(image)
            } else {
                completion(nil)
            }
        }.resume()
        
}

當我們呼叫 fetchImage，傳入 closure 時，我們必須自己輸入 closure { } 裡的參數名字。在這個例子裡，我們通常會在 UIImage? 的框框輸入 image，因為它代表抓取的圖片。

如果不想自己輸入 closure 的參數名字，其實有個方法。我們可以在宣告 function 型別參數時搭配參數名字。以下例子的 completion 型別改成 (_ image: UIImage?) -> Void，它的外部參數名省略，內部參數名為 image。

func fetchImage(url: URL, completion: @escaping (_ image: UIImage?) -> Void) {
        URLSession.shared.dataTask(with: url) { data, response, error in
            if let data,
               let image = UIImage(data: data) {
                completion(image)
            } else {
                completion(nil)
            }
        }.resume()
        
}

在此我們一定要省略外部參數名，因為 function type 不能有外部參數名(argument label)。

之後當我們呼叫 fetchImage，在 completion 的框框按下 enter 時，Xcode 將幫我們輸入 closure 接收的參數名 。
