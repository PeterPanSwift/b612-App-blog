設定 SwiftUI App 統一的背景

最近有同學問到是否能讓整個 App 有統一的背景，只要設定在一個地方，即可讓 App 的每個頁面都有一樣的背景顏色或背景圖片。

開發 SwiftUI App 時，通常我們會用 TabView & NavigationStack 管理多個頁面，此時要特別記得 TabView & NavigationStack 本身也是有顏色的，因此以下利用 ZStack 設定背景的做法不可行，因為 Color.yellow 會被 TabView 和 NavigationStack 本身的顏色蓋住。

struct ContentView: View {
    var body: some View {
        ZStack {
            Color.yellow
                .ignoresSafeArea()
            TabView {
                Tab("Books", systemImage: "book") {
                    BooksView()
                }
                Tab("Songs", systemImage: "music.pages") {
                    SongsView()
                }
                Tab("Movies", systemImage: "film") {
                    MoviesView()
                }
            }
        }
    }
}
struct ContentView: View {
    let names = [
        "小王子",
        "彼得潘",
        "愛麗絲夢遊仙境",
        "綠野仙蹤",
    ]

    var body: some View {
        ZStack {
            Color.yellow
                .ignoresSafeArea()
            NavigationStack {
                ScrollView {
                    VStack {
                        ForEach(names, id: \.self) { name in
                            Button(name) {

                            }
                            .font(.largeTitle)
                            .buttonStyle(.borderedProminent)
                            .buttonSizing(.flexible)

                        }
                    }
                    .padding()
                }
            }
        }
    }
}

為了讓 App 的每個頁面有一樣的背景，我們必須在每個頁面設定，比方以下例子。

  • 利用 ZStack 設定背景。
struct BookDetailView: View {
    var body: some View {
        ZStack {
            Color.yellow
                .ignoresSafeArea()
            VStack(spacing: 20) {
                Text("小王子")
                    .font(.largeTitle)
                Text("《小王子》的故事描述一位年輕的王子拜訪多個星球（包括地球）的歷程，探討孤獨、友誼、愛與失落等主題。雖然表面上是一本兒童讀物，卻對人生、成年人和人性提供了深刻的洞察。")
                Spacer()
            }
            .padding()
        }
    }
}
  • 設定 ScrollView 的背景。
struct BooksView: View {
    let names = [
        "小王子",
        "彼得潘",
        "愛麗絲夢遊仙境",
        "綠野仙蹤"
    ]

        ScrollView {

  • 設定 List 的背景。

List 本身也會有預設的背景，所以要先用 scrollContentBackground(.hidden) 隱藏。

struct BooksView: View {
    let names = [
        "小王子",
        "彼得潘",
        "愛麗絲夢遊仙境",
        "綠野仙蹤"
    ]
    var body: some View {
        List(names, id: \.self) { name in
            Text(name)
                .font(.largeTitle)
            
        }
        .scrollContentBackground(.hidden)
        .background(.yellow)
    }
}

不過如果頁面很多，懶得重覆輸入背景的程式，我們可以透過自訂 modifier 或容器型別簡化程式。

  • 自訂 modifier。

參考以下連結的方法自訂 modifier。

簡化大量 modifier 程式的兩個方法，自訂組合 modifier 的 function 和透過 ViewModifier 實作 custom modifier
開發 SwiftUI App 時，我們時常透過大量的 modifier 客製元件，將畫面變成賞心悅目的樣子。不過 App 裡很多元件的都是類似的設計，比方以下 SongList & LyricsView 裡的文字都是白色粗體加陰影，透過…medium.com

自訂設定背景的 appBackground。

struct AppBackgroundModifier: ViewModifier {
    func body(content: Content) -> some View {
        ZStack {
            Color.yellow
                .ignoresSafeArea()
            content
        }
    }
}

extension View {
    func appBackground() -> some View {
        modifier(AppBackgroundModifier())
    }
}

  • 自訂容器型別。

參考以下連結的方法自訂容器型別。

宣告方便接收多個 view 的 @ViewBuilder property
設計 SwiftUI App 時，VStack、HStack、ScrollView 等容器元件可以方便地以 closure { } 傳入多個 view，例如以下例子。medium.com

自訂包含 App 背景的 AppBackgroundContainer。

import SwiftUI

struct AppBackgroundContainer<Content: View>: View {
    @ViewBuilder var content: Content

    var body: some View {
        ZStack {
            Color.yellow
                .ignoresSafeArea()
            content
        }
    }
}
struct BookDetailView: View {
    var body: some View {
        AppBackgroundContainer {
            VStack(spacing: 20) {
                Text("小王子")
                    .font(.largeTitle)
                Text("《小王子》的故事描述一位年輕的王子拜訪多個星球（包括地球）的歷程，探討孤獨、友誼、愛與失落等主題。雖然表面上是一本兒童讀物，卻對人生、成年人和人性提供了深刻的洞察。")
                Spacer()
            }
            .padding()
        }
    }
}
