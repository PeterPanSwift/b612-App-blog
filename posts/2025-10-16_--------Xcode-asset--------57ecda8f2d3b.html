透過資料夾管理 Xcode asset 的圖片和顏色

開發 App 時，我們會在 Xcode asset 存放 App 需要的素材，像是圖片和顏色，不過當素材很多時，asset 裡的東西將變得十分凌亂，此時可透過資料夾分類管理。

在 asset 新增資料夾的方法有以下四種。

  • New Folder。
  • New Folder from Selection。
  • 將資料夾加到 asset。
  • New Folder with Namespace。

New Folder

點選 Assets.xcassets，然後透過以下方法新增資料夾。

  • 從 Quick Actions 點選 Add New Asset > Folder。
  • 從右鍵選單點選 New Folder。
  • 點選左下的 + > Folder。

New Folder from Selection

選取多個素材後，從右鍵選單點選 New Folder from Selection。

選取的圖片將被加到新增的資料夾。

將資料夾加到 asset

將包含多張圖片的資料夾拖曳或複製貼上到 asset。

New Folder with Namespace

新增包含 namespace 的資料夾。

跟一般資料夾的差別在於它勾選了 Provides Namespace。

勾選 Namespace 的好處在於我們可有同名的圖片，然後搭配不同的資料夾區分。

沒有 namespace 時，同名的圖片將造成警告 The image set name xxx is used by multiple image sets.。

有了 namespace，程式裡生成圖片或顏色時也要包含資料夾的名稱，例如以下例子。

  • SwiftUI 寫法 1。
struct ContentView: View {
    var body: some View {
        VStack {
            Image(.Fox.pic1)
                .resizable()
                .scaledToFit()
            Image(.Prince.pic1)
                .resizable()
                .scaledToFit()
        }
    }
}
  • SwiftUI 寫法 2。
struct ContentView: View {
    var body: some View {
        VStack {
            Image("fox/pic1")
                .resizable()
                .scaledToFit()
            Image("prince/pic1")
                .resizable()
                .scaledToFit()
        }
    }
}
  • UIKit。
class ViewController: UIViewController {

    override func viewDidLoad() {
        super.viewDidLoad()

        let foxImageView = UIImageView(frame: CGRect(x: 10, y: 60, width: 300, height: 300))
        foxImageView.image = .Fox.pic1
        foxImageView.contentMode = .scaleAspectFit
        view.addSubview(foxImageView)
        
        let princeImageView = UIImageView(frame: CGRect(x: 10, y: 380, width: 300, height: 300))
        princeImageView.image = UIImage(named: "prince/pic1")
        princeImageView.contentMode = .scaleAspectFit
        view.addSubview(princeImageView)
        
    }
}
