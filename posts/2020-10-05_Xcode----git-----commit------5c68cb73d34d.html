Xcode 多個 git 帳號的 commit 名字設定

就像狡兔有三窟，我們通常會有很多的 FB，IG，Gmail 帳號，所以 git 帳號也不例外，而 Xcode 當然也支援設定多個 git 帳號，比方下圖的 AppPeterPan & deeplovepan。

將專案程式上傳到 GitHub 時，我們可選擇要用哪個帳號上傳。

不過當 Xcode 設定多個 git 帳號時，commit 的名字卻可能錯亂，例如以下例子。

將專案傳到 deeplovepan 的 GitHub。

GitHub 網頁上顯示 commit 的名字和頭像卻是 AppPeterPan。

怎麼會這樣呢 ?

這是因為 commit 的名字跟 Xcode 設定的 Author Email 有關，我們可照以下步驟查詢 & 設定 Author Email。

設定 Author Email

  • 點選 Xcode menu 的 Xcode > Preferences。
  • 切換到 Source Control 的 Git 分頁。

如下圖所示，此時 Author Email 為 apppeterpan@gmail.com，難怪 GitHub 上顯示的 commit 名字是 AppPeterPan。

我們只要將它改成 deeplovepan@gmail.com，即可讓之後 commit 的名字變成 deeplovepan。

值得注意的，之前 commit 的名字並不會變，只有新 commit 的名字會受影響。當我們之後建立新的 Xcode 專案時，一開始的 Initial Commit 名字將以剛剛 Author Email的設定為準。
