請 AI 用 table format 說明遞迴(recursive) function 運算的過程

最近發現請 AI 用 table format 說明遞迴 function 運算的過程滿方便的，它會鉅細靡遺地列出過程中呼叫的 function。雖然 AI 還是偶爾會出錯，不過經過一兩次修正還是能成功，大幅省下我們一個個寫下遞迴過程的時間。

以下我們以 APCS 考題為例，請 GPT 4 用 table format 說明遞迴 function 運算的過程。

範例 1

用 table format 說明 f(10) 運算的過程

int f (int i) {
    if (i > 0) {
        if (((i/2) % 2) == 0) {
            return f(i-2)*i;
        } else {
            return f(i-2)*(-i);
        }
    } else {
        return 1;
    }
}

AI 犯了明顯的錯誤，每一次除 2 的餘數都是 0。後來彼得潘再經過兩次 prompt 後，AI 終於給出正確答案。

  • 第一次修正的 prompt。
你算錯了
  • 第二次修正的 prompt。
(10/2)%2 應該是 1 耶

正確解答。

範例 2

用 table format 說明 a(13, 15) 運算的過程

int a(int n, int m) {
    if (n < 10) {
        if (m < 10) {
            return n + m;
        } else {
            return a (n, m-2) + m ;
        }
    } else {
        return a (n-1, m) + n;
    }
}

AI 在欄位的分支都寫 else，令人分不清是哪個 else。沒關係，我們請他寫清楚一點。

欄位”執行的分支” 的內容可以寫得更清楚嗎

AI 改成執行的條件與動作，更清楚地說明程式執行的段落。
