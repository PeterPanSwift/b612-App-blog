讓 Interface Builder 的 UIView 元件拉 IBAction

當我們在 Interface Builder 製作 App 畫面時，IBAction 可以讓我們設定使用者觸碰元件時觸發的 function，比方按鈕點擊時執行的 function。不過很可惜，並非所有元件都能拉 action，我們經常使用的 UIView 元件就無法。

如下圖所示，橘色的 View 元件只能拉 outlet 和 outlet collection，不能拉 action。

如果希望點擊橘色區塊執行某段程式，比方發出 kiwi 的聲音，我們有許多變通的方法，比方透過 UIButton 元件或是加上 tap gesture recognizer。

然後還有第三種特別的方法。我們可以自訂繼承 UIControl 的類別 ，比方類別 KiwiView，然後將畫面上的 UIView 元件設為類別 KiwiView，如此即可用 UIView 拉 IBAction。

為什麼要繼承 UIControl 呢 ? 因為 UIControl 類別可設定 action，因此繼承 UIControl 的類別都可拉 action。

UIControl - UIKit | Apple Developer Documentation
You do not create instances of this class directly. The class is a subclassing point that you extend to implement…developer.apple.com

而 UIControl 又繼承 UIView，因此繼承 UIControl 的 KiwiView 將可設為 UIView 元件的 class ，讓 Interface Builder 裡的 UIView 元件能拉 action。(當我們從 Interface Builder 設定 UIView 元件的 class 時，只能設定 UIView 或繼承自 UIView 的類別。）

接下來就讓我們一步步示範 UIView 元件拉 IBAction 的步驟吧。

1 在 controller 的畫面上加入 UIView 元件，如下圖的橘色長方形區塊。
2 自訂繼承 UIControl 的類別 KiwiView。
class KiwiView: UIControl {
}
3 將橘色 View 元件的類別設為 KiwiView。

現在我們的 View 已經可以拉 action 了。如下圖所示，我們產生 IBAction viewTapped，並將 event 設為 Touch Up Inside。

接著我們在 function 裡列印訊息 viewTapped。

@IBAction func viewTapped(_ sender: Any) {
   print("viewTapped")
}

App 執行後，每當我們點擊橘色 View，即可像 button 一樣觸發 event Touch Up Inside，呼叫 functin viewTapped，印出訊息 viewTapped。
