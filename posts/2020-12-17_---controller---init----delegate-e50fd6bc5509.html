定義 controller 的 init 設定 delegate

當我們有多個頁面時，利用 delegate 在 controller 間溝通 & 傳資料是常見的方法。比方以下例子的 SecondViewController 透過 delegate 將輸入的文字傳給 FirstViewController。

protocol SecondViewControllerDelegate: AnyObject {
    func update(_ controller: SecondViewController, name: String)
}

class SecondViewController: UIViewController {
    @IBOutlet weak var textField: UITextField!
    weak var delegate: SecondViewControllerDelegate?

    @IBAction func done(_ sender: Any) {
        delegate?.update(self, name: textField.text!)
    }

通常設定 delegate 的方法如下，在準備切換到下一頁時設定 delegate = self。

@IBSegueAction func showSecondView(_ coder: NSCoder) -> SecondViewController? {
    let secondViewController = SecondViewController(coder: coder)
    secondViewController?.delegate = self
    return secondViewController
}

不過如果想讓程式更簡潔，我們也可以另外定義 controller 的 init，在它的 init 傳入 delegate，如此也比較不會忘了設定 delegate。

定義 controller 的 init 設定 delegate

在 SecondViewController 定義搭配參數 delegate 的 init。（ps: 當我們在 controller 定義 init 時，也要同時定義 init?(coder: NSCoder)，因為它是 required)

class SecondViewController: UIViewController {
    
    init?(coder: NSCoder, delegate: SecondViewControllerDelegate) {
        self.delegate = delegate
        super.init(coder: coder)
    }
    
    required init?(coder: NSCoder) {
        fatalError("init(coder:) has not been implemented")
    }

有了剛剛定義的 init，之後當我們要切換到 SecondViewController 時，將可直接傳入 delegate，比方以下 2 個例子。

IBSegueAction

透過 segue 切換時，在 IBSegueAction 生成 SecondViewController。

@IBSegueAction func showSecondView(_ coder: NSCoder) -> SecondViewController? {
     SecondViewController(coder: coder, delegate: self)
}

instantiateViewController(identifier:creator:)

  • 不透過 segue，直接從程式呼叫 instantiateViewController(identifier:creator:) 產生 SecondViewController。
guard let controller = storyboard?.instantiateInitialViewController(creator: { coder in
    SecondViewController(coder: coder, delegate: self)
}) else { return }
