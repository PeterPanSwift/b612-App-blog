function 名稱打錯 — 初學者易犯的程式錯誤

開發 iOS App 時，如果希望某件事發生時執行某段程式碼，時常需要定義事件發生時呼叫的 function。這些 function 一般名字都很長，不是正常人可以背起來的，就算背起來，打字時 99% 的機率會打錯，所以一定要搭配 Xcode 的自動完成。

比方我們想在表格點選時執行某段程式碼，因此輸入 function tableView(_:didSelectRowAt:) ，從選單裡找到目標後按 enter 完成輸入。

func tableView(_ tableView: UITableView, didSelectRowAt indexPath: IndexPath) {
}

如果不小心打錯 function，比方將剛剛 function 的參數 indexPath 打成 path，並不會產生任何錯誤，但這樣反而更不好 !

func tableView(_ tableView: UITableView, didSelectRowAt path: IndexPath) {
}

因為此時 Xcode 將以為我們剛剛輸入的 function 是另一個 function，並不是表格點選時會呼叫的 function。雖然只是 indexPath 變成 path，但只要差一個字，就是不一樣，就好像劉德華跟劉的華帥度差很多一樣。
