SwiftUI 設定內容間距的 contentMargins — iOS 17 新功能

SwiftUI 在 iOS 17 推出新的 modifier contentMargins，我們可以用它調整捲動內容的間距。

以下我們以 ScrollView 為例，先看看未添加 contentMargins 時 ScrollView 包 Text 顯示統一獅加油歌王者無敵的模樣。

如圖所示 ScrollView 的內容緊靠 ScrollView 的邊緣，讓人覺得有點擁擠。

此問題有許多解法，而 iOS 17 的 contentMargins 是其中最方便的解法。contentMargins 專門設定內容的 margin，完全正中我們的需求，我們想調整 ScrollView 內容的 margin，讓文字不要那麼靠邊。

nonisolated
func contentMargins(
    _ length: CGFloat,
    for placement: ContentMarginPlacement = .automatic
) -> some View
contentMargins(_:for:) | Apple Developer Documentation
Configures the content margin for a provided placement.developer.apple.com

我們在 ScrollView 後呼叫 contentMargins(20)，讓 ScrollView 的內容 Text 和 ScrollView 間有著 20 points 的間距。

.contentMargins(20)

為了更清楚看到 contentMargins 的效果，下圖我們將 Text 的背景設為橘色，明顯看到 Text 跟 ScrollView 的左右邊界有著 20 points 的間距。值得注意的，Text 跟 ScrollView 上下邊界的間距超過 20，它還包含 Safe Area 保留的空間，因為 ScrollView 原本就會考慮 Safe Area 防止內容被檔到。

有時我們只想在某些位罝有 margin，或是想要不同位置有不同的 margin，此時可透過參數 edges 指定 margin 的位置。

nonisolated
func contentMargins(
    _ edges: Edge.Set = .all,
    _ length: CGFloat?,
    for placement: ContentMarginPlacement = .automatic
) -> some View

以下例子傳入 .horizontal，只在水平方向添加 20 的 margin。

.contentMargins(.horizontal, 20)

contentMargins 還有個特別的 placement 參數，它可以設定 margin 添加的地方。

當我們沒指定 placement 時，它預設會同時添加 margin 到內容和 scroll bar(scroll indicator)，因此設定 .contentMargins(20) 的畫面如下，scroll bar 上方也多出 20 points 的間距。

設定 .contentMargins(20, for: .scrollContent)時，將只有內容的上方多出 20 points 的間距，scroll bar 不受影響。

剛剛我們都以 ScrollView 為例，同樣的概念也能套用到其它捲動的元件，比方 List、Form 和 TextEditor。

import SwiftUI

struct ContentView: View {
    var body: some View {
        List {
            Text("Ho Ho Ho Ho Ho Ho HoHo")
            Text("Ho Ho Ho Ho Ho Ho")
            Text("Ho~~~~~  Ho~~~  王者無敵")
        }
        .listStyle(.plain)
        .contentMargins(30)
    }
}
