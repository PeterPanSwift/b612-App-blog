搭配 trailing closure，在結尾輸入 { } 生成 SwiftUI 元件

SwiftUI 很多 UI 元件 init 的最後一個參數是 function 型別，因此我們時常看到 SwiftUI 生成元件時採用 trailing closure 語法，於結尾接 { } ，尤其常見於 stack，tab view 之類的容器元件。

Swift 最後一名的 trailing closure
小時候唸書時，我們都不喜歡當最後一名，因為只有壞事，沒有好事。然而在 Swift 的世界，如果傳入 function 的 closure 參數剛好是最後一個參數，反而會有好事發生呢! 我們可以採用 trailing closure…medium.com

接下來就讓我們看看一些常見的例子。

VStack，HStack，ZStack 等 stack 元件

struct ContentView: View {
    var body: some View {
        VStack {
            Text("私奔到月球")
            Text("離開地球表面")
            Text("月亮惹的禍")
        }
    }
}

搭配的 init 如下，前兩個參數有預設值可省略，因此只要用 { } 傳入第三個參數。

init(alignment: HorizontalAlignment = .center, spacing: CGFloat? = nil, @ViewBuilder content: () -> Content)

通常我們會使用 trailing closure 的省略寫法，不過另一種無省略的寫法也可以參考。

List

struct ContentView: View {
    var body: some View {
        List {
            Text("私奔到月球")
            Text("離開地球表面")
            Text("月亮惹的禍")
        }
    }
}

搭配的 init。

init(@ViewBuilder content: () -> Content)
struct ContentView: View {
    let songs = ["私奔到月球", "離開地球表面", "月亮惹的禍"]
    
    var body: some View {
        List(songs, id: \.self) { song in
            Text(song)
        }
    }
}

搭配的 init。

init<Data, ID, RowContent>(_ data: Data, id: KeyPath<Data.Element, ID>, @ViewBuilder rowContent: @escaping (Data.Element) -> RowContent) where Content == ForEach<Data, ID, HStack<RowContent>>, Data : RandomAccessCollection, ID : Hashable, RowContent : View

Group

struct ContentView: View {
    
    var body: some View {
        VStack {
            Text("私奔到月球")
            Group {
                Text("離開地球表面")
                Text("月亮惹的禍")
            }
        }
    }
}

NavigationStack

struct ContentView: View {
    
    var body: some View {
        NavigationStack {
            List {
                Text("私奔到月球")
                Text("離開地球表面")
                Text("月亮惹的禍")
            }
            .navigationTitle("星球情歌")
        }
    }
}

搭配的 init。

init(@ViewBuilder content: () -> Content)

Button

struct ContentView: View {
    
    var body: some View {
        Button("follow") {
            print("follow")
        }
    }
}

搭配的 init。

init<S>(_ title: S, action: @escaping () -> Void) where S : StringProtocol

Form

struct ContentView: View {
    @State private var account = ""
    @State private var password = ""
    
    var body: some View {
        Form {
            TextField("account", text: $account)
            TextField("password", text: $password)
        }
    }
}

搭配的 init。

init(@ViewBuilder content: () -> Content)

結尾輸入 { } 生成 SwiftUI 元件

熟悉在結尾輸入 { } 生成 SwiftUI 元件的寫法後，未來我們在輸入程式生成 SwiftUI 元件時，可以視情況直接輸入 { } 加快開發速度。比方要生成 VStack 時，不是用傳統的 VStack 加 ( 輸入，而是直接輸入 VStack { }。
