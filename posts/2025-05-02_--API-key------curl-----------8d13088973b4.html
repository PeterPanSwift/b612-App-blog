將 API key 存在方便 curl 指令存取的環境變數

串接 API 時，我們時常利用 curl 指令測試 API。當 API 必須包含 API key 時，輸入 curl 指令變得麻煩許多，因為要輸入冗長無比的 API key。

以 OpenAI 聊天的 responses API 為例，它的 curl 指令格式如下，Authorization: Bearer 之後要接 API key。

curl https://api.openai.com/v1/responses \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer API key" \
    -d '{
        "model": "gpt-4.1",
        "input": "如何成為超帥的 iOS App 工程師"
    }'

比方當 API key 是 12345abcde 時，-H 後的字串變成 "Authorization: Bearer 12345abcde"。

為了簡化 curl 指令，我們可將 API key 存在環境變數，之後輸入 curl 指令時 API key 只要打 $變數名即可。

在 Mac 上操作時，我們可從 ~/.zshenv 或 ~/.zshrc. 設定環境變數，將 API key 存在 OPENAI_API_KEY。

從 Terminal 輸入以下指令，將 API key 換成真正的 API key，在 ~/.zshenv 加入儲存 API key 的 OPENAI_API_KEY。

echo 'export OPENAI_API_KEY=API key' >> ~/.zshenv

之後在 curl 指令輸入 $OPENAI_API_KEY 即可取得真正的 API key。

curl https://api.openai.com/v1/responses \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer $OPENAI_API_KEY" \
    -d '{
        "model": "gpt-4.1",
        "input": "如何成為超帥的 iOS App 工程師"
    }'
