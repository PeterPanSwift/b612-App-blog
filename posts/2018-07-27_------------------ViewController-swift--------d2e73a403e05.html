為什麼要新增檔案定義型別? 何不在 ViewController.swift 定義型別就好

開發 iOS App 時，我們時常需要新增檔案，然後在檔案裡定義 class, struct 或 enum。

不過有朋友問到，為什麼要這麼麻煩。建立新專案時，Xcode 已經幫我們產生 ContentView.swift 或 ViewController.swift，何不把所有的型別都定義在裡面呢 ?

  • ContentView.swift
struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundColor(.accentColor)
            Text("Hello, world!")
        }
        .padding()
    }
}

struct Baby {
    
}
  • ViewController.swift
class LoginViewController: UIViewController {

}
class ViewController: UIViewController {

}

struct Baby {

}

的確，我們可以在一個檔案裡定義很多型別，你要在一個檔案裡定義一百個型別也沒問題。但是這樣會造成檔案裡的程式宇宙無敵多，比方一個檔案就有一萬行程式，之後非常不方便找尋和修改某個型別的某個 function。

因此比較好的做法，還是儘量讓一個檔案包含一個型別，檔案名就是型別的名字，方便日後的查詢和修改。
