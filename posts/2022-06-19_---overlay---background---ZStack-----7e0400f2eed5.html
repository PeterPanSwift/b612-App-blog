使用 overlay / background & ZStack 的差別

使用 SwiftUI 製作 App 畫面時，overlay / background & ZStack 都能實現堆疊的效果，比方以下三種寫法都可實現文字保護我蓋在 Penny 圖片上的酷酷畫面:

  • 使用 ZStack
struct ContentView: View {
    var body: some View {
        ZStack(alignment: .bottom) {
            Image("penny")
                .resizable()
                .scaledToFill()
                .frame(width: 300, height: 300)
                .clipped()
            Text("保護我")
                .font(.largeTitle)
                .padding()
                .background(.gray.opacity(0.7))
        }
    }
}
  • 使用 overlay
struct ContentView: View {
    var body: some View {
        Image("penny")
            .resizable()
            .scaledToFill()
            .frame(width: 300, height: 300)
            .clipped()
            .overlay(alignment: .bottom) {
                Text("保護我")
                    .font(.largeTitle)
                    .padding()
                    .background(.gray.opacity(0.7))
            }
    }
}
  • 使用 background
struct ContentView: View {
    var body: some View {
        Text("保護我")
            .font(.largeTitle)
            .padding()
            .background(.gray.opacity(0.7))
            .background(alignment: .bottom) {
                Image("penny")
                    .resizable()
                    .scaledToFill()
                    .frame(width: 300, height: 300)
                    .clipped()
            }
    }
}

不過有些時候，使用 overlay / background & ZStack 還是有差別的。使用 ZStack 時，ZStack 裡東西的大小不會互相影響。使用 overlay / background 時，overlay / background 的大小將依據它作用的元件。

以下我們將文字改成字數較多的保護我歌詞說明。

  • 使用 ZStack
struct ContentView: View {
    var body: some View {
        ZStack(alignment: .bottom) {
            Image("penny")
                .resizable()
                .scaledToFill()
                .frame(width: 300, height: 300)
                .clipped()
            Text("我假裝翻來覆去 翻來覆去 不讓你靠近我 不讓你找到我的耳朵 說愛我 我看你走來走去 走來走去 像找不到出口的狗 最後又緊緊的抱著你 對你說～ 我要你保護我 我要你保護我")
                .padding()
                .background(.gray.opacity(0.7))
        }
    }
}

ZStack 裡東西的大小不會互相影響，因此 Text 的寬度等於螢幕的寬度，不受圖片的影響。

  • 使用 overlay
struct ContentView: View {
    var body: some View {
        Image("penny")
            .resizable()
            .scaledToFill()
            .frame(width: 300, height: 300)
            .clipped()
            .overlay(alignment: .bottom) {
                Text("我假裝翻來覆去 翻來覆去 不讓你靠近我 不讓你找到我的耳朵 說愛我 我看你走來走去 走來走去 像找不到出口的狗 最後又緊緊的抱著你 對你說～ 我要你保護我 我要你保護我")
                    .padding()
                    .background(.gray.opacity(0.7))
            }        
    }
}

overlay 的大小將依據它作用的元件，由於 overlay 作用在 Image，因此文字的寬度將等於圖片的寬度。

我們也可以額外設定 overlay 元件的大小，讓它依據我們指定的大小，而不是 overlay 作用的元件大小。因此以下例子的文字寬度將是 350，而不是圖片的 300。

struct ContentView: View {
    var body: some View {
        Image("penny")
            .resizable()
            .scaledToFill()
            .frame(width: 300, height: 300)
            .clipped()
            .overlay(alignment: .bottom) {
                Text("我假裝翻來覆去 翻來覆去 不讓你靠近我 不讓你找到我的耳朵 說愛我 我看你走來走去 走來走去 像找不到出口的狗 最後又緊緊的抱著你 對你說～ 我要你保護我 我要你保護我")
                    .padding()
                    .frame(width: 350)
                    .background(.gray.opacity(0.7))
            }
    }
}

background 的大小將依據它作用的元件，由於 background 作用在文字，因此圖片的寬度將等於文字的寬度。

我們也可以額外設定 background 元件的大小，讓它依據我們指定的大小，而不是 background 作用的元件大小。因此以下例子的圖片寬度將是 300，而不是文字的寬度。

struct ContentView: View {
    var body: some View {
