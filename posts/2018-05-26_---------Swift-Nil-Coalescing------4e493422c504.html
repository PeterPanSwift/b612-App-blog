初學者容易誤解的 Swift Nil-Coalescing 語法範例

最近在看 iOS App 開發教學影片時，發現範例裡有個初學者容易看不懂的寫法，程式如下:

let isInputValid = textField.text?.count ?? 0 > 0

它使用了 ??，Nil-Coalescing 的技巧。利用它，程式將判斷 optional 是否有值，如果有的話將讀取 optional 的內容，無值的話將使用 ?? 後的值。

問題出在剛剛的程式又多了一個 > 0，所以有的人可能誤以為後面的 0 > 0 要合起來看，因此覺得奇怪， 0 > 0 是 false，而 count 的型別卻是整數， ?? 的左邊和右邊的內容應該要同一個型別呀。

其實 0 > 0 不能合起來看，一旦我們為 0 > 0 加上 ( )，程式馬上會顯示紅色錯誤。

所以剛剛程式的正確解讀應該是 textField.text?.count ?? 0 要合起來看，它們運算的結果再來跟 0 比大小。如果想讓程式更清楚，可以改成以下寫法

let isInputValid = (textField.text?.count ?? 0) > 0

這樣子就清楚多了，此行程式的目的是先讀取文字的字數，如果沒有文字，字數將設為 0，然後再判斷是否大於 0，若大於 0 即表示使用者有輸入文字。
