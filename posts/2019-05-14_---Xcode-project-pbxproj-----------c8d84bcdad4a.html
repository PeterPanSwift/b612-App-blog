2. project.pbxproj 檔

也許滿多人好奇，我們的 Xcode 專案裡有這個傢伙嗎 ? 的確你不容易發現它，它藏在 .xcodeproj 資料夾下，比方我們點選下圖的 DemoApp.xcodeproj，從右鍵選單選取 Show Package Contents，即可發現 project.pbxproj。

project.pbxproj 檔的內容決定專案 Project navigator 顯示的檔案清單和順序，因此這也解釋了為何多人合作時它容易產生衝突。因為每個人會新增刪除檔案，有時還會調整檔案的順序，於是 git 就不知該如何合併，它不知該保留刪除哪個檔案，哪個檔案要在前面，哪個要在後面。

project.pbxproj 檔發生衝突的畫面

當然遇到衝突也不是世界末日，比方我們可以參考以下連結的做法手動修正。

一步一腳印，修正 Xcode App 專案的版本管理衝突(conflict)
Git 版本管理大部分的時候都很聰明，儘管我們的 App 是多人一起開發，它還是可以將大家的修改自動合併，不用我們操心。medium.com

但是由於每個人都認真地寫著 App，新增了很多檔案，因此 project.pbxproj 往往會有很多衝突，一個個解決十分辛苦。

其實 project.pbxproj 大部份的衝突解法都是同時保留本機端和遠端的檔案，畢竟你不想刪除自己辛苦寫的程式，也不敢刪除隊友辛苦寫的程式，所以我們可以透過 .gitattributes 檔，告訴 git 在 project.pbxproj 遇到衝突時，同時保留本機端和遠端的修改，如此即可解決 project.pbxproj 大部分的衝突。

詳細的操作步驟如下:

1 在 Xcode 的專案資料夾下新增檔案 .gitattributes。

如果原本就有 .gitattributes 則不用新增。

(1) 點選 File > New > File。

點選 iOS 下的 Empty。

輸入檔名 .gitattributes。

由於檔名以 . 開頭，所以會出現警告視窗。不過我們就是想要一意孤行地用 . 開頭，所以請點選 Use . 。

順利地產生 .gitattributes。

2 在 .gitattributes 裡加入以下內容。

*.pbxproj merge=union

3 點選 Source Control > Commit 記錄剛剛的修改。

最後將剛剛的修改 push 到 GitHub 後，即可上傳 .gitattributes，到時候其它隊友 pull 專案時也能取得 .gitattributes。之後有了 .gitattributes 的幫忙，我們終於不再為了 project.pbxproj 版本衝突的問題煩心了 !
