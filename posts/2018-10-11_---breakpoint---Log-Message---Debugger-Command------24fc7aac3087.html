利用 breakpoint 的 Log Message 和 Debugger Command 列印資訊

開發 Swift iOS App 時，我們時常會用 print 列印資訊，查詢某個變數的內容，或是檢查某個 function 是否執行。但使用 print 有個缺點，我們在 App 上架前，還要花時間移除這些不必要的 print。

其實有個更方便的方法。利用 breakpoint 的 Log Message 和 Debugger Command，我們不只可以列印資訊，也不需要煩惱移除 print 的問題呢。

1 加入中斷點。

比方我們希望檢查 AppDelegate 的 applicationDidEnterBackground(_:) 何時執行。點擊 func applicationDidEnterBackground 左邊的區塊，加入表示中斷點的藍色箭頭。

2 編輯中斷點。

對藍色箭頭按右鍵 ，從右鍵選單點選 Edit Breakpoint。

3 設定 App 在中斷點停下時執行動作 Log Message。

點擊 Add Action。

有許多 action 可以選擇。比方 Sound 會發出聲音。不過現在我們想要的是在 Console 列印資訊，所以請選擇 Log Message。

檢查被選取的是 Log message to console，而不是 Speak message。

5 印出資訊後讓 App 繼續執行，不要停在中斷點。

我們只想印出資訊，並不想 App 中斷執行，所以請勾選 Automatically continue after evaluating actions。

5 印出中斷點所在的 function 名稱。

輸入 %B 將印出中斷點的名字，它將印出中斷點所在的 function 名字。

在剛剛的例子，當 App 進到背景時，Console 將印出 applicationDidEnterBackground(_:)。

如下圖所示，若我們於第 35 & 36 行設定中斷點，它印出的訊息也會是 applicationDidEnterBackground(_:)，因為它們都在 function applicationDidEnterBackground(_:) 裡。

6 印出中斷點觸發的次數。

%H 代表中斷點觸發的次數

當 App 第二次進入背景時，印出 applicationDidEnterBackground(_:) 2。

7 印出變數的內容

在 @@ 裡可包含想執行的程式，因此 @cuteBaby.age@ 將印出 age 的數字。
