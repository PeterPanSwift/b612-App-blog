if and switch expressions in Swift 5.9

在 Swift 5.9 if 和 switch 這對好兄弟變得更厲害了 ! 當 if { } 和 switch case 裡的程式只有一行時，那行程式的運算結果可以直接存入變數或回傳。

以下我們以撲克牌花色轉換成分數的例子說明，示範以下兩種應用的 case。

  • 設定變數的內容(也可以在宣告變數時設定內容)。
  • function，computed property 和 closure 的回傳結果。

設定變數的內容(也可以在宣告變數時設定內容)

  • if

在舊版的 Swift，我們必須在每個 if { } 裡用 suitValue = 設定內容。

func printSuitValue(suit: String) {
    let suitValue: Int
    if suit == "♠︎" {
        suitValue = 4
    } else if suit == "♥︎" {
        suitValue = 3
    } else if suit == "♦︎" {
        suitValue = 2
    } else {
        suitValue = 1
    }
    print(suitValue)
}

新版簡單多了，if { } 裡程式的運算結果可以直接存入變數。

func printSuitValue(suit: String) {
    let suitValue =
        if suit == "♠︎" { 4 }
        else if suit == "♥︎" { 3 }
        else if suit == "♦︎" { 2 }
        else { 1 }
    print(suitValue)
}

不過有一點要特別注意，if { } 裡的程式只有一行時才能這樣寫，因此以下程式會產生錯誤。

  • switch

在舊版的 Swift，我們必須在每個 case 裡用 suitValue = 設定內容。

func printSuitValue(suit: String) {
    let suitValue: Int
    switch suit {
    case "♠︎":
        suitValue = 4
    case "♥︎":
        suitValue = 3
    case "♦︎":
        suitValue = 2
    default:
        suitValue = 1
    }
    print(suitValue)
}

新版簡單多了，case 裡程式的運算結果可以直接存入變數。(ps: 同樣有一行的限制)

func printSuitValue(suit: String) {
    let suitValue = switch suit {
        case "♠︎": 4
        case "♥︎": 3
        case "♦︎": 2
        default: 1
    }
    print(suitValue)
}

function，computed property 和 closure 的回傳結果

  • if

在舊版的 Swift，我們必須在每個 if { } 裡用 return 回傳結果。

func getSuitValue(suit: String) -> Int {
    if suit == "♠︎" { return 4 }
    else if suit == "♥︎" { return 3 }
    else if suit == "♦︎" { return 2 }
    else { return 1 }
}

新版簡單多了，if { } 裡程式的運算結果可以直接回傳。(ps: 同樣有一行的限制)

func getSuitValue(suit: String) -> Int {
    if suit == "♠︎" { 4 }
    else if suit == "♥︎" { 3 }
    else if suit == "♦︎" { 2 }
    else { 1 }
}

在舊版的 Swift，我們必須在每個 case 裡用 return 回傳結果。

func getSuitValue(suit: String) -> Int {

    case "♠︎":
        return 4
    case "♥︎":
        return 3
    case "♦︎":
        return 2
    default:
        return 1
    }
}
