使用 GitHub Copilot 生成 SwiftUI preview 需要的假資料

設計 SwiftUI 畫面時，我們時常在 preview 生成畫面需要的假資料。以下我們示範使用 VS Code 搭配 GitHub Copilot 生成 SwiftUI preview 需要的假資料。

以下為 SwiftUI 的範例畫面 AnimalRow。

import SwiftUI

struct AnimalRow: View {
    let name: String
    let emoji: String
    let description: String
    
    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    AnimalRow()
}

我們加了三個 property，因此產生錯誤 Missing arguments。

以下我們使用 GitHub Copilot 生成 AnimalRow 需要的假資料。

  • 範例 1。

點選 Fix using Copilot。

AI 隨機生成某個動物的資料。

  • 範例 2。

使用 inline chat(cmd + i) 產生 AnimalRow。

  • 範例 3。

點選 Modify Using Copilot。

請 AI 生成企鵝相關的資料。

  • 範例 4。

使用 inline chat(cmd + i) 產生 AnimalRow，指定生成企鵝相關的資料。

  • 範例 5。

在 preview 傳入自訂的 Animal 資料。

import SwiftUI

struct AnimalRow: View {
    let animal: Animal

    var body: some View {
        Text("Hello, World!")
    }
}

#Preview {
    AnimalRow()
}

點選 Modify Using Copilot。

加入 Animal.swift，讓 AI 知道 Animal 的定義。

  • 範例 6。

我們也常將畫面顯示的資料定義成型別，以下我們使用 inline chat(cmd + i) 在 Animal 裡生成 preview 需要的假資料。

使用 inline chat(cmd + i) 生成多筆資料的 array。
