Invalid redeclaration 的重覆宣告問題

撰寫 Swift 程式時，我們可以產生很多的變數跟常數，但請記得名字不能重覆，比方以下例子宣告了三個名字都是 coolName 的變數，因此產生紅色錯誤。

var coolName = "彼得潘"
var coolName = "忠孝新生張智霖"
var coolName = "忠孝新生Cool魔"

此時紅色的錯誤訊息顯示

Invalid redeclaration of 'coolName'

Invalid 是無效的，redeclaration 是重覆宣告，因此意思是我們重覆宣告了 coolName，造成第 4 行跟第 6 行的程式是無效的。

從 Xcode playground 下方的 Debug area 也可以看到更詳細的錯誤說明，它說我們已經在 var coolName = "彼得潘" 宣告 coolName，所以不能用 var coolName = "忠孝新生張智霖" 宣告另一個也叫 coolName 的變數。

error: MyPlayground.playground:4:5: error: invalid redeclaration of 'name'
var name = "忠孝新生張智霖"
MyPlayground.playground:2:5: note: 'name' previously declared here
var name = "彼得潘"

如果我們將剛剛的例子改成以下程式則是可以的，此時只有一個名字叫 coolName 的變數，它的內容一開始是彼得潘，後來先被改成忠孝新生張智霖，接著再改成忠孝新生Cool魔。

var coolName = "彼得潘"
coolName = "忠孝新生張智霖"
coolName = "忠孝新生Cool魔"
