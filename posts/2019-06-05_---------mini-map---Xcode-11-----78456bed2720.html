快速瀏覽程式的 mini map — Xcode 11 新功能

當我們在 Xcode 11 撰寫程式時，右手邊多了一個叫 mini map 的區塊。

什麼是 mini map ?

你可以把它想成程式碼的縮圖。當我們程式碼很多時，它可以標示出其中重要的段落，幫助我們快速瀏覽整個程式。

如下圖所示，縮圖顯示的顏色跟縮排，完全對應左邊的程式碼。因此上方的灰色區塊是注解，接著是 import 的段落。

當滑鼠游標停在 mini map 上

它將標示出對應的東西，比方型別，property，method。

當滑鼠游標停在 mini map 上時，若我們還按住 command 鍵，此時將如下圖所示，標示出檔案裡所有定義的型別，property & method。

點選 mini map 的某個區塊

點選後程式將跳到對應的段落，比方下圖的 function viewDidLoad

放大突顯 mark 標示的文字

如下圖所示，mini map 裡 Properties & Methods 特別明顯，因為它是以 // MARK: 標示的文字。// MARK: 標示一般是為了將程式區分成不同區塊，因此突顯在 mini map 裡也能幫助我們更方便找尋對應的段落。

mini map 標示的其它小地方

mini map 還會貼心地標示出程式裡出現的警告，錯誤，中斷點，程式碼修改的段落(對應版本管理) & 搜尋文字出現的地方。比方以下例子，我們搜尋 notification 時，對應的文字將被標示為黃色，此時右邊的 mini map 也可以看到黃色的區塊。
