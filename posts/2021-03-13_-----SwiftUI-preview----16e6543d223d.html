設定多個 SwiftUI preview 畫面

利用 SwiftUI preview 我們可以方便地預覽 App 畫面，而且我們還可以設定條件，預覽不同 case 下的 App 畫面。

假設原本的 SwiftUI 程式如下。

struct ContentView: View {
    var body: some View {
        Image("community")
            .resizable()
            .scaledToFit()
            .clipShape(Circle())
    }
}
struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        Group {
            ContentView()
          
        }
    }
}

此時我們將看到一個預覽畫面，顯示好笑影集 community 的圖片。點選下圖右上的 + 將新增預覽畫面。

新增預覽畫面後，preview 的 body 變成以 Group 包含兩個 ContentView，因此將出現兩個預預覽畫面。

struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        Group {
            ContentView()
            ContentView()
          
        }
    }
}

不過現在兩個預覽畫面一模一樣，因此通常我們會再點選下圖紅色框框的 Inspect Preview 調整畫面尺寸，顏色，文字尺寸等。

比方將 Device 設為 iPad Air，Color Scheme 設為 Dark。

struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        Group {
            ContentView()
            ContentView()
                .previewDevice("iPad Air (4th generation)")
                .preferredColorScheme(.dark)
          
        }
    }
}

Cool，現在我們可以一口氣看到白色的 iPhone 畫面跟黑色的 iPad 畫面。
