請 AI 建立 App 的畫面程式檔和設定 storyboard

開發 SwiftUI / UIKit / Flutter App 時，我們時常在新增畫面時選擇合適的 template 新增檔案。現在這些小事也可以請 AI 代勞了，而且在 UIKit 還可以請 AI 連 storyboard 都幫我們加好畫面呢 !

UIKit App

從前沒有 AI 時，建立一個新畫面往往要 3 個步驟。

  • new Cocoa Touch Class。
  • 在 storyboard 加入 view controller。
  • 從 Identity inspector 設定 view controller 的 class name。

現在透過 AI 我們可以一句 prompt 搞定。以下我們用 VS Code 搭配 Codex 示範。(ps: Xcode 的 Coding Assistant 還做不到，因為它不會修改 storyboard。)

prompt: create LoginViewController class file, then add LoginViewController in storyboard

AI 的回答。

Added a dedicated login controller and scene wired into the storyboard.

  • New LoginViewController subclass with a basic setup so the view renders with the system background (Demo2/LoginViewController.swift:1).
  • Inserted a Login scene in Main.storyboard, bound to LoginViewController and given a storyboard identifier for future navigation hooks (Demo2/Base.lproj/Main.storyboard:23).

AI 新增了 LoginViewController.swift。

而且也幫我們在 storyboard 加入 LoginViewController。

SwiftUI App

使用 Xcode 的 Coding Assistant。

prompt: new SwiftUI view LoginView with Text Login

Flutter App
