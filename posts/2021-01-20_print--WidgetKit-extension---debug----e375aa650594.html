print WidgetKit extension 的 debug 訊息

開發 iOS App 時，我們時常透過 print 列印訊息 debug，不過當我們在 WidgetKit extension 的程式列印訊息，然而在模擬器執行時，它卻會時靈時不靈，就像天氣一樣難以捉摸。

比方我們在以下 widget 程式加上 print。

func getTimeline(in context: Context, completion: @escaping (Timeline<Entry>) -> ()) {
        var entries: [SimpleEntry] = []
        let currentDate = Date()
        for hourOffset in 0 ..< 5 {
            let entryDate = Calendar.current.date(byAdding: .hour, value: hourOffset, to: currentDate)!
            let entry = SimpleEntry(date: entryDate)
            entries.append(entry)
            print("add \(entry.date)")
        }
        let timeline = Timeline(entries: entries, policy: .atEnd)
        completion(timeline)
}

如下圖所示，選擇 widget 的 scheme & 模擬器，然後點選三角形啟動 widget。

運氣不好時，右下的 console 將一片空白，完全看不到訊息印出。

為什麼會這樣呢? 從下圖的 Xcode menu 可發現 Attach to Process 沒有連結到 widget，因此訊息不會顯示在 console。

不過如果運氣好的話，我們有時也能看到訊息印出。

因爲 Attach to Process 有自動連結到 widget。

解法

這個問題應該算從 Xcode 的模擬器啟動 widget 的 bug，若是從實機啟動，將可避免此問題，看到訊息順利印出。
