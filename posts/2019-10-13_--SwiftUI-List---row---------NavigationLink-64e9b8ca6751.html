讓 SwiftUI List 的 row 有多個可點選的 NavigationLink

當我們使用 iOS App 時，一般常見的表格頁面通常可以點選某個 row 切換到下一頁顯示詳細資料，例如點選 TV App 的 Spider-Man: Far From Home 進入此生必看的蜘蛛人電影頁面。

不過有些 App 的表格頁面更為複雜，點選 row 裡不同地方將切換到不同頁面，例如下圖 Medium App 的文章列表，點選 row 裡的作者名字將進入作者的個人頁面，點選 row 裡其它地方將進入文章頁面。

接下來就讓我們利用 SwiftUI 的 List，NavigationView & NavigationLink 實現這樣的功能吧。(ps: 切換頁面的相關說明可參考以下連結)

利用 NavigationView & NavigationLink 切換頁面
SwiftUI 利用 NavigationView 管理切換多層頁面，它會在畫面的上方長出一條 navigation bar，bar 的左上角則有 back 返回的按鈕，類似以下的 Music App 畫面。medium.com

設計 iOS App 時，我們時常利用表格(List)顯示豐富的資料內容。如下圖所示，小說精選列表顯示一本本精彩的小說，點選作者名跳到作者頁面，點選整本書的 row 跳到書本頁面。medium.com

其它解法: 利用 scroll view 包住 row

我們將 List 裡的每個 row 以 HStack 設計，在 HStack 裡容納兩個 Text，點選電影名稱進入電影頁面 MovieDetail，點選演員名稱進入演員頁面 ActorView。

struct Movie {
   let name: String
   let actor: String
}
struct MovieDetail: View {
   let name: String
   var body: some View {
      Text(name)
   }
}
struct ActorView: View {
   let name: String
   var body: some View {
      Text(name)
   }
}
struct ContentView: View {
   let movies = [Movie(name: "蜘蛛人", actor: "Tom"), Movie(name: "小丑", actor: "Joaquin")]
   var body: some View {
      NavigationView {
         List(movies, id: \.name) { (movie) in
            HStack {
               NavigationLink(destination: MovieDetail(name: movie.name)) {
                  Text(movie.name)
               }
               NavigationLink(destination: ActorView(name: movie.actor)) {
                  Text(movie.actor)
               }
            } 
         }
      }
   }
}

然而出來的 App 畫面卻慘不忍睹，NavigationLink 在 Text 後會補上 >，所以現在一個 row 變成有兩個 >，而且點選也不太正常，比方點選蜘蛛人時，整行 row 都會被點選，因此也會進入 Tom 的頁面。

利用 scroll view 包住 row

目前此問題的其中一種解法是另外用 ScrollView 包住 row 的內容，因此我們將 HStack 包在 ScrollView 裡。

NavigationView {
   List(movies, id: \.name) { (movie) in
      ScrollView {
         HStack {
            NavigationLink(destination: MovieDetail(name: movie.name)) {
               Text(movie.name)
            }
            NavigationLink(destination: ActorView(name: movie.actor)) {
               Text(movie.actor)
            }
         }
      }
   }
}

如下圖所示，現在畫面正常多了，而且電影名和演員名可分別點選，不過加了 ScrollView 卻可能產生其它問題。如下圖所示，此時 cell 的高度增加了一點，文字看起來有點偏上，沒有置中。

此問題可再透過以下方法修正，從 ScrollView 呼叫 fixedSize。

若希望電影名稱 & 演員名稱分開，則要調整 2 個地方。除了在中間加入 Spacer 外，還要從 ScrollView 呼叫 fixedSize(horizontal: false, vertical: true)，讓水平的 fixedSize 為 false，如此 ScrollView 的寬度才會變成幾乎等於螢幕的寬度。(不會剛好等於，因為預設 row 的內容和表格邊邊會有段安全距離。)

struct ContentView: View {
   let movies = [Movie(name: "蜘蛛人", actor: "Tom"), Movie(name: "小丑", actor: "Joaquin")]
   var body: some View {
      NavigationView {
         List(movies, id: \.name) { (movie) in
            ScrollView {
                HStack {
                   NavigationLink(destination: MovieDetail(name: movie.name)) {
                      Text(movie.name)
                   }
                   Spacer()
                   NavigationLink(destination: ActorView(name: movie.actor)) {
                      Text(movie.actor)
                   }
                }
            }
            .fixedSize(horizontal: false, vertical: true)
         }
      }
   }
}
