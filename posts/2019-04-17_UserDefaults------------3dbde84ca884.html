UserDefaults 存太多資料會怎樣 ?

iOS App 有許多儲存資料的方法，其中 UserDefaults 應該是其中最簡單的方法。因為它可以儲存 Data 型別的資料，所以我們只要將資料轉換成 Data 型別(比方利用 PropertyListEncoder 轉換)，任何資料都能存入 UserDefaults。

UserDefaults | Apple Developer Documentation
An interface to the user's defaults database, where you store key-value pairs persistently across launches of your app.developer.apple.com

那如果我們將網路上抓下來的圖片都存在 UserDefaults，在 UserDefaults 存太多資料會怎樣呢 ?

就讓我們用以下程式做個小實驗吧。假設我們按了多次 button，觸發 function saveData 抓圖並存入 UserDefaults。

@IBAction func saveData(_ sender: Any) {
   if let data = try? Data(contentsOf: URL(string: "https://bit.ly/2UlVU5t")!) {
       UserDefaults.standard.set(data, forKey: UUID().uuidString)
   }
}
啟動 App

App 初次執行時，還是苗條的身材，只有 48.3 MB。

當我們按下 button 時，程式將抓圖並存入 UserDefaults，此時 UserDefaults 也會佔據愈多的記憶體。

更可怕的是，下一次我們重新啟動 App 時， App 一開始就佔據 100 MB 的記憶體。這是因為 App 啟動時 UserDefaults 將讀取 plist 檔儲存的資料(UserDefaults 的資料其實是存在 plist 檔裡)，將資料全部載入到記憶體，所以當你儲存的資料愈多，佔據的記憶體也會愈多。

因此記得不要儲存太多東西在 UserDefaults，因為它將增加 App 使用的記憶體。當 App 使用太多記憶體時，可是被會 iOS 殺掉閃退的。
