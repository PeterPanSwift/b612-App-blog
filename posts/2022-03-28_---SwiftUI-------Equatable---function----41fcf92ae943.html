影響 SwiftUI 畫面更新的 Equatable & function ==

資料改變時更新畫面是 SwiftUI 十分方便的功能，不過當我們自訂判斷資料是否相等的 function == 時，卻可能會遇到問題。

以下我們以實際的例子說明。

資料型別 Love 的定義如下，count 記錄有幾個愛心。

struct Love {
    var count: Int
    var name: String
}

LoveCountView 將顯示愛心的數量。

struct LoveCountView: View {
    let love: Love
    
    var body: some View {
        HStack {
            Image(systemName: "heart.fill")
            Text("\(love.count)")
        }
        .font(.largeTitle)
    }
}

顯示 App 畫面的 ContentView。

struct ContentView: View {
    @State private var love = Love(count: 1, name: "Wendy")
    
    var body: some View {
        VStack {
            Image(.love)
                .resizable()
                .scaledToFit()
            LoveCountView(love: love)
            Button("每天愛你多一些") {
                love.count += 1
            }
        }
    }
}

執行 App 後，點選 button 每天愛你多一些會增加愛心的數量，如下圖所示，畫面將即時更新顯示最新的愛心數量。

自訂判斷資料是否相等的 function ==

接著我們修改 Love，讓它遵從 protocol Equatable，定義 function ==，判斷當兩個 Love 名稱一樣時為相等。

struct Love: Equatable {
    var count: Int
    var name: String
    
    static func == (lhs: Self, rhs: Self) -> Bool {
        lhs.name == rhs.name
    }
}

執行 App 後，結果將大大不同。點選 button 每天愛你多一些會增加 Love 的 count，但是畫面卻不會更新，數量固定顯示 1。怎麼會這樣呢 ?

這是因為 SwiftUI 利用 Love 的 function == 判斷資料是否改變，有改變才更新畫面。由於我們在 function ==裡判斷 name 一樣即為相等，而剛剛例子的 name 固定是 Wendy，只有 count 改變，因此 SwiftUI 會覺得資料沒變。

我們也可以在 func == 加入 print，檢查它是否被觸發。

static func == (lhs: Self, rhs: Self) -> Bool {
    print("compare", lhs, rhs)
    return lhs.name == rhs.name
}

執行 App 後，點選 button 時將印出以下訊息，證明 SwiftUI 會透過 Love 的 function == 判斷資料是否改變。

compare Love(count: 1, name: "Wendy") Love(count: 2, name: "Wendy")

剛剛畫面更新的問題出在 function == 。不過如果我們還是想維持以 Love name 一樣判斷相等，能不能在改變 Love count 時讓畫面更新呢？

可以的 ! 以下我們將介紹兩種方法。

  • 方法 1: 利用 @Observable (iOS 17 以上)。
  • 方法 2: 利用 ObservableObject。

方法 1: 利用 @Observable (iOS 17 以上)

  • 定義加上 @Observable 的 LoveData。

之後它的 property love 改變時將觸發畫面更新。

@Observable class LoveData {
    var love: Love
    
    init(love: Love) {
        self.love = love
    }
}
  • LoveCountView 的 property love 加上 Binding。

為了讓 love 改變時 LoveCountView 更新，此處必須加上 Binding。

struct LoveCountView: View {
    @Binding var love: Love
    
    var body: some View {
        HStack {
            Image(systemName: "heart.fill")
            Text("\(love.count)")
        }
        .font(.largeTitle)
    }
}
  • 產生 LoveCountView 時傳入 Love 的 Binding。
struct ContentView: View {
    @State private var loveData = LoveData(love: Love(count: 1, name: "Wendy"))
    
    var body: some View {
        VStack {
            Image(.love)
                .resizable()
                .scaledToFit()
            LoveCountView(love: $loveData.love)
            Button("每天愛你多一些") {
                loveData.love.count += 1
            }
        }
    }
}

方法 2: 利用 ObservableObject

  • 定義遵從 protocol ObservableObject 的 LoveData。

property love 宣告為 Published，love 改變時將觸發畫面更新。

class LoveData: ObservableObject {
    @Published var love: Love
    
    init(love: Love) {
        self.love = love
    }
}
  • LoveCountView 的 property loveData 加上 ObservedObject。

為了讓 love 改變時 LoveCountView 更新，此處必須加上 ObservedObject。在 LoveCountView 裡我們宣告 property 的型別是 LoveData，而不是 Love，因為宣告為 ＠Binding 搭配 Love 時一樣無法觸發畫面更新。

struct LoveCountView: View {
    @ObservedObject var loveData: LoveData
    
    var body: some View {
        HStack {
            Image(systemName: "heart.fill")
            Text("\(loveData.love.count)")
        }
        .font(.largeTitle)
    }
}
  • 產生 LoveCountView 時傳入 loveData。
struct ContentView: View {
    @StateObject var loveData = LoveData(love: Love(count: 1, name: "Wendy"))
    
    var body: some View {
        VStack {
            Image(.love)
                .resizable()
                .scaledToFit()
            LoveCountView(loveData: loveData)
            Button("每天愛你多一些") {
                loveData.love.count += 1
            }
        }
    }
}

Cool，終於正常了 ! 執行 App 後，點選 button 每天愛你多一些會增加愛心的數量，而且畫面也會即時更新顯示最新的愛心數量。
