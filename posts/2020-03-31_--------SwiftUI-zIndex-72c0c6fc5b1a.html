調整元件階層的 SwiftUI zIndex

當我們用 SwiftUI 的 VStack，HStack，ZStack 排列元件時，元件的階層順序將和 { } 裡元件的順序一致，{ } 裡愈先寫的元件會在愈下層，因此以下範例的 head 會在下層，body 會在上層。

struct ContentView: View {
    var body: some View {
        VStack {
            Image("head")
                .resizable()
                .scaledToFit()
            Image("body")
                .resizable()
                .scaledToFit()
                .offset(x: -20, y: -95)
        }
    }
}

有時我們會想改變元件的階層順序，像在點選元件時改變圖層的順序，或是以上圖為例，頭的圖片蓋住身體會比較好看，而不是身體的圖片蓋住頭。

SwiftUI 有個 zIndex modifier 可以幫我們調整元件的階層順序。

zIndex(_:) | Apple Developer Documentation
Controls the display order of overlapping views.developer.apple.com

每個 UI 元件都有一個控制階層的 z index，預設為 0。z index 數字愈大的元件將在愈上層，因此當我們從 haed 呼叫 zIndex(1)，讓它的 zIndex 大於 body 時，即可讓頭蓋住身體。

struct ContentView: View {
   var body: some View {
        VStack {
            Image("head")
                .resizable()
                .scaledToFit()
                .zIndex(1)
            
            Image("body")
                .resizable()
                .scaledToFit()
                .offset(x: -20, y: -95)
        }
    }
}
