利用 SwiftUI 的 scrollTargetBehavior(.paging) 實現分頁捲動效果

開發 SwiftUI App 時，我們有兩種方法實現水平捲動的分頁效果，一種是 TabView 搭配 tabViewStyle(.page)，一種是 ScrollView 搭配 scrollTargetBehavior(.paging)。

利用 SwiftUI 的 scrollTargetBehavior(.paging) 實現分頁捲動效果

開發 SwiftUI App 時，我們有兩種方法實現水平捲動的分頁效果，一種是 TabView 搭配 tabViewStyle(.page)，一種是 ScrollView 搭配 scrollTargetBehavior(.paging)。

TabView 的做法比較簡單，它會讓 TabView { } 裡的元件自動分配到每一頁。比方以下例子，每一頁剛好顯示一張圖片，而且圖片的寬度也剛好等於分頁的寬度。

struct ContentView: View {
    var body: some View {
        TabView {
            ForEach(0..<10) { index in
                Image(index.isMultiple(of: 2) ? .prince : .fox)
                    .resizable()
                    .scaledToFit()
            }
        }
        .tabViewStyle(.page)
    }
}
利用 SwiftUI 的 page 實現分頁水平滑動瀏覽
利用 iOS 14 的 page 我們可以將顯示 tab 分頁的 TabView 變成水平滑動的分頁照片瀏覽。接下來就讓我們看它表演吧。medium.com

相較之下 ScrollView 搭配 scrollTargetBehavior(.paging) 的做法較為複雜，我們要另外設定元件的大小讓它剛好塞在分頁裡，不過它可實現一些特別的捲動效果，而且除了水平捲動，它還能垂直捲動呢。

ScrollTargetBehavior | Apple Developer Documentation
A type that defines the scroll behavior of a scrollable view.developer.apple.com

以下我們示範幾種 scrollTargetBehavior(.paging) 的應用。

  • 水平捲動的分頁效果。

從 ScrollView 呼叫 scrollTargetBehavior(.paging)，因此它變成水平捲動的分頁效果，分頁的寬度等於 ScrollView 的寬度。(ps: 因此也等於螢幕的寬度)

struct ContentView: View {
    var body: some View {
        ScrollView(.horizontal) {
            HStack(spacing: 0) {
                ForEach(0..<10) { index in
                    Image(index.isMultiple(of: 2) ? .prince : .fox)
                        .resizable()
                        .scaledToFit()
                }
            }
        }
        .scrollTargetBehavior(.paging)
    }
}

然而 App 呈現的效果差強人意，因為圖片的寬度並沒有剛好等於分頁的寬度，它甚至超過了一頁的寬度 !

滑到第二頁時，還能看到小王子圖片的右半部。

  • 水平捲動的分頁效果，一頁顯示一張圖片

我們希望一頁顯示一張圖片，圖片的寬度等於分頁的寬度。實現的方法有很多，最簡單的方法是透過 modifier containerRelativeFrame(_:alignment:)。

containerRelativeFrame(_:alignment:) | Apple Developer Documentation
Positions this view within an invisible frame with a size relative to the nearest container.developer.apple.com
struct ContentView: View {
    var body: some View {
        ScrollView(.horizontal) {
            HStack(spacing: 0) {
                ForEach(0..<10) { index in
                    Image(index.isMultiple(of: 2) ? .prince : .fox)
                        .resizable()
                        .scaledToFit()
                        .containerRelativeFrame(.horizontal)
                }
            }
        }
        .scrollTargetBehavior(.paging)
    }
}

containerRelativeFrame 將依據容器的大小設定元件的大小，剛剛程式的 containerRelativeFrame(.horizontal) 將讓圖片的寬度等於容器 ScrollView 的寬度，實現一個分頁顯示一張圖的效果。

  • 使用效能更好的 LazyHStack。

將 HStack 改成 LazyHStack。

更有效率的 LazyHStack & LazyVStack
開發 SwiftUI App 時，我們時常用 HStack & VStack 排版。雖然它們很方便，不過卻有個令人小小在意的小缺點，當 stack 裡裝了很多 view 時，它們會一次全部生成，影響 App 的效能。medium.com
struct ContentView: View {
    var body: some View {
        ScrollView(.horizontal) {
            LazyHStack(spacing: 0) {
                ForEach(0..<10) { index in
                    Image(index.isMultiple(of: 2) ? .prince : .fox)
                        .resizable()
                        .scaledToFit()
                        .containerRelativeFrame(.horizontal)
                }
            }
        }
        .scrollTargetBehavior(.paging)
    }
}
  • 一頁顯示兩張圖片。

containerRelativeFrame(.horizontal, count: 2, spacing: 0) 表示容器 ScrollView 的寬度裡可塞兩張圖片。

struct ContentView: View {
    var body: some View {
        ScrollView(.horizontal) {
            LazyHStack(spacing: 0) {
                ForEach(0..<10) { index in
                    Image(index.isMultiple(of: 2) ? .prince : .fox)
                        .resizable()
                        .scaledToFit()
                        .containerRelativeFrame(
                            .horizontal,
                            count: 2,
                            spacing: 0
                        )
                }
            }
        }
        .scrollTargetBehavior(.paging)

    }
}
  • 讓圖片和邊界間有間距。

在 containerRelativeFrame 前加上 padding。

struct ContentView: View {
    var body: some View {
        ScrollView(.horizontal) {
            LazyHStack(spacing: 0) {
                ForEach(0..<10) { index in
                    Image(index.isMultiple(of: 2) ? .prince : .fox)
                        .resizable()
                        .scaledToFit()
                        .padding()
                        .containerRelativeFrame(.horizontal)
                }
            }
        }
        .scrollTargetBehavior(.paging)
    }
}
  • 圖片寬度等於螢幕寬度，滑動時可看到兩張圖間的空白間距。

利用 GeometryReader 的 proxy.size.width 取得螢幕寬度，然後在 ScrollView 呼叫 frame(width: proxy.size.width + 20) 讓分頁的寬度成為螢幕寬度加 20。而 Image 呼叫 padding(.trailing, 20) 將讓圖片間多出 20 的間距。

struct ContentView: View {
    var body: some View {
        GeometryReader { proxy in
            ScrollView(.horizontal) {
                LazyHStack(spacing: 0) {
                    ForEach(0..<10) { index in
                        Image(index.isMultiple(of: 2) ? .prince : .fox)
                            .resizable()
                            .scaledToFit()
                            .padding(.trailing, 20)
                            .containerRelativeFrame(.horizontal)
                    }
                }
            }
            .frame(width: proxy.size.width + 20)
            .scrollTargetBehavior(.paging)
        }
    }
}

滑動時可看到兩張圖間的空白間距。
