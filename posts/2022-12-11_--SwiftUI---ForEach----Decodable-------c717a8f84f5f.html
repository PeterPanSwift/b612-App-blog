用 SwiftUI 的 ForEach 顯示 Decodable 型別的資料

當我們透過網路 API 抓取 JSON 資料，將它轉換成 Decodable 型別後，時常會用 SwiftUI 的 ForEach 顯示到畫面上。

假設 JSON 資料解碼成型別 [Song] 的 array songs，此時我們可用以下幾種方法讓 ForEach 顯示 Decodable 型別的資料。

  • 遵從 Identifiable，Decodable 型別的 property id 對應 JSON 的 id 欄位。
  • ForEach 使用某個 property 當 id。
  • 遵從 Identifiable，用 UUID 當 id。
  • 遵從 Identifiable，用某個 property 當 id。

遵從 Identifiable，Decodable 型別的 property id 對應 JSON 的 id 欄位

struct Song: Decodable, Identifiable {
    let name: String
    let singer: String
    let id: Int
}

ForEach(songs) { song in
    Text(song.name)
}

使用某個 property 當 id

ForEach(songs, id: \.name) { song in
    Text(song.name)
}

遵從 Identifiable，用 UUID 當 id

使用 UUID 當 id 時，id 必須宣告為常數，宣告為變數會造成解碼失敗。

struct Song: Decodable, Identifiable {
    let id = UUID()
    let name: String
    let singer: String
}

以上程式會出現警告訊息 Immutable property will not be decoded because it is declared with an initial value which cannot be overwritten，不過可以不理它。

若是想消除警告，我們也可以定義 Coding Keys，不要寫到 id 的 case，讓 JSONDecoder 解碼時跳過 id。

struct Song: Decodable, Identifiable {
    let id = UUID()
    let name: String
    let singer: String
    
    enum CodingKeys: String, CodingKey {
        case name
        case singer
    }
}

遵從 Identifiable，用某個 property 當 id

將 id 宣告為 computed property，用某個 property 當 id。

struct Song: Decodable, Identifiable {
    var id: String { name }
    let name: String
    let singer: String
}
