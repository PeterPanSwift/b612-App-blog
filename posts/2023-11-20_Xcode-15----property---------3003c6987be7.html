Xcode 15 設定 property 內容的自動完成

最近發現在 Xcode 15 設定 property 內容時有更方便的自動完成了 ! 當我們將 property 設為 enum 或 static/class computed property 時，Xcode 的自動完成會出現 = 的選項，幫我們同時輸入 property 和內容。

以下我們分別實驗幾種不同的例子說明。

  • 用 SDK 內建的 enum 或 static/class computed property 設定 property。
  • 用自訂的 enum 或 static/class computed property 設定 property。
  • 不會有自動完成的兩種 case。

用 SDK 內建的 enum 和 static/class computed property 設定 property

  • 設定 backgroundColor。

輸入 property 和內容的關鍵字，比方 bacred，出現將 backgroundColor 設為 .red 或 .systemRed 的選項。

輸入內容的關鍵字，比方輸入 gree，出現設為 .green 或 .systemGreen 的選項。

  • 設定 contentMode scaleAspectFit。

輸入 confit，出現將 contentMode 設為 scaleAspectFit 的選項。

  • 設定 textAlignment。

輸入 alile，出現將 textAlignment 設為 .left 的選項。

用自訂的 enum 設定 property

enum Pet {
    case cat, rabbit, dog
}

struct Baby {
    var cutePet: Pet
}

輸入 c 時出現每個 case 的選項。

輸入 ca，出現將 cutePet 設為 .cat 的選項。

用自訂的 static/class computed property 設定 property

struct Pet {
    let name: String
    
    static var rabbit: Pet {
        Pet(name: "rabbit")
    }
    static var cat: Pet {
        Pet(name: "cat")
    }
}

struct Baby {
    var cutePet: Pet
}

輸入 c 時出現 static computed property 的選項。

輸入 ca，出現將 cutePet 設為 .cat 的選項。

不會有自動完成的兩種 case

  • static stored property 不會有自動完成。
struct Pet {
    let name: String
    
    static let rabbit = Pet(name: "rabbit")
    static let cat = Pet(name: "cat")
    static let dog = Pet(name: "dog")
}

struct Baby {
    var cutePet: Pet
}

輸入 c 沒有出現 static stored property 的選項。

輸入 ca 沒有出現設定為 cat 的選項。

  • asset 自動生成的 computed property 不會有自動完成。
let imageView = UIImageView()
imageView.image = .peter

輸入 imapet 不會有自動完成。
