使用 NavigationView 設計兩欄的 iPad 畫面，如何讓主畫面一開始顯示內容

當我們使用 NavigationView & NavigationLink 製作 App 畫面時，它在 iPad 會自動變成兩欄的分割畫面。

以下面的埃及神話為例，左邊區塊會顯示埃及九柱神的列表，右邊的主畫面區塊會依據我們在左邊點選的神顯示 NavigationLink 的目標頁面。

struct ContentView: View {
    let names = ["舒", "塞特", "拉", "泰芙努特", "蓋布", "努特", "俄賽里斯", "伊西絲", "奈芙蒂斯"]
    
    var body: some View {
        NavigationView {
            List(names, id: \.self) { name in
                NavigationLink {
                    Image(name)
                        .resizable()
                        .scaledToFit()
                } label: {
                    Text(name)
                }
            }
            .navigationTitle("埃及九柱神")
        }
        
    }
}

不過當 App 畫面第一次出現，我們還沒選擇喜歡的神時，右邊將會是一片空白。

直到我們點選左邊的神後，右邊才會顯示對應的畫面。

讓主畫面一開始顯示預設內容(placeholder)

如果不喜歡主畫面一開始一片空白，我們可以在 NavigationView 的 { } 裡加入一開始想在主畫面顯示的內容(placeholder)，比方以下例子 NavigationView 的 { } 裡有 List & Text， Text("點選喜歡的神吧")將成為主畫面一開始呈現的內容。

struct ContentView: View {
    let names = ["舒", "塞特", "拉", "泰芙努特", "蓋布", "努特", "俄賽里斯", "伊西絲", "奈芙蒂斯"]
    
    var body: some View {
        NavigationView {
            List(names, id: \.self) { name in
                NavigationLink {
                    Image(name)
                        .resizable()
                        .scaledToFit()
                } label: {
                    Text(name)
                }
            }
            .navigationTitle("埃及九柱神")
            
            Text("點選喜歡的神吧")
　　　　 }
        
    }
}

當 App 畫面第一次出現，我們還沒選擇喜歡的神時，右邊將顯示文字點選喜歡的神吧。

若是 App 在 iPhone 執行，剛剛加入的 Text("點選喜歡的神吧") 不會有任何作用，因為 iPhone 不會有分割畫面，一開始只會看到 List 的畫面。

讓主畫面一開始顯示列表裡第一個項目的內容

struct ContentView: View {
    let names = ["舒", "塞特", "拉", "泰芙努特", "蓋布", "努特", "俄賽里斯", "伊西絲", "奈芙蒂斯"]
    
    @State private var selectName: String? = "舒"
   
    var body: some View {
        NavigationView {
            List(names, id: \.self) { name in
                NavigationLink(tag: name, selection: $selectName) {
                    Image(name)
                        .resizable()
                        .scaledToFit()
                } label: {
                    Text(name)
                }
            }
            .navigationTitle("埃及九柱神")
            
        }
        
    }
}

NavigationLink 可以搭配參數 selection 控制它的目標頁面是否顯示。selection 的型別是 optional，當 optional 有值且內容等於 tag 時將顯示目標頁面。以上例子的 selectName 一開始是舒，而 List 第一個 row 的 tag 為舒，因此一開始將直接選取舒，在主畫面顯示舒的圖片。
