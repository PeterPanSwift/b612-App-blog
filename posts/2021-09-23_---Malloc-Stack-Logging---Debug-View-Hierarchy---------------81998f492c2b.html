使用 Malloc Stack Logging & Debug View Hierarchy 找出物件在程式裡產生的地方

最近朋友問到一個問題，他接手了公司前輩博大精深的 iOS App 專案，裡面的畫面都是用程式碼產生。當他想修改畫面裡某個元件的大小位置時，要如何快速找到程式修改的地方呢 ?

使用 Malloc Stack Logging & Debug View Hierarchy，我們可以快速找出物件在程式裡產生的地方。

接下來我們將以畫面上有兩個骰子圖片的簡單範例說明。

view controller 的程式

import UIKit

class ViewController: UIViewController {
    
    let diceImageView1: UIImageView = {
        let imageView = UIImageView()
        imageView.translatesAutoresizingMaskIntoConstraints = false
        imageView.contentMode = .scaleAspectFit
        imageView.tintColor = .orange
        return imageView
    }()
    
    let diceImageView2: UIImageView = {
        let imageView = UIImageView()
        imageView.translatesAutoresizingMaskIntoConstraints = false
        imageView.contentMode = .scaleAspectFit
        imageView.tintColor = .orange
        return imageView
    }()
    
    override func viewDidLoad() {
        super.viewDidLoad()
        
        view.addSubview(diceImageView1)
        view.addSubview(diceImageView2)
        configureConstraints()
        diceImageView1.image = UIImage(systemName: "die.face.\(Int.random(in: 1...6)).fill")
        diceImageView2.image = UIImage(systemName: "die.face.\(Int.random(in: 1...6)).fill")
        
    }
    
    func configureConstraints() {
        NSLayoutConstraint.activate([
            diceImageView1.leadingAnchor.constraint(equalTo: view.safeAreaLayoutGuide.leadingAnchor),
            diceImageView1.trailingAnchor.constraint(equalTo: view.safeAreaLayoutGuide.trailingAnchor),
            diceImageView1.topAnchor.constraint(equalTo: view.safeAreaLayoutGuide.topAnchor),
            diceImageView1.widthAnchor.constraint(equalTo: diceImageView1.heightAnchor),
            
            diceImageView2.leadingAnchor.constraint(equalTo: diceImageView1.leadingAnchor),
            diceImageView2.trailingAnchor.constraint(equalTo: diceImageView1.trailingAnchor),
            diceImageView2.topAnchor.constraint(equalTo: diceImageView1.bottomAnchor),
            diceImageView2.widthAnchor.constraint(equalTo: diceImageView2.heightAnchor),
        ])
    }
}

App 執行的畫面

我們想要修改下面骰子圖片的顏色，使用 Malloc Stack Logging & Debug View Hierarchy，我們可以快速找出它是 diceImageView2。

編輯 scheme

點選紅色框框的 App 名稱。

點選 Edit Scheme。

切換到 Run > Diagnostics 分頁，勾選 Malloc Stack Logging，選擇 All Allocation and Free History

勾選 Malloc Stack Logging 後，Xcode 將紀錄程式裡產生物件的地方，方便我們之後查詢畫面上的元件在哪裡產生。

執行 App，進入 Debug View Hierarchy 模式

點選紅色框框的 Debug View Hierarchy。

點選下方的骰子圖片，右邊切換到紅色框框的 Object inspector 分頁，此分頁將顯示 image view 的記憶體位置 & 詳細資訊。

在 Object inspector 分頁往下滑，從 Backtrace 區塊可看到產生 image view 時，程式呼叫過那些 function。

從 Backtrace 區塊看出產生 image view 的地方應該是下圖紅色框框標示的 closure，點選右邊的箭頭可跳到程式碼。

Cool，才幾秒鐘的時間，我們就找到下方的骰子圖片是 diceImageView2 ~
