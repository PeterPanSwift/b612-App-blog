iOS 15 方便設定 SF Symbol 樣式的 variant

SF Symbol 包含五花八門的圖案， iOS App 時常使用的圖案它都幫我們設計好了。不過使用 SF Symbol 時有個小小的麻煩，我們必須輸入它的名字，所以要先花時間查一下名字。

比方下圖的愛心圖案有多種樣式，雖然命名有一定的規則，我們手動輸入時還是有可能打錯。

從 iOS 15 開始，我們可以利用 variant 設定 SF Symbol 的樣式，比方圓形，方形，填滿，斜線等，再也不用手動輸入超長的圖片名字了

以剛剛的愛心為例，我們只要記得它的名字是 heart 就好。接下來讓我們瞧瞧 variant 在 SwiftUI & UIKit 的設定方法。

SwiftUI

呼叫 symbolVariant，傳入型別 SymbolVariants 的參數控制樣式，在此我們傳入 .slash 產生斜線樣式。

Image(systemName: "heart")
    .resizable()
    .symbolVariant(.slash)
    .frame(width: 200, height: 200)

從 .circle 得到圓形樣式後，接著讀取 fill，實現圓形填滿的樣式。

Image(systemName: "heart")
    .resizable()
    .symbolVariant(.circle.fill)
    .frame(width: 200, height: 200)

圓形填滿並搭配斜線的樣式。

Image(systemName: "heart")
    .resizable()
    .symbolVariant(.circle.fill.slash)
    .frame(width: 200, height: 200)

若是使用不存在的樣式，則會有以下幾種情況:

heart 沒有同時包含長方形和斜線的樣式，因此將顯示斜線的樣式。

Image(systemName: "heart")
    .resizable()
    .symbolVariant(.rectangle.slash)
    .frame(width: 200, height: 200)

book 沒有長方形的樣式，因此將採用預設的 book 樣式。

Image(systemName: "book")
    .resizable()
    .symbolVariant(.rectangle)
    .frame(width: 200, height: 200)

UIKit

產生 UIImage 時傳入型別 Variant 的參數控制樣式，在此我們傳入 .slash 產生斜線樣式。

let image = UIImage(systemName: "heart", variant: .slash)
let imageView = UIImageView(image: image)
imageView.frame = CGRect(x: 100, y: 100, width: 100, height: 100)
view.addSubview(imageView)

從 .circle 得到圓形樣式後，接著讀取 fill，實現圓形填滿的樣式。

let image = UIImage(systemName: "heart", variant: .circle.fill)
let imageView = UIImageView(image: image)
imageView.frame = CGRect(x: 100, y: 100, width: 100, height: 100)
view.addSubview(imageView)

圓形填滿並搭配斜線的樣式。

let image = UIImage(systemName: "heart", variant: .circle.fill.slash)
let imageView = UIImageView(image: image)
imageView.frame = CGRect(x: 100, y: 100, width: 100, height: 100)
view.addSubview(imageView)

若是使用不存在的樣式，圖片將無法顯示，比方以下的 heart 並沒有同時包含長方形和斜線的樣式。

let image = UIImage(systemName: "heart", variant: .rectangle.slash)
let imageView = UIImageView(image: image)
imageView.frame = CGRect(x: 100, y: 100, width: 100, height: 100)
view.addSubview(imageView)
