遇到此問題，其中一種解法是自訂遵從 CodingKey 的型別 CodingKeys，然後在裡面將小寫開頭的 property 名稱對應到 JSON 大寫開頭的 key。

struct BusRoute: Codable {
    let departureStopNameZh: String
    let destinationStopNameZh: String
    
    enum CodingKeys: String, CodingKey {
        case departureStopNameZh = "DepartureStopNameZh"
        case destinationStopNameZh = "DestinationStopNameZh"
    }
}

不過如果有很多 key 要宣告，寫起來也是滿辛苦的。因此接下來讓我們試試另一種方法，以公車 API 回傳的 JSON 為例，自訂 custom 的 keyDecodingStrategy 將 JSON 裡大寫開頭的 key 變小寫。

  • 定義 JSON 對應的 BusRoute 型別。

property 名稱以小寫開頭，跟 JSON 裡大寫開頭的 key 不合。不過沒關係，待會我們將自訂 custom 的 keyDecodingStrategy 做轉換。

struct BusRoute: Codable {
    let departureStopNameZh: String
    let destinationStopNameZh: String
}
  • 以 struct 定義遵從 CodingKey 的型別

遵從 CodingKey，然後定義 CodingKey 需實作的變數和 init，此型別將影響待會 JSONDecoder 解碼用到的 key。

struct CustomCodingKey: CodingKey {
    var stringValue: String
    
    init?(stringValue: String) {
        self.stringValue = stringValue
    }
    var intValue: Int?
    
    init?(intValue: Int) {
        return nil
    }
}

此處最重要的是以下程式，它將讓參數 stringValue 成為到時候解碼用到的 key。

init?(stringValue: String) {
    self.stringValue = stringValue
}
  • 自訂 custom 的 keyDecodingStrategy

讓 JSONDecoder 解碼時使用小寫開頭的英文當 key。為了方便說明，我們加入一些 print，印出 codingPath，lastComponent & key。

do {
   let decoder = JSONDecoder()
   decoder.keyDecodingStrategy = .custom({ codingPath in
      print(codingPath)
      let lastComponent = codingPath.last!.stringValue
      print("lastComponent", lastComponent)
      let key = lastComponent.prefix(1).lowercased() + lastComponent.dropFirst()
      print("key", key)
      return CustomCodingKey(stringValue: key)!
   })
   let busRoutes = try decoder.decode([BusRoute].self, from: data)
} catch  {
   print(error)
}

當 JSONDecoder 解碼時，它將執行 custom 傳入的 closure 程式，closure 裡回傳的 CustomCodingKey 決定解碼時使用的 key。從上圖的結果可看出 closure 將呼叫多次，JSON 裡的每個 key 都會呼叫一次。第一次呼叫時，codingPath 是 [_JSONKey(stringValue: "Index 0", intValue: 0), _JSONKey(stringValue: "BusRouteType", intValue: nil)]，因此它找到的 JSON key 是 BusRouteType。我們以 lastComponent.prefix(1).lowercased() + lastComponent.dropFirst() 將它變成小寫開頭的 busRouteType，然後回傳 CustomCodingKey(stringValue: key)!。

利用此方法，我們可將 JSON 裡所有大寫開頭的 key 變成小寫開頭，如此即可讓 JSON 裡原本大寫開頭的 key 對應到 BusRoute 裡小寫開頭的 property。

定義 JSONDecoder.KeyDecodingStrategy 的 extension

為了方便之後使用，我們可以定義 JSONDecoder.KeyDecodingStrategy 的 extension，宣告將大寫 key 轉小寫的 bigCaseToSmall。

import Foundation
extension JSONDecoder.KeyDecodingStrategy {
    
    struct BigCaseToSmallCodingKey: CodingKey {
        var stringValue: String
        
        init?(stringValue: String) {
            self.stringValue = stringValue
        }
        var intValue: Int?
        
        init?(intValue: Int) {
            return nil
        }
    }
    
    static var bigCaseToSmall: Self {
        .custom { codingPath in
            let lastComponent = codingPath.last!.stringValue
            let key = lastComponent.prefix(1).lowercased() + lastComponent.dropFirst()
            return BigCaseToSmallCodingKey(stringValue: key)!
        }
    }
}

之後解析 JSON 時，只要將 keyDecodingStrategy 設成 .bigCaseToSmall 即可。

let decoder = JSONDecoder()
decoder.keyDecodingStrategy = .bigCaseToSmall
