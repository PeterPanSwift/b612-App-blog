防止 URLSessionDataTask 忘了呼叫 resume 的寫法

當我們利用 URLSessionDataTask 下載 / 上傳資料時，有件十分重要的事情。一定要記得呼叫 function，它才會真正開始下載 / 上傳資料，例如以下例子:

let task = URLSession.shared.dataTask(with: URL(string: "https://some-random-api.ml/meme")!) { data, response, error in

}
task.resume()

可惜我們不是神，所以我們常常知道卻忘記做到，忘了加上 task.resume()。其實只要改個寫法，不只能讓我們更容易記得 resume，還可以讓程式更精簡呢。

URLSession.shared.dataTask(with: URL(string: "https://some-random-api.ml/meme")!) { data, response, error in

}.resume()

我們在呼叫 dataTask 的 { } 後直接加 .resume()，在生成 URLSessionDataTask 後馬上呼叫 resume。
