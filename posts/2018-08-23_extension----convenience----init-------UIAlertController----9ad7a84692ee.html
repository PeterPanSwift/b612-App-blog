extension 裡用 convenience 定義 init — 以建立 UIAlertController 為例

透過 extension 我們可以擴充型別的功能，除了常見的定義 function 外，我們還可以定義 convenience init，讓我們更方便地建立東西。

比方在 Apple 的 HomeKit 範例程式裡，它在 UIViewController+Convenience.swift 裡透過 extension 幫 UIAlertController 定義了兩個 init，一個建立 App 裡常看到的 OK 按鈕 alert，一個建立可輸入文字的 alert。

接下來就讓我們模仿 Apple 的範例，實現更方便建立的 UIAlertController 吧。

建立搭配 OK 按鈕 alert
let controller = UIAlertController(title: "好久不見", body: "你會不會忽然的出現，在街角的咖啡店")
present(controller, animated: true)

extension UIAlertController 裡的 init 定義。

convenience init(title: String, body: String) {
   self.init(title: title, message: body, preferredStyle: .alert)
   let okAction = UIAlertAction(title: "Okay", style: .default, handler: nil)
   addAction(okAction)
}

另外記得 extension 只能定義 convenience init，不能定義 designated initializer，忘了加上 convenience 會產生可怕的紅色錯誤。

Designated initializer cannot be declared in an extension
建立搭配 text field 的 alert
let controller = UIAlertController(title: "請輸入你的生辰八字",   placeholder: "生辰八字") { (text) in
   print(text)
}
present(controller, animated: true)

extension UIAlertController 裡的 init 定義。

convenience init(title: String, placeholder: String, completionHandler: @escaping (_ name: String) -> Void) {
   self.init(title: title, message: nil, preferredStyle: .alert)
   addTextField { (textField) in
      textField.placeholder = placeholder
   }
   let cancelAction = UIAlertAction(title: "Cancel", style: .cancel, handler: nil)
   let okAction = UIAlertAction(title: "OK", style: .default) { (_) in
      if let name = self.textFields?.first?.text {
         let trimmedName = name.trimmingCharacters(in:   CharacterSet.whitespacesAndNewlines)
         completionHandler(trimmedName)
      }
   }
   addAction(cancelAction)
   addAction(okAction)
}
