SwiftUI stack & frame 控制對齊的 alignment

設計 SwiftUI App 畫面時，我們常常用 stack 排版畫面，而 stack & frame modifier 都有著控制對齊的參數 alignment，它們有什麼差別呢 ?

以下我們以圖庫網站 pexels 上可愛的企鵝為例，透過一些例子介紹它們的用法和差異。

https://www.pexels.com/photo/penguins-standing-on-the-snow-during-daytime-52509/

以 VStack 為例，元件上下排列時預設會對齊中間。

VStack {
    Image(.penguin)
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("企鵝")
}
.background(.yellow)

透過參數 alignment 可控制內容元件對齊的方式。

.leading 讓元件靠前對齊。(ps: 當系統的語言是由左到右的語言時，前面是左邊。 當系統的語言是由右到左的語言時，前面是右邊。)

VStack(alignment: .leading) {
    Image(.penguin)
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("企鵝")
}
.background(.yellow)

.trailing 會靠後對齊

VStack(alignment: .trailing) {
    Image(.penguin)
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("企鵝")
}
.background(.yellow)

stack 預設的大小將等於內容的大小，然而我們也可透過 frame 調整大小。frame alignment 將在 stack 大小和內容大小不同時發揮作用。

以下我們將 stack 的寬度改成 300，此時 stack 的內容元件將在 stack 的中間。因此 stack 的 alignment 只影響元件的對齊，不影響元件在 stack 框框裡的位置。

VStack(alignment: .trailing) {
    Image(.penguin)
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("企鵝")
}
.frame(width: 300)
.background(.yellow)

透過 frame 的參數 alignment 可調整內容在 stack 框框裡的位置

.leading 讓內容靠前對齊，當系統語言是左到右時對齊 stack 框框的左邊。

VStack(alignment: .trailing) {
    Image(.penguin)
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("企鵝")
}
.frame(width: 300, alignment: .leading)
.background(.yellow)

.trailing 讓內容靠後對齊，當系統語言是左到右時對齊 stack 框框的右邊。

VStack(alignment: .trailing) {
    Image(.penguin)
        .resizable()
        .scaledToFit()
        .frame(width: 100)
    Text("企鵝")
}
.frame(width: 300, alignment: .trailing)
.background(.yellow)
