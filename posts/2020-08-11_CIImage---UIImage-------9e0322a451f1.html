CIImage 變 UIImage 的轉向問題

開發 iOS App 時，圖片的型別通常是 UIImage，不過有時我們會遇到型別 CIImage 的圖片，需要將它轉換成 UIImage 後才能顯示在 UIImageView 上。

比方以下例子，我們使用 CIFilter 在圖片上添加濾鏡，然而 CIFilter 只能作用於 CIImage，因此我們必須處理型別 CIImage 的圖片。

#74 利用 CIFilter 實現美麗的圖片濾鏡
iOS 上照片類的 App 十分受歡迎，尤其是 App 裡各式各樣幫我們做出漂亮照片的濾鏡功能。其實基本的濾鏡功能不難做到，因為 iOS SDK 早已為我們提供實現濾鏡功能的 CIFilter。medium.com

當照片是利用 iPhone 相機拍照，或是從相簿選擇的照片是當初從 iPhone 拍照的照片時，當我們將它從 UIImage 變成 CIImage，然後再從 CIImage 轉成 UIImage 時，卻可能會遇到轉向問題。

接下來就讓我們親手實驗跟解決問題吧。

在 storyboard 加入 2 個 image view

上面顯示從相簿選擇的照片，下面顯示從 CIImage 轉成 UIImage 的圖片。

連結 outlet

上面圖片的 outlet 是 originalImageView，下面圖片的 outlet 是 newImageView。

@IBOutlet weak var newImageView: UIImageView!
@IBOutlet weak var originalImageView: UIImageView!

選擇照片後，將照片顯示到 image view

extension ViewController: UIImagePickerControllerDelegate & UINavigationControllerDelegate {
    
    func imagePickerController(_ picker: UIImagePickerController, didFinishPickingMediaWithInfo info: [UIImagePickerController.InfoKey : Any]) {
        guard let originalUIImage = info[.originalImage] as? UIImage else { return }
        guard let ciImage = CIImage(image: originalUIImage) else { return }
        originalImageView.image = originalUIImage
        newImageView.image = UIImage(ciImage: ciImage)
        dismiss(animated: true)
    }
}

結果

將選擇的照片從 UIImage 變成 CIImage，然後再從 CIImage 變成 UIImage 時，可怕的事發生了 ! 照片轉向了 ~

使用 CIFilter 的濾鏡效果

extension ViewController: UIImagePickerControllerDelegate & UINavigationControllerDelegate {
    
    func imagePickerController(_ picker: UIImagePickerController, didFinishPickingMediaWithInfo info: [UIImagePickerController.InfoKey : Any]) {
        guard let originalUIImage = info[.originalImage] as? UIImage else { return }
        guard let ciImage = CIImage(image: originalUIImage) else { return }
        let filter = CIFilter.colorMonochrome()
        filter.inputImage = ciImage
        filter.intensity = 0.9
        guard let outputCIImage = filter.outputImage else { return }
        
        originalImageView.image = originalUIImage
        newImageView.image = UIImage(ciImage: outputCIImage)
        dismiss(animated: true)
    }
}

結果

一樣的，可怕的事發生了 ! 照片轉向了 ~

解決 CIImage 變 UIImage 的轉向問題

extension CGImagePropertyOrientation {
    init(_ uiOrientation: UIImage.Orientation) {
        switch uiOrientation {
            case .up: self = .up
            case .upMirrored: self = .upMirrored
            case .down: self = .down
            case .downMirrored: self = .downMirrored
            case .left: self = .left
            case .leftMirrored: self = .leftMirrored
            case .right: self = .right
            case .rightMirrored: self = .rightMirrored
        @unknown default:
            self = .up
        }
    }
}

extension ViewController: UIImagePickerControllerDelegate & UINavigationControllerDelegate {
    
    func imagePickerController(_ picker: UIImagePickerController, didFinishPickingMediaWithInfo info: [UIImagePickerController.InfoKey : Any]) {
        guard let originalUIImage = info[.originalImage] as? UIImage else { return }
        guard let ciImage = CIImage(image: originalUIImage) else { return }
        let filter = CIFilter.colorMonochrome()
        filter.inputImage = ciImage
        filter.intensity = 0.9
        guard let outputCIImage = filter.outputImage else { return }
        
        let rotateCIImage = outputCIImage.oriented(CGImagePropertyOrientation(originalUIImage.imageOrientation))
        originalImageView.image = originalUIImage
        newImageView.image = UIImage(ciImage: rotateCIImage)
        dismiss(animated: true)
    }
}

說明

從 CIImage 呼叫 function oriented(_:) 將圖片轉成正確的方向。從 UIImage 的 property imageOrientation 可取得圖片原本的方向，不過它的型別是 UIImage.Orientation，而呼叫 oriented(_:) 時參數 orientation 的型別是 CGImagePropertyOrientation，所以我們要先進行型別的轉換。

func oriented(_ orientation: CGImagePropertyOrientation) -> CIImage
