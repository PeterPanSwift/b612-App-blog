在 function，closure，computed property 裡省略 return — Swift 5.1

在 Swift 5.1，體貼的 Apple 為了讓我們少打一點字，決定不只 closure 裡只有一行程式時可以省略 return，當 function，closure，computed property getter， subscript & init( failable initializer) 的程式只有一行時，都可以省略 return，比方以下例子:

function 省略 return
func add(number1: Int, number2: Int) -> Int {
   number1 + number2
}
add(number1: 10, number2: 20)
computed property getter 省略 return
struct Baby {
   var height: Double
   var weight: Double
   var bmi: Double {
      weight / (height * height)
   }
}
closure 省略 return
let array = ["1", "2", "3"]
let numbers = array.map { $0 }

在 SwiftUI 的程式其實就運用了省略 return 的技巧讓程式更簡潔，比方以下例子。

例1:

struct ContentView : View {
   var body: some View {
      Text("Hello World")
   }
}

例2:

struct ContentView : View {
   var body: some View {
      VStack {
         Text("I Love U")
           .color(.red)
         Text("Wendy")
      }
   }
}
