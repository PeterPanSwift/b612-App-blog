利用 labelsHidden 隱藏 SwiftUI 元件的 label

SwiftUI 很多元件在顯示時都會在左邊顯示輔助的內容(label)，比方以下 DatePicker 的左邊顯示文字 peter pan & peter parker 認識的那天。不過有時為了讓畫面更好看，我們會想隱藏左邊顯示的文字，接下來就讓我們一步步說明輔助內容造成的問題和幫我們隱藏 label 的 labelsHidden。

@State private var meetDate = Date()

var body: some View {
   DatePicker("peter pan & peter parker 認識的那天", selection: $meetDate, displayedComponents: .date)
}

如上圖所示，DatePicker 的說明文字完全跟邊邊靠在一起，所以我們通常會加個 padding，讓它跟邊界保持一點距離。

DatePicker("peter pan & peter parker 認識的那天", selection: $meetDate, displayedComponents: .date)
   .padding()

此時文字的確沒有跟邊邊靠在一起了，可是簡單的一句話變成好幾行實在不太好看。

比較好的做法應該是用 VStack 容納 DatePicker，在它的上方顯示說明的 Text。

VStack {
    Text("peter pan & peter parker 認識的那天")
    DatePicker("", selection: $meetDate, displayedComponents: .date)
}

我們在 DatePicker 的字串參數傳入空字串，請它別在左邊顯示說明文字。如下圖所示，現在它的確左邊沒字了，但它還是保留著顯示文字的空間，造成 DatePicker 偏右顯示。

我們希望 DatePicker 左邊顯示輔助內容的區塊完全消失，如此 DatePicker 才能置中顯示 。此時就是 labelsHidden 派上用場的時候了，它專門把說明元件用途的 label 隱藏。

labelsHidden() | Apple Developer Documentation
Hides the labels of any controls contained within this view.developer.apple.com
VStack {
    Text("peter pan & peter parker 認識的那天")
    DatePicker("peter pan & peter parker 認識的那天", selection: $meetDate, displayedComponents: .date)
        .labelsHidden()
}

值得注意的，雖然我們不想在 DatePicker 的左邊看到文字，但在產生 DatePicker 時最好還是傳入文字，因為它可以幫助開啟 VoiceOver 功能的使用者。當使用者點選 DatePicker 時 iOS 會唸出我們傳入的文字，告訴使用者元件的用途。

除了 DataPicker ，SwiftUI 還有很多元件都可以搭配 labelsHidden，例如以下例子:

Toggle

VStack {
   Text("下雨嗎")
   Toggle("下雨嗎", isOn: $isRain)
      .labelsHidden()
}

Stepper

VStack {
   Text("買 \(number) 隻 iPhone")
   Stepper("買 iPhone", value: $number)
      .labelsHidden()
}
