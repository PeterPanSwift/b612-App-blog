同時 hot reload iOS & Android Flutter App

Flutter App 有許多不錯的優點，比方支援跨平台和 hot reload 功能，因此彼得潘在開發 Flutter App 時想到如果能同時 hot reload iOS & Android App，修改程式後馬上看到雙平台更新的畫面該有多好。

研究一下後，發現 VS Code 的 Multi device debugging 可以實現 hot reload 多個裝置，以下彼得潘一步步示範實現的步驟。

Multi device debugging in VS Code
Flutter makes it easy and fast to build beautiful apps for mobile and beyond — Multi device debugging in VS Code ·…github.com

啟動 iOS & Android 模擬器

點選 Start iOS Simulator。

點選 Start Android 模擬器，比方下圖的 Pixel_3a。

點選 VS Code menu 的 Run > Add Configuration

生成 .vscode 下的 launch.json。

在 launch.json 加入將 App 同時執行在 iOS & Android 模擬器的設定

{
    // Use IntelliSense to learn about possible attributes.
    // Hover to view descriptions of existing attributes.
    // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "name": "demo",
            "request": "launch",
            "type": "dart"
        },
        {
            "name": "demo (profile mode)",
            "request": "launch",
            "type": "dart",
            "flutterMode": "profile"
        },
        {
            "name": "demo (release mode)",
            "request": "launch",
            "type": "dart",
            "flutterMode": "release"
        },
        {
            "name": "iPhone",
            "request": "launch",
            "type": "dart",
            "deviceId": "D79C5310-266D-4865-831B-30969488A903"
        },
        {
            "name": "Android",
            "request": "launch",
            "type": "dart",
            "deviceId": "emulator-5554"
        },
    ],
    "compounds": [
        {
            "name": "All Devices",
            "configurations": [
                "Android",
                "iPhone"
            ],
        }
    ]
}

我們新增的區塊如下。

  • 在 configurations 下加入 iPhone & Android 裝置的設定。
{
    "name": "iPhone",
    "request": "launch",
    "type": "dart",
    "deviceId": "D79C5310-266D-4865-831B-30969488A903"
},
{
    "name": "Android",
    "request": "launch",
    "type": "dart",
    "deviceId": "emulator-5554"
}

重點在 deviceId，先用 flutter devices 指令找出 iOS 和 Android 模擬器的 device id，然後輸入在 JSON 的 deviceID 欄位。

  • 新增 compounds 區塊。

取名為 All Devices，configurations 裡的 Android 和 iPhone 對應剛剛在 configurations 設定的 name。

"compounds": [
    {
        "name": "All Devices",
        "configurations": [
            "Android",
            "iPhone"
        ]
    }
]

將 App 同時執行在 iOS & Android 模擬器

點選側邊欄的 Run and Debug。

從 RUN AND DEBUG 頁面的上方選單點選 All Devices，All Devices 是我們剛剛在 launch.json compounds 區塊設定的名字。

點選三角形啟動 App。

耶，App 成功啟動到 iOS & Android 模擬器 !

更令人開心的，修改程式後 hot reload 也會同時更新 iOS & Android App。如果螢幕夠大且電腦效能夠好的朋友，開發時可以考慮將螢幕分割成三個視窗，一邊用 VS Code 編輯程式，一邊觀看即時更新的的 iOS & Android App。
