自動完成暗藏殺機的 Implicitly Unwrapped Optional

熟悉 optional 的朋友都知道，宣告 optional 有兩種方法，一種是加問號，一種是加驚嘆號。加驚嘆號的又稱為 Implicitly Unwrapped Optional，因為到時候讀取它的內容時不用再加驚嘆號，可以自動取值。

簡單地說， Implicitly Unwrapped Optional 就像披著羊皮的狼，我們在使用它時就像使用傳統非 optional 的變數，但是它卻暗藏殺機，比問號宣告的 optional 危險許多。比方我們先看看善良問號宣告的例子:

如上圖所示，我們想讓寶寶去唱歌，因此呼叫 function sing，當我們輸入後，Xcode 的自動完成將幫我們補上問號，變成更安全的 optional chaining，這樣即使 cuteBaby 是 nil，我們的程式也不會閃退。

cuteBaby?.sing()

但是如果當初 cuteBaby 是以 ! 宣告，結果可就大不同了。

var cuteBaby: Baby! = nil
cuteBaby.sing()

以 ! 宣告的 cuteBaby，當我們呼叫 function sing 時，Xcode 的自動完成並不會補上問號，將變成 cuteBaby.sing()，因此當 cuteBaby 是 nil 時，你的程式只能光榮的死去。
