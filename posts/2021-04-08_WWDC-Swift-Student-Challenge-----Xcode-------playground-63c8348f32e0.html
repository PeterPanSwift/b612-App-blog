彼得潘已經不是學生，無法參加 Swift Student Challenge，不過還是滿好奇怎麼樣才能參加比賽。根據 Apple 的比賽文件，作品需滿足以下條件:

  • 以 Swift playground 開發。

必須以 iPadOS 的 Swift Playgrounds，macOS 的 Swift Playgrounds 或 Xcode 的 playground 開發，因此不能是 Xcode project。

  • playground 壓縮成 zip 上傳，檔案的大小不能超過 25 MB。
  • 單機版可實現的功能，不能連網路。

看來對一般習慣使用 Xcode project 開發 App 的朋友來說，最大的難題在於如何利用 Swift Playgrounds 開發 App。彼得潘研究了歷年的 WWDC scholarship 作品後，發現使用 Xcode 專案開發的 App 其實也可以移植到 playground ，只是需要進行一些小小調整。

打點專案 Scrumdinger.xcodeproj

打開 playground 的 navigator 畫面

點選下圖紅色的框框，打開 playground 的 navigator 畫面。

將 project 的 swift 檔加到 playground 的 Sources 下

將專案的圖片 & 影音檔加到 playground 的 Resources 下

從 Xcode 專案開發 App 時，我們通常會將圖片放在 assets 裡。然而 playground 不能直接讀取 assets，因此若 assets 裡有圖片，我們必須做以下調整:

  • 將圖片和影音檔放到 playground 的 Resources 下。
  • Resources 下的圖片必須有附檔名，比方 png & jpg。
  • 若要對應不同的 iPhone 解析度，圖檔的名字還要加上 @2x & @3x。

剛好剛剛的 Scrumdinger 專案沒有用到圖片，因此我們不須加入圖檔。

從 UIImage 建立 Image

使用 SwiftUI 開發 App 時，我們通常以 Image("Peter") 產生圖片，傳入的字串 Peter 是圖片在 assets 裡的名字。然而 playground 不能讀取 assets，只能讀取 Resources 資料夾下的圖片，因此我們必須先以 Resources 下的圖生成 UIImage，然後再從 UIImage 建立 Image。

比方圖片是 Bald.png，那麼在 SwiftUI 裡產生圖片的寫法如下:

Image(uiImage: UIImage(named: "Bald.png")!)

若想少打一點字，也可以利用 extension 定義 Image 的 init，在裡面利用 UIImage 生成 Image。

extension Image {
    init(uiImageName name: String) {
        self.init(uiImage: UIImage(named: name)!)
    }
}

定義在專案 asset 設定的顏色

Scrumdinger 原本在 asset 設定顏色，但是 playground 無法使用 asset。

因此我們修改 DailyScrum.swift，將顏色改成內建的顏色。

extension DailyScrum {
    static var data: [DailyScrum] {
        [
            DailyScrum(title: "Design", attendees: ["Cathy", "Daisy", "Simon", "Jonathan"], lengthInMinutes: 10, color: .red),
            DailyScrum(title: "App Dev", attendees: ["Katie", "Gray", "Euna", "Luis", "Darla"], lengthInMinutes: 5, color: .green),
            DailyScrum(title: "Web Dev", attendees: ["Chella", "Chris", "Christina", "Eden", "Karla", "Lindsey", "Aga", "Chad", "Jenn", "Sarah"], lengthInMinutes: 1, color: .blue)
        ]
    }
}

將第一個畫面的型別宣告為 public

為了在 playground 使用 Sources 下檔案定義的型別，我們必須將型別宣告為 public。雖然我們的 App 定義了很多型別，但我們只須將 playground 使用到的型別宣告成 public 即可。

從 struct ScrumdingerApp 我們發現 App 的第一個畫面是 NavigationView。

import SwiftUI
@main
struct ScrumdingerApp: App {
    
    @ObservedObject private var data = ScrumData()
    
    var body: some Scene {
        WindowGroup {
            NavigationView {
                ScrumsView(scrums: $data.scrums) {
                    data.save()
                }
            }
            .onAppear {
                data.load()
            }
        }
    }
}

為了方便待會在 playground 顯示第一個畫面，我們新增 AppView，將第一個畫面的內容設定在它的 body 裡。

import SwiftUI
struct AppView: View {
    
    @ObservedObject private var data = ScrumData()
    
    var body: some View {
        NavigationView {
            ScrumsView(scrums: $data.scrums) {
                data.save()
            }
        }
        .onAppear {
            data.load()
        }
        
    }
}

將 ScrumsView 宣告為 public，如此才能在 playground 使用。

public struct AppView: View {
    
    @ObservedObject private var data = ScrumData()
    
    public init() {
    }
    
    public var body: some View {
        NavigationView {
            ScrumsView(scrums: $data.scrums) {
                data.save()
            }
        }
        .onAppear {
            data.load()
        }
        
    }
}

值得注意的，body 也要加上 public，而且還要定義 public init，如此才能在 playground 生成 AppView。

ps: 若是第一個畫面的型別是 view controller 定義的型別，則要將它的型別和一些 method 宣告為 public，比方以下的 viewDidLoad。

public class DemoViewController: UIViewController {
    public override func viewDidLoad() {
        super.viewDidLoad()
view.backgroundColor = UIColor.yellow
    }
}

在 playground 顯示第一個畫面

import SwiftUI
import PlaygroundSupport
PlaygroundPage.current.liveView = UIHostingController(rootView: AppView())

設定 PlaygroundPage 的 liveView，將它設為 UIHostingController。在生成 UIHostingController 時傳入 AppView，讓 AppView 成為第一個畫面。

利用 Xcode playground 的 live view 顯示 App 畫面
Xcode 的 playground 除了適合當新手的 Swift 程式練功場，其實也可以利用 controller 顯示 App 畫面，然後在畫面上點選按鈕進行互動，實驗 App 的相關功能。medium.com

執行 playground

點選三角形後，即可在 playground 的右邊操作 App。

let controller = UIStoryboard(name: "Main", bundle: nil).instantiateInitialViewController()
