定義 protocol function 或覆寫父類別 function 時多用關鍵字加自動完成

iOS App 很多功能都是透過 iOS SDK 裡類別或 protocol 宣告的 function 實現的，因此我們常常需要覆寫類別的 function，或是定義 protocol 的 function。不過這些 function 為了讓我們容易了解它的意思，往往 function 名和參數名十分地長，如果每個字都自己打，很容易打錯字。

比方我們想在 table view cell 點選時做某件事，因此定義 UITableViewDelegate 的 function tableView(_:didSelectRowAt:)。然而我們因為太相信自己，每個字都自己打，不小心打錯字，少打了 _，tableView 打成 tableview。

func tableView(tableview: UITableView, didSelectRowAt indexPath: IndexPath) {
   print("select \(indexPath)")
}

雖然打錯字，但 Xcode 卻沒給任何的警告或錯誤，因為它以為這是我們全新發明的 function。所以 App 還是可以執行，只是點選 cell 時，此 function 不會被呼叫，function 裡的程式白寫了。(ps: 有朋友因為這樣的問題卡了兩天，因為打錯字的問題往往旁觀者清，自己像是被鬼遮眼般看不出來)

請多相信 Xcode，多用關鍵字加自動完成。因此剛剛的例子我們只要輸入 didse，將可看到 function tableView(_:didSelectRowAt:) 的提示，按下 enter 即可輸入完整的 function。

另外若想看到 function 的自動完成提示，有一點要特別注意。請記得先繼承類別或遵從 protocol，自動完成才會顯示相關的 function。

class MovieViewController: UIViewController, UITableViewDelegate, UITableViewDataSource  {
}

因此剛剛的例子如果我們沒有遵從 protocol UITableViewDelegate，輸入 dids 後，等一百年也等不到 function tableView(_:didSelectRowAt:) 的提示。
