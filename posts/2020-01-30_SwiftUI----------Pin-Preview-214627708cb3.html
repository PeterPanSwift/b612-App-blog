SwiftUI 將畫面牢牢釘住的 Pin Preview

SwiftUI preview 的左下角有個長得像大頭針的 Pin Preview 按鈕，它就像大頭針一樣，可以將東西牢牢釘住。

但是它要釘什麼呢? 它專門釘住 preview 畫面，讓某個畫面的 preview 固定在右邊，不管我們切換到哪一個檔案。

接下來讓我們從實際的例子了解 Pin Preview 的應用。

釘住某個 SwiftUI 畫面

ContentView 裡顯示世界知名的三本童書。

struct ContentView: View {
    let books = ["Where the Wild Things Are", "The Snowy Day", "Goodnight Moon"]
    
    var body: some View {
        ScrollView {
            VStack {
                ForEach(books, id: \.self) { book in
                    BookRow(name: book)
                }
            }
        }
    }
}

而 BookRow 定義在另一個檔案。

struct BookRow: View {
    let name: String
    
    var body: some View {
        HStack {
            Image(name)
                .resizable()
                .scaledToFill()
                .frame(width: 150, height: 150)
                .clipShape(Circle())
            Text(name)
            Spacer()
        }
    }
}

#Preview(traits: .sizeThatFitsLayout) {
    BookRow(name: "Where the Wild Things Are")
}

當我們調整 BookRow 的設計時，也想看到它在 ContentView 列表顯示的模樣。當然我們可以設計完 BookRow 後再切回 ContentView，不過其實有更簡單的方法。

先切換到 ContentView 頁面，點擊 preview 左上的 Pin Preview 將 ContentView 釘住。

接著切回 BookRow 時，preview 的上方將出現分頁標籤，一個是剛剛被釘住的 Preview(ContentView.swift)，一個是 BookRow 的 Preview。

如下圖所示，當我們在圖片加上黃色邊框時，馬上能在右邊看到 ContentView 的 preview。
