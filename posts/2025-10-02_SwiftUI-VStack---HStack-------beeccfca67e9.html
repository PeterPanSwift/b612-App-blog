SwiftUI VStack & HStack 的間距問題

當我們使用 SwiftUI 的 VStack & HStack 排版時，若是沒有指定間距，它將有個預設的間距，而不是零間距。

  • 預設有間距的 VStack。
struct ContentView: View {
    var body: some View {
        VStack {
            Rectangle()
            Rectangle()
        }
    }
}
  • 預設有間距的 HStack。
struct ContentView: View {
    var body: some View {
        HStack {
            Rectangle()
            Rectangle()
        }
    }
}

而且在某些情況預設的間距還會增加，比方以下例子 Text 字體愈大時，間距也會愈大。(ps: 此現象會發生在 Text 下是佔滿空間的 view，比方 Rectangle、ScrollView、List。)

struct ContentView: View {
    var body: some View {
        VStack {
            Text("B612")
                .font(.system(size: 100))
                .border(.red, width: 3)
            Rectangle()
        }
    }
}

因此若遇到畫面的間距比想像中大時，記得要另外指定間距，而不是採用預設間距。

  • 指定零間距。
struct ContentView: View {
    var body: some View {
        VStack(spacing: 0) {
            Text("B612")
                .font(.system(size: 100))
                .border(.red, width: 3)
            Rectangle()
        }
    }
}
  • 指定間距 10。
struct ContentView: View {
    var body: some View {
        VStack(spacing: 10) {
            Text("B612")
                .font(.system(size: 100))
                .border(.red, width: 3)
            Rectangle()
        }
    }
}
