SwiftUI 控制浮點數顯示的 string interpolation

開發 iOS App 時，我們時常遇到帶有小數點的浮點數。若是直接將浮點數變成字串顯示在畫面上，往往會顯示成一個超長的數字，例如以下程式的 iOS App 工程師年薪 1000000.878 變成 1000000.878000。

var body: some View {
   let salary = 1000000.878
   return Text("傳說 iOS App 工程師都是年薪 \(salary)")
}

我們通常會透過一些方法指定顯示時顯示到小數點後第幾位數，讓畫面看起來比較好看。在 SwiftUI 控制浮點數的顯示變得更加容易，因為它在 string interolation 增添了參數 specifier，我們可透過它控制浮點數如何顯示。

appendInterpolation(_:specifier:) | Apple Developer Documentation
Appends a type, convertible to a string with a format specifier, to a string interpolation.developer.apple.com

比方我們傳入 "%.2f"，試試它的效果吧。

var body: some View {
   let salary = 1000000.878
   return Text("傳說 iOS App 工程師都是年薪 \(salary, specifier: "%.2f")")
}

如下圖所示，%.2f 將帶來以下影響:

  • %.2f 表示它將顯示到小數點以下第二位，因此顯示到第三位是 %.3f，其它以此類推。
  • 它將四捨五入，因此 1000000.878 變成 1000000.88。
  • 整數部分每三位數加上逗號分隔，因此 1000000.878 變成 1,000,000.88。
