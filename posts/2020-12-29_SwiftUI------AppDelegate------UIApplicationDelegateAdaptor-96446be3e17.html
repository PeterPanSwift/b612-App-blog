SwiftUI 如何搭配 AppDelegate — 利用 UIApplicationDelegateAdaptor

開發 SwiftUI App 時，App 的設定寫在遵從 protocol App 的型別，而不是以前的 AppDelegate。

不過目前很多第三方功能一開始的設定都寫在 AppDelegate 的application(_:didFinishLaunchingWithOptions:)，它將在 App 啟動後呼叫，比方 Facebook 登入跟 Firebase，難道我們要放棄它們嗎?

現在說放棄還太早，其實 Apple 留了一條後路給我們繼續使用 AppDelegate，方法如下:

新增 AppDelegate，定義它的 application(_:didFinishLaunchingWithOptions:)

import UIKit

class AppDelegate: NSObject, UIApplicationDelegate {
    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey : Any]? = nil) -> Bool {
        return true 
    }
}

以登入 FB 為例，我們可以在其中加入 ApplicationDelegate.shared.application(application, didFinishLaunchingWithOptions: launchOptions) 完成 FB 的相關設定。

import UIKit
import FacebookCore

class AppDelegate: NSObject, UIApplicationDelegate {
    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey : Any]? = nil) -> Bool {
        ApplicationDelegate.shared.application(application, didFinishLaunchingWithOptions: launchOptions)
        return true
    }
}
