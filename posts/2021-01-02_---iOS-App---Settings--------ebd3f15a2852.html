打開 iOS App 的 Settings 畫面設定權限

為了使用者的安全著想，iOS App 很多功能都要經過使用者同意。不過如果使用者不同意給予 App 權限，後來想讓他同意將麻煩許多，就像追求一開始對我們印象不好的對象一樣。

如下圖所示，如果我們不同意 LINE 的拍照權限，後來當我們想從 App 拍照時，它將無法直接打開權限。它只能跳到 iOS 的 Settings 頁面，麻煩使用者自己打開 LINE App 的拍照權限。

好吧，無魚，蝦也好。既然不能從 App 打開使用者拒絕的權限，能夠跳到 Setting App 的設定畫面也不錯，方法如下:

import SwiftUI
struct ContentView: View {
    
    
    var body: some View {
        Button(action: {
            let url = URL(string: UIApplication.openSettingsURLString)!
            UIApplication.shared.open(url)
            
        }) {
            
            Text("帶我去 Setting")
        }
        
    }
}

透過 UIApplication.openSettingsURLString 取得 App 的 Settings 畫面路徑，然後利用 UIApplication 的 open 跳到 Settings App。(ps: 並非所有 App 都會在 Settings 下有設定畫面，只有曾要求權限或設定 Settings.bundle 的 App 會在 Settings 下有設定畫面，App 沒有設定頁面的話以上程式一樣會打開 Settings App )
