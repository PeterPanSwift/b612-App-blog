利用 UIViewPropertyAnimator 實現反轉動畫

開發 iOS App 時，有時我們想實現自動反轉的動畫效果，比方為了欣賞 peter 奔跑的英姿，下圖的 peter 將從左邊移動到右邊，然後再從右邊移動到左邊。

這樣的動畫效果可以透過型別 UIView.AnimationOptions 的 autoreverse 實現，可惜 Apple 文件說 UIViewPropertyAnimator 不支援跟動畫方向有關的 option，因此我們不能結合 UIViewPropertyAnimator & autoreverse。

就算沒有 autoreverse，我們還是能實現反轉動畫。接下來讓我們試試幾種不同的方法。

在動畫完成時呼叫的 completion closure 設定返回初始狀態的動畫

在 completion closure 將 image view 的條件 constant 設回原本的 constant。

let originalConstant = leadingConstraint.constant

UIViewPropertyAnimator.runningPropertyAnimator(withDuration: 2, delay: 0) {

利用 addCompletion 加入返回初始狀態的動畫

let originalConstant = leadingConstraint.constant
let animator = UIViewPropertyAnimator(duration: 2, curve: .easeInOut) {
            self.leadingConstraint.constant = 200
            self.view.layoutIfNeeded()
}
animator.addCompletion { (_) in
            let reverseAnimator = UIViewPropertyAnimator(duration: 2, curve: .easeInOut) {
                self.leadingConstraint.constant = originalConstant
                self.view.layoutIfNeeded()
            }
            reverseAnimator.startAnimation()
}

利用 isReversed


class ViewController: UIViewController {
    
    @IBOutlet weak var leadingConstraint: NSLayoutConstraint!
    var animator: UIViewPropertyAnimator?
@IBAction func tap(_ sender: Any) {
        
        
        animator = UIViewPropertyAnimator(duration: 2, curve: .easeInOut) {
            self.leadingConstraint.constant = 200
            self.view.layoutIfNeeded()
        }
        animator?.pausesOnCompletion = true
    
        animator?.addObserver(self, forKeyPath: #keyPath(UIViewPropertyAnimator.isRunning), options: [.new], context: nil)
        animator?.startAnimation()
        
    }
   
    override func observeValue(forKeyPath keyPath: String?, of object: Any?, change: [NSKeyValueChangeKey : Any]?, context: UnsafeMutableRawPointer?) {
        
        if change?[NSKeyValueChangeKey.newKey] as? Int == 0 {
            self.animator?.isReversed = true
            self.animator?.startAnimation()
            animator?.removeObserver(self, forKeyPath: keyPath!)
            animator?.pausesOnCompletion = false
        }
        
    }
}
