利用 lldb，搭配 breakpoint 或 Debug View Hierarchy 找到 button 觸發的 function

開發 iOS App 時，我們時常研究或維護別人開發的 App。此時如何快速找出 App 功能的相關程式碼成了影響我們能否提早下班的大事。

許多的 App 功能都是在點擊按鈕時觸發的，如果當初是以 IBAction 設定觸發的 function，而且已經在 Interface Builder 找到畫面，我們將可從 Connections inspector 直接取得觸發的 function 名稱。

然而如果不清楚 button 是否以 IBAction 設定，或是專案太過複雜，無法馬上找到畫面的相關檔案的話，也可以找 lldb 幫忙，搭配 breakpoint 或 Debug View Hierarchy 找到 button 點擊時觸發的 function。

方法 2: 利用 symbolic breakpoint。

利用 symbolic breakpoint，我們可以設定按鈕點擊時觸發中斷點，然後再用 LLDB 找出呼叫的 function 名稱。

1 新增 Symbolic Breakpoint。

2 在 Symbol 欄位輸入以下文字。

-[UIControl sendAction:to:forEvent:]

此 symbol 將讓 App 在 UI 元件發生特別事件時被中斷，因為當使用者操作 UI 元件時將觸發 UIControl 的 sendAction(_:to:for:) function，然後透過 target 執行 action 對應的 function。

sendAction(_:to:for:) - UIControl | Apple Developer Documentation
This method takes the provided information and forwards it to the singleton object for dispatching. If a valid target…developer.apple.com
func sendAction(_ action: Selector, 
             to target: Any?, 
            for event: UIEvent?)

3 點擊中斷點右邊的箭頭，將它 disable。

如此 App 啟動後，才不會因為我們在畫面上操作產生的事件而中斷。

4 啟動 App，進入 CHECK button 所在的 App 頁面。

5 點擊中斷點右邊的箭頭，將它 enable。

待會我們將點擊 CHECK button，所以是時候打開中斷點，讓 App 迎接中斷的命運了。

6 勇敢地按下 CHECK button !

Cool，此時 App 立馬中斷，右下角的 Console 出現方便我們 debug 的 lldb。lldb 提供許多幫助我們 debug 的指令，待會我們將輸入一些指令找出 CHECK button 按下時觸發的 function。

7 輸入以下指令印出觸發的 action 的 function 名稱。

po (SEL)$arg3

原來 function 的名字是 checkAnswer:。此時我們可在 Find Navigator 裡輸入 checkAnswer，找尋 checkAnswer 出沒之處。

也許有人好奇，為何要輸入 $arg3 呢 ? 讓我們先回過頭來瞧瞧 $arg1 & $arg2。當中斷點停在 function sendAction(_:to:for:) 時，$arg1 代表執行 function 的元件，如下圖所示，$arg1 理所當然是 UIButton，因為當我們點擊 CHECK button 造成 App 中斷時，將讓 UIButton 呼叫 function sendAction(_:to:for:)。

而 $arg2 代表呼叫的 function，因此它應該是 sendAction(_:to:for:)。不過當我們輸入 po $arg2 時，它卻印出奇怪的數字。

這是因為 lldb 不知道 $arg2 是 function，因此要在 $arg2 前加上 (SEL)，讓 lldb 知道它是 function。

了解 $arg1 & $arg2 後，接著的 $arg3，$arg4 & $arg5 即為參數 action，target 和 event，因此剛剛我們以 $arg3 取得觸發的 action 名稱。

func sendAction(_ action: Selector, 
             to target: Any?, 
            for event: UIEvent?)

倘若運氣好，叫 checkAnswer 的 function 只有一個，我們現在已找到目標，可以翹腳休息了。等等，就算這樣也還不能休息啦，我們還得研究 checkAnswer { } 裡的程式碼。

而當運氣不好，很多 function 都叫 checkAnswer 時，將如下圖所示，我們得先找到哪個才是我們要的。就好像金城武有很多，有內湖金城武，板橋金城武，只有一個是我們的白馬王子一樣。

這麼多的 checkAnswer，我們想到頭髮都白了也猜不到，還是再請 lldb 幫忙吧。

8 按鈕點擊時將呼叫 target 的 function，因此我們輸入以下指令印出 target 的資訊。

po $arg4

只要找到 target 的型別，即可明白 function 定義在何處。如下圖所示，target 的型別為 MultipleSelectReviewViewController，因此按鈕點擊時將觸發 MultipleSelectReviewViewController 的 checkAnswer function。
