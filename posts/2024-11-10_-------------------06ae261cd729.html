定義型別名稱時，不要用到內建的型別名

最近有同學跟彼得潘說，他的程式出現奇怪的錯誤，SwiftUI 的 Image 竟然無法使用，出現一堆莫名其妙的錯誤，明明程式完全沒有錯。

彼得潘看了一下程式後，發現同學重覆定義了 Image，因此程式變成只認得新定義的 Image，而不是 SwiftUI 的 Image。

struct Image {
    let name: String
    let width: Double
    let height: Double
}

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

此問題的其中一種解法是使用 SwiftUI 的 Image 時寫成 SwiftUI.Image，不過更好的解法應該是定義型別名稱時，不要用到內建的型別名，即可避免許多奇怪的問題跟麻煩。

struct ContentView: View {
    var body: some View {
        VStack {
            SwiftUI.Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}
