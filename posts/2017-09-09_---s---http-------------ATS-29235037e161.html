錯誤訊息的重點是以下這句，提到 App Transport Security 要求 App 採用安全的連結。

The resource could not be loaded because the App Transport Security policy requires the use of a secure connection

不過如果我們想連接不安全的 http，還是有方法。以下介紹兩種方法。

  • 新增 Capability 設定 App Transport Security Exception。
  • 從 Info 頁面設定 App Transport Security Settings。

方法 1: 新增 Capability 設定 App Transport Security Exception

方法 2: 從 Info 頁面設定 App Transport Security Settings

操作步驟

1 從 Project Navigator 頁面點選專案檔。

2 點選 TARGETS 下的 App 後，切換到 Info 頁面。

3 滑鼠游標移到表格裡任一筆資料，將看到 + 出現，點選 + 新增一列

4 從選單裡選擇 App Transport Security Settings。

5 點選 App Transport Security Settings 旁邊的三角形，變成三角形朝下時再點選 +，從選單選擇 Allow Arbitrary Loads。

6 將 Allow Arbitrary Loads 的 Value 設為 YES。

Cocoa Keys
Describes the keys for a bundle's Info.plist file.developer.apple.com

不過請不要開心的太早，為了怕你做壞事，Apple 說當你將 Allow Arbitrary Loads 設為 YES 時，需要跟他解釋為什麼你要用到不安全的 http 連線。只有當你的理由合情合理，通過 Apple 的審核後才能上架。

Setting this key’s value to YES triggers App Store review and requires justification.

不過將 Allow Arbitrary Loads 設為 YES 將允許抓取任何地方的資料，所以會比較危險，因此 App Transport Security Settings 還提供了其它兩種選項。

  • Allow Arbitrary Loads in Web Content

只允許 web view 抓取不安全的資料。

  • Exception Domains

利用 Exception Domains 指定例外的網址，只有這些網址的資料可以用不安全的方式抓取。
