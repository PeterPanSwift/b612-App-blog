SwiftUI List Row 的 button 點選

iOS App 在顯示表格時，常常一筆資料裡會有多個不同功能的 button，比方以下的 IG App 我們可針對每則 po 文按愛心或留言。

不過當我們用 SwiftUI 的 List 製作表格頁面，想實現類似的 button 功能時卻會產生嚴重的問題。接下來就讓我們以最近上映的電影陪你很久很久為例說明。

struct ContentView: View {
    var body: some View {
        List(0 ..< 5) { (index) in
            VStack {
                Image("陪你很久很久")
                    .resizable()
                    .scaledToFit()
                HStack(spacing: 20) {
                    Button(action: {
                        print("like")
                    }) {
                        Image(systemName: "heart")
                    }
                    Button(action: {
                        print("comment")
                    }) {
                        Image(systemName: "bubble.right")
                    }
                    Spacer()
                }
            }
        }
    }
}

如下圖所示，電影的下方有按讚 & 留言的按鈕，但不管我們點擊哪個 button，都會變成整個電影的 row 被點選，不只背景變灰色，兩個 button 點選對應的程式都會被觸發 !

或許這也滿好的，買一送一，按留言順便增加讚。

不過有些人可能會不喜歡這樣的功能，我們還是來想想解決的方法吧。

解法 1: 利用 buttonStyle & EmptyView

設計 iOS App 時，我們時常利用表格(List)顯示豐富的資料內容。如下圖所示，小說精選列表顯示一本本精彩的小說，點選作者名跳到作者頁面，點選整本書的 row 跳到書本頁面。medium.com

解法 2: 放棄 button，改用 onTapGesture

其實這一切的罪魁禍首是 button。button 本身很 ok，但當它出現在 List 的 row 時卻會變成點選後整個 row 被點選，甚至害同個 row 的其它 button 被點選。

所以我們只好放棄 button 了。我們可以改用 onTapGesture 設定點選後執行的程式。

HStack(spacing: 20) {
   Image(systemName: "heart")
      .onTapGesture {
         print("like")
   }
   Image(systemName: "bubble.right")
      .onTapGesture {
         print("comment")
   }
   Spacer()
}

如下圖所示，現在點選愛心和留言都不會讓整個 row 變成被點選的灰色，也不會再發生點選一個 button 讓其它 button 被點選的連鎖反應。
