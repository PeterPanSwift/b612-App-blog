protocol 的 optional function 定義不要寫在另一個 function 裡

iOS SDK 裡有許多 protocol 的 function 都是 optional，代表我們在遵從 protocol 時不一定要定義它，比方 protocol UITableViewDelegate 的 tableView(_:didSelectRowAt:)。

optional func tableView(_ tableView: UITableView, didSelectRowAt indexPath: IndexPath)

此 function 將在 cell 被點選時呼叫，除非我們想在 cell 點選時做事，才需要定義此 function。

有些朋友在定義這些 optional function 時，不小心將它寫在另一個 function 裡，例如以下例子:

override func viewDidLoad() {
   super.viewDidLoad()
   func tableView(_ tableView: UITableView, didSelectRowAt indexPath: IndexPath) {
      print("didSelectRowAt")
   }
}

由於 Swift 允許 function 裡定義 function，因此 App 還是可以順利執行，不會有任何錯誤。但此時 function 將不會再被呼叫，因為它被藏在另一個 function 裡，protocol 的眼睛沒那麼大，他會找不到它，以為我們沒有定義呀。
