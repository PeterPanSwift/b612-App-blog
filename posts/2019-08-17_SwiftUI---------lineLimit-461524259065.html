SwiftUI 控制文字行數的 lineLimit

SwiftUI 的 function lineLimit(_:) 控制文字可顯示的最大行數，因此參數 number 傳入 1 可顯示 1 行， 傳入 2 可顯示 2 行。傳入 nil 呢 ? nil 代表沒有任何限制，顯示一百行，一千行都不是問題。

SwiftUI 控制文字行數的 lineLimit 和固定行高的 reservesSpace

SwiftUI 的 modifier lineLimit(_:) 控制文字可顯示的最大行數，因此參數 number 傳入 1 可顯示 1 行， 傳入 2 可顯示 2 行。傳入 nil 呢 ? nil 代表沒有任何限制，顯示一百行，一千行都不是問題。

func lineLimit(_ number: Int?) -> some View
lineLimit(_:) | Apple Developer Documentation
Sets the maximum number of lines that text can occupy in this view.developer.apple.com

Text 元件預設的 lineLimit number 為 nil，因此 Text 預設可顯示多行文字，比方以下例子用 Text 顯示電影玩具總動員的名言。

struct ContentView: View {
   var body: some View {
      Text("To the world you may be just one person, but to one person you may be the whole world.")
         .background(.yellow)
   }
}

若想限制行數，則可呼叫 lineLimit(_:)，比方限制行數為一行。

struct ContentView: View {
   var body: some View {
      Text("To the world you may be just one person, but to one person you may be the whole world.")
         .background(.yellow)
         .lineLimit(1)
   }
}

在 iOS 16 SwiftUI 推出了更厲害的 lineLimit(_:reservesSpace:)，當 reservesSpace 為 true 時，Text 可固定行高，不管它現在有多少字。

lineLimit(_:reservesSpace:) | Apple Developer Documentation
Sets a limit for the number of lines text can occupy in this view.developer.apple.com

因為 lineLimit(2, reservesSpace: true)，以下例子的 Peter Pan 雖然只有一行字，但卻可以佔據 2 行的空間。

struct ContentView: View {
   var body: some View {
      Text("Peter Pan")
         .background(.yellow)
         .lineLimit(2, reservesSpace: true)
   }
}
