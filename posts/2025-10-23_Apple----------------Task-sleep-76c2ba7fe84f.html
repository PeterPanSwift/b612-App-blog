Apple 的人物對話動畫範例 — 使用 Task.sleep

最近彼得潘研究 Apple 在 WWDC25 的範例 Generate dynamic game content with guided generation and tools，發現它實現了一個遊戲常見的功能 — 人物對話。

Generate dynamic game content with guided generation and tools | Apple Developer Documentation
Make gameplay more lively with AI generated dialog and encounters personalized to the player.developer.apple.com

查看程式碼後，發現原來 Apple 利用 Task.sleep 實現對話一字一字出現的動畫效果。

以下我們將 Appe 的程式精簡成以下範例說明。

struct ContentView: View {
    @State private var isTalking = false
    let text = "All grown-ups were once children... but only few of them remember it."
    @State private var renderedText = ""
    
    var body: some View {
        VStack(alignment: .leading, spacing: 0) {
            Text("Little Prince")
                .fontWeight(.bold)
            HStack {
                Text(renderedText)
                    .onChange(of: renderedText, typingAnimation)
                Spacer()
            }
        }
        .frame(width: 250)
        .padding()
        .background(.brown.secondary)
        .border(.brown, width: 6)
        .onAppear(perform: typingAnimation)
    }
    
    func typingAnimation() {
        if renderedText.count < text.count {
            isTalking = true
            Task {
                try? await Task.sleep(for: .seconds(0.025))
                if renderedText.count < text.count {
                    let next = text[renderedText.endIndex]
                    renderedText.append(next)
                } else {
                    isTalking = false
                }
            }
        } else {
            isTalking = false
        }
    }
}

text 是對話的完整內容，renderedText 是畫面上看到的對話，我們透過 function typingAnimation 讓 renderedText 一字一字增加到跟 text 一模一樣，利用 Task.sleep(for: .seconds(0.025))稍做停頓，實現對話慢慢長出的動畫效果。

呼叫 typingAnimation 的地方有兩種，一個是 onAppear，一個是 Text(renderedText) 的 onChange(of:initial:_:)，因此畫面一出現就會觸發 typingAnimation，然後在 renderedText 改變時繼續觸發 typingAnimation。
