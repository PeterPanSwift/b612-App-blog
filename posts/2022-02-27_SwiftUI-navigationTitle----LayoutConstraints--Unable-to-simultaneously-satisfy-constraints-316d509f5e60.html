SwiftUI navigationTitle 的 [LayoutConstraints] Unable to simultaneously satisfy constraints

開發 iOS App 時，我們時常使用 NavigationView & navigationTitle 設定 navigation bar 的標題，例如以下例子。

struct ContentView: View {
    var body: some View {
        NavigationView {
            Image("apple watch")
            .navigationTitle("我只是想加個 title")
        }
    }
}

App 畫面看來十分完美，不過 Xcode 的 Console 卻印出一長串的警告訊息，告訴我們畫面的排版有問題。

[LayoutConstraints] Unable to simultaneously satisfy constraints.
 Probably at least one of the constraints in the following list is one you don't want.

其實我們可以不管這個警告訊息，因為 App 的畫面跟功能都是正常的，不過如果不想看到以上煩人的訊息，也可以在 NavigationView 後加上 modifier navigationViewStyle，將樣式指定為 stack 即可消除警告訊息。

struct ContentView: View {
    var body: some View {
        NavigationView {
            Image("apple watch")
            .navigationTitle("我只是想加個 title")
        }
        .navigationViewStyle(.stack)
    }
}
