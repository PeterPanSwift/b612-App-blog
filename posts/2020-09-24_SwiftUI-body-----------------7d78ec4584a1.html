SwiftUI body 裡有多個元件時，記得包在容器裡

開發 SwiftUI App 時，我們透過變數 body 設計畫面的內容。當 body 裡有多個 UI 元件時，記得要將它們包在某個容器元件的 { } 裡，這樣才能明確表達我們想如何排列元件。

比方以下例子用 VStack 裝著 Text & Image，所以 Text & Image 會由上而下排列。

struct ContentView: View {
    var body: some View {
        VStack {
            Text("我能想到最浪漫的事\n就是和 Swift 一起慢慢變老")
            Image("book")
        }
    }
}

若是忘了用容器裝著多個元件，那會世界末日嗎 ?

struct ContentView: View {
    var body: some View {
        Text("我能想到最浪漫的事\n就是和 Swift 一起慢慢變老")
        Image("book")
    }
}

還好，並不會世界末日，也沒有紅色錯誤，此時 SwiftUI 會聰明地由上而下排版元件。不過還是建議 body 裡有多個元件時，記得包在容器裡，如此比較能讓自己和別人看程式時一眼看出畫面的排版。
