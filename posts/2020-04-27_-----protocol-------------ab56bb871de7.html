限定遵從 protocol 的型別也要繼承某某型別

protocol 可以繼承 protocol，因此當 protocol Idol 繼承 protocol Cute 時，遵從 protocol 的 Man 需要定義 smile ＆ sing。

protocol Cute {
    func smile()
}
protocol Idol: Cute {
    func sing()
}
class Man: Idol {
    func sing() {
    }
    
    func smile() {
    }    
}

不過 protocol 的冒號後不只可以接 protocol，還可以接 class，實現限定遵從 protocol 的型別也要繼承某某型別的效果。比方以下例子代表遵從 Idol 的型別也要繼承 Animal。

class Animal {
    func eat() {
    }
}
protocol Idol: Animal {
    func sing()
}

除了 protocol Idol: Animal，另一種搭配 where 的寫法也有一樣的效果。

protocol Idol where Self: Animal {
    func sing()
}

當 Man 遵從 Idol 的型別卻沒繼承 Animal 時將產生錯誤。

class Man: Animal, Idol {
    func sing() {
        
    }
    
}
