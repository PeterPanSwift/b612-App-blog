讓 Flutter 的 iOS App 使用 SPM 安裝套件

Flutter 最近開始支援 SPM(Swift Package Manager)，我們可以設定 iOS App 使用 SPM 安裝套件，而不是 CocoaPods。

Swift Package Manager for app developers
How to use Swift Package Manager for native iOS or macOS dependenciesdocs.flutter.dev

以下我們以播放聲音的套件 just_audio 為例，說明如何打開 Flutter 的 SPM 功能。

just_audio | Flutter package
A feature-rich audio player for Flutter. Loop, clip and concatenate any sound from any source (asset/file/URL/stream)…pub.dev

使用 just_audio 播放音檔的範例如下。

import 'package:flutter/material.dart';
import 'package:just_audio/just_audio.dart';

void main() => runApp(MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(title: 'Just Audio 範例', home: AudioPlayerScreen());
  }
}

class AudioPlayerScreen extends StatefulWidget {
  const AudioPlayerScreen({super.key});

  @override
  State<AudioPlayerScreen> createState() => _AudioPlayerScreenState();
}

class _AudioPlayerScreenState extends State<AudioPlayerScreen> {
  final AudioPlayer _player = AudioPlayer();
  final String audioUrl =
      'https://www.google.com/logos/fnbx/animal_sounds/cat.mp3';

  @override
  void initState() {
    super.initState();
    _initAudio();
  }

  Future<void> _initAudio() async {
    try {
      await _player.setUrl(audioUrl);
    } catch (e) {
      print("載入音源時發生錯誤：$e");
    }
  }

  @override
  void dispose() {
    _player.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Just Audio 範例')),
      body: Center(
        child: IconButton(
          icon: Icon(Icons.play_arrow, size: 36),
          onPressed: () {
            _player.seek(Duration.zero);
            _player.play();
          },
        ),
      ),
    );
  }
}

目前 Flutter 預設關閉 SPM 功能，因此 iOS App 將用 CocoaPods 安裝套件。加入套件 just_audio 後，iOS 資料夾下將加入 Podfile & CocoaPods 的相關設定，

為了使用 SPM，我們必須打開 Flutter 的 SPM 功能。

請在 Terminal 輸入以下指令。

flutter config --enable-swift-package-manager

結果。

Setting "enable-swift-package-manager" value to "true".

You may need to restart any open editors for them to read new settings.

打開 SPM 功能後，新建立的 Flutter 專案將預設使用 SPM 安裝 iOS App 的套件，因此 iOS 資料夾下不再產生 CocoaPods 的相關設定。

用 Xcode 打開 Flutter 生成的 iOS 專案後，從 Package Dependencies 可看到 SPM 安裝的套件。

如果 Flutter 專案在 SPM 功能打開前建立，執行 App 時將自動改成搭配 SPM 安裝套件。不過 CONSOLE 會顯示建議我們移除專案裡 CocoaPods 設定的訊息，移除可加快 App 編譯的速度。

All plugins found for ios are Swift Packages, but your project still has CocoaPods integration. To remove CocoaPods integration, complete the following steps:
  * In the ios/ directory run "pod deintegrate"
  * Also in the ios/ directory, delete the Podfile
  * Remove the include to "Pods/Target Support Files/Pods-Runner/Pods-Runner.debug.xcconfig" in your ios/Flutter/Debug.xcconfig
  * Remove the include to "Pods/Target Support Files/Pods-Runner/Pods-Runner.release.xcconfig" in your ios/Flutter/Release.xcconfig

Removing CocoaPods integration will improve the project's build time.

ps:

  • 有些尚未支援 SPM 的 Flutter 套件仍然要以 CocoaPods 安裝，比方以下連結的 audioplayers。
audioplayers | Flutter package
A Flutter plugin to play multiple audio files simultaneouslypub.dev
  • 關閉 SPM 功能的指令如下。
flutter config --no-enable-swift-package-manager
