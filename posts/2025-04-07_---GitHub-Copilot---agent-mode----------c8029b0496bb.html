透過 GitHub Copilot 的 agent mode 自動搜索專案程式

最近許多 IDE 的 AI 功能都推出了自動化的 agent mode，AI 可以自動地執行指令和搜索研究專案的檔案程式。以下彼得潘以 GitHub Copilot 的 agent mode 為例，說明開發新功能時 agent mode 自動搜索專案程式的強大之處。

以下我們以 SwiftUI 開發的 tab 分頁 App 為例，第一個 tab 分頁顯示好看的書已實作完成，接下來我們將請 AI 幫忙實作第二個 tab 分頁顯示好看的電影。

struct ContentView: View {

    var body: some View {
        TabView {
            Tab("Books", systemImage: "books.vertical") {
                BookList()
            }
            Tab("Movies", systemImage: "film") {
                
            }
        }
    }
}

首先我們先試試沒有 Agent mode 的 Edit Mode，它將有修改程式的功能，但缺乏自動化功能。

prompt: 模仿第一個 tab 的 book 頁面，實作第二個 tab 的 movie 相關頁面。

沒有 agent 的 AI 不夠聰明，它不會自動搜索研究專案程式，因此當我們沒有指定參考的 swift 檔時，它將不明白如何修改，產生了完全錯誤的回答，變成生成網頁相關的 tsx 檔。

以下我們試試 agent mode 的威力。請先將 prompt 的輸入改成 Agent mode。

ps: 若是沒出現 Agent 選項，請先打開 Settings 勾選 Enable Agent mode。

Agent mode 的 AI 果然厲害多了，自動找出 book 的相關檔案 Book.swift、BookList.swift、BookRow.swift、BookDetail.swift、ContentView.swift。

AI 模仿 book 的相關檔案新增 Movie.swift、MovieDetail.swift、MovieList.swift、MovieRow.swift。

agent mode 的 AI 成功幫我們的 App 增添了電影頁面。

ps: 記得手動在 asset 加入電影圖片。
