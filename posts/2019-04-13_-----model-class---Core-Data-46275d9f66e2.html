自動生成 model class 的 Core Data

開發 iOS App 時，Core Data 是其中一種我們經常使用的資料儲存技術。我們可以在 .xcdatamodeld 檔定義資料(entity)的名稱和欄位(attribute)，比方下圖定義的 Movie entity 。

之後在 Swift 程式裡我們即可利用類別 Movie 產生電影物件。但是類別 Movie 需要自己定義嗎 ? 如果 Movie 有很多欄位，親手輸入每個欄位對應的 property 可是十分辛苦的事。

其實不用這麼辛苦，現在 Xcode 將幫我們自動產生 .xcdatamodeld 檔裡定義的 entity 類別。如下圖所示，我們可以直接輸入 Movie( ) 生東西。

也許有人好奇 Movie 的類別寫在哪裡 ? 明明 Xcode 左邊的 project navigator 沒看到它呀。

其實它真的存在，只是要多繞一點路才能看到。如下圖所示，請將滑鼠游標移到 Movie 上，按住 cmd 鍵再按下滑鼠左鍵後，點選 Jump to Definition 即可看到 class Movie 的定義。

至於為什麼 Xcode 能夠自動產生 model 的 class 呢 ? 答案就藏在 .xcdatamodeld 檔裡。

請點選 .xcdatamodeld 檔，然後選擇某個 entity，點選上方的第三個 icon (Data Model inspector)，將可看到 model 裡有個叫 Codegen 的欄位，正因為它的 value 是 Class Definition，所以能夠自動產生類別。

相反的，當我們改成 Manual/None 時，它就不會自動生成了。

此時我們只能自己辛苦地手動輸入，或是從 Xcode menu 的 Editor > Create NSManagedObject Subclass 產生類別。
