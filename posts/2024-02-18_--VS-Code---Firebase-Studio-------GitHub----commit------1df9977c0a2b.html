從 VS Code / Firebase Studio 上傳程式到 GitHub 和用 commit 記錄修改

VS Code / Firebase Studio 內建版本管理功能，因此我們可以方便地對專案進行 git 操作。以下文章我們將針對兩個部份做簡單地說明。

  • 上傳程式到 GitHub。
  • 用 commit 記錄修改的程式並上傳到 GitHub。

上傳程式到 GitHub

點選 Publish to GitHub

  • 方法 1。

從 Command Palette(cmd+shift+p)選單點選 Publish to GitHub。

  • 方法 2。

點選下圖紅色框框的 Source Control Icon 切換到版本管理頁面。

點選 Publish to GitHub。

點選 Allow 打開 GitHub 網頁

登入 GitHub 網頁

點選 Authorize Visual-Studio-Code 同意 GitHub for VS Code 的存取權限

同意權限讓 VS Code 能存取 GitHub 上的 repository 和進行 git 相關操作。

點選 Allow 打開 VS Code

點選 Publish to GitHub 上傳程式

可依是否公開程式選擇 public 或 private。

ps: 如果是交作業的同學，請選擇 public repository，如此別人才能看到你的程式。

剛剛一連串的步驟只須設定一次，未來新的專案上傳到 GitHub 時，只要點選 Publish to GitHub，即可從選單點選 Publish to GitHub。

查看專案在 GitHub 的網址

打開 branch 選單。

  • 方法 1: 點選左下的分支圖示。
  • 方法 2: 從版本管理頁面點選 repository 旁的分支圖示。

滑鼠游標移到 branch 名字上，點選 GitHub icon 開啟網頁。

開啟的網頁是專案在 GitHub 的 repository，從上方的網址列可看到網址。

用 commit 記錄修改的程式並上傳到 GitHub

將專案的程式上傳到 GitHub 後，我們可能還會做修改。

當程式修改到某個段落時我們可執行 git 的 commit 做記錄。如下圖所示，切換到 Source Control 分頁，Changes 的下方將列出目前被修改的相關檔案。

在 Message 框框輸入訊息說明做了哪些修改，然後點選 Commit 做記錄。如果想請 AI 幫忙，也可以點選星星圖示，請 Copilot 生成訊息。

點選 Yes，表示我們要將所有的修改加入暫存區並做記錄。

點選 Sync Changes 將記錄的修改上傳到 GitHub。

此動作會進行 pull & push，因此若 GitHub 上有更新的程式也會被抓下來和本機端的程式合併。

以上只是 VS Code 基本的 git 操作介紹，想再探究進階功能的朋友可參考官方網站的說明。
