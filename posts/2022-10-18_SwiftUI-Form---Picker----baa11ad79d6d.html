SwiftUI Form 的 Picker 問題

開發 SwiftUI App 時，當我們在 Form 下的 Picker 選擇項目時，在不同的版本會有不同的行為。在 iOS 16 以上是正常的，在 iOS 15 之前則要做一些調整。

SwiftUI Form 的 Picker 問題

開發 SwiftUI App 時，當我們在 Form 下的 Picker 選擇項目時，在不同的版本會有不同的行為。在 iOS 16 以上是正常的，在 iOS 15 之前則要做一些調整。

接下來我們分別在 iOS 16 & 15 測試以下 Form 搭配 Picker 的程式。

struct ContentView: View {
    let roles = ["李白", "李嘉誠", "彼得潘", "奇妙仙子"]
    @State private var selectedName = "李白"
    
    var body: some View {
        Form {
            Picker(selection: $selectedName) {
                ForEach(roles, id: \.self) { role in
                    Text(role)
                }
            } label: {
                Text("選擇角色")
            }
        }
    }
}

iOS 16 的 Form & Picker

點選 picker 後將出現選單，預設採用的 picker 樣式是 menu。

iOS 15 之前的 Form & Picker

超慘的，picker 完全不能點選。

修正方法

  • 方法 1: 加入 NavigationView

iOS 15 Form 下的 Picker 預設採用跳到下一頁選擇的方式，因此我們要加入 NavigationView，讓它能跳到跳下一頁。

struct ContentView: View {
    let roles = ["李白", "李嘉誠", "彼得潘", "奇妙仙子"]
    @State private var selectedName = "李白"
    
    var body: some View {
        NavigationView {
            Form {
                Picker(selection: $selectedName) {
                    ForEach(roles, id: \.self) { role in
                        Text(role)
                    }
                } label: {
                    Text("選擇角色")
                }
            }
        }
    }
}

ps: iOS 16 若想採用切換到下一頁選擇的方式，可將 pickerStyle 設成 navigationLink。

.pickerStyle(.navigationLink)
