multi-statement closure parameter/result type inference — Swift 5.7 新功能

從 Swift 5.7(搭配 Xcode 14)開始，swift 變聰明了! 當 closure 裡的程式有多行時，swift 也能推理出 closure 的型別。

以下我們以 function map 傳入的 closure 為例說明。

當 closure 裡的程式只有一行時，swift 可以輕鬆推理出 closure 回傳的型別是字串。

func test() {
        let numbers = [1, 2, 3]
        let numberStrings = numbers.map { number in
            "\(number)"
        }
        print(numberStrings)
}

不過當 closure 裡有多行程式時，舊版的 Swift 就看不懂了，它無法推理出 closure 回傳的型別是字串。

func test() {
        let numbers = [1, 2, 3]
        let numberStrings = numbers.map { number in
            let numberString = "\(number)"
            return numberString
        }
        print(numberStrings)
}

錯誤訊息顯示 Cannot infer return type for closure with multiple statements; add explicit type to disambiguate。

在舊版的 Swift，我們可以用以下兩種方法修正問題:

  • 解法一: 寫出 closure 的回傳型別
func test() {
    let numbers = [1, 2, 3]
    let numberStrings = numbers.map { number -> String in
         let numberString = "\(number)"
         return numberString
    }
    print(numberStrings)
}
  • 解法二: 宣告 numberStrings 的型別是 [String]，讓 swift 知道 closure 回傳的型別是 String
func test() {
    let numbers = [1, 2, 3]
    let numberStrings: [String] = numbers.map { number in
         let numberString = "\(number)"
         return numberString
    }
    print(numberStrings)
}

從 Swift 5.7 開始，不用再這麼辛苦了。就算 closure 裡有多行程式， swift 也能秒速推理出回傳的型別。

func test() {
        let numbers = [1, 2, 3]
        let numberStrings = numbers.map { number in
            let numberString = "\(number)"
            return numberString
        }
        print(numberStrings)
}
