Swift 5.4 的 Extend implicit member syntax to cover chains of member references

當我們存取 type property & method 時，型別省略的寫法可讓程式更精簡。然而原本它是有限制的，如果在 type property & method 後繼續存取下一層的 property & method，它將不能再省略，比方以下程式將產生紅色錯誤。

這個限制在 Swift 5.4 被打破了，以下程式將不再產生任何錯誤。(ps: Xcode 12.5 開始搭配 Swift 5.4)

Image("peter")
   .shadow(color: .blue.opacity(0.5), radius: 10)

現在我們省略型別時，想要存取到第五，第六層的 property & method 都不是問題，只要省略的型別跟最後回傳資料的型別一致。

struct Mother {
    func createBaby() -> Baby {
        return Baby()
    }
}
struct Baby {
    var mother = Mother()
    static var cuteBaby = Baby()
}
var baby: Baby = .cuteBaby.mother.createBaby()

.cuteBaby.mother.createBaby() 省略型別 Baby，最後回傳的資料是 Baby，因此符合條件。值得注意的，過程中的資料可以是別的型別，因此剛剛例子裡的 Mother 是 ok 的。
