FAQ: 關於 Swift 編程語言，您需要知道的 8 個問題

本常見問題解答將解答有關 Swift 編程語言的主要主題和概念。

1. Swift 語言的主要特點是什麼？

Swift 是一種現代的編程語言，設計目的是簡潔、快速且安全。一些主要特點包括：

  • 簡潔易讀： 語法簡潔，不需要分號結尾，讓程式碼更容易閱讀和編寫。
  • 類型安全： Swift 強制執行類型安全，有助於在編譯時捕獲錯誤，減少運行時錯誤的可能性。
  • 快速高效： Swift 的設計目標之一是快速，它使用 LLVM 編譯器，可以生成高效的機器碼。
  • 安全可靠： Swift 具有內置的安全機制，例如自動内存管理（ARC）和可選類型，有助於防止常見的編程錯誤。

2. 如何在 Swift 中声明常量和变量？

  • 使用 let 關鍵字声明常量，常量一旦被賦值就不能再更改。
  • 使用 var 關鍵字声明變量，變量的值可以多次更改。

例如：

let myConstant = 10 // 声明一个常量

var myVariable = 20 // 声明一个变量

3. 可選類型（Optional）是什麼？如何在 Swift 中處理可選類型？

可選類型表示一個變量可能存在值，也可能不存在值（nil）。

  • 在類型後面添加問號 ? 來声明一個可選類型。
  • 使用可選綁定（Optional Binding）來安全地解開可選類型，例如使用 if let 或 guard let 語句。
  • 使用空合運算符 ?? 提供一個默認值，以防可選類型為 nil。

例如：

var optionalName: String? = “John”

// 可選綁定

if let name = optionalName {

print(“Hello, \(name)”)

} else {

print(“Name is nil”)

}

// 空合運算符

let greeting = “Hello, \(optionalName ?? “Guest”)”

4. 如何在 Swift 中创建和使用函數？

  • 使用 func 關鍵字声明函數。
  • 使用 () 包裹参数列表，使用 -> 指定返回類型。

例如：

func greet(person: String, day: String) -> String {

return “Hello \(person), today is \(day).”

}

let greetingMessage = greet(person: “Alice”, day: “Monday”)

5. 闭包（Closure）是什麼？如何在 Swift 中使用闭包？

闭包是可以在代码中传递和使用的代码块。

  • 闭包可以使用簡化的語法，例如省略参数类型、使用 in 关键字区分参数和返回值等。
  • Swift 中许多高阶函数，例如 map、filter、reduce 等，都接受闭包作为参数。

例如：

let numbers = [1, 2, 3, 4, 5]

let doubledNumbers = numbers.map { $0 * 2 } // 使用闭包将数组中的每个元素乘以 2

6. Swift 中的類別（Class）和結構（Struct）有什麼區別？

  • 类別是引用類型，结构是值類型。
  • 类別支持繼承，结构不支持。
  • 类別实例在堆上分配内存，结构实例在栈上分配内存。

一般來說，如果需要使用繼承或需要在多个地方共享和修改数据，则应该使用类。如果数据结构相对简单，并且希望每次传递时都复制一份新的数据，则应该使用结构。

7. 如何在 Swift 中處理錯誤？

  • 使用 Error 协议来表示错误类型。
  • 使用 throw 关键字抛出错误。
  • 使用 try 关键字调用可能抛出错误的函数。
  • 使用 do-catch 语句捕获和处理错误。

例如：

enum NetworkError: Error {

case noConnection

case invalidURL

}

func fetchData() throws -> Data {

// … 模拟网络请求，可能抛出错误

}

do {

let data = try fetchData()

// … 处理数据

} catch NetworkError.noConnection {

// … 处理网络连接错误

} catch {

// … 处理其他错误

}

8. 什麼是協議（Protocol）？如何在 Swift 中使用協議？

協議定義了一组方法或属性的蓝图，任何类型只要实现了协议中定义的所有方法和属性，就被认为是符合该协议的。

  • 使用 protocol 关键字定义协议。
  • 使用 extension 为现有的类型添加协议实现。
  • 使用协议作为类型约束，限制函数或方法的参数类型。

例如：

protocol Drawable {

func draw()

}

struct Circle: Drawable {

func draw() {

// … 绘制圆形

}

}

let myCircle = Circle()

myCircle.draw()

簡答題答案

  1. let 用於宣告常數，常數的值在初始化後無法更改；而 var 用於宣告變數，變數的值可以多次更改。例如： let myConstant = 10 宣告一個常數，其值為 10，而 var myVariable = “Hello” 宣告一個變數，其值初始為 “Hello”，之後可以更改。
  2. 型別推斷是指編譯器可以根據值的類型自動推斷變數或常數的資料類型。例如，當您寫入 let myNumber = 42 時，Swift 編譯器會推斷出 myNumber 的類型為 Int。
  3. 使用 \() 語法可以在 Swift 中使用字串插值。例如：let name = “John”，print(“Hello, \(name)!”) 會印出 “Hello, John!”。
  4. 陣列是有序的值的集合，而字典是無序的鍵值對的集合。例如，可以使用陣列儲存待辦事項清單，使用字典儲存學生姓名和分數的對應關係。
  5. if let 語句用於安全地解開可選類型。如果可選類型包含值，則將值賦給 if let 語句中定義的常數或變數，並執行語句塊。例如，if let myValue = optionalValue { print(myValue) }，如果 optionalValue 包含值，則會印出該值。
  6. 函式是有名稱的程式碼塊，可以通過名稱調用，而閉包是沒有名稱的程式碼塊，可以作為參數傳遞給函式或儲存在變數中。
  7. 類別是定義物件模板的藍圖，物件是類別的實例。例如，可以定義一個名為 Car 的類別，它具有顏色、品牌和型號等屬性，然後可以創建多個 Car 物件，每個物件都有自己的顏色、品牌和型號。
  8. 結構是值類型，當您傳遞結構時，實際上是創建了一個副本；而類別是引用類型，當您傳遞類別時，傳遞的是對同一個物件的引用。例如，可以使用結構表示日期、時間或座標等值類型，使用類別表示更複雜的物件，例如使用者、網路請求或遊戲角色等。
  9. 協定定義了一組方法、屬性或其他需求的藍圖，類別、結構或列舉可以遵循這些藍圖。例如，可以定義一個名為 Drawable 的協定，它要求遵循該協定的類型實現一個 draw() 方法，然後可以創建多個遵循 Drawable 協定的類型，例如 Circle、Square 或 Triangle 等，每個類型都以自己的方式實現 draw() 方法。
  10. 錯誤處理允許您處理程式運行時可能發生的錯誤。do-catch 語句用於處理可能拋出錯誤的程式碼。do 塊包含可能拋出錯誤的程式碼，catch 塊用於處理特定類型的錯誤。例如：do { try performOperation() } catch SomeError.invalidInput { // 處理無效輸入錯誤 } catch { // 處理其他錯誤 }。

Briefing Doc: Swift 程式語言導覽：主要主題與重要概念

本文件總結了“A Swift Tour | Documentation.pdf” 的主要主題和重要概念，並在適當的地方引用了原文。

一、基本語法

  • Swift 的語法類似於其他流行的程式語言，但有一些關鍵差異。
  • 使用 print 函數輸出文字到螢幕，例如：
  • print(“Hello, world!”) // 輸出 “Hello, world!”
  • 不需要匯入函式庫來使用輸出文字或處理字串等功能。
  • 全局範圍的程式碼作為程式的入口點，因此不需要 main() 函數。
  • 每行程式碼結尾不需要分號。

二、簡單值

  • 使用 let 聲明常數，使用 var 聲明變數。例如：
  • let myConstant = 42
  • var myVariable = 42
  • 常數的值不需要在編譯時知道，但必須且只能賦值一次。
  • 變數可以多次賦值。
  • 可以讓編譯器根據初始值推斷變數或常數的類型，也可以明確指定類型。例如：
  • let implicitInteger = 70 // 推斷為 Int
  • let explicitDouble: Double = 70 // 明確指定為 Double
  • 值永遠不會被隱式轉換為其他類型，需要明確轉換。例如：
  • let width = 94
  • let widthLabel = “The width is “ + String(width)
  • 可以使用字符串插值將值包含在字符串中。例如：
  • let apples = 3
  • let appleSummary = “I have \(apples) apples.”
  • 使用三個雙引號 (“””) 表示多行字符串。

三、集合類型

  • 使用方括號 ([]) 創建数组和字典。
  • 可以通過索引或鍵訪問数组和字典中的元素。
  • 数组會自動增長。
  • 可以使用方括號創建空数组和字典，但需要明確指定類型。例如：
  • let emptyArray: [String] = []
  • let emptyDictionary: [String: Float] = [:]

四、控制流

  • 使用 if 和 switch 進行條件判斷。
  • 使用 for-in 迭代集合類型，例如：
  • for score in individualScores {
  • // …
  • }
  • 使用 while 和 repeat-while 進行循環。
  • if 語句中的條件必須是布林表达式。
  • 可以使用 if 和 let 來處理可能缺失的值（可選值）。例如：
  • if let name = optionalName {
  • greeting = “Hello, \(name)”
  • }
  • 使用 ?? 操作符提供默認值，以處理可選值。
  • switch 支持任何类型的數據和各種比較操作。
  • 在 switch 語句中，程式碼執行完匹配的 case 後會退出 switch 語句，不會繼續執行下一個 case。
  • 可以使用 ..< 和 … 創建區間，用於迭代或其他操作。

五、函數和閉包

  • 使用 func 聲明函數。例如：
  • func greet(person: String, day: String) -> String {
  • return “Hello \(person), today is \(day).”
  • }
  • 使用 -> 分隔参数列表和返回类型。
  • 可以使用自定義参数标签或使用 _ 忽略参数标签。
  • 使用元組返回多個值。
  • 函數可以嵌套，嵌套函數可以訪問外部函數中聲明的變量。
  • 函數是一等公民，可以作為參數傳遞給其他函數，也可以作為返回值返回。
  • 閉包是可調用的程式碼塊，可以使用 in 關鍵字分隔参数列表/返回類型和函數體。
  • 可以使用簡寫語法來編寫閉包。

六、對象和類別

  • 使用 class 創建類別。例如：
  • class Shape {
  • // …
  • }
  • 類別的屬性和方法聲明方式與常數和變數類似，但位於類別的上下文中。
  • 使用 init 創建初始化器，使用 deinit 创建析構函數。
  • 使用 self 區分類別的屬性和初始化器的参数。
  • 子類別在類別名稱後面使用冒號，繼承父類別。
  • 使用 override 關鍵字標記覆蓋父類別方法的子類別方法。
  • 屬性可以有 getter 和 setter。
  • 可以使用 willSet 和 didSet 監控屬性值的变化。
  • 使用 ? 操作符鏈式調用可選值的方法和屬性。

七、枚舉和結構

  • 使用 enum 創建枚舉。例如：
  • enum Rank: Int {
  • // …
  • }
  • 枚舉可以像類別一樣具有方法。
  • 可以為枚舉成員指定原始值，也可以不指定。
  • 可以使用 rawValue 屬性訪問枚舉成員的原始值。
  • 枚舉成員可以有關聯值。
  • 使用 struct 創建結構。
  • 結構和類別的行為類似，但結構是值類型，而類別是引用類型。

八、並發

  • 使用 async 標記異步函數。
  • 使用 await 調用異步函數。
  • 使用 async let 并行調用多個異步函數。
  • 使用 Task 從同步代码中調用異步函數。
  • 使用任務組 (task group) 組織并发代碼。
  • actor 類似於類別，但可以確保在同一個 actor 实例上，不同的異步函數可以安全地并发執行。

九、協議和擴展

  • 使用 protocol 聲明協議。例如：
  • protocol ExampleProtocol {
  • // …
  • }
  • 類別、枚舉和結構都可以遵循協議。
  • 使用 extension 為現有類型添加功能，例如新的方法和計算屬性。
  • 可以使用擴展為類型添加協議一致性。
  • 可以使用協議名稱作為類型，例如創建一個包含不同類型但都遵循同一個協議的對象集合。

十、錯誤處理

  • 使用遵循 Error 協議的類型表示錯誤。
  • 使用 throw 拋出錯誤，使用 throws 標記可以拋出錯誤的函數。
  • 使用 do-catch 處理錯誤。
  • 可以使用多個 catch 塊處理特定錯誤。
  • 可以使用 try? 將結果轉換為可選值，如果拋出錯誤則為 nil。
  • 使用 defer 编写在函數返回前執行的程式碼，無論是否拋出錯誤都會執行。

十一、泛型

  • 使用尖括號 (<>) 中的名称创建泛型函數或類型。例如：
  • func makeArray<Item>(repeating item: Item, numberOfTimes: Int) -> [Item] {
  • // …
  • }
  • 可以使用 where 子句指定泛型類型的約束。
