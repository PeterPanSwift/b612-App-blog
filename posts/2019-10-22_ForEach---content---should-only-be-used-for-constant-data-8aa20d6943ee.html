ForEach(_:content:) should only be used for constant data

利用 SwiftUI 的 ForEach，我們可以快速產生多個 UI 元件，不過當 ForEach 搭配 range 型別的參數時，它的數量只能是固定的，否則會產生錯誤，例如以下例子。

struct ContentView: View {
    
    @State private var count = 3
    
    var body: some View {
        NavigationView {
            List {
                ForEach(0 ..< count) { index in
                    Text("一天一點愛戀")
                }
            }
            .toolbar(content: {
                ToolbarItem {
                    Button {
                        count += 1
                    } label: {
                        Image(systemName: "plus.circle.fill")
                    }
                }
            })
            
        }
    }
}

當我們點選右上方的 +，增加 List 裡 Text 的數量時，發現它完全不聽話，畫面依然只有一開始的 3 個 Text。

此時 Console 顯示以下錯誤訊息。

ForEach<Range<Int>, Int, Text> count (4) != its initial count (3). `ForEach(_:content:)` should only be used for *constant* data. Instead conform data to `Identifiable` or use `ForEach(_:id:content:)` and provide an explicit `id`!

若我們改搭配 array，點選 + 時增加 array 的數量，還是會有一樣的問題。

struct ContentView: View {
    
    @State private var loves = ["一天1點愛戀", "一天2點愛戀", "一天3點愛戀"]
    
    var body: some View {
        NavigationView {
            List {
                ForEach(loves.indices) { index in
                    Text(loves[index])
                }
            }
            .toolbar(content: {
                ToolbarItem {
                    Button {
                        loves.append("一天\(loves.count+1)點愛戀")
                    } label: {
                        Image(systemName: "plus.circle.fill")
                    }
                }
            })
            
        }
    }
}

不管我們按了幾次 +，畫面還是只有一開始的 3 個 Text。

解法其實就藏在 Console 的錯誤訊息裡，讓我們再仔細看看它。

它告訴我們搭配 range 的 ForEach 只適合用在數量固定的資料。若我們想要資料可以增減，我們應該使用搭配參數 id 或使用資料遵從 protocol Identifiable 的 ForEach，因為它必須能區分 array 裡的成員，到時增減資料時才知道如何更新畫面，比方移除資料時只須移除資料對應的 view，而不用重新生成每個元件。

ForEach<Range<Int>, Int, Text> count (4) != its initial count (3). `ForEach(_:content:)` should only be used for *constant* data. Instead conform data to `Identifiable` or use `ForEach(_:id:content:)` and provide an explicit `id`!

接下來就讓我們分別試試這兩種解法吧。

  • 方法 1: 搭配參數 id。(此方法可能產生特別的問題，尤其牽扯到動畫時。)
  • 方法 2: 資料遵從 protocol Identifiable。

方法 1: 搭配參數 id

以 array 裡的字串當 id。

List {
    ForEach(loves, id: \.self) { love in
         Text(love)
    }
}

值得注意的，此方法可能產生特別的問題，尤其牽扯到動畫時。

使用 array 的 index 當 ForEach id 的特別問題
當我們用 SwiftUI 的 ForEach 顯示 array 的內容時，通常需要指定區分成員的 id。medium.com

方法 2: 資料遵從 protocol Identifiable

由於 String 是已經發明的型別，所以我們利用 extension 讓它遵從 protocol Identifiable。

extension String: Identifiable {
    public var id: String {
        self
    }
    
}

此時生成 ForEach 不須再傳入參數 id，因為它可由資料本身的 id 屬性區分。

List {
   ForEach(loves) { love in
      Text(love)
   }
}
