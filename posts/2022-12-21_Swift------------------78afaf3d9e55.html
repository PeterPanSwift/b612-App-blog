Swift 父類別的變數可以儲存子類別的物件

Swift 對型別有著嚴格的檢查，因此當以下的常數 color 宣告型別為 UIColor 時，它只能儲存型別 UIColor 的資料。

import UIKit

let color: UIColor = UIColor(red: 1, green: 0, blue: 0, alpha: 1)

若我們硬塞型別 UIView 的資料給它，將產生紅色錯誤，顯示 Cannot convert value of type 'UIView' to specified type 'UIColor'。

不過以下寫法卻是可以的，view 的型別宣告為 UIView，我們卻可以存入型別 UILabel 的資料。

import UIKit

let view: UIView = UILabel()

這是因為當變數的型別是父類別時，它也可以儲存型別為子類別的資料。以下我們以可愛的 Baby 類別說明。

class Baby {
    var age = 1
    
    func sleep() {
        print("睡到自然醒")
    }
}

class SuperBaby: Baby {
    var magic = "隱形"
}

class SmartBaby: Baby {
    var programmingLanguage = "swift"
}

var cuteBaby: Baby = SuperBaby()
cuteBaby.sleep()
cuteBaby.age = 2
cuteBaby = SmartBaby()
cuteBaby.sleep()
cuteBaby.age = 2

當 cuteBaby 型別為 Baby 時，它可以儲存子類別 SuperBaby 生成的資料。因此程式會變得比較彈性，cuteBaby 可以存 SuperBaby，也可以存 SmartBaby。

以白話來說，SuperBaby 和 SmartBaby 都繼承 Baby，所以 SuperBaby 和 SmartBaby 也算是 Baby，因此這樣寫是 ok 的。

我們也可以這樣理解。當 cuteBaby 型別為 Baby 時，表示它必須可以存取 Baby 的 property & method。而 SuperBaby 和 SmartBaby 繼承 Baby，它將得到來自 Baby 的 property & method，因此 cuteBaby 可以順利地存取 sleep & age。

值得注意的，雖然 cuteBaby 存著 SuperBaby，但是它並不能存取 SuperBaby 的 property & method。它的型別是 Baby，因此它只能存取 Baby 裡定義的 property & method。

了解以上概念後，我們再看一個 function 參數的例子。addSubview 的參數型別是 UIView，因此我們可傳入 UIView 或繼承 UIView 的型別資料。如此 addSubView 將變得更方便，可以加入繼承 UIView 的 label、image view 和 button，但還是有一些限制，比方傳入字串會有錯誤，因為字串沒有繼承 UIView。

因此以下程式傳入 label 是 ok 的，label 將順利地加到 view 的身上。

let label = UILabel()
let view = UIView()
view.addSubview(label)

那如果反過來，變數的型別是子類別，它可以儲存型別為父類別的資料嗎 ?

很遺憾的，不行 !

當 cuteBaby 型別為 SuperBaby 時，表示它必須可以存取 SuperBaby 的 magic。但是 Baby( ) 生成的寶寶只是普通的寶寶，他完全不會魔法呀，因此這樣的寫法是不行的。
