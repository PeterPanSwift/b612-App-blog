SwiftUI modifier 太多造成的 Attributes inspector 編輯問題

SwiftUI 提供豐富的 modifier 幫我們客製元件的樣式，不過當 modifier 加太多時，將造成小小的 Attributes inspector 編輯問題，比方我們看看以下例子。

struct ContentView: View {
    var body: some View {
        Image("peter")
            .resizable(resizingMode: .stretch)
            .aspectRatio(contentMode: .fill)
            .frame(width: 300.0, height: 300.0)
            .clipped()
            .background(Color.blue)
            .cornerRadius(40.0)
    }
}

此時 Image 的 Attributes inspector 如下。

我們可以順利地編輯，也可以從 Add Modifier 視窗加入新的 modifier。

接著我們再多加一個 modifier rotationEffect 試試。

struct ContentView: View {
    var body: some View {
        Image("peter")
            .resizable(resizingMode: .stretch)
            .aspectRatio(contentMode: .fill)
            .frame(width: 300.0, height: 300.0)
            .clipped()
            .background(Color.blue)
            .cornerRadius(40.0)
            .rotationEffect(.degrees(20))
    }
}

此時悲慘的事發生了 ! 因為畫面太長，所以 Attributes inspector 無法完整顯示，上方的圖片名稱跟下方的 Add Modifier 框框都看不到了。雖然 Attributes inspector 可以上下捲動，但是目前捲動功能有點問題，所以無法順序捲動露出被切除的內容。

當然熟練程式後，我們習慣直接從程式輸入 modifier，不過還是希望未來新版的 Xcode 能夠修正這個小小的 bug。
