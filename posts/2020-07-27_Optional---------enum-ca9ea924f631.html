Optional 的真面目原來是 enum

撰寫 Swift 程式時，我們時常用到 optional，為了它使用 ? & !。現在是時候揭開它的真面目了。

Optional 由 enum 定義

相關的程式片段如下:

@frozen public enum Optional<Wrapped> : ~Copyable where Wrapped : ~Copyable {

    /// The absence of a value.
    ///
    /// In code, the absence of a value is typically written using the `nil`
    /// literal rather than the explicit `.none` enumeration case.
    case none

    /// The presence of a value, stored as `Wrapped`.
    case some(Wrapped)
Optional | Apple Developer Documentation
A type that represents either a wrapped value or the absence of a value.developer.apple.com

enum 裡分兩個 case，none 表示沒東西，some(Wrapped) 表示有東西，可從 Wrapped 型別的參數取得東西。由於 Wrapped 是以 generic 定義的型別代號，因此 Optional 可以儲存任何型別的資料，存在 case some Wrapped 型別的參數裡。

ps: 關於 enum case 搭配 (參數) 的寫法，可參考以下連結 associated value 的說明。

enum 儲存相關聯資料的 associated value
Swift 的 enum 有個特別的 associated value 語法，它將讓 enum 的 case 具有儲存相關聯資料的特異功能。接下來讓我們透過實際的例子好好認識它吧。medium.com

利用 Switch 判斷 optional 是否有值

了解 Optional 是 enum 後，我們也可用 switch 比對 enum 的方法判斷 optional 是否有值。

var age: Int? = 18
switch age {
case .none:
    print("nil")
case let .some(value):
    print("永遠\(value)歲")
}

以 Optional( ) 生成 optional

var age = Optional(18)
age = Optional.some(18)
age = Optional.none
age = .some(18)
age = .none
