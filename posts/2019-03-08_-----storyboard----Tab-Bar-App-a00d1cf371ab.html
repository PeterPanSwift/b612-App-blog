利用多個 storyboard 製作 Tab Bar App

使用 iOS App 時，我們時常看到利用 tab bar 切換頁面的 App，比方 Music App & Clock App。

感謝 iOS SDK 提供的 tab bar controller，我們可以快速打造 Tab Bar App。如果剛好 App 是多人開發，每個人負責其中一兩個 tab 的頁面，不妨考慮將每個 tab 的畫面分配給一個 storyboard，使用多個 storyboard 製作 Tab Bar App，降低大家合作時 git 版本管理發生衝突的機率。

ps: 關於多個 storyboard 裡畫面串接的方法，相關說明可參考以下連結。

利用多個 storyboard 和 storyboard reference 製作 App 畫面
Xcode 的 Storyboard 可以讓我們快速製作 App 畫面，但是當頁面一多時，將如下圖所示，一個 storyboard 塞了滿滿的畫面，產生以下問題:medium.com

假設我們想製作包含 2 個 tab 的推理小說 App，分別顯示彼得潘的最愛，伊坂幸太郎和乙一的作品，接下來就讓我們一步步示範如何利用多個 storyboard 製作 Tab Bar App 吧。

設計 App 架構

此 App 將有 3 個 storyboard，分別是負責伊坂幸太郎 tab 頁面的伊坂幸太郎.storyboard，負責乙一 tab 頁面的乙一.storyboard，以及利用 tab bar controller 串接其它 tab 頁面的 Main.storyboard。

建立伊坂幸太郎.storyboard，設計伊坂幸太郎的 tab 頁面

  • 建立伊坂幸太郎.storyboard。

從 Xcode 的 menu 點選 File > New > File from Template。

選擇 iOS 下的 Storyboard。

取名伊坂幸太郎。

  • 在伊坂幸太郎.storyboard 加入 Navigation Controller，設計伊坂幸太郎的頁面。
  • 在 Navigation Controller 加入 tab bar item，設定 tab 的文字和 icon。
  • 設定 storyboard 裡開頭箭頭指到的 controller。

勾選 Navigation Controller 的 Is Initial View Controller。

建立乙一.storyboard，設計乙一的 tab 頁面

以同樣的方法建立乙一.storyboard 並設計它的畫面。

在 Main.storyboard 利用 tab bar controller 串接其它 tab 的頁面

  • 刪除原本的 controller，加入 Tab Bar Controller。
  • 點選 Tab Bar Controller，勾選它的 Is Initial View Controller。
  • 刪除 Tab Bar Controller 原本連接的 controller，讓它成為孤單一人。
  • 加入對應伊坂幸太郎.storyboard 的 storyboard reference。

將 storyboard reference 的 Storyboard 設為伊坂幸太郎。(從下拉選單選取)

  • 加入對應乙一.storyboard 的 storyboard reference。
  • 連接 tab bar controller & storyboard reference，記得 segue 要選擇 Relationship Segue 下的 view controllers。

現在已經大功告成，只要執行 App，即可看到 2 個 tab 的推理小說 App。

查看 & 調整 tab 的順序

也許有朋友覺得不滿意，因為在 Main.storyboard 看不出 tab 的 icon 和文字，無法分辨 tab，之後該如何調整 tab 順序呢 ?

別擔心，其實可以看出順序，只要用你的真心 ! 從 tab bar controller 的 connections inspector，我們將可看到 view controllers 的連線，由上而下排列的順序即為 tab 的順序。

  • 點選 tab bar controller。
  • 從 connections inspector 查看 tab 順序。

由上而下的順序即為 tab 的順序。
