自動幫遵從 Equatable 的 struct & enum 定義 == function — Swift 4.1

在 Swift 的世界，並不是任何東西都可以用 == 判斷相等，比方以下我們自訂的 Man 就不行。

struct Man {
   var name: String
   var height: Double
}

let man1 = Man(name: "彼得潘", height: 180.5)
let man2 = Man(name: "金城武", height: 180.5)
if man1 == man2 {

}

想要用 ==比較相等，其實也沒這麼難，只要讓型別遵從 protocol Equatable，定義 static function ==，例如以下例子:

struct Man: Equatable {
   var name: String
   var height: Double

   static func == (lhs: Man, rhs: Man) -> Bool {
      return lhs.height == rhs.height
   }
}

在 func == 裡我們判斷身高，當身高一樣時當成兩個人相等，因此以下的例子，身高都是 180.5 的彼得潘 & 金城武是同一類人。

let man1 = Man(name: "彼得潘", height: 180.5)
let man2 = Man(name: "金城武", height: 180.5)
if man1 == man2 {
   print("\(man1.name) & \(man2.name)是同一類人")
} else {
   print("\(man1.name) & \(man2.name)是不同類人")
}

不過如果想判斷每個 property 都一樣才算相等，而且型別的 property 又有很多時，定義 func == 時可是會讓人寫到快瘋掉，例如以下例子:

struct Man: Equatable {
   var name: String
   var height: Double
   var weight: Double
   var age: Int

   static func == (lhs: Man, rhs: Man) -> Bool {
      return lhs.height == rhs.height && lhs.name == rhs.name && lhs.weight == rhs.weight && lhs.age == rhs.age
   }
}

例子裡的 Man 有 4 個 property，假設有 10 個的話，我們在 func == 裡得比較更多的 property。這樣子的程式不太需要經過大腦思考就能寫出，但是需要無與倫比的耐心，一個一個比。

Swift 明白我們什麼都有，就是沒有耐心。所以好心的它在 4.1 加了一個貼心的功能，只要 struct 遵從 protocol Equatable，它就會在背後幫我們定義比對每一個 property 的 fuction ==，只有全部相等時才會回傳 true。

struct Man: Equatable {
   var name: String
   var height: Double
   var weight: Double
   var age: Int
}

不過只有當 Man 的 property 都能比較相等時， Swift 才能自動幫我們定義 function ==。如以下例子，由於它無法比較型別 Woman 的 girlFriend，因此我們必須自己定義 function ==。

如果是 enum 的話 ，當有 associated value 時，比對也很麻煩，因為要一一比對每個 value，例如以下例子:

enum Status {
   case onTime
   case delayed(minute: Int, second: Int)

   static func == (lhs: Status, rhs: Status) -> Bool {
      switch (lhs, rhs) {
      case (.onTime, .onTime):
         return true
      case (.delayed(let lhsMinute, let lhsSecond), .delayed(let rhsMinute, let rhsSecond)):
         return lhsMinute == rhsMinute && lhsSecond == rhsSecond
      default:
         return false
      }
   }
}

一樣的，只要讓 enum 遵從 protocol Equatable，它就會在背後幫我們定義比對每一個 case 的 fuction ==，包含比對 associated value，只有全部相等時才會回傳 true。

enum Status: Equatable {
   case onTime
   case delayed(minute: Int, second: Int)
}

let status1 = Status.delayed(minute: 60, second: 10)
let status2 = Status.delayed(minute: 60, second: 10)
if status1 == status2 {
   print("真巧，我們的遲到時間一樣")
} else {
   print("可惜，我們遲到的時間不一樣")
}

那如果是 class 呢 ? 很遺憾的，他沒有自動定義 == 的功能。

然而大部分的時候，我們想比的會是 class 生成的物件是否為同一個物件，我們想比的是物件的記憶體是否一樣，而不是它們的 property 是否相等。

由於 class NSObject 遵從 protocol Equatable，並且定義了物件的記憶體一樣時 == 回傳 true，所以我們想讓自訂的 class 型別生成的物件也能比較是否為同一個物件的話，只要繼承 NSObject 即可。

extension NSObject : Equatable, Hashable

例子

class Man: NSObject {
   var name: String = "金城武"
   var height: Double = 180.5
   var weight: Double = 70
   var age: Int = 35
}

let man1 = Man()
let man2 = man1
if man1 == man2 {
   print("man1 & man2 是同一個物件")
} else {
   print("man1 & man2 是不同的物件")
}

答案: man1 & man2 是同一個物件。

let man1 = Man()
let man2 = Man()
if man1 == man2 {
   print("man1 & man2 是同一個物件")
} else {
   print("man1 & man2 是不同的物件")
}

答案: man1 & man2 是不同的物件
