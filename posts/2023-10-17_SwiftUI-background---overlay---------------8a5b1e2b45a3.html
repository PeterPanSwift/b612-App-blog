SwiftUI background / overlay ( ) 和 { } 的差異

開發 SwiftUI App 時，background modifier 可以幫元件加上背景，overlay 可以將東西覆蓋在元件上。不過當我們輸入 background 或 overlay 時，Xcode 的提示選單會出現許多選項，主要分為以下兩種。

ps: 以下我們以 background 舉例說明，overlay 也是一樣的做法。

  • 參數型別是 ShapeStyle，輸入時要在 ( ) 裡傳入資料。
  • 參數型別是 ( ) → View，輸入時傳入 closure，因此要在 { } 裡傳入要當背景的 view。

所以我們到底要用哪一個呢 ?

  • 設定用顏色、漸層、圖片填滿形狀的背景。
利用 SwiftUI 的 background(_:in:fillStyle:) 設定背景，用顏色、漸層、圖片填滿形狀
開發 iOS App 時，我們時常會替元件加上某種樣式的背景，比方背景為黃色圓角的按鈕。從 iOS 15 開始，利用 modifier background(_:in:fillStyle:)…medium.com
  • 使用 ( )。

想用 Color、Material、HierarchicalShapeStyle、漸層當背景。

  • 使用 { }。

想用某個 view 當背景，而且它的型別沒有遵從 ShapeStyle。

使用 ( )

當我們想用 Color、Material、HierarchicalShapeStyle、漸層當背景時，建議用 ( )。

  • 顏色當背景。
struct ContentView: View {
    var body: some View {
        Image(.peter)
            .background(.red)
           
    }
}
  • 漸層當背景。
struct ContentView: View {
    var body: some View {
        Image(.peter)
            .background(.red.gradient)
           
    }
}
  • Material 當背景。
struct ContentView: View {
    var body: some View {
        ZStack {
            Color.red
            Image(.peter)
                .background(.ultraThinMaterial)
        }
        .ignoresSafeArea()
    }
}

使用 { }

當我們想用某個 view 當背景，而且它的型別沒有遵從 ShapeStyle 時，建議使用 { }。

  • 圖片當背景。
struct ContentView: View {
    var body: some View {
        Image(.peter)
            .background {
                Image(systemName: "star")
                    .resizable()
                    .scaledToFit()
                    .foregroundStyle(.orange)
            }
    }
}
  • 形狀當背景
struct ContentView: View {
    var body: some View {
        Image(.peter)
            .background {
                Circle()
            }
    }
}

Deprecated 警告

有的寫法雖然跑起來沒問題，但卻會有 Deprecated 的警告。

struct ContentView: View {
    var body: some View {
        Image(.peter)
            .background(Circle())
    }
}

以上程式在 background( ) 裡傳入 Circle()，因為它沒有遵從 ShapeStyle，因此呼叫的是下圖 deprecated 的 background。

型別不合的紅色錯誤

struct ContentView: View {
    var body: some View {
        Image(.peter)
            .background {
                Color.red.gradient
            }
    }
}

Color.red.gradient 的型別是 AnyGradient，它沒有遵從 View，因此會有紅色錯誤，因為 background { } 裡一定要生成 view。
