調整 Installed 隱藏元件，方便我們在 Interface Builder 設計畫面

有些時候 view controller 裡會有多層 view，造成我們在 Interface Builder 設計畫面時不太容易，比方 Apple 範例的心理測驗 App，以下三種題型是在同一個 controller 設計。

view controller 透過控制三個 stack view 的 hidden，決定顯示的題型。

此時我們會遇到以下問題:

  • 底層的 stack view 被其它 stack view 蓋到，因此我們無法直接從畫布點選。
  • 三個 stack view 重疊的畫面看起來很亂，不方便看到單一題型顯示的樣子。

以上問題可透過取消元件的 Installed 勾選解決。當元件的 Installed 沒有勾選時，它將不會被產生加到畫面上。因此當我們想專心設計某區塊時，可先取消其它元件的 Installed，暫時隱藏其它區塊，方便我們在 Interface Builder 設計畫面。

比方我們想調整剛剛範例的 Single Stack View，因此先將 Multiple Stack View & Ranged Stack View 的 Installed 取消勾選。

此時畫面變得乾淨許多，只顯示 Single Stack View 的四個 button，而且我們也能直接從畫布點選 button。

設計完 Single Stack View 後，記得要把 Multiple Stack View & Ranged Stack View 的 Installed 勾選，不然最後 App 執行時將完全看不到它們。
