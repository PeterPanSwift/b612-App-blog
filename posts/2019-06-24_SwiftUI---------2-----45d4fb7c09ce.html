SwiftUI 快速取代元件的 2 個方法

開發 iOS App 時，有時我們想將原本的元件換成另一種元件，比方將 Text 換成 Image。除了笨笨地先將原本元件刪除，然後加入新的元件，其實有另外 2 個更快的方法。

接下來就讓我們假設原本的畫面顯示文字 Hello World!，我們想將它換成可愛的彼得潘圖片。

struct ContentView : View {
   var body: some View {
      Text("Hello World!")
   }
}

方法 1: 雙擊大法

  • 從 Preview 畫面點選要被取代的元件。

點選 Text 元件。

  • 從 Library 找到要加到畫面上的元件後雙擊觸控板。

雙擊可愛的彼得潘圖片。

此時 Image 將取代 Text，顯示可愛的彼得潘。

方法 2: 按住 cmd 鍵

當我們從 Library 視窗拖曳元件到 Preview 畫面裡的某個元件附近時，Xcode 預設會認為我們希望這兩個元件親密地靠在一起，比方下圖拖曳圖片到 Text 文字時，圖片和文字將一起包在 VStack 裡。

Xcode 的提示文字顯示 Add Image to a new Vertical Stack along with existing Text。

不過若我們在拖曳元件時搭配 cmd 鍵，它將變成取代的效果，步驟如下:

1 將圖片拖曳到文字上。
2 按住 cmd 鍵後將手指離開觸控板。

此時 Xcode 的提示文字變成 Replace Text with Image

文字果然被圖片取代了，程式碼從 Text 變成 Image。

struct PhotoView : View {
   var body: some View { 
      Image("peter")
   }
}
