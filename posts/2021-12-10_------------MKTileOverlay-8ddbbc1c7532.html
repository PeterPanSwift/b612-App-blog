客製地圖模樣的地圖圖層 MKTileOverlay

Apple Map 很漂亮，不過看久了也是會膩，接下來讓我們試試改造它吧。我們將加入 MKTileOverlay，它是一塊加在地圖上的圖層，將讓地圖徹底變臉。

加入 Map View

連結 outlet

記得要 import MapKit。

@IBOutlet weak var mapView: MKMapView!

加入 MKTileOverlay

class ViewController: UIViewController {
    
    @IBOutlet weak var mapView: MKMapView!
    
    override func viewDidLoad() {
        super.viewDidLoad()
        // Do any additional setup after loading the view.
        
        let template = "http://tile.thunderforest.com/outdoors/{z}/{x}/{y}.png"
        let overlay = MKTileOverlay(urlTemplate: template)
        overlay.canReplaceMapContent = true
        mapView.addOverlay(overlay, level: .aboveLabels)
        mapView.delegate = self
    }
}

說明。

let template = "http://tile.thunderforest.com/outdoors/{z}/{x}/{y}.png"

overlay.canReplaceMapContent = true

canReplaceMapContent 設為 true 將讓加入的圖層蓋住原本的地圖內容。

mapView.addOverlay(overlay, level: .aboveLabels)

呼叫 addOverlay 加入 MKTileOverlay。

mapView.delegate = self

地圖顯示 overlay 時必須呼叫 delegate 的 function mapView(_:rendererFor:)。因此我們將 view controller 設為地圖的 delegate，之後才能定義 function mapView(_:rendererFor:)。

ps: 也可以從 Interface Builder 設定地圖的 delegate 是 view controller。

定義 MKMapViewDelegate 的 mapView(_:rendererFor:)

extension ViewController: MKMapViewDelegate {
    func mapView(_ mapView: MKMapView, rendererFor overlay: MKOverlay) -> MKOverlayRenderer {
        if overlay is MKTileOverlay {
            return MKTileOverlayRenderer(overlay: overlay)
        } else {
            return MKTileOverlayRenderer()
        }
    }
}

地圖顯示 overlay 時會呼叫 delegate 的 function mapView(_:rendererFor:)，它回傳的 MKOverlayRenderer 控制著地圖上圖層的繪製。在此我們檢查 overlay 的型別，發現它是 MKTileOverlay 時回傳包含 overlay 的 MKOverlayRenderer。

設定 ATS

由於剛剛 tile 圖片的網址是 http，不是 https，因此我們必須另外設定 ATS，將 App Transport Security Settings > Allow Arbitrary Loads 設為 YES。
