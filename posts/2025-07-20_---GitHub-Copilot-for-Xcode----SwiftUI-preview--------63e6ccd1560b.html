使用 GitHub Copilot for Xcode 生成 SwiftUI preview 需要的假資料

SwiftUI preview 顯示畫面時常常需要搭配一些假資料，以下我們示範使用 GitHub Copilot for Xcode 生成 preview 需要的假資料。

ps: GitHub Copilot for Xcode 的安裝方法可參考以下連結。

使用 GitHub Copilot for Xcode 幫忙寫程式
最近聽說 GitHub 推出了厲害的 AI 程式碼工具 Copilot，它可以提供不錯的程式碼建議，大幅加快程式的開發速度。medium.com

以下我們用顯示可愛動物的 AnimalRow 說明。

import SwiftUI

struct AnimalRow: View {
    let name: String
    let emoji: String
    let description: String
    
    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            HStack {
                Text(emoji)
                    .font(.largeTitle)
                Text(name)
                    .font(.title)
                    .bold()
            }
            Text(description)
                .font(.body)
        }
        .padding()
    }
}

#Preview {
    AnimalRow()
}

我們加了三個 property，因此產生錯誤 Missing arguments。

從 Quick Actions 點選 GitHub Copilot > Open Chat。

在 GitHub Copilot 的聊天框框選擇 Agent mode，輸入修正 AnimalRow 的 Missing Arguments。

AI 成功輸入叢林之王獅子的相關資料。

我們也可以請 AI 生成指定的資料，以下我們請 AI 生成企鵝的相關資料，description 包含 100 字的中文。

當 AnimalRow 顯示的資料型別另外定義在 Animal.swift 時，AI 也能聰明地在 Preview 生成假資料。

比方以下分別為 Animal.swift & AnimalRow.swift 的內容。

struct Animal {
    let name: String
    let emoji: String
    let description: String
}
import SwiftUI

struct AnimalRow: View {
    let animal: Animal
    
    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            HStack {
                Text(animal.emoji)
                    .font(.largeTitle)
                Text(animal.name)
                    .font(.title)
                    .bold()
            }
            Text(animal.description)
                .font(.body)
        }
        .padding()
    }
}

#Preview {
    AnimalRow()
}

AI 聰明地找出相關的 Animal.swift，生成企鵝的相關資料。

我們也常在資料的型別裡以 static 定義資料，以下我們在 Animal 生成 preview 需要的假資料，輸入 create sample data，企鵝，100 字的中文 description。

請 AI 生成包含多筆資料的 array 也不是問題，以下我們修改 struct SampleData，輸入宣告 animals 儲存 5 種動物，型別為 [Animal]。

AI 成功生成 animals，儲存企鵝、獅子、大象、熊貓和老虎。
