API 代表網路上提供的服務

當 API 跟網路有關時，它代表網路上提供的服務，讓我們以下圖說明。

圖片裡有 2 個主角，一個是 client，一個是 server。通常 client 代表我們的手機或電腦，server 則是某台遠方的電腦，它將為我們提供某種服務，比方告訴我們本週最新的電影。

client 跟 server 都是沒有生命的機器，它們如何溝通呢 ? 其實跟我們人類很像，我們現在都是透過 LINE / FB 跟朋友聊天吵架，client 跟 server 也是透過網路溝通。client 送出 request 封包給 server，然後 server 再回傳 response 封包給 client，至於 request 跟 response 的格式跟內容，則由提供服務的 server 訂定，App 在撰寫跟 server 溝通的程式時，要照著 server 訂定的格式，這樣彼此才能理解對方說的話，才不會雞同鴨講。

API 主要的三種資訊

讓我們看一個實際的 API 例子，News API 提供抓取新聞資料的服務，以下為相關的文件說明。

Top headlines - Documentation - News API
This endpoint provides live top and breaking headlines for a country, specific category in a country, single source, or…newsapi.org

文件裡主要說明三種重要的資訊:

  • API 的網址。
  • request 的參數和資料上傳的欄位。
  • response 回傳的內容。

根據文件描述的內容，我們即可撰寫出 iOS App 跟 server 順利溝通的程式，發送 request 跟 server 說我們想取得熱門的新聞，然後 server 回傳 response 告訴我們有哪些熱門的新聞。

  • 網址開頭是 https://newsapi.org/v2/top-headlines。
  • request 參數有 country，category，sources 等。
  • 回傳資料的 articles 是新聞文章，它的內容有 source，author，title 等。

API 可以為我們提供各種服務

API 是一種網路上提供的服務，而服務的範圍很廣，所以它可以提供各式各樣的服務，不只是常見的抓取資料跟上傳圖片，比方轉檔，去背，翻譯，照片處理都難不倒它。

我們可以把 server 想成無所不能的工具人或神燈巨人，透過 API 我們可以傳送 request 拜託 server 做事。server 如果願意幫我們，就會回傳實現我們請求的 response，比方將上傳的照片去背後回傳。不過它也是可能不幫我們的，畢竟我們也不是她的誰。

下圖的 remove.bg API 可以幫我們去背，但是去背很辛苦的，所以它一個月只願免費幫我們去背 50 張圖片，想去背更多圖片請刷卡付錢 ～
