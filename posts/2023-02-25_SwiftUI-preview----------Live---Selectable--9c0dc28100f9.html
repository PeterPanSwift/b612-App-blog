方便預覽程式畫面和執行 App 的 SwiftUI preview

開發 SwiftUI App 時，我們可以有更快的產出，更早下班去約會，因為我們可以從 preview 預覽程式設計的畫面和執行 App。

以下我們一步步介紹 SwiftUI preview 的各種功能和應用。

preview 顯示的 iPhone 機型

Xcode preview 顯示的 iPhone 機型
Xcode 11 貼心地在右邊的 Canvas 頁面顯示 App 在 iPhone 上 preview 的模樣，方便我們從程式或 preview 設計 App 畫面。但是 iPhone & iPad 有那麼多種，它怎麼知道要顯示哪一種呢…medium.com

SwiftUI 的 preview 不見了? 請打開 Canvas

負責顯示預覽的區塊稱為 Canvas(畫布)，若是看不到 preview，可參考以下連結打開 Canvas。

SwiftUI 的 preview 不見了? 請打開 Canvas
使用 SwiftUI 開發 iOS App 有許多開心的事，其中彼得潘覺得最開心的，莫過於可以在 preview 看到並操作程式的預覽畫面。medium.com

Refresh Canvas

有時 canvas 會停止更新，顯示下圖的 Canvas paused。

此時我們可以重新啟動 canvas。

方法 1: 點選 Canvas paused 右邊的重新整理按鈕。

方法 2: 從 Quick Actions 點選 Refresh Canvas。(快速鍵 cmd + option + p)

縮放 preview 畫面的大小

縮放觸碰版或點選右下的四個按鈕。

四顆按鈕的功能如下。

  1. 縮小。
  2. 以 100% 顯示。
  3. Zoom to Fit，縮放至剛好能在 Canvas 顯示的大小。
  4. 放大。

preview 的 live mode

SwiftUI preview 預設是 live mode，方便我們修改程式後立即操作測試 App。

preview 左下的三角形被選取表示 preview 目前為 live mode。

比方輸入以下程式生成骰子畫面後，我們馬上能在 SwiftUI preview 點擊骰子改變點數。

struct ContentView: View {
    @State private var number = 1
    
    var body: some View {
        Image(systemName: "die.face.\(number).fill")
            .resizable()
            .scaledToFit()
            .onTapGesture {
                number = .random(in: 1...6)
            }
    }
}

preview 的 selectable mode

點選下圖的箭頭可讓 preview 切換到 Selectable mode。

在 Selectable mode 時 preview 無法執行 App，但是卻多了點選功能。點選 preview 的元件將突顯對應的程式，點選程式將在 preview 突顯對應的元件。

讓 preview 的畫面顯示 previewLayout 指定的大小

指定 SwiftUI preview 尺寸的 previewLayout
SwiftUI preview 預設會顯示整個 iPhone 尺寸的大小。因此以下的 MovieRow 雖然只是設計表格裡某部電影的畫面，preview 依然會傻傻地顯示整個 iPhone 大小。medium.com

#Preview macro 搭配 iOS 的版本問題

#Preview macro 搭配 iOS 的版本問題
透過 #Preview macro 我們可以方便地預覽 App 畫面，不過當專案的 Minimum Deployments 小於 iOS 17 時，在某些情況會遇到版本問題。medium.com
