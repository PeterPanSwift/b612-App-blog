Liquid Glass 的 Tab Bar 問題 — iOS 26.0 bug

最近有同學發現 iOS 26 Tab Bar 的 Liquid Glass 效果漂亮歸漂亮，但當頁面有上下捲動的頁面時，卻會在切換分頁時產生奇怪的 bug。

讓我們用以下 SwiftUI App 例子說明。(ps: UIKit App 也有一樣的問題)

App 由兩個 tab 分頁組成，星星分頁顯示數字 List，愛心分頁顯示可愛的愛心。

struct ContentView: View {
    var body: some View {
        TabView {
            Tab("star", systemImage: "star") {
                List {
                    ForEach(0..<100) { index in
                        Text(index.formatted())
                    }
                }
            }
            Tab("heart", systemImage: "heart") {
                ZStack {
                    Color.black
                        .ignoresSafeArea()
                    Image(systemName: "heart")
                        .resizable()
                        .scaledToFit()
                        .frame(width: 300, height: 300)
                        .foregroundStyle(.pink)
                }
            }
        }
    }
}

一開始啟動 App 時，一切看似正常，星星分頁和愛心分頁的 tab bar 都正常顯示成跟背景色搭配的透明效果。

然而當我們切回 star 頁面滑動一下再進入 heart 頁面時，tab bar 卻變成跟背景色不搭的灰色背景 !

不過別擔心，此問題在 iOS 26.1 測試是正常的，所以應該只是 26.0 的 bug。
