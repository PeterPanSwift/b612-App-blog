cell 連 segue 切換頁面時，判斷點選的 cell 傳資料到下一頁

開發 iOS App 時，我們時常在點選 table view cell / collection view cell 時切換到下一頁，顯示點選 cell 的詳細內容，就像下圖的 YouTube App，點選 Pentatonix 的 My Favorite Things 跳到下一頁播放好聽的 MV。

當 cell 連 segue 到 detail 頁面時，我們有許多方法可判斷點選的 cell，然後傳遞 cell 對應的資料到下一頁。以下我們以歌舞片電影為例說明各種不同的方法。

  • table view cell
  1. 透過 IBSegueAction 傳資料，判斷 table view 的 indexPathForSelectedRow。
  2. 透過 IBSegueAction 傳資料，判斷參數 sender 對應的 IndexPath。
  3. 透過 prepare 傳資料，判斷 table view 的 indexPathForSelectedRow。
  4. 透過 prepare 傳資料，判斷參數 sender 對應的 IndexPath。
  • collection view cell
  1. 透過 IBSegueAction 傳資料，判斷 collection view 的 indexPathsForSelectedItems?.first。
  2. 透過 IBSegueAction 傳資料，判斷參數 sender 對應的 IndexPath。
  3. 透過 prepare 傳資料，判斷 collection view 的 indexPathsForSelectedItems?.first。
  4. 透過 prepare 傳資料，判斷參數 sender 對應的 IndexPath。

定義歌舞片電影的 array

 let musicalFilms = ["媽媽咪呀", "歌劇魅影", "芝加哥", "真善美", "貓", "悲慘世界", "窈窕淑女", "萬花嬉春", "綠野仙蹤", "秋水伊人"]

table view cell

  • 透過 IBSegueAction 傳資料，判斷 table view 的 indexPathForSelectedRow。

從 indexPathForSelectedRow 判斷點選的是第幾個 cell。

@IBSegueAction func showMusicalFilm(_ coder: NSCoder) -> DetailViewController? {
    guard let row = tableView.indexPathForSelectedRow?.row else { 
        return nil 
    }
    return DetailViewController(coder: coder, musicalFilm: musicalFilms[row])
}
  • 透過 IBSegueAction 傳資料，判斷參數 sender 對應的 IndexPath。

從 cell 連 segue 時，參數 sender 就是 cell，因此我們可從 cell 取得它對應的 IndexPath。

@IBSegueAction func showMusicalFilm(_ coder: NSCoder, sender: Any?) -> DetailViewController? {
    guard let cell = sender as? UITableViewCell,
          let row = tableView.indexPath(for: cell)?.row else {
        return nil
    }
    return DetailViewController(coder: coder, musicalFilm: musicalFilms[row])
}
  • 透過 prepare 傳資料，判斷 table view 的 indexPathForSelectedRow。

從 indexPathForSelectedRow 判斷點選的是第幾個 cell。

override func prepare(for segue: UIStoryboardSegue, sender: Any?) {
    guard let detailViewController = segue.destination as? DetailViewController,
          let row = tableView.indexPathForSelectedRow?.row else { 
        return 
    }
    detailViewController.musicalFilm = musicalFilms[row]
}
  • 透過 prepare 傳資料，判斷參數 sender 對應的 IndexPath。

從 cell 連 segue 時，參數 sender 就是 cell，因此我們可從 cell 取得它對應的 IndexPath。

override func prepare(for segue: UIStoryboardSegue, sender: Any?) {
    guard let detailViewController = segue.destination as? DetailViewController,
          let cell = sender as? UITableViewCell,
          let row = tableView.indexPath(for: cell)?.row else { 
        return 
    }
    detailViewController.musicalFilm = musicalFilms[row]
}

collection view cell

  • 透過 IBSegueAction 傳資料，判斷 collection view 的 indexPathsForSelectedItems?.first。

從 indexPathsForSelectedItems?.first 判斷點選的是第幾個 cell。

@IBSegueAction func showMusicalFilm(_ coder: NSCoder) -> DetailViewController? {
    guard let item = collectionView.indexPathsForSelectedItems?.first?.item else {
        return nil
    }
    return DetailViewController(coder: coder, musicalFilm: musicalFilms[item])
