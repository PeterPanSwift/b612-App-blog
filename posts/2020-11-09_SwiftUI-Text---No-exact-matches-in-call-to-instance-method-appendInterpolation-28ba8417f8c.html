SwiftUI Text 的 No exact matches in call to instance method appendInterpolation

當我們在 SwiftUI 的 Text 裡顯示變數內容時，時常會使用 string interpolation 的 \() 語法。

struct ContentView: View {
    var body: some View {
        let name = "peter"
        let age = 3
        return Text("name:\(name) age:\(age)")
    }
}

當 \() 裡是 Int，Double，String，Date 等常用的基本型別資料時，這樣的寫法完全沒有問題。然而當 Text 的 \() 裡傳入較複雜的型別資料，比方我們自訂的 Baby 型別時，將產生紅色錯誤。

struct Baby {
    var name: String
    var age: Int
}

struct ContentView: View {
    var body: some View {
        let cuteBaby = Baby(name: "peter", age: 1)
        return Text("\(cuteBaby)")
    }
}

錯誤訊息顯示 No exact matches in call to instance method appendInterpolation。

解法1: 利用 init(verbatim:)

struct ContentView: View {
    var body: some View {
        let cuteBaby = Baby(name: "peter", age: 1)
        return Text(verbatim: "\(cuteBaby)")
    }
}

解法2: as String

struct ContentView: View {
    var body: some View {
        let cuteBaby = Baby(name: "peter", age: 1)
        return Text("\(cuteBaby)" as String)
    }
}
