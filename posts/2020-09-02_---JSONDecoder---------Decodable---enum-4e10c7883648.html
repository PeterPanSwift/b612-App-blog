利用 JSONDecoder 將資料變成遵從 Decodable 的 enum

當我們從 API 抓取 JSON 資料時，有些欄位的內容雖然是字串或數字，但卻很適合變成 enum，提高程式的可讀性跟安全性。

比方以下 Dcard API 的例子，JSON 裡的 gender 欄位內容有 M，F，D 三種可能，因此很適合以 enum 表達。

當我們利用 JSONDecoder 解析 JSON 時，以下程式可將 Dcard API 回傳資料的 gender 內容變成型別 enum Gender 的資料。

enum Gender: String, Decodable {
    case male = "M", female = "F", other = "D"
}
struct DcardData: Decodable {
    let title: String
    let excerpt: String
    let gender: Gender
}
URLSession.shared.dataTask(with: URL(string: "https://dcard.tw/_api/posts")!) { (data, response, error) in
    if let data = data,
       let dcards = try? JSONDecoder().decode([DcardData].self, from: data) {
        dump(dcards)
    } else {
        print("error")
    }
}.resume()

說明

我們定義 enum Gender 對應 gender 欄位的資料型別。由於 gender 的欄位內容為字串，因此我們在 Gender 後接 : String，讓它的 raw value 型別為 String。(ps: 若是資料是整數，enum 的 raw value 型別則為 Int )

此時我們定義以下 enum 對應 JSON 的 gender 欄位內容，case 的名稱對應 gender 的內容。

enum Gender: String, Decodable {
    case M, F, D
}

不過 case 名稱 M, F, D 不易看懂，而且 Swift 習慣 case 以小寫開頭，因此我們可自訂 case 名稱，然後設定 case 的 raw value 對應 gender 的內容。

enum Gender: String, Decodable {
    case male = "M", female = "F", other = "D"
}

結果

JSONDecoder 成功解析 JSON，而且還將原本是字串的 gender 內容變成 enum 型別了。
