Date 印出的時間是時區 +0，開發 App 時通常會用 DateFormatter 或 formatted 將 Date 變字串

開發 Swift 程式時，我們通常會用 Date 型別的資料表示時間，例如 Date( ) 可以生成現在時間的物件。

import Foundation

let now = Date()
print(now)

以上程式在 playground 執行時，let now = Date( ) 的右邊果然看到現在時間下午 2:24，但是 print 顯示的結果卻有點奇怪，它顯示的時間少了 8 小時，顯示早上 6:24。

難道是 print 壞掉了嗎? 不要錯怪它了，它沒有錯，它只是印出時區 +0 的時間，因此它的後面顯示 +0000。台灣的時區是 +8，因此它才會少了 8 個小時。

開發 App 時需要特別處理 print 的時區問題嗎 ? 其實是不用擔心的，因為 App 畫面上顯示的時間是字串，我們會從程式將 Date 變成字串，然後再將它顯示到畫面上。

將 Date 變成字串有兩種做法。

  • 方法 1: 使用 DateFormatter。
  • 方法 2: 使用 Date 的 formatted function。

方法 1: 使用 DateFormatter

DateFormatter 預設就會依據手機的時區生成字串。

let now = Date()
let formatter = DateFormatter()
formatter.dateFormat = "yyyy/MM/dd HH:mm"
let dateString = formatter.string(from: now)

彼得潘的時區是台灣時區，因此以上程式將得到台灣時間的字串。

若有需要也可以另外從程式設定時區。以下程式將 DateFormatter 的 timeZone 設為英國倫敦的時區，因此得到的時間字串是倫敦的時間。

let now = Date()
let formatter = DateFormatter()
formatter.timeZone = TimeZone(identifier: "Europe/London")
formatter.dateFormat = "yyyy/MM/dd HH:mm"
let dateString = formatter.string(from: now)

方法 2: 使用 Date 的 formatted function

let now = Date()
let dateString = now.formatted()

formatted 預設產生的格式如下。

10/03/2024, 7:05 PM

若想調整格式，可再進一步研究 formatted 搭配的參數。
