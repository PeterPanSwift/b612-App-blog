用 extension & enum 定義 Notification 的名稱和 userInfo 的 key

利用 NotificationCenter 發送和接收通知時，我們有很多地方要輸入字串，像是通知的名稱和 userInfo 的 key。

輸入字串其實是不太好的做法，因為不小心打錯時也不會出現紅色錯誤，要等 App 執行時才能發現問題。

最近彼得潘在研究 Apple 官方範例 Updating Collection Views Using Diffable Data Sources 時剛好看到一個不錯的寫法。

Updating Collection Views Using Diffable Data Sources | Apple Developer Documentation
Streamline the display and update of data in a collection view using a diffable data source that contains identifiers.developer.apple.com

Apple 在 NotificationName.swift 定義了通知的名稱和 userInfo 的 key。

  • 通知的名稱。

在 NSNotification.Name 的 extension 宣告 type property 儲存自訂的通知名稱。

  • userInfo 的 key。

用 raw value 為字串的 enum 定義 userInfo 的 key。

import Foundation

extension NSNotification.Name {
    /// - Tag: NSNotificationName
    static let recipeDidAdd = Notification.Name("com.example.apple-samplecode.DiffableDataSourceSample.recipeDidAdd")
    static let recipeDidChange = Notification.Name("com.example.apple-samplecode.DiffableDataSourceSample.recipeDidChange")
    static let recipeDidDelete = Notification.Name("com.example.apple-samplecode.DiffableDataSourceSample.recipeDidDelete")

    static let selectedRecipesDidChange = Notification.Name("com.example.apple-samplecode.DiffableDataSourceSample.selectedRecipesDidChange")

    static let recipeCollectionsDidChange = Notification.Name("com.example.apple-samplecode.DiffableDataSourceSample.recipeCollectionsDidChange")
}

// Custom keys to use with userInfo dictionaries.
enum NotificationKeys: String {
    case recipe
    case recipeId
    case recipeCollections
    case selectedRecipeIds
}

之後在接收和發送通知時只要輸入點，然後再從選單選擇。

輸入 userInfo 的 key 時也可以方便地輸入 NotificationKeys.，然後再從選單選擇。
