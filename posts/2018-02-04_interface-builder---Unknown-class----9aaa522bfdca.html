interface builder 的 Unknown class 問題

在 interface builder 上設計 App 時，我們時常將 controller 或 UI 元件的 class 設為自訂的類別，如此才能從程式存取控制畫面上的元件。

不過如果你後來將 class 檔刪除，但卻忘了修改 storyboard 裡元件的 class，執行 App 時將看到類似以下的錯誤訊息，顯示 Unknown class …，表示 Xcode 找不到這個 class。

Unknown class MovieController in Interface Builder file.

controller 的 class，MovieController 並不存在。如果你點選 class 旁的箭頭，將發現無法跳到相關的程式檔案，因為它根本不存在 !

此錯誤不見得會造成 App 閃退，但最好記得將 class 改成真實存在的類別。如果你想寫程式實作相關功能，請將它改成自訂類別。如果你不想寫程式實作相關功能，也請將 Class 欄位的文字全部刪除，如此它才能回到元件預設的類別，比方下圖的 UIViewController。
