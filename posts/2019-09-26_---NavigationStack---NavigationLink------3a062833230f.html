利用 NavigationStack & NavigationLink 切換頁面

SwiftUI 利用 NavigationStack 管理切換多層頁面，它會在畫面的上方長出一條 navigation bar(toolbar)，bar 的左上角則有箭頭的返回按鈕，類似以下的 Music App 畫面。

接下來就讓我們看看 SwiftUI 如何實現這樣的功能吧。我們希望從下圖的那個人是你嗎頁面點選文字或圖片跳到下一頁，顯示對的人的照片。(ps: 聽說對的人名字叫 peter)

第一頁畫面 ContentView 的程式

struct ContentView: View {
   var body: some View {
      VStack {
         Text("對的時間點")
              .font(.largeTitle)
         Image(systemName: "clock")
            .resizable()
            .scaledToFit()
            .frame(width: 200)
      }
   }
}

第二頁畫面 RightPersonView 的程式

顯示著命中注定的人，假設 peter 就是 Mr.Right，程式如下。

struct RightPersonView: View {
    var body: some View {
        Image(.peter)
            .resizable()
            .scaledToFit()
    }
}

利用 NavigationStack 包住它管理的第一頁畫面

為了利用 NavigationStack 管理切換多層頁面，我們必須用 NavigationStack 包住它管理的第一頁畫面，之後即可切換到第二頁，第三頁。

NavigationStack | Apple Developer Documentation
A view that displays a root view and enables you to present additional views over the root view.developer.apple.com

我們想要它管理的第一頁是 ContentView，因此我們可以在 ContentView 的 body 生成 NavigationStack，然後用 NavigationStack 的 { } 包含 ContentView 要顯示的內容。

輸入 NavigationStack 的方法很多，比方利用 embed in 加入，詳細步驟可參考以下連結:

利用 Embed in 加入 SwiftUI 元件
SwiftUI 有一些特別的元件，專門以 { } 容納其它的元件，比方 HStack，VStack，List 等，Xcode 貼心地在 Code Actions 選單提供 Embed in 功能，方便我們快速加入這些元件。medium.com

加了 NavigationStack 後，程式如下，原本的 VStack 整個被包在 NavigationStack 的 { } 裡。

struct ContentView: View {
   var body: some View {
       NavigationStack {
           VStack {
             Text("對的時間點")
                  .font(.largeTitle)
             Image(systemName: "clock")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
           }
       }
   }
}

此時畫面上方已長出一條 navigation bar，只是因為顏色透明所以我們看不到。

利用 navigationTitle 設定 bar 上的標題

我們可以呼叫 function navigationTitle 設定 bar 上的標題，因此我們從 VStack 呼叫 navigationTitle，將標題設為那個人是你嗎。

struct ContentView: View {
   var body: some View {
       NavigationStack {
           VStack {
             Text("對的時間點")
                  .font(.largeTitle)
             Image(systemName: "clock")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
           }
           .navigationTitle("那個人是你嗎")
       }
   }
}

設定標題時，記得要從 NavigationStack { } 裡的元件呼叫 navigationTitle，若是直接從 NavigationStack 呼叫 navigationTitle 是不會產生任何標題的。

struct ContentView: View {
   var body: some View {
       NavigationStack {
           VStack {
             Text("對的時間點")
                  .font(.largeTitle)
             Image(systemName: "clock")
                .resizable()
                .scaledToFit()
                .frame(width: 200)
           }
       }
       .navigationTitle("那個人是你嗎")
   }
}

接著讓我們幫第二頁的 RightPersonView 也加個標題，在它的 Image 呼叫 .navigationTitle("Peter Pan")。

struct RightPersonView: View {
    var body: some View {
        Image(.peter)
            .resizable()
            .scaledToFit()
            .navigationTitle("Peter Pan")
    }
}

此時 RightPersonView 的 preview 並不會顯示標題 Peter Pan。這是正常的，因為要有 NavigationStack 才會有標題，而 preview 畫面沒那麼聰明，它不知道我們會透過 NavigationStack 從 ContentView 進入 RightPersonView。但當我們執行 App 從 ContentView 進入 RightPersonView 時，卻是可以看到標題的。

若想在 RightPersonView 的 preview 也能看到 navigation bar & 標題，可以在 preview 的程式加上 NavigationStack，將 RightPersonView 包在 NavigationStack 裡。

#Preview {
    NavigationStack {
        RightPersonView()
    }
}

利用 inline 縮短 navigation bar 的高度

在設定 navigation bar 標題時，我們也可以縮短 navigation bar 的高度。呼叫 navigationBarTitleDisplayMode，傳入 .inline讓 navigation bar 的高度縮短。

struct RightPersonView: View {
    var body: some View {
        Image(.peter)
            .resizable()
            .scaledToFit()
            .navigationTitle("Peter Pan")
            .navigationBarTitleDisplayMode(.inline)
    }
}

設定 navigation bar 的副標題

設定 SwiftUI & UIKit App navigation bar 的副標題 — iOS 26 新功能
從 iOS 26 開始，我們也可以在 navigation bar 顯示副標題了。以下分別介紹 SwiftUI & UIKit App 的做法。medium.com

利用 NavigationLink 切換頁面

使用者通常透過點選 button 切換頁面。當我們透過 NavigationStack 管理多層頁面時，它將結合名為 NavigationLink 的 button 切換頁面。我們只要將元件包在 NavigationLink 裡，即可實現點選元件切換頁面的功能。比方我們將 Text 包在 NavigationLink 裡，點選 Text 即可切換頁面。

NavigationLink | Apple Developer Documentation
A view that controls a navigation presentation.developer.apple.com

點選文字切換頁面

接下來就讓我們將文字對的時間點包在 NavigationLink 裡，實現點選文字切換頁面的功能。

  • 在 Text("對的時間點") 前輸入參數是 destination & label 的 NavigationLink。

剛剛生成的 NavigationLink 搭配的 init 如下，參數 destination 代表點選後會前往的頁面，label 代表 NavigationLink 顯示的內容。

init(@ViewBuilder destination: () -> Destination, @ViewBuilder label: () -> Label)

在參數 destination 的框框按 enter，此時 NavigationLink 自動變成下圖的格式。

我們想要前往的頁面是 RightPersonView，因此我們在第一個 { } 傳入 RightPersonView()，然後在 label 參數的 { } 裡傳入要顯示的內容 Text("對的時間點") 。

NavigationLink {
    RightPersonView()
} label: {
    Text("對的時間點")
        .font(.largeTitle)
}

此時 ContentView 的完整程式如下。

struct ContentView: View {
    var body: some View {
        NavigationStack {
            VStack {
                NavigationLink {
                    RightPersonView()
                } label: {
                    Text("對的時間點")
                        .font(.largeTitle)
                }
                Image(systemName: "clock")
                    .resizable()
                    .scaledToFit()
                    .frame(width: 200)
            }
            .navigationTitle("那個人是你嗎")
        }
    }
}

如下圖所示，此時文字還會神奇地變成藍色，因為 iOS 預設會將可點選的文字變成藍色。

現在 App 已經可以切換頁面了。點選對的時間點後，即可看到帥氣的 Peter !

顯示文字的 NavigationLink

由於點選文字切換是最常見的應用，所以 NavigationLink 還提供另一個方便的 init，我們可以直接傳入字串當顯示的文字，不用再生成 Text。

NavigationLink("對的時間點") {
    RightPersonView()
}

若想調整文字樣式，可從 NavigationLink 直接呼叫 modifier。

NavigationLink("對的時間點") {
    RightPersonView()
}
.font(.largeTitle)

客製的 NavigationLink

NavigationLink 裡可以包含任何的 view，因此我們也可將 VStack 包在 NavigationLink 裡，變成圖片跟文字都能點選。

struct ContentView: View {
    var body: some View {
        NavigationStack {
            VStack {
                NavigationLink {
                    RightPersonView()
                } label: {
                    VStack {
                        Text("對的時間點")
                            .font(.largeTitle)
                        Image(systemName: "clock")
                            .resizable()
                            .scaledToFit()
                            .frame(width: 200)
                    }
                }
            }
            .navigationTitle("那個人是你嗎")
        }
    }
}

切換多層頁面

熟悉 NavigationStack & NavigationLink 後，我們將可隨心所欲地切換頁面，比方在 RightPersonView 裡加上文字錯的時間點，點選後切換到 WrongPersonView 頁面。

struct RightPersonView: View {
    var body: some View {
        VStack {
            Image(.peter)
                .resizable()
                .scaledToFit()
            NavigationLink("錯的時間點") {
                WrongPersonView()
            }
            .font(.largeTitle)
        }
        .navigationTitle("Peter Pan")
    }
}

WrongPersonView 顯示討厭的 hook。

struct WrongPersonView: View {
    var body: some View {
        Image(.hook)
            .resizable()
            .scaledToFit()
            .navigationTitle("hook")
    }
}

點選對的時間點後，看到帥氣的 spider man ! 點選錯的時間點後，看到可怕的 hook。

容易遇到的問題

最後關於 NavigationStack & NavigationLink 切換頁面的故事，還有兩個容易遇到問題要稍微注意一下:

  • NavigationStack 通常只需要一個。

當我們想透過 NavigationStack 從 ContentView 切換到 RightPersonView 時，只須在 ContentView 生成 NavigationStack，透過一個 NavigationStack 管理即可。若以 TabView 實現分頁 App，則可在每個分頁搭配 NavigationStack 管理各分頁的頁面切換。

  • 利用 NavigationLink 切換頁面一定要搭配 NavigationStack。

NavigationStack 控制著頁面的切換。若沒有 NavigationStack，只使用 NavigationLink 將無法點選切換頁面。

struct ContentView: View {
    var body: some View {
        VStack {
            NavigationLink {
                RightPersonView()
            } label: {
                VStack {
                    Text("對的時間點")
                        .font(.largeTitle)
                    Image(systemName: "clock")
                        .resizable()
                        .scaledToFit()
                        .frame(width: 200)
                }
            }
        }
        .navigationTitle("那個人是你嗎")
        
    }
}
