開發 SwiftUI App 時，我們有時會需要在某些情況動態隱藏 / 顯示元件。常見的做法有以下三種。

利用 opacity，hidden 或 if 動態隱藏 / 顯示 SwiftUI 元件

開發 SwiftUI App 時，我們有時會需要在某些情況動態隱藏 / 顯示元件。常見的做法有以下三種。

  • 使用 modifier opacity。
  • 使用 modifier hidden。
  • 使用 if 條件。

以上三種方法都能動態隱藏 / 顯示元件，不過它們對畫面排版的影響卻有很大差異。

使用 opacity & hidden 隱藏元件時，元件雖然看不到，但它還是存在，就像鬼一樣。這些被隱藏的元件會佔空間，當它隱藏時，畫面的排版不受絲毫影響。

相反的，當 if 條件不成立，SwiftUI 不會產生 if { } 裡的元件，所以空間不會被佔據，畫面的排版將受影響。

接下來讓我們以超可愛的心碎狐 & 利綺的藏不住歌詞為例說明。

使用 modifier opacity

struct ContentView: View {
    @State private var show = false

    var body: some View {
        VStack(spacing: 20) {
            Image(.sadFox)
                .resizable()
                .scaledToFit()
                .padding()
                .opacity(show ? 1 : 0)
            Text(
                """
                藏不住對你的愛 無法掩飾這痛苦
                只好眼看著幸福 越來越遠 墮落在深谷
                愛怎麼能就此打住
                我想變得清楚 不必在乎 不再受苦
                """
            )
            .opacity(show ? 1 : 0)
            
            Button(show ? "藏不住" : "藏住了") {
                show.toggle()
            }
            .font(.largeTitle)
            .bold()
            .foregroundStyle(.yellow)
        }
    }
}

一開始 show 是 false，因此 opacity 為 0，表示圖片和歌詞完全透明，沒人看得到。不過它們還是佔空間，所以 button 被擠到下方。

點選藏住了， show 變成 true，文字變成藏不住。此時 opacity 為 1，表示圖片和歌詞完全不透明，大家都看得到。圖片和歌詞顯示後畫面的排版不受影響，因為 SwiftUI 原本就為它們保留了空間。(ps: 就像楊過一樣，不管小龍女是否在他身邊，他的心裡都會為小龍女保留位置。）

使用 modifier hidden

struct ContentView: View {
    @State private var show = false
    
    var body: some View {
        VStack(spacing: 20) {
            Image(.sadFox)
                .resizable()
                .scaledToFit()
                .padding()
                .hidden()
            Text(
                """
                藏不住對你的愛 無法掩飾這痛苦
                只好眼看著幸福 越來越遠 墮落在深谷
                愛怎麼能就此打住
                我想變得清楚 不必在乎 不再受苦
                """
            )
            .hidden()
            
            Button("藏住了") {
                
            }
            .font(.largeTitle)
            .bold()
            .foregroundStyle(.yellow)
        }
    }
}

hidden() 作用的元件將被隱藏，因此圖片和歌詞不會顯示。不過它們還是佔空間，所以 button 被擠到下方。

使用 if 條件

struct ContentView: View {
    @State private var show = false
    
    var body: some View {
        VStack(spacing: 20) {
            if show {
                Image(.sadFox)
                    .resizable()
                    .scaledToFit()
                    .padding()
                Text(
                """
                藏不住對你的愛 無法掩飾這痛苦
                只好眼看著幸福 越來越遠 墮落在深谷
                愛怎麼能就此打住
                我想變得清楚 不必在乎 不再受苦
                """
                )
            }
            Button(show ? "藏不住" : "藏住了") {
                show.toggle()
            }
            .font(.largeTitle)
            .bold()
            .foregroundStyle(.yellow)
        }
    }
}

一開始 show 是 false，因此 if 條件不成立，圖片和歌詞不會產生。既然它們不存在世上，當然不會佔空間，button 開心地置中顯示。

點選藏住了後， show 變成 true，文字變成藏不住。此時 if 條件成立， 圖片和歌詞將產生並顯示在畫面上。此時它們將佔據部分空間，因此影響畫面的排版，button 被擠到下方。

透過以上例子，我們了解了使用 opacity，hidden 或 if 隱藏元件對畫面排版的影響，因此我們應該依畫面的設計需求選擇合適的做法。想要隱藏時保留空間讓排版不受影響，可使用 opacity 或 hidden。想要隱藏時不佔空間，可使用 if。
