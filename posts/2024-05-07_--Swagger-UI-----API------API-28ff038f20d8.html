從 Swagger UI 創建的 API 文件測試 API

研究第三方 API 串接時，我們時常看到用 Swagger UI 創建的 API 文件，比方下圖的 ReqRes API 文件。

使用 Swagger UI 創建 API 文件有很多好處，它不只可以看，還能讓我們直接在網頁上測試 API。

以下我們以 ReqRes API 為例說明測試的方法。

連到 ReqRes API 的網頁

Swagger UI
Edit descriptionreqres.in

點選 fetches a user 的 API

點選右上方的 Try it out

輸入 API 需要的參數，點選 Execute

下方的 Response 區塊出現 API 回傳的結果，包含了 Response body & Response headers，而且也提供剛剛發送的 API 對應的 curl 指令。

輸入 HTTP Header 的 API key

有些 API 必須在 HTTP Header 包含 API key，比方 The Blue Alliance API。

The Blue Alliance
Watch webcasts and get team information, competition results, and more from the FIRST Robotics Competition.www.thebluealliance.com

從網頁的右上方可看到 Authorize 按鈕。

點擊它可輸入 API key。
