從 xcloc 設定 iOS App 多國語言的翻譯文字

開發 iOS App 時，我們時常使用 xcloc (Xcode Localization Catalog) 設定多國語言的翻譯文字。不過它的格式是 xml，編輯起來並不是很方便。

令人開心的，從 Xcode 13 開始，Xcode 可以方便地編輯 xcloc。最近快要過年了，接下來我們將以過年吉祥話的翻譯為例，示範如何設定 iOS App 的多國語言並從 Xcode 編輯 xcloc。

預設英文版的 App 畫面

import SwiftUI
struct ContentView: View {
    var body: some View {
        VStack(alignment: .leading, spacing: 10) {
            Image("吉祥如意")
                .resizable()
                .scaledToFit()
            Text("Wishing you good wealth and prosperity")
            Text("May there be bounty every year")
        }
        .font(.largeTitle)
    }
}

新增繁體中文為 App 支援的語言

切換到 project 的 Info 頁面，點選 Localizations 區塊下方的 +，新增 Chinese, Traditional(zh-Hant)

輸出包含多國語言資料的 xcloc

點選 Product > Export Localizations。

Xcode 將聰明地找出 App 裡需要翻譯的文字，產生對應的 xcloc，比方剛剛程式裡寫到 Text("Wishing you good wealth and prosperity")&Text("May there be bounty every year")，因此 xcloc 裡翻譯的文字將出現 Wishing you good wealth and prosperity ＆May there be bounty every year。

勾選 English & Chinese，點選 Export，為每個語言產生對應的 xcloc。如下圖所示，這些 xcloc 將存放在資料夾 Demo Localizations。

編輯 xcloc 設定翻譯的文字

我們想要加入中文翻譯，因此打開剛剛輸出的資料夾 Demo Localizations，編輯 zh-Hant.xcloc。

點選側邊欄的 Localizable，從 Localizable 的表格設定 App 需要翻譯的文字。

在 Chinese, Traditional 欄位輸入中文翻譯。

輸入包含多國語言資料的 xcloc

點選 Product > Import Localizations。

選擇 zh-Hant.xcloc，然後點選 Import。

由於 InfoPlist 沒有設定翻譯，所以會出現下圖的警告，不過在此我們可以先不管，直接點選右下的 Import。

ps: 編輯 zh-Hant.xcloc 時可在 InfoPlist 設定 App 名字的翻譯，比方讓 App 有英文版跟中文版的名字。

輸入成功後，Xcode 將產生檔案 Localizable.strings，裡頭包含我們在 xcloc Localizable 設定的中文翻譯。

若 App 支援多個語言，則會有多個 Localizable.strings，存在不同的資料夾下。比方繁體中文的 Localizable.strings 將存在 zh-Hant.lproj 資料夾下。

App 執行時將從 Localizable.strings 找尋翻譯的內容。當手機設定的語言是繁體中文時，App 將從 zh-Hant.lproj/Localizable.strings 找尋對應的翻譯。

測試 App 的多國語言

從模擬器測試 App 的多國語言時，我們可以將 iOS 的語言改成特定語言，比方設為繁體中文，然後再測試 App 是否正確顯示中文。

不過其實有更簡單的方法，不需要修改 iOS 的語言就能測試，方法如下:

點選 Edit Scheme。

切換到 Run 的 Options 頁面，點選 App Language 設定 App 啟動時 iOS 採用的語言。

選擇 Chinese, Traditional。

Cool，App 成功地顯示中文的過年吉祥話。

One more thing，xcloc 是個資料夾

xcloc 其實是個資料夾，裡頭包含了 App 多國語言的相關資料。有興趣的朋友可從 Finder 的右鍵選單 Show Package Contents 打開它，一探它的廬山真面目。

我們在 Localizable 輸入的翻譯存在 Localized Contents 下的 xliff 檔。
