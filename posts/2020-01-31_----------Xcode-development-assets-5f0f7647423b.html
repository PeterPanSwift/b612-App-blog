方便存放測試檔案的 Xcode development assets

開發 iOS App 時，我們時常會在專案裡準備一些方便測試的圖片，JSON 檔和程式，這樣當後台還不能串接或是彼得潘回 Neverland 過年沒有網路時，依然可以測試 App。

這些測試的檔案其實不該出現在上架 App Store 的 App 裡，有方法把它們當工具人，需要的時候使用，不需要時甩掉嗎 ?

可以的 ! 我們可以將測試的檔案放在 Xcode 的 development assets 裡，它們將不會包含在我們用 archive 製作的 App 裡。

從 App 的 General 分頁可看到 Development Assets 區塊。

如果一開始建立的是 SwiftUI App 專案，Xcode 將自動幫我們產生存放 preview 測試檔案的資料夾 Preview Content，將它加到 development assets 下，而且還貼心地包含了存放圖片的 Preview Assets.xcassets。

以下讓我們看看 Development Assets 的幾種應用。

存放測試圖片

存放 JSON 或其它素材檔

圖片以外的檔案，比方 JSON 檔，文字檔，影音檔，都可加到 Preview Assets.xcassets，或是單獨放在 Preview Content 下。

  • 將 lyrics.json 放在 Preview Assets.xcassets。
  • 將 lyrics.json 放在 Preview Content 下。

存放程式檔

在 Song+Preview.swift 定義測試的假資料。

extension Song {
    static let sampleSongs = [
        Song(name: "對的時間點", singer: "林俊傑", lyrics: "如果 愛情是場遠程的渦旋 儘管繞著圈子 也要走向前 不離心太遠 我要面朝最藍的 晴天 不脫離軌道 有你在身邊"),
        Song(name: "想見你想見你想見你", singer: "八三夭", lyrics: "想見你 只想見你 未來過去 我只想見你 穿越了 千個萬個 時間線裡 人海裡相依")
    ]
    
    static let sampleSong = sampleSongs[0]
}

手動新增 Development Assets

如果不是採用 SwiftUI，而是選擇 Storyboard 建立的專案呢 ? 此時 Development Assets 區塊預設將是空的。

我們可以點選 Development Assets 下方的 +，自己加入想測試的檔案。

比方我們先用 New Folder 建立存放測試檔案的資料夾 Preview Content。

在 Preview Content 裡加入 Preview Assets.xcassets & song.json。

然後點選 Development Assets 下方的 +，加入 Preview Content。
