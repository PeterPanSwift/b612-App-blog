從 Firebase console 手動輸入 Cloud Firestore 的 database 內容

Cloud Firestore 的 database 主要分成 collection & document 兩種東西。document 代表我們的資料，比方我們可以用 document 儲存每首歌的相關資訊，像是歌名，歌手等，所以十首歌將有十個 document。而 document 會被裝在 collection 裡，所以我們可以用一個 Songs collection 儲存 10 首歌的 document。

以建立一秒落淚情歌歌單為例。

1 連到 Firebase 的專案頁面，切換到 Firestore Database 的 Data 頁面。

https://console.firebase.google.com

2 點選 Add collection 的 +，新增一個 collection。

database 裡可以有很多 collection，每個 collection 代表某一類東西的集合，比方儲存 10 首歌的 songs collection，儲存 10 部電影的 movies collection。

3 輸入 collection 的名字 songs。

collection songs 將儲存一秒落淚的情歌歌單。

輸入名字 songs。一般 collection 的結尾會加 s，代表複數。

上圖黑色說明的文字還提到了 document。collection 裡存的東西其實是 document，所以 songs 裡倘若存了 10 首歌，即代表有 10 個 document，每首歌都是一個 document。

4 輸入第一首歌。(add first document)

每個 document 都有一個 Document ID。我們可自己設定，也可以不填，讓它自動生成。在此我們選擇讓它自動生成。

我們可透過 filed 設定一首歌有哪些資訊。Field 代表欄位名稱。如下圖所示，我們輸入 name 代表歌名。然後將 Type 設為 string，value 填入 你好不好。

資料的內容不一定是 string。如下圖所示，我們可選擇的 Type 有 string，number，boolean，map，array，null，timestamp， geopoint，reference。

點選 Add field 的 + 新增欄位，設定 singer 為周興哲。

點選 Add field 的 + 新增欄位，設定 rate 為 5。

點選 Save 儲存，成功輸入第一首歌後的 database 頁面。

5 輸入第二首歌。

點選 Add document 輸入第二首歌。
