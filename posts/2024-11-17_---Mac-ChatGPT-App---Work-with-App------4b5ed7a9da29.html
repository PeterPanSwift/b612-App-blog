使用 Mac ChatGPT App 的 Work with App 讀取程式

最近 Mac ChatGPT App 推出新功能 Work with App，透過它我們可以直接讀取 App 正在編輯的程式，不用再辛苦地複製貼上程式或上傳檔案。

https://help.openai.com/en/articles/10119604-work-with-apps-on-macos

以下我們一步步介紹 Work with App 的相關操作。

  • 打開 ChatGPT 的 work with Apps 和 Accessibility 權限。
  • ChatGPT 支援的 App。
  • 讀取 Xcode 的內容。
  • 讀取 Terminal 的內容。
  • 讀取 VS Code 的內容。
  • 讀取多個 App 的內容。
  • 讀取多個檔案。

打開 ChatGPT work with Apps 和 Accessibility 權限

使用 work with Apps 前必須先開啟權限。請打開 Mac ChatGPT App，點選左下的個人頭像，點選 Settings。

打開 Enable Work with Apps。

ChatGPT App 的輸入框出現 work with Apps 的按鈕，不過既然是搭配 App 使用，我們更常的操作是在操作 App 時按 option + 空白鍵叫出 ChatGPT 的輸入框。

點選 work with Apps 按鈕，出現可選擇的 App 清單。清單將顯示目前已啟動且可搭配的 App，比方下圖的 Xcode、TextEdit & Terminal。

可搭配但沒啟動的 App 則會出現在 Inactive Apps 裡，比方下圖的 VS Code & Iterm2。

App 右邊的按鈕顯示 Enable 表示 Accessibility 的權限尚未打開。請先點選 任一 App 旁的 Enable，跳出 Enable Accessibility Permissions 的視窗，然後點選 Open System Settings。

打開 Accessibility 的 ChatGPT 開關。

經過以上步驟，我們終於可以使用 ChatGPT 讀取 App 的內容。App 右邊的按鈕變成了 +，點選 + 即可讓 ChatGPT 讀取 App 的內容。

ChatGPT 支援的 App

  • code editors: Xcode, VS Code (包含 Code, Code Insiders, VSCodium, Cursor, Windsurf), Jetbrains (包含 Android Studio, IntelliJ, PyCharm, WebStorm, PHPStorm, CLion, Rider, RubyMine, AppCode, GoLand, DataGrip), Nova。
  • text editors: TextEdit, Notes, Notion。
  • terminals: Terminal, iTerm, Warp, Prompt。

自動配對正在操作的 App

除了手動選取配對的 App，我們也可以在操作 App 時按下快速鍵 option + shift + 1，ChatGPT 將自動配對 App。

比方彼得潘在操作 Xcode 時按下 option + shift + 1，因此 ChatGPT 將自動設定 Working with Xcode。

讀取 Xcode 的內容

點選 Xcode 旁的 +，ChatGPT 輸入框的上方出現 Working with Xcode，表示它可讀取 Xcode 的內容，點選 Stop 可停止讀取。(ps: project & playground 皆可讀取)

此時 ChatGPT 將讀取 Xcode 正在編輯的檔案內容，游標移到 Working with Xcode 可看到目前讀取的檔案。

以下我們請 ChatGPT 說明 Xcode 正在編輯的 SwiftUI 程式。

ChatGPT 的回答。

這個 SwiftUI 程式非常簡單，是用來展示基本的視圖結構。讓我用幽默的方式跟你說明吧，這樣會更容易理解。

📝 程式解說

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundStyle(.tint)
            Text("Hello, world!")
        }
        .padding()
    }
}

 1. struct ContentView: 定義了一個 ContentView 結構，它遵循 View 協定。這裡我們在定義一個簡單的界面，說穿了就像在準備一盤美味的程式餐點一樣！
 2. var body: some View: 這是程式的靈魂，也就是說明這個視圖會長什麼樣子。你可以想像成設計一個可愛的小卡片，卡片上的內容就定義在這裡。
 3. VStack: 這個是用來垂直排列子視圖的容器，簡單來說，VStack 就是個可以疊放不同元素的”蛋糕架子”！這裡面我們放了一個圖片和一段文字。
 4. Image(systemName: “globe”): 這行是顯示一個地球圖示，來自 Apple 的系統圖標庫。這就像在說：「嘿，我們來環遊世界吧！」
 5. .imageScale(.large): 這是設定圖片大小的修飾器，把圖片調整成大尺寸，讓地球看起來壯觀一點。
 6. .foregroundStyle(.tint): 這個設定用來修改圖片的前景色，讓顏色符合 App 設定的 tint 色調，說白了就是讓地球圖示看起來更美美噠！
 7. Text(“Hello, world!”): 一段顯示文字，這是開發者界經典的問候語句，就像是說：「程式世界，你好！」🤓
 8. .padding(): 最後給整個 VStack 加上一些內間距，避免內容貼得太緊，這樣整個畫面更有空氣感。

預覽區域

#Preview {
    ContentView()
}

這行會讓 Xcode 自動預覽這個視圖，讓你即時看到你創建的美麗界面，簡直像魔法一樣酷炫！✨

以下我們請 ChatGPT 加入更多 modifier 讓 globe 更漂亮。

ChatGPT 的回答。

可以！我們可以用更多 SwiftUI 的 modifier 讓這個地球圖標更有吸引力。來加一些修飾吧！

Image(systemName: "globe")
    .imageScale(.large)
    .foregroundStyle(.tint)
    .shadow(radius: 10) // 加上陰影，讓圖標更有層次感
    .background(
        Circle() // 背景加上圓形
            .fill(Color.blue.opacity(0.3)) // 用半透明藍色填滿
            .frame(width: 100, height: 100) // 控制圓形的大小
    )
    .padding(10) // 添加內間距
    .overlay(
        Circle() // 再加一層圓形作為邊框
            .stroke(Color.white, lineWidth: 2) // 白色邊框，線寬 2
    )

這樣修改之後，globe 圖標就會有更多的視覺效果，包括陰影、背景和邊框！希望這樣的裝飾能讓你的地球圖標更漂亮，像個時尚 icon 一樣閃耀～

讀取 Terminal 的內容

請 ChatGPT 說明 Terminal 目前輸入的指令，以下為 Terminal 目前顯示的畫面。

ChatGPT 的回答。

當然！你在終端機中使用的指令有：
 1. cd [目錄名稱]：用來改變目前的工作目錄，cd Desktop 意思是將當前路徑切換到桌面，而 cd Demo 是進入名為 “Demo” 的目錄。
 2. ls：列出當前目錄下的所有檔案和目錄。ls 會顯示 “Demo” 和 “Demo.xcodeproj” 兩個項目，表示這些是目前 “Demo” 目錄中的內容。

讀取 VS Code 的內容

讀取 VS Code 前必須先安裝 openai-chatgpt-latest.vsix。

請從以下連結下載 openai-chatgpt.vsix。

https://persistent.oaistatic.com/pair-with-ai/openai-chatgpt-latest.vsix

在 VS Code 的 Command Palette 點選 Extensions: Install from VSIX。

選擇 openai-chatgpt.vsix。

以下我們請 ChatGPT 說明 VS Code 正在編輯的 Flutter 程式。

ChatGPT 的回答。

這個程式是一個簡單的 Flutter 應用程式，讓使用者可以點擊按鈕來增加一個計數器。下面是程式的分解說明：

程式概要

 • 主要入口點：void main() 函數是應用程式的入口，執行 runApp() 並傳入 MyApp 作為參數，啟動整個 Flutter 應用程式。
 • MyApp 類別：這是一個繼承自 StatelessWidget 的 Widget，定義了應用程式的主體架構。MaterialApp 用來設定應用的標題、主題，以及主頁面 MyHomePage。

主題設置

 • MaterialApp 使用了 Material Design 的風格，並定義了一個主題，將 seedColor 設置為 Colors.deepPurple，使用 Material 3 的設計原則。
 • 你可以通過調整主題中的顏色或其他屬性來更改應用的外觀。

MyHomePage 類別

 • 這是一個繼承自 StatefulWidget 的類別，代表應用的主頁面。它使用 title 參數來顯示應用標題，並建立一個 State 對象 _MyHomePageState，用來管理計數器的狀態。

_MyHomePageState 類別

 • int _counter = 0;：這個變數用來儲存計數器的值。
 • _incrementCounter 方法：當按下浮動按鈕時，這個方法會被呼叫，並使用 setState() 方法來通知 Flutter 框架計數器值已經改變。這樣，畫面會自動更新並重新執行 build 方法。

build 方法

 • Scaffold 小部件：這是一個 Material Design 的頁面結構，包含 AppBar、body 和 floatingActionButton。
 • AppBar：顯示應用的標題，背景顏色設置為主題的 inversePrimary 顏色。
 • body：主要的內容顯示區域，包含一個 Column 小部件，排列了兩個子小部件：
 • Text：顯示說明文字 “You have pushed the button this many times:”
 • Text：顯示計數器的值 _counter，使用主題中的文字樣式。
 • floatingActionButton：一個浮動按鈕，當被按下時，會呼叫 _incrementCounter 方法並增加計數器的值。

運作方式

 1. 啟動應用：顯示一個帶有按鈕和文字的頁面，初始計數器值為 0。
 2. 點擊按鈕：每次按下浮動按鈕，計數器會增加 1，畫面即時更新並顯示新的計數器值。

希望這樣的說明能讓你理解程式邏輯！記得試著用 “hot reload” 來快速預覽改變，讓開發變得更有趣！

讀取多個檔案

ChatGPT 預設只會讀取一個正在編輯的檔案。若想讓它讀取多個檔案，必須同時開啟多個檔案，以下我們分別介紹 Xcode & VS Code 的做法。

  • Xcode

透過 Add Editor 同時開啟多個檔案，因此 ChatGPT 能讀到 TeamView.swift & BaseballTeam.swift。

  • VS Code

透過 Open to the Side 同時開啟多個檔案，因此 ChatGPT 能讀到 main.dart & song.dart。
