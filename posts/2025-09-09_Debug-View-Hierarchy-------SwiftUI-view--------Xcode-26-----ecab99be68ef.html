Debug View Hierarchy 終於能看到 SwiftUI view 的型別了 — Xcode 26 新功能

研究 App 畫面時，我們時常利用 Debug View Hierarchy 快速找出畫面對應的型別。這一招在 UIKit App 很好用，但在舊版 Xcode 卻往往看不到 SwiftUI view 的型別。

以預設專案的 Hello world 畫面為例，從下圖完全看不出定義畫面的型別是 ContentView。

令人開心的，在 Xcode 26 我們終於可以看到 SwiftUI view 的型別了。同樣地打開 Debug View Hierarchy，這次一眼就能看到 ContentView。

若是比較複雜的畫面，已經堆疊好幾頁畫面，左邊的列表將難以一眼看出畫面對應的元件。

此時我們可在下方輸入 hosting view 搜尋，因為自訂的 SwiftUI view 通常會出現在 hosting view 附近。

賓果 ! 看來應該是 Hosting view 下方的 LandmarkDetail。

回到專案查看 LandmarkDetail 的 preview，果然沒錯 !
